Looking at this problem, I need to:

1. Find which wall was built first to surround a part of the country
2. A wall "surrounds" a part if removing it leaves the territory open (i.e., the wall is essential to forming a closed region)

The key insight is that we need to find the first wall that, when added to the set of walls, creates a cycle. Since walls can only cross at endpoints, we're looking for a cycle in a graph where:
- Nodes are endpoints of walls
- Edges are the walls themselves

The first wall that creates a cycle (forms a closed region) is the answer. We can use a Union-Find (Disjoint Set Union) data structure to detect when adding a wall creates a cycle.

The algorithm:
1. For each wall in order, check if its two endpoints are already connected in the DSU
2. If they are connected, this wall creates a cycle - it's the answer
3. If no wall creates a cycle, output 0

Let me trace through the example:
- Wall 1: (0,0)-(1,0) - connects two new nodes, no cycle
- Wall 2: (0,1)-(0,0) - connects a new node to previous, no cycle
- Wall 3: (1,0)-(0,1) - connects two already-connected components, creates a cycle!
- Wall 4: (2,2)-(5,7) - separate component

So the answer is 3.

```cpp
#include <bits/stdc++.h>
using namespace std;

class DSU {
public:
    map<pair<long long, long long>, pair<long long, long long>> parent;
    
    pair<long long, long long> find(pair<long long, long long> x) {
        if (parent.find(x) == parent.end()) {
            parent[x] = x;
        }
        if (parent[x] != x) {
            parent[x] = find(parent[x]);
        }
        return parent[x];
    }
    
    bool unite(pair<long long, long long> x, pair<long long, long long> y) {
        x = find(x);
        y = find(y);
        if (x == y) {
            return false; // already connected, creates cycle
        }
        parent[x] = y;
        return true;
    }
};

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int M;
    cin >> M;
    
    DSU dsu;
    int answer = 0;
    
    for (int i = 1; i <= M; i++) {
        long long x1, y1, x2, y2;
        cin >> x1 >> y1 >> x2 >> y2;
        
        pair<long long, long long> p1 = {x1, y1};
        pair<long long, long long> p2 = {x2, y2};
        
        if (!dsu.unite(p1, p2)) {
            // Wall creates a cycle
            answer = i;
            break;
        }
    }
    
    cout << answer << endl;
    
    return 0;
}
```

The solution uses Union-Find to track connected components:
- Each endpoint of a wall is a node
- When we add a wall, we try to unite its two endpoints
- If they're already in the same component, the wall creates a cycle (closes a region)
- The first wall that creates a cycle is the answer
- If no wall creates a cycle, we output 0

Time complexity: O(M * α(M)) where α is the inverse Ackermann function (practically constant)
Space complexity: O(M) for storing endpoints
