Looking at this problem, I need to find the number of non-isomorphic Abelian groups of order n.

Key insights from the problem:
1. Every finite Abelian group is isomorphic to a direct sum of cyclic groups
2. By the Chinese Remainder Theorem, if n = p₁^a₁ × p₂^a₂ × ... × pₖ^aₖ (prime factorization), then the number of Abelian groups of order n is the product of the number of Abelian groups of order p₁^a₁, p₂^a₂, etc.
3. For a prime power p^a, the number of non-isomorphic Abelian groups equals the number of partitions of a (denoted p(a))

So the algorithm is:
1. Find the prime factorization of n
2. For each prime power p^a in the factorization, find the number of partitions of a
3. Multiply all these partition counts together

The number of partitions p(a) can be computed using dynamic programming. For small values of a (which won't exceed log₂(10¹²) ≈ 40), we can compute partitions.

Let me trace through the examples:
- n=5: 5 = 5¹, so we need p(1) = 1. Answer: 1 ✓
- n=4: 4 = 2², so we need p(2) = 2 (partitions: {2}, {1,1}). Answer: 2 ✓
- n=12: 12 = 2² × 3¹, so we need p(2) × p(1) = 2 × 1 = 2. Answer: 2 ✓

```cpp
#include<bits/stdc++.h>
using namespace std;

// Compute number of partitions of n
long long countPartitions(int n) {
    vector<vector<long long>> dp(n + 1, vector<long long>(n + 1, 0));
    
    // dp[n][k] = number of partitions of n using parts of size at most k
    for (int i = 0; i <= n; i++) {
        dp[i][0] = 0;
    }
    dp[0][0] = 1;
    
    for (int i = 1; i <= n; i++) {
        for (int k = 1; k <= n; k++) {
            dp[i][k] = dp[i][k-1];
            if (i >= k) {
                dp[i][k] += dp[i-k][k];
            }
        }
    }
    
    return dp[n][n];
}

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    long long n;
    cin >> n;
    
    // Find prime factorization
    map<long long, int> factors;
    long long temp = n;
    
    for (long long i = 2; i * i <= temp; i++) {
        while (temp % i == 0) {
            factors[i]++;
            temp /= i;
        }
    }
    if (temp > 1) {
        factors[temp]++;
    }
    
    long long result = 1;
    
    // For each prime power, multiply by the number of partitions of its exponent
    for (auto& p : factors) {
        int exponent = p.second;
        result *= countPartitions(exponent);
    }
    
    cout << result << endl;
    
    return 0;
}
```

The solution works as follows:

1. **Prime Factorization**: I factorize n by trial division up to √n
2. **Partition Counting**: For each exponent in the prime factorization, I compute the number of integer partitions using dynamic programming
3. **Result**: I multiply together the partition counts for all exponents

The partition DP works with the recurrence: the number of partitions of n using parts of size at most k equals the number of partitions using parts of size at most k-1, plus the number of partitions of (n-k) using parts of size at most k.
