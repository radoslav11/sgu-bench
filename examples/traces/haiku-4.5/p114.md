Looking at this problem, I need to find the optimal position for a TV station on a 1D axis that minimizes the sum of (population × distance) for all cities.

Let me think about this mathematically:
- We have cities at positions x_i with populations p_i
- We want to find position s that minimizes: f(s) = Σ p_i * |s - x_i|

This is a classic problem in statistics - finding the weighted median!

The key insight is that f(s) is a piecewise linear function (sum of absolute values), and its minimum occurs at one of the city positions or at a weighted median.

For a weighted median problem:
- We sort cities by position
- We find the position where the cumulative weight from left equals the cumulative weight from right

However, let me verify with the example:
- Cities: (1,3), (2,1), (5,2), (6,2)
- Total population: 3+1+2+2 = 8
- We need cumulative weight = 4 (half of 8)

Sorted by position: (1,3), (2,1), (5,2), (6,2)
- At x=1: cumulative = 3 (need 4)
- At x=2: cumulative = 4 (perfect!)
- At x=5: cumulative = 6
- At x=6: cumulative = 8

So the weighted median is at position 2, but the answer is 3.0. Let me reconsider...

Actually, since the function is piecewise linear and convex, I can use ternary search or calculate the derivative. The derivative of f(s) is:
f'(s) = Σ p_i * sign(s - x_i)

The minimum occurs where f'(s) changes from negative to positive, which happens around the weighted median, but we might need to check the actual minimum more carefully.

Let me use ternary search since the function is unimodal (convex):

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    int n;
    cin >> n;
    
    vector<double> x(n), p(n);
    double minX = 1e9, maxX = 0;
    
    for(int i = 0; i < n; i++){
        cin >> x[i] >> p[i];
        minX = min(minX, x[i]);
        maxX = max(maxX, x[i]);
    }
    
    // Function to calculate total displeasure at position pos
    auto calcDispleasure = [&](double pos) -> double {
        double sum = 0;
        for(int i = 0; i < n; i++){
            sum += p[i] * abs(pos - x[i]);
        }
        return sum;
    };
    
    // Ternary search
    double left = minX - 1000;
    double right = maxX + 1000;
    
    while(right - left > 1e-6){
        double mid1 = left + (right - left) / 3;
        double mid2 = right - (right - left) / 3;
        
        if(calcDispleasure(mid1) > calcDispleasure(mid2)){
            left = mid1;
        } else {
            right = mid2;
        }
    }
    
    double answer = (left + right) / 2;
    cout << fixed << setprecision(5) << answer << endl;
    
    return 0;
}
```

Wait, let me verify once more with the sample. Using ternary search should find that around 3.0 gives minimum displeasure:
- At x=3: 3*|3-1| + 1*|3-2| + 2*|3-5| + 2*|3-6| = 3*2 + 1*1 + 2*2 + 2*3 = 6+1+4+6 = 17
- At x=2: 3*|2-1| + 1*|2-2| + 2*|2-5| + 2*|2-6| = 3*1 + 0 + 2*3 + 2*4 = 3+0+6+8 = 17
- At x=3.5: 3*2.5 + 1*1.5 + 2*0.5 + 2*0.5 = 7.5+1.5+1+1 = 11
