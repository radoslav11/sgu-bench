Looking at this problem, I need to:

1. Understand what matrix A is: It's an N × M incidence matrix where each column represents an edge and each row represents a vertex. For edge j, there are exactly two 1s (at the positions of its endpoints).

2. Understand what A^T × A is: This is an M × M matrix. Element (i,j) of A^T × A is the dot product of column i and column j of A.

3. Find the sum of all elements in A^T × A.

Let me think about what A^T × A looks like:
- For diagonal elements (i,i): This is the dot product of edge i with itself. Since each edge has exactly two 1s, the diagonal element is 2.
- For off-diagonal elements (i,j) where i ≠ j: This is the dot product of edge i and edge j. This equals the number of vertices that are endpoints of both edges.

Two edges can share:
- 0 vertices (dot product = 0)
- 1 vertex (dot product = 1)
- 2 vertices (dot product = 2) - but the problem says "there are no loops" and "All edges are different", so two edges can share at most 1 vertex

So the sum of A^T × A equals:
- Sum of diagonal elements: 2M (each of M edges contributes 2)
- Sum of off-diagonal elements: for each pair of edges, count 1 if they share a vertex

To count pairs of edges sharing a vertex:
- For each vertex, if it has degree d, then there are d edges incident to it
- The number of pairs of these edges is C(d,2) = d(d-1)/2
- So the contribution from this vertex is d(d-1)/2

Wait, let me reconsider. If edges i and j share a vertex v, their dot product contributes 1. So we need to count how many pairs of edges share at least one vertex.

Actually, if two edges share exactly one vertex, they contribute 1 to the sum (since their dot product is 1 at that vertex). If they share two vertices, they contribute 2, but that's impossible given the constraints.

For each vertex v with degree d_v, the number of edge pairs incident to v is C(d_v, 2) = d_v(d_v-1)/2.

So the answer is:
- 2M (diagonal elements)
- Plus sum over all vertices v of d_v(d_v-1)/2 (off-diagonal elements)

This can be rewritten as:
2M + Σ(d_v(d_v-1)/2) = 2M + (Σ d_v^2 - Σ d_v)/2 = 2M + (Σ d_v^2 - 2M)/2 = M + Σ d_v^2/2

Let me verify with the example:
- N=4, M=4
- Edges: (1,2), (1,3), (2,3), (2,4)
- Degrees: vertex 1 has degree 2, vertex 2 has degree 3, vertex 3 has degree 2, vertex 4 has degree 1
- Sum = 2*4 + [2*1/2 + 3*2/2 + 2*1/2 + 1*0/2] = 8 + [1 + 3 + 1 + 0] = 8 + 5 = 13

Actually, let me reconsider: Σ d_v = 2M (handshaking lemma), so each edge is counted twice in the degree sum.

Let me recompute: 2M + Σ d_v(d_v-1)/2 = 8 + (2*1 + 3*2 + 2*1 + 1*0)/2 = 8 + (2+6+2+0)/2 = 8 + 5 = 13

Σ d_v^2 = 4 + 9 + 4 + 1 = 18

Ah! The answer is simply Σ d_v^2 where d_v is the degree of vertex v!

This makes sense: the sum equals Σ d_v^2.

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
  ios_base::sync_with_stdio(false);
  cin.tie(nullptr);
  
  int N, M;
  cin >> N >> M;
  
  vector<long long> degree(N + 1, 0);
  
  for(int i = 0; i < M; i++){
    int u, v;
    cin >> u >> v;
    degree[u]++;
    degree[v]++;
  }
  
  long long sum = 0;
  for(int i = 1; i <= N; i++){
    sum += degree[i] * degree[i];
  }
  
  cout << sum << endl;
  
  return 0;
}
```
