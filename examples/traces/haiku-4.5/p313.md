Looking at this problem, I need to:

1. Match n houses to n offices on a circular railway of length L
2. Minimize total travel time
3. Travel time between two stations on a circle is the minimum of clockwise and counterclockwise distances

Key insight: This is an assignment problem. For a circular arrangement, a good heuristic is to sort both houses and offices, then try different rotations of the matching to find the minimum cost.

The approach:
1. Calculate the distance function for a circular railway
2. Sort houses and offices (keeping track of original indices)
3. Try all possible rotations of matching sorted houses to sorted offices
4. Pick the rotation with minimum total cost
5. Map back to original indices

Let me think about the distance formula:
- Distance between station a and station b on a circle of length L:
  - Clockwise: (b - a + L) % L
  - Counterclockwise: (a - b + L) % L
  - Minimum: min of the two

Actually, the minimum distance is simply: min((b-a+L)%L, (a-b+L)%L)

For the matching strategy: if we sort both arrays and try rotating one relative to the other, we should find a good solution. The optimal solution for a circular problem often involves matching sorted sequences with some rotation.

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, L;
    cin >> n >> L;
    
    vector<pair<int,int>> houses(n), offices(n);
    
    for(int i = 0; i < n; i++){
        cin >> houses[i].first;
        houses[i].second = i;
    }
    
    for(int i = 0; i < n; i++){
        cin >> offices[i].first;
        offices[i].second = i;
    }
    
    auto dist = [&](int a, int b) -> long long {
        long long d1 = (b - a + L) % L;
        long long d2 = (a - b + L) % L;
        return min(d1, d2);
    };
    
    // Sort houses and offices by location
    sort(houses.begin(), houses.end());
    sort(offices.begin(), offices.end());
    
    long long minCost = LLONG_MAX;
    int bestRotation = 0;
    
    // Try all rotations
    for(int rotation = 0; rotation < n; rotation++){
        long long cost = 0;
        for(int i = 0; i < n; i++){
            cost += dist(houses[i].first, offices[(i + rotation) % n].first);
        }
        if(cost < minCost){
            minCost = cost;
            bestRotation = rotation;
        }
    }
    
    // Create result array
    vector<int> result(n);
    
    for(int i = 0; i < n; i++){
        int houseOriginalIdx = houses[i].second;
        int officeIdx = (i + bestRotation) % n;
        int officeOriginalIdx = offices[officeIdx].second;
        result[houseOriginalIdx] = officeOriginalIdx + 1; // 1-indexed
    }
    
    cout << minCost << "\n";
    for(int i = 0; i < n; i++){
        cout << result[i];
        if(i < n-1) cout << " ";
    }
    cout << "\n";
    
    return 0;
}
```

The algorithm:
1. Read houses and offices with their original indices
2. Sort both by location
3. Try all n rotations - for each rotation, match house[i] with office[(i+rotation)%n]
4. Pick the rotation with minimum total distance
5. Map back to original indices for the output

Time complexity: O(n² + n log n) which is acceptable for n ≤ 50000.
