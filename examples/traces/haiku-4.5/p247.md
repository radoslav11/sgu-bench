Looking at this problem, I need to find the number of ways to choose P balls from 2P balls (numbered 1 to 2P) such that the sum of chosen numbers is divisible by P.

Let me think about this mathematically:
- We have balls numbered 1 to 2P
- We need to choose exactly P balls
- The sum of chosen ball numbers must be divisible by P
- P is an odd prime

Key insight: The sum of all numbers from 1 to 2P is:
$$\text{Sum} = \frac{2P(2P+1)}{2} = P(2P+1)$$

This sum is divisible by P.

If we choose P balls with sum S, the remaining P balls have sum P(2P+1) - S.

For S ≡ 0 (mod P), we need P(2P+1) - S ≡ 0 (mod P), which means S ≡ 0 (mod P).

So the condition is symmetric - if a subset of P balls has sum divisible by P, the complementary subset also does.

Now, let's think about the distribution of sums modulo P. 

For any subset of P elements chosen from {1, 2, ..., 2P}, the sum modulo P can take values 0, 1, ..., P-1.

By a symmetry argument (or direct counting), when we choose P items from 2P items where the items are labeled 1 to 2P:
- The total number of ways to choose P items from 2P is C(2P, P)
- These choices distribute uniformly (or near-uniformly) across the P residue classes modulo P

For an odd prime P, the number of P-subsets with sum ≡ 0 (mod P) is:
$$\frac{C(2P, P) - (-1)^P}{P}$$ or $$\frac{C(2P, P) + 1}{P}$$

Actually, let me use a different approach. By Lucas' theorem and properties of binomial coefficients modulo primes:

For odd prime P, the number of ways to choose P items from 2P items such that their sum is divisible by P is:
$$\frac{1}{P}\left(\binom{2P}{P} + (P-1)(-1)^P\right)$$

Since P is odd, $(-1)^P = -1$:
$$\frac{1}{P}\left(\binom{2P}{P} - (P-1)\right)$$

Let me verify with P=3:
- C(6,3) = 20
- Answer = (20 - 2)/3 = 18/3 = 6... but expected is 8

Let me reconsider. Perhaps the formula is:
$$\frac{\binom{2P}{P} + (P-1)}{P}$$

For P=3: (20 + 2)/3 = 22/3... doesn't work either.

Let me try a different formula. For odd prime P:
$$\frac{1}{P}\left(\binom{2P}{P} + P - 1\right)$$

For P=3: (20 + 3 - 1)/3 = 22/3... still no.

Actually, the correct formula for odd primes should be:
$$\text{count} = \frac{\binom{2P}{P} - (-1)^P}{P}$$

For P=3 (odd): count = (20 - (-1))/3 = 21/3 = 7... close but not 8.

After research, the answer for odd prime P is:
$$\frac{\binom{2P}{P} + P - 1}{P}$$

For P=3: (20 + 2)/3 = 22/3 ≈ 7.33... 

Let me just implement a direct solution using dynamic programming to count subsets:

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    int N;
    cin >> N;
    
    while(N--) {
        int P;
        cin >> P;
        
        int total = 2 * P;
        
        // dp[i][j][s] = number of ways to choose j items from first i items with sum % P == s
        vector<vector<vector<long long>>> dp(total + 1, vector<vector<long long>>(P + 1, vector<long long>(P, 0)));
        
        dp[0][0][0] = 1;
        
        for(int i = 1; i <= total; i++) {
            for(int j = 0; j <= P; j++) {
                for(int s = 0; s < P; s++) {
                    // Don't take ball i
                    dp[i][j][s] += dp[i-1][j][s];
                    
                    // Take ball i
                    if(j > 0) {
                        int prev_s = (s - i % P + P) % P;
                        dp[i][j][s] += dp[i-1][j-1][prev_s];
                    }
                }
            }
        }
        
        cout << dp[total][P][0] << "\n";
    }
    
    return 0;
}
```
