Looking at this problem, I need to determine if a cop can catch a thief on a grid where they alternate moves (one starting first).

Key observations:
1. The cop catches the thief if they're in the same cell
2. Both play optimally
3. The thief wants to avoid capture as long as possible, the cop wants to catch quickly
4. Both can see each other and play optimally

Let me think about the conditions:
- The thief can escape if there's enough space to move around
- The cop needs to be able to corner or catch the thief
- The distance and parity matter

The thief can escape indefinitely if:
1. The thief can always move away from the cop, maintaining distance
2. On a finite grid, the thief will eventually be caught if the cop plays optimally, UNLESS the thief can reach a "safe" position

Key insight: 
- If the cop moves first, the cop needs to catch within a reasonable number of moves
- If the thief moves first, the thief gets one extra move advantage
- On a grid, if the initial Manhattan distance is even and it's the cop's turn, the cop can potentially catch the thief
- The thief can escape if they can maintain an odd distance when it's the cop's turn to move (or even distance when it's the thief's turn)

Let me reconsider with parity:
- Manhattan distance: `dist = |Rc - Rt| + |Cc - Ct|`
- If cop moves first and dist is even, cop can catch (they move on the same parity positions)
- If cop moves first and dist is odd, they're on opposite parity
- The catch happens when they're on the same cell

If the thief plays optimally:
- If initial distance is odd and cop moves first: cop is on even parity cells, thief on odd parity cells. Thief can stay on odd parity forever → escape (output 0)
- If initial distance is even and cop moves first: both can be on same parity. Cop can corner the thief.
- If thief moves first and distance is odd: after thief moves, distance becomes even, cop can pursue
- If thief moves first and distance is even: after thief moves, distance becomes odd, thief can escape

Actually, let me reconsider more carefully with the examples:
- Example 1: 2x2 grid, cop at (1,2), thief at (2,1), cop moves first. Distance = |1-2|+|2-1| = 2 (even). Output: 0
- Example 2: Same but thief moves first. Output: 2

When thief moves first with distance 2:
- Move 1: Thief moves (say to (1,1)), distance becomes 1 or 3
- Move 2: Cop moves and catches

If the grid is small (bounded), the thief can be trapped. But with a large grid and thief moving first, maybe the thief can escape more easily.

After analysis: The cop can catch the thief if and only if:
1. The cop moves first AND the Manhattan distance is even, OR
2. The thief moves first AND the Manhattan distance is odd

When caught, it's after distance moves (optimal play).

#include<bits/stdc++.h>
using cmp = long long;

    ios_base::sync_with_stdio(false);

    char first;
    
    cin >> H >> W >> Rc >> Cc >> Rt >> Ct >> first;
    
    long long dist = abs(Rc - Rt) + abs(Cc - Ct);
    
    bool cop_catches = false;
    
    if(first == 'C'){
        // Cop moves first
        // Cop catches if distance is even
        if(dist % 2 == 0){
            cop_catches = true;
        }
    } else {
        // Thief moves first
        // Cop catches if distance is odd
        if(dist % 2 == 1){
            cop_catches = true;
        }
    }

    } else {
        cout << dist << endl;
    }

}
