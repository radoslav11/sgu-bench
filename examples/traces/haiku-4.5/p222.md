Looking at this problem, I need to find the number of ways to place k non-attacking rooks on an n×n chessboard.

Key insights:
1. Two rooks attack each other if they're in the same row or column
2. So to place k non-attacking rooks, I need to:
   - Choose k rows from n rows
   - Choose k columns from n columns
   - Arrange k rooks on the selected k×k grid such that there's exactly one rook per row and one per column

The last part is equivalent to finding permutations of k elements, which is k!

So the formula is:
- Choose k rows: C(n, k)
- Choose k columns: C(n, k)
- Arrange rooks in selected rows/columns: k!
- Total = C(n, k) × C(n, k) × k!

Let me verify with the example:
- n=4, k=4
- C(4,4) = 1
- C(4,4) = 1
- 4! = 24
- Total = 1 × 1 × 24 = 24 ✓

Let me also check edge cases:
- k=0: C(n,0) × C(n,0) × 0! = 1 × 1 × 1 = 1 (one way to place zero rooks)
- k=1, n=2: C(2,1) × C(2,1) × 1! = 2 × 2 × 1 = 4 (place on (0,0), (0,1), (1,0), (1,1))

This approach seems correct!

```cpp
#include <iostream>
using namespace std;

long long factorial(int n) {
    long long result = 1;
    for (int i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}

long long binomial(int n, int k) {
    if (k > n || k < 0) return 0;
    if (k == 0 || k == n) return 1;
    
    long long result = 1;
    for (int i = 0; i < k; i++) {
        result *= (n - i);
        result /= (i + 1);
    }
    return result;
}

int main() {
    int n, k;
    cin >> n >> k;
    
    if (k > n) {
        cout << 0 << endl;
        return 0;
    }
    
    long long ways = binomial(n, k) * binomial(n, k) * factorial(k);
    cout << ways << endl;
    
    return 0;
}
```

The solution works as follows:

1. **Read input**: Get n (board size) and k (number of rooks)

2. **Edge case**: If k > n, it's impossible to place k non-attacking rooks (we can't have more than n rooks without some sharing a row or column), so output 0

3. **Calculate the answer**:
   - `C(n, k)` ways to choose which k rows to use
   - `C(n, k)` ways to choose which k columns to use  
   - `k!` ways to arrange the k rooks within the selected k×k subgrid (this is the number of permutations)
   - Multiply all three together

4. **Helper functions**:
   - `factorial(n)`: Computes n!
   - `binomial(n, k)`: Computes C(n, k) using the formula with division to avoid overflow
