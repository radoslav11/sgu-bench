Looking at this problem, I need to find the number of integer solutions (x, y) to the equation ax + by + c = 0 where x1 ≤ x ≤ x2 and y1 ≤ y ≤ y2.

Let me break down the approach:

1. The equation is ax + by + c = 0, which means by = -ax - c, so y = (-ax - c)/b (when b ≠ 0)
2. For a solution to be valid:
   - x must be an integer in [x1, x2]
   - y = (-ax - c)/b must be an integer in [y1, y2]
   - This means (-ax - c) must be divisible by b

Special cases:
- If a = 0 and b = 0: either no solutions (if c ≠ 0) or infinite solutions (if c = 0)
- If a = 0 and b ≠ 0: y = -c/b is fixed, check if it's an integer in [y1, y2]
- If b = 0 and a ≠ 0: x = -c/a is fixed, check if it's an integer in [x1, x2]
- If a ≠ 0 and b ≠ 0: iterate through valid x values and check if y is valid

Let me trace through the example:
- a=1, b=1, c=-3
- Equation: x + y - 3 = 0, so y = 3 - x
- x in [0,4], y in [0,4]
- x=0: y=3 ✓
- x=1: y=2 ✓
- x=2: y=1 ✓
- x=3: y=0 ✓
- x=4: y=-1 ✗
- Answer: 4 ✓

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    long long a, b, c, x1, x2, y1, y2;
    cin >> a >> b >> c;
    cin >> x1 >> x2;
    cin >> y1 >> y2;
    
    long long count = 0;
    
    // Special case: both a and b are 0
    if(a == 0 && b == 0){
        if(c == 0){
            // All points in the rectangle are solutions
            count = (x2 - x1 + 1) * (y2 - y1 + 1);
        }
        // else c != 0, no solutions
    }
    // Special case: a = 0, b != 0
    else if(a == 0){
        // by + c = 0, so y = -c/b
        if(c % b == 0){
            long long y = -c / b;
            if(y >= y1 && y <= y2){
                count = x2 - x1 + 1;
            }
        }
    }
    // Special case: b = 0, a != 0
    else if(b == 0){
        // ax + c = 0, so x = -c/a
        if(c % a == 0){
            long long x = -c / a;
            if(x >= x1 && x <= x2){
                count = y2 - y1 + 1;
            }
        }
    }
    // General case: a != 0 and b != 0
    else {
        for(long long x = x1; x <= x2; x++){
            long long numerator = -a * x - c;
            if(numerator % b == 0){
                long long y = numerator / b;
                if(y >= y1 && y <= y2){
                    count++;
                }
            }
        }
    }
    
    cout << count << endl;
    
    return 0;
}
```

The solution handles all cases:
1. **Both a and b are 0**: If c=0, entire rectangle is solution; otherwise no solutions
2. **a=0, b≠0**: y is fixed at -c/b, count valid x values if y is in range
3. **b=0, a≠0**: x is fixed at -c/a, count valid y values if x is in range
4. **General case**: Iterate through all x in [x1, x2], check if corresponding y = (-ax-c)/b is an integer and in [y1, y2]

Time complexity is O(x2-x1) in the worst case, which should be acceptable given the constraints.
