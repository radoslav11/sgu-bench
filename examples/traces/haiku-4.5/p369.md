Looking at this problem, I need to:

1. Start with K black cells
2. Repeatedly find rectangles where 3 vertices are black and 1 is white, then paint the white vertex black
3. Continue until no more moves can be made
4. Return the final count of black cells

The key insight is that if we have black cells at positions (x1, y1), (x1, y2), and (x2, y1), then (x2, y2) must become black.

More generally, if we think about it in terms of which x-coordinates and y-coordinates have black cells:
- If there are black cells at (x1, y), (x2, y) for some row y, and black cells at (x1, y'), (x2, y') for another row y', then we'd eventually fill in all 4 corners and more.

Actually, a simpler way to think about it: if we have black cells in certain x-coordinates and certain y-coordinates, the game will eventually fill in all cells that are at the intersection of any "active" x-coordinate and "active" y-coordinate.

The question is: which x and y coordinates become "active"?

An x-coordinate becomes active if there exists some black cell in that column. Similarly for y-coordinates. But the game continues until we reach a fixed point.

Let me think differently:
- If we have a black cell at (x, y), both x and y are "involved"
- If we have black cells at (x1, y1) and (x1, y2), then x1 appears twice in different rows, so both y1 and y2 are connected
- If two rows have a common column with black cells, those two rows will eventually have black cells in all columns that appear in either row

This is essentially a union-find problem! Two rows are in the same group if they share an x-coordinate where there's a black cell. Similarly for columns.

The algorithm:
1. For each black cell (x, y), union the x-coordinate with a special "x-set" and y-coordinate with a "y-set"
2. After processing all cells, find which x-coordinates are in the same connected component as other x-coordinates through y-coordinates (and vice versa)
3. The answer is the count of distinct (representative_x, representative_y) pairs in the connected components

Let me reconsider: we need to find connected components where x-coordinates and y-coordinates form a bipartite graph, and a black cell at (x, y) connects x and y. The final answer is: for each connected component, if it has `nx` distinct x-coordinates and `ny` distinct y-coordinates, it contributes `nx * ny` black cells.

```cpp
#include<bits/stdc++.h>
using namespace std;

class DSU {
public:
    map<long long, long long> parent;
    
    long long find(long long x) {
        if (parent.find(x) == parent.end()) {
            parent[x] = x;
        }
        if (parent[x] != x) {
            parent[x] = find(parent[x]);
        }
        return parent[x];
    }
    
    void unite(long long x, long long y) {
        x = find(x);
        y = find(y);
        if (x != y) {
            parent[x] = y;
        }
    }
};

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int K;
    cin >> K;
    
    DSU dsu;
    set<long long> xs, ys;
    
    for (int i = 0; i < K; i++) {
        long long x, y;
        cin >> x >> y;
        xs.insert(x);
        ys.insert(y);
        
        // Use negative numbers for y to distinguish from x
        dsu.unite(x, -y - 1);
    }
    
    // Group by component
    map<long long, pair<set<long long>, set<long long>>> components;
    
    for (long long x : xs) {
        long long comp = dsu.find(x);
        components[comp].first.insert(x);
    }
    
    for (long long y : ys) {
        long long comp = dsu.find(-y - 1);
        components[comp].second.insert(y);
    }
    
    long long answer = 0;
    for (auto& p : components) {
        long long nx = p.second.first.size();
        long long ny = p.second.second.size();
        answer += nx * ny;
    }
    
    cout << answer << endl;
    
    return 0;
}
```
