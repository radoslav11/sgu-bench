Looking at this problem, I need to:

1. Read N, M, K from the first line
2. Read N numbers from the second line
3. Count how many numbers, when raised to the M-th power, are divisible by K
4. Output the count

Let me trace through the example:
- N=4, M=2, K=50
- Numbers: 9, 10, 11, 12
- Check each number raised to power M:
  - 9^2 = 81, 81 % 50 = 31 (not divisible)
  - 10^2 = 100, 100 % 50 = 0 (divisible!) ✓
  - 11^2 = 121, 121 % 50 = 21 (not divisible)
  - 12^2 = 144, 144 % 50 = 44 (not divisible)
- Count = 1 ✓

The straightforward approach would be to compute num^M for each number and check if it's divisible by K. However, with M up to 10000 and numbers up to 10001, this could overflow. 

The key insight is that we only care about divisibility by K, not the actual value. We can use modular exponentiation: we compute (num^M) % K and check if it equals 0.

For modular exponentiation, we can use the property:
(a * b) % K = ((a % K) * (b % K)) % K

This way we never deal with numbers larger than K^2 during computation.

```cpp
#include <iostream>
#include <cmath>
using namespace std;

long long modpow(long long base, long long exp, long long mod) {
    long long result = 1;
    base %= mod;
    while (exp > 0) {
        if (exp % 2 == 1) {
            result = (result * base) % mod;
        }
        base = (base * base) % mod;
        exp /= 2;
    }
    return result;
}

int main() {
    int N, M, K;
    cin >> N >> M >> K;
    
    int count = 0;
    for (int i = 0; i < N; i++) {
        int num;
        cin >> num;
        
        long long remainder = modpow(num, M, K);
        if (remainder == 0) {
            count++;
        }
    }
    
    cout << count << endl;
    
    return 0;
}
```

The solution uses fast modular exponentiation to compute (num^M) % K efficiently:
- Time complexity per number: O(log M)
- Space complexity: O(1)

This handles all edge cases and avoids overflow issues by keeping intermediate results within bounds.
