Looking at this problem, I need to recognize that this is a game theory problem that can be solved using the Sprague-Grundy theorem.

The key observations are:
1. This is an impartial game (both players have the same moves available)
2. Players alternate turns
3. The player who cannot move loses
4. Players can only take from the first or last heap

Since n ≤ 5 and the constraint on sum is not too large, I can use dynamic programming with memoization to calculate the Grundy numbers (nim values) for each game state.

A state is defined by the current first heap index and last heap index (left and right pointers). For each state, I need to:
1. Find all possible moves (take any number from first or last heap)
2. Calculate the Grundy number for each resulting state
3. The Grundy number of current state is the MEX (minimum excludant) of all reachable states

If the Grundy number of the initial state is non-zero, the first player (Fedor) wins; otherwise, the second player (Sergey) wins.

```cpp
#include<bits/stdc++.h>
using namespace std;

int n;
vector<int> a;
map<tuple<int,int,vector<int>>, int> memo;

int grundy(int left, int right, vector<int> heaps) {
    // Base case: no heaps left
    if (left > right) {
        return 0;
    }
    
    // Create state key
    auto state = make_tuple(left, right, heaps);
    if (memo.count(state)) {
        return memo[state];
    }
    
    set<int> reachable;
    
    // Try taking from left heap
    for (int take = 1; take <= heaps[left]; take++) {
        vector<int> newHeaps = heaps;
        newHeaps[left] -= take;
        
        int newLeft = left;
        while (newLeft <= right && newHeaps[newLeft] == 0) {
            newLeft++;
        }
        
        int g = grundy(newLeft, right, newHeaps);
        reachable.insert(g);
    }
    
    // Try taking from right heap
    for (int take = 1; take <= heaps[right]; take++) {
        vector<int> newHeaps = heaps;
        newHeaps[right] -= take;
        
        int newRight = right;
        while (newRight >= left && newHeaps[newRight] == 0) {
            newRight--;
        }
        
        int g = grundy(left, newRight, newHeaps);
        reachable.insert(g);
    }
    
    // Calculate MEX
    int mex = 0;
    while (reachable.count(mex)) {
        mex++;
    }
    
    memo[state] = mex;
    return mex;
}

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    cin >> n;
    a.resize(n);
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }
    
    int g = grundy(0, n - 1, a);
    
    if (g != 0) {
        cout << "FEDOR" << endl;
    } else {
        cout << "SERGEY" << endl;
    }
    
    return 0;
}
```

The solution works as follows:

1. The `grundy` function computes the Grundy number for a game state defined by:
   - `left`: index of the leftmost non-empty heap
   - `right`: index of the rightmost non-empty heap  
   - `heaps`: current state of all heaps

2. For each state, we try all possible moves:
   - Take any number from the leftmost heap (1 to all of it)
   - Take any number from the rightmost heap (1 to all of it)

3. After each move, we find the new leftmost/rightmost heap indices (skipping empty heaps)

4. We collect Grundy numbers of all reachable states in a set and compute the MEX (minimum excludant)

5. If the initial state has a non-zero Grundy number, the first player (Fedor) has a winning strategy; otherwise Sergey wins

For the first example (5 5 5): The Grundy number is non-zero, so Fedor wins.
For the second example (3 1 2 3): The Grundy number is zero, so Sergey wins.
