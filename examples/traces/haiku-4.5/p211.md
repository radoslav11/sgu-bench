Looking at this problem, I need to:

1. Maintain N registers, each containing 0, 1, or 2
2. For each operation, add 2^i to the counter
3. Output the changes made, with at most 4 registers changed per operation

The key insight is that when we add 2^i:
- We increment register i by 1
- If a register becomes 3, we need to "carry" - set it to 0 and add 1 to the next register
- The constraint that at most 4 registers change suggests we need an efficient carrying strategy

Let me think about the carrying process:
- If register i becomes 3, we set it to 0 and carry 1 to register i+1
- If register i becomes 4 (from carry), we set it to 1 and carry 1 to register i+1 (since 4 = 2^2, we get 1*2 on position i and 1 on position i+1)

Actually, let me reconsider: each register can hold 0, 1, or 2. When we add 2^i to a register:
- If it was 0 → becomes 1
- If it was 1 → becomes 2
- If it was 2 → becomes 3, which is invalid, so we need to carry

When a register reaches 3:
- Set it to 0
- Add 1 to the next register

This is essentially base-3 addition with digits {0, 1, 2}.

Let me trace through the example:
- Start: [0,0,0,0,0]
- Add 2^0: register 0: 0→1. Output: "1 0 1" ✓
- Add 2^0: register 0: 1→2. Output: "1 0 2" ✓
- Add 2^0: register 0: 2→3, carry. Register 0→0, register 1: 0→1. Output: "2 0 0 1 1" but expected "2 0 1 1 1"

Wait, looking at the expected output "2 0 1 1 1" - that's 2 changes: register 0 becomes 1, register 1 becomes 1. That doesn't match my logic.

Let me re-read... Oh! When register 0 goes from 2 to 3, instead of setting it to 0 and carrying, we could set it to 1 and add 2 to the next register? No, that doesn't make sense either.

Actually, looking more carefully at the output format and the third operation:
- We're at [1, 1, 0, 0, 0] after 2 operations
- Adding 2^0 should give us [2, 1, 0, 0, 0], output "1 0 2"

Wait, let me recount: after first two additions of 2^0:
- [1, 0, 0, 0, 0]
- [2, 0, 0, 0, 0]

Third operation adds 2^0 again:
- We'd have [3, 0, 0, 0, 0]
- This carries: [0, 1, 0, 0, 0] but that means we changed 2 registers
- Expected output is "2 0 1 1 1" which means register 0 becomes 1, register 1 becomes 1

So it seems like [3,0,0,0,0] → [1,1,0,0,0]? That would mean 3 = 1 + 2*1, so register 0 becomes 1 and register 1 gets +1.

So the rule is: when adding 1 to a register that's already 2, instead of carrying a full 1, we adjust to keep each register ≤ 2. This is balanced ternary!

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, M;
    cin >> N >> M;
    
    vector<int> reg(N, 0);
    
    for(int op = 0; op < M; op++){
        int pos;
        cin >> pos;
        
        vector<pair<int,int>> changes;
        reg[pos]++;
        changes.push_back({pos, reg[pos]});
        
        // Handle carries
        for(int i = pos; i < N-1; i++){
            if(reg[i] >= 3){
                changes.push_back({i, reg[i] % 3});
                reg[i] %= 3;
                reg[i+1]++;
                if(i+1 < N) changes.push_back({i+1, reg[i+1]});
            }
        }
        
        // Remove duplicates and keep only last value for each register
        map<int, int> final_changes;
        for(auto [idx, val] : changes){
            final_changes[idx] = val;
        }
        
        cout << final_changes.size();
        for(auto [idx, val] : final_changes){
            cout << " " << idx << " " << val;
        }
        cout << "\n";
    }
    
    return 0;
}
```
