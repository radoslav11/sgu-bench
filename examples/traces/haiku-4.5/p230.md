Looking at this problem, I need to:

1. Assign N coins (with weights 1 to N grams) to N boxes such that:
   - Each coin type goes to exactly one box
   - Each box gets exactly one coin
   - All weighing constraints are satisfied (if box P is lighter than box Q, the coin in P must be lighter than the coin in Q)

2. A weighing constraint "P Q" means box P is lighter than box Q, so weight[P] < weight[Q]

This is essentially a topological sort problem with constraints, but we need to find a valid assignment of coin weights to boxes.

Approach:
1. Build a directed graph where an edge from P to Q means P < Q (P must have a lighter coin than Q)
2. Check if there's a cycle (if yes, no solution)
3. Do a topological sort to get the ordering
4. Assign the lightest available coins to boxes in topological order

Let me think differently:
- We have boxes 1 to N
- We need to assign coin values 1 to N to these boxes
- Constraint: if P < Q (in ordering), then coin_in_box[P] < coin_in_box[Q]

Better approach:
1. Build a directed graph where edge P→Q means box P must be lighter than box Q
2. Check for cycles using DFS (if cycle exists, no solution)
3. Use topological sort to order boxes
4. Assign coins 1,2,3,...,N to boxes in topological order

Let me verify with the example:
- N=3, M=2
- Constraints: box 2 < box 1, box 1 < box 3
- So: box 2 < box 1 < box 3
- Topological order: 2, 1, 3
- Assign: box 2 gets coin 1, box 1 gets coin 2, box 3 gets coin 3
- Output: coin in box 1 = 2, coin in box 2 = 1, coin in box 3 = 3
- Answer: "2 1 3" ✓

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, M;
    cin >> N >> M;
    
    vector<vector<int>> adj(N + 1);
    vector<int> indegree(N + 1, 0);
    
    for(int i = 0; i < M; i++){
        int P, Q;
        cin >> P >> Q;
        // P is lighter than Q, so P comes before Q in topological order
        adj[P].push_back(Q);
        indegree[Q]++;
    }
    
    // Topological sort using Kahn's algorithm
    queue<int> q;
    for(int i = 1; i <= N; i++){
        if(indegree[i] == 0){
            q.push(i);
        }
    }
    
    vector<int> topo_order;
    while(!q.empty()){
        int u = q.front();
        q.pop();
        topo_order.push_back(u);
        
        for(int v : adj[u]){
            indegree[v]--;
            if(indegree[v] == 0){
                q.push(v);
            }
        }
    }
    
    // Check if there's a cycle
    if(topo_order.size() != N){
        cout << "No solution\n";
        return 0;
    }
    
    // Assign coins to boxes based on topological order
    vector<int> result(N + 1);
    for(int i = 0; i < N; i++){
        result[topo_order[i]] = i + 1;
    }
    
    // Output
    for(int i = 1; i <= N; i++){
        if(i > 1) cout << " ";
        cout << result[i];
    }
    cout << "\n";
    
    return 0;
}
```

The solution works as follows:
1. Build a directed graph where edge P→Q means box P must contain a lighter coin than box Q
2. Use Kahn's algorithm for topological sorting to detect cycles
3. If we can't sort all N nodes topologically, there's a cycle → "No solution"
4. Assign coin values 1, 2, ..., N to boxes in topological order
5. Output the assignment for each box

Time complexity: O(N + M)
