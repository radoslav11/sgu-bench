Looking at this problem, I need to find the minimum time for Red to travel from the castle to the village.

Key observations:
1. Red travels along a polyline defined by n points
2. He can walk on the surface with speed `vw`
3. He can tunnel horizontally through mountains with speed `vc`
4. We need to find the minimum time

The path consists of line segments between consecutive points. For each segment, Red can choose to:
- Walk along the surface (following the segment)
- Tunnel horizontally through the mountain

Let me think about what "tunneling horizontally through a mountain" means. If Red is at point i and wants to reach point i+1:
- Walking along the surface means traveling the actual distance at speed vw
- Tunneling horizontally means moving horizontally (change in x) and let the mountain elevation change happen without additional cost (or covered by horizontal tunneling), traveling at speed vc

Actually, re-reading: "move strictly inside the mountain with speed vc" and "tunnel through a mountain horizontally" suggests:
- He can move horizontally at speed vc (saving time on elevation changes)
- The horizontal distance should be covered at speed vc
- The straight-line distance should be at speed vw

For a segment from (x_i, y_i) to (x_{i+1}, y_{i+1}):
- Walking the full straight line: distance = sqrt((x_{i+1}-x_i)^2 + (y_{i+1}-y_i)^2), time = distance/vw
- Tunneling horizontally: horizontal distance = |x_{i+1}-x_i|, time = |x_{i+1}-x_i|/vc, but we still need to account for vertical movement somehow...

Let me reconsider: If Red tunnels horizontally, he covers the horizontal distance at speed vc. The vertical component... maybe he climbs/descends at no additional cost? Or perhaps the tunnel is at a constant altitude?

Looking at the examples more carefully:
- Example 1: (0,0) to (50,50) to (100,0), vw=2, vc=1, answer=70.71
- Straight line distance from (0,0) to (100,0) = 100, time at vw=2 is 50
- But answer is 70.71 ≈ 100/√2

Let me try another interpretation: when tunneling, Red moves horizontally at speed vc, and the vertical changes are "free" (part of the mountain traversal). So for each segment, the time is max(horizontal_distance/vc, straight_line_distance/vw)?

Let me verify with example 1:
- Segment 1: (0,0) to (50,50): h_dist=50, v_dist=50, line_dist=70.71
  - Option 1: walk = 70.71/2 = 35.35
  - Option 2: tunnel = 50/1 = 50
  - Min = 35.35
- Segment 2: (50,50) to (100,0): h_dist=50, v_dist=50, line_dist=70.71
  - Option 1: walk = 70.71/2 = 35.35
  - Option 2: tunnel = 50/1 = 50
  - Min = 35.35
- Total = 70.70 ✓

This matches! So for each segment, we choose minimum of:
- Walking the segment at vw
- Tunneling horizontally at vc (covering just the x-distance)

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    double vw, vc;
    cin >> n >> vw >> vc;
    
    vector<double> x(n), y(n);
    for (int i = 0; i < n; i++) {
        cin >> x[i] >> y[i];
    }
    
    double total_time = 0.0;
    
    for (int i = 0; i < n - 1; i++) {
        double dx = x[i + 1] - x[i];
        double dy = y[i + 1] - y[i];
        
        // Distance along the surface (straight line)
        double line_distance = sqrt(dx * dx + dy * dy);
        double walk_time = line_distance / vw;
        
        // Horizontal tunneling
        double tunnel_time = fabs(dx) / vc;
        
        // Choose the minimum time for this segment
        total_time += min(walk_time, tunnel_time);
    }
    
    cout << fixed << setprecision(6) << total_time << endl;
    
    return 0;
}
```
