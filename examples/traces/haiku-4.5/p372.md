1. Select K tea types from N available types (each type used at most once)
2. Minimize total cost
3. Ensure no three consecutive rounds use the same kind (black/green)
4. All served tea types must be different

- We need K tea types total
- Each type is used exactly once
- No three consecutive teas can be of the same kind (0, 0, 0 or 1, 1, 1 not allowed)
- We want minimum cost

The key insight is that we need to alternate or vary between black (1) and green (0) tea such that we never have three consecutive of the same kind.

Valid patterns avoid: ...0,0,0,... and ...1,1,1,...

- We can use a greedy approach with dynamic programming or backtracking
