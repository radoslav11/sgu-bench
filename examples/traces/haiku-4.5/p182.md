1. Parse a boolean expression with proper operator precedence
2. Remove all brackets by applying distributive laws
3. Output the simplified expression without brackets

Key observations:
- Negation (!) has highest priority
- Conjunction (&) has middle priority
- Disjunction (||), implication (=>), equality (<=>), XOR (#) have lowest priority
- Need to apply distributive law: `a&(b||c) = a&b||a&c`

1. Parse the expression into an AST (Abstract Syntax Tree)
