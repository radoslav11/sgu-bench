Looking at this problem, I need to:

1. Read N (number of petals) and M (number of phrases)
2. Read M phrases
3. Simulate picking petals - for each petal, we say the next phrase in cyclic order
4. After N petals, output the phrase we just said

The key insight is that if we have M phrases and N petals, the phrase we end up with is the one at position (N-1) % M in our 0-indexed phrase list.

Let me trace through the examples:

Example 1:
- N=6, M=4
- Phrases: "loves" (0), "doesnt" (1), "sincerely" (2), "doubts" (3)
- Petals: 1→loves, 2→doesnt, 3→sincerely, 4→doubts, 5→loves, 6→doesnt
- Position: (6-1) % 4 = 5 % 4 = 1 → "doesnt" ✓

Example 2:
- N=9, M=3
- Phrases: "loves" (0), "doesnt" (1), "hates" (2)
- Petals: 1→loves, 2→doesnt, 3→hates, 4→loves, 5→doesnt, 6→hates, 7→loves, 8→doesnt, 9→hates
- Position: (9-1) % 3 = 8 % 3 = 2 → "hates" ✓

```cpp
#include <iostream>
#include <vector>
#include <string>

using namespace std;

int main() {
    int N, M;
    cin >> N >> M;
    
    vector<string> phrases(M);
    for (int i = 0; i < M; i++) {
        cin >> phrases[i];
    }
    
    int resultIndex = (N - 1) % M;
    cout << phrases[resultIndex] << endl;
    
    return 0;
}
```
