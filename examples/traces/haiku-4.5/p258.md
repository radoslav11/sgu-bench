Looking at this problem, I need to:

1. Understand what "lucky" means: A number with 2N digits where the sum of first N digits equals the sum of last N digits.
2. Understand what "almost lucky" means: A number where changing exactly one digit makes it lucky.
3. Count almost lucky numbers in range [A, B].

Key observations:
- Lucky numbers have even number of digits (2N)
- For a number to be almost lucky, we need to be able to change one digit to make the left sum equal to the right sum.
- If a number has 2N digits with left sum L and right sum R:
  - If we change a digit in the left half from d to d', the new left sum is L - d + d'
  - If we change a digit in the right half from d to d', the new right sum is R - d + d'
  - We need L - d + d' = R (if changing left) or L = R - d + d' (if changing right)

Strategy:
- For each number in [A, B], check if it's almost lucky
- A number is almost lucky if:
  1. It has even digits (2N)
  2. We can change exactly one digit to make left_sum == right_sum
  3. The changed digit must be valid (0-9, and not leading zero)

```cpp
#include<bits/stdc++.h>
using namespace std;

bool isAlmostLucky(long long num) {
    string s = to_string(num);
    int len = s.length();
    
    // Must have even number of digits
    if (len % 2 != 0) return false;
    
    int n = len / 2;
    
    // Calculate sums
    int leftSum = 0, rightSum = 0;
    for (int i = 0; i < n; i++) {
        leftSum += s[i] - '0';
    }
    for (int i = n; i < len; i++) {
        rightSum += s[i] - '0';
    }
    
    // Try changing each digit
    for (int i = 0; i < len; i++) {
        int oldDigit = s[i] - '0';
        
        // Try all possible new digits
        for (int newDigit = 0; newDigit <= 9; newDigit++) {
            // Skip if no change
            if (newDigit == oldDigit) continue;
            
            // Skip leading zeros
            if (i == 0 && newDigit == 0) continue;
            
            int newLeftSum = leftSum;
            int newRightSum = rightSum;
            
            if (i < n) {
                // Changing left half
                newLeftSum = leftSum - oldDigit + newDigit;
            } else {
                // Changing right half
                newRightSum = rightSum - oldDigit + newDigit;
            }
            
            if (newLeftSum == newRightSum) {
                return true;
            }
        }
    }
    
    return false;
}

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    long long A, B;
    cin >> A >> B;
    
    long long count = 0;
    
    for (long long i = A; i <= B; i++) {
        if (isAlmostLucky(i)) {
            count++;
        }
    }
    
    cout << count << endl;
    
    return 0;
}
```

The solution:
1. For each number in [A, B], I check if it's almost lucky
2. A number is almost lucky if:
   - It has an even number of digits
   - By changing exactly one digit, we can make the left sum equal to the right sum
3. For each position, I try changing it to each digit (0-9)
4. I check if the new configuration makes the sums equal
5. I handle the leading zero constraint properly

For the example [1, 99]:
- All 2-digit numbers (10-99) have 2 digits
- We check each: if left digit != right digit, we can usually change one to make them equal
- This gives us 81 almost lucky numbers (all except the 9 lucky ones like 11, 22, ..., 99)
