Looking at this problem, I need to:

1. Given an (H-1) × (W-1) fingerprint matrix where each element is the sum of a 2×2 block in the original key
2. Reconstruct the original H × W key matrix of 0's and 1's
3. Output the key or "CORRUPT" if impossible

Key insight: This is a system of linear equations over the integers where each variable must be 0 or 1.

Let me denote the key matrix as A and fingerprint as B. We have:
- B[i][j] = A[i][j] + A[i+1][j] + A[i][j+1] + A[i+1][j+1]

Strategy:
1. Try all possible values for the first row (2^W possibilities, but W ≤ 300 is too large)
2. Actually, try all possible values for just the first row and first column
3. Once first row and column are fixed, we can determine all other values using the constraints

Better approach:
- Try all 2^W possibilities for the first row
- For each first row, try all 2^(H-1) possibilities for the first column
- Once first row and column are determined, calculate what the rest of the matrix must be
- Check if all calculated values are 0 or 1

Wait, that's still potentially large. Let me think differently:

- Fix the first row arbitrarily (try all 2^W values)
- Given the first row and a constraint equation B[0][j] = A[0][j] + A[1][j] + A[0][j+1] + A[1][j+1]
- We can express A[1][j] in terms of A[0][j], A[0][j+1] and known B[0][j]
- Continue this way to fill the entire matrix row by row
- Check if all values are valid (0 or 1)

Since W can be up to 300, trying all 2^W first rows is infeasible. But we can use a different approach:
- Try all 2^W possibilities only when W is small enough
- For larger W, we need a smarter method

Actually, re-reading: W ≤ 300 but trying 2^W is impossible. Let me think of this as a constraint satisfaction problem more carefully.

If we fix just the first element A[0][0], we can propagate through the matrix:
- From B[i][j] = A[i][j] + A[i+1][j] + A[i][j+1] + A[i+1][j+1], we get:
- A[i+1][j+1] = B[i][j] - A[i][j] - A[i+1][j] - A[i][j+1]

So if we fix the first row, we can compute everything else row by row, and just need to check validity.

Let me try all 2^W possibilities for the first row (only feasible if W is small, but let's see):

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
  int H, W;
  cin >> H >> W;
  
  vector<string> fingerprint(H-1);
  for(int i = 0; i < H-1; i++){
    cin >> fingerprint[i];
  }
  
  // Try all possible first rows
  for(int mask = 0; mask < (1 << W); mask++){
    vector<vector<int>> key(H, vector<int>(W));
    
    // Set first row based on mask
    for(int j = 0; j < W; j++){
      key[0][j] = (mask >> j) & 1;
    }
    
    bool valid = true;
    
    // Fill the rest of the matrix
    for(int i = 0; i < H-1 && valid; i++){
      for(int j = 0; j < W-1 && valid; j++){
        // B[i][j] = A[i][j] + A[i+1][j] + A[i][j+1] + A[i+1][j+1]
        // A[i+1][j+1] = B[i][j] - A[i][j] - A[i+1][j] - A[i][j+1]
        int needed = (fingerprint[i][j] - '0') - key[i][j] - key[i+1][j] - key[i][j+1];
        if(needed < 0 || needed > 1){
          valid = false;
        } else {
          key[i+1][j+1] = needed;
        }
      }
    }
    
    if(valid){
      for(int i = 0; i < H; i++){
        for(int j = 0; j < W; j++){
          cout << key[i][j];
        }
        cout << "\n";
      }
      return 0;
    }
  }
  
  cout << "CORRUPT\n";
  return 0;
}
```
