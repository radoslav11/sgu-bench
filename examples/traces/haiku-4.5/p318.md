Looking at this problem, I need to understand what's being asked:

1. We have N resources and M users
2. Each user i wants access to a specific set of resources Si
3. We need to create user groups where:
   - Each resource belongs to at most one group
   - A user in a group gets access to all resources in that group
   - Each user must have access to exactly their desired resources (no more, no less)
   - Minimize the number of groups

Key insight: Two users can be in the same group if and only if they want the exact same set of resources. Why? Because if user A wants resources {1,3} and user B wants resources {3,4,5}, we can't put them in the same group - they'd both get access to the intersection and union of their desired resources.

So users can only be in the same group if they want the same exact set of resources.

But resource 3 is in both groups - that violates rule 1!

So we need to partition the resources. Each resource can only be in one group. But each user needs access to all their desired resources.

This is more complex. We need to find a partition of resources into groups such that:
- Each group contains some subset of resources
- For each user, there exists a subset of groups they belong to that covers exactly their desired resources

This is actually a set cover variant. We want to find the minimum number of groups (subsets of resources) such that each user's desired resources can be expressed as a union of groups they belong to, and each resource appears in exactly one group.

Actually, I think the key insight is: we need to find groups such that no two groups share resources, and for each user, the union of their groups' resources equals their desired resources.

The minimal solution is to group resources that must always appear together. We can use a union-find or graph coloring approach based on which resources appear together in user requirements.

Let me think of it as a graph problem: resources are nodes. If two resources appear in the same user's requirement, they could potentially be in the same group. We want to partition resources into groups minimizing the number of groups while satisfying user constraints.

After further thought: We need to find the minimum number of sets such that each user's requirement is a union of some of these sets. This is equivalent to finding a minimal base of the user requirement sets.

#include<bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, M;
    cin >> N >> M;
    
    vector<set<int>> users(M);
    set<set<int>> uniqueSets;
    
    for(int i = 0; i < M; i++){
        int size;
        cin >> size;
        for(int j = 0; j < size; j++){
            int resource;
            cin >> resource;
            users[i].insert(resource);
        }
        uniqueSets.insert(users[i]);
    }
    
    // Greedy: start with individual resources as groups
    // Try to merge groups that don't violate constraints
    vector<set<int>> groups;
    for(int i = 1; i <= N; i++){
        set<int> g;
        g.insert(i);
        groups.push_back(g);
    }
    
    // Try to merge groups while maintaining validity
    bool changed = true;
    while(changed){
        changed = false;
        for(int i = 0; i < groups.size() && !changed; i++){
            for(int j = i+1; j < groups.size() && !changed; j++){
                // Try merging i and j
                set<int> merged = groups[i];
                merged.insert(groups[j].begin(), groups[j].end());
                
                // Check if this merge is valid
                bool valid = true;
                for(auto& user : users){
                    int countMatching = 0;
                    for(int g = 0; g < groups.size(); g++){
                        if(g != i && g != j){
                            set<int> intersection;
                            set_intersection(user.begin(), user.end(), groups[g].begin(), groups[g].end(), inserter(intersection, intersection.begin()));
                            if(intersection.size() > 0) countMatching++;
                        }
                    }
                    if(countMatching > 0){
                        valid = false;
                        break;
                    }
                }
                
                if(valid){
                    groups[i] = merged;
                    groups.erase(groups.begin() + j);
                    changed = true;
                }
            }
        }
    }
    
    cout << groups.size() << endl;
    
    return 0;
}
