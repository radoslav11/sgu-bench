Looking at this problem, I need to:

1. Understand Huffman encoding: Build a tree by repeatedly combining the two smallest frequency nodes until one tree remains
2. Calculate the total encoded length: For each character, multiply its frequency by its depth in the tree
3. The total bits = sum of (frequency[i] × depth[i]) for all characters

The key insight is that instead of building the actual tree and computing depths, I can use the fact that **every time we merge two nodes, we add their combined frequency to the total cost**. This is because in Huffman coding, each merge operation represents combining two subtrees, and the nodes being merged will each have their depth increased by 1 when they become children of the new parent node.

Let me trace through the example:
- Frequencies: [1, 1, 4]
- Step 1: Merge two nodes with frequency 1 and 1 → creates node with frequency 2. Cost += 1+1 = 2
- Step 2: Merge node with frequency 2 and node with frequency 4 → creates node with frequency 6. Cost += 2+4 = 6
- Total cost: 2 + 6 = 8 ✓

The algorithm:
1. Use a min-heap (priority queue) to store all active nodes
2. Repeatedly extract the two minimum elements, merge them (add their frequencies)
3. Add the merged frequency back to the heap
4. Track the total cost by adding each merged frequency to a running sum

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    cin >> n;
    
    priority_queue<long long, vector<long long>, greater<long long>> pq;
    
    for (int i = 0; i < n; i++) {
        long long p;
        cin >> p;
        pq.push(p);
    }
    
    long long totalCost = 0;
    
    while (pq.size() > 1) {
        long long first = pq.top();
        pq.pop();
        long long second = pq.top();
        pq.pop();
        
        long long merged = first + second;
        totalCost += merged;
        pq.push(merged);
    }
    
    cout << totalCost << endl;
    
    return 0;
}
```

The solution works because:
- When we merge two nodes with frequencies f1 and f2, every character in those subtrees now has a code that's 1 bit longer
- So the cost of merging is f1 + f2 (that's how many extra bits we need)
- By doing this greedily (always merging the smallest), we minimize total cost
- The sum of all merge costs equals the total bits needed

Time complexity: O(n log n) due to heap operations
Space complexity: O(n) for the heap
