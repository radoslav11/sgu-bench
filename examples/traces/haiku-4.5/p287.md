Looking at this problem, I need to understand the Qc Machine's behavior:

1. The machine selects a number from 1 to q
2. When you ask the i-th question, you get the answer to the (i-c)-th question
3. We need to find the minimum number of questions needed in the worst case

Key observations:
- For c=0 (no delay), this is standard binary search: we need ceil(log2(q)) + 1 questions
- For c>0, the first c questions get no answer, so we're "blind" for the first c steps
- After c questions, we start getting answers (delayed by c steps)
- We need to maintain c "active" ranges that we're narrowing down simultaneously

Let me think about the strategy:
- For the first c questions, we get no feedback, so we're essentially blind
- Starting from question c+1, each question gets an answer about an earlier question
- In the worst case, we need to handle the scenario where answers don't help us narrow down as much as possible

The key insight is that:
- During the first c steps, we can't use binary search effectively since we get no feedback
- We need to think of this as managing c simultaneous binary searches
- Each question contributes to narrowing down one of the c ranges

This is similar to a problem where we need to find the worst-case number of queries needed when we can only get c results at a time.

#include<bits/stdc++.h>
using namespace std;

int main(){
    long long q, c;
    cin >> q >> c;
    
    if(c == 0) {
        // Standard binary search
        long long ans = 0;
        long long temp = q;
        while(temp > 0) {
            temp /= 2;
            ans++;
        }
        cout << ans + 1 << endl;
        return 0;
    }
    
    // For c > 0, we use dynamic programming/simulation
    // dp[i] = max q that can be solved in i steps with delay c
    vector<long long> dp(5000001, 0);
    dp[0] = 1;
    
    for(int i = 1; i <= 5000000; i++) {
        if(i <= c) {
            dp[i] = 1; // Can't get useful info in first c steps
        } else {
            // After c steps, we can use binary search on previous answer
            dp[i] = dp[i - c - 1] + dp[i - c] + 1;
            if(dp[i] > q) {
                cout << i << endl;
                return 0;
            }
        }
    }
    
    return 0;
}
