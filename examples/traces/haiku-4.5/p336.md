Looking at this problem, I need to:

1. Track which parties/blocks know information about which other parties/blocks
2. Handle two operations:
   - Query: Does party/block `a` know information about party/block `b`?
   - Merge: Combine two parties/blocks into a new one, merging their information

Key insights:
- Initially, we have N parties (numbered 1 to N)
- We're given initial information relationships
- When two entities merge, the new block:
  - Has all information that both original entities had
  - Becomes the target of information that anyone had about either original entity
- Need to track "who knows what about whom"

I'll use:
- `knows[a]` = set of entities that `a` knows something about
- `knownBy[b]` = set of entities that know something about `b`
- When merging `a` and `b` into new block `c`:
  - `knows[c]` = `knows[a]` ∪ `knows[b]` (remove `a` and `b` from these sets, add `c`)
  - For each entity that knew about `a` or `b`, update to know about `c`

Let me trace through the example:
- Initial: 1→2, 1→3, 3→2, 4→4
- Query `1 3 4`: Does 3 know about 4? NO (3 only knows about 2)
- Merge `2 3` into block `5`: 
  - Block 5 knows what 2 and 3 knew (nothing for 2, 2 for 3) → 5 knows {2}
  - Who knew about 2 or 3? (1 knew about both) → 1 now knows about 5
  - Who knew about 4? 4 itself
- Query `1 5 4`: Does 5 know about 4? NO
- Query `1 4 5`: Does 4 know about 5? YES (4 knew about 4, and 1 now knows about 5... wait)

Let me reconsider. When we merge 2 and 3:
- Everything that knew about 2 or 3 now knows about the new block (5)
- The new block knows everything that 2 or 3 knew

After merge of 2,3→5:

Wait, let me re-read. "All the information that was known by some parties or blocks about a and b now concerns the newly created block."

So if X knew about A or B, then X now knows about the new block.

Let me retrace:
- Merge 2,3 → block 5:
  - knows[5] = knows[2] ∪ knows[3] = {} ∪ {2} = {2}
  - Any entity X where X knew about 2 or 3: 1 knew about both, 4 knew about 4 (not 2 or 3)
  - Update: 1 now knows about 5 instead of 2,3
  - Update: knows[5] shouldn't include 2 anymore since 2 is merged

Actually, I think the new block's knowledge should NOT include the merged entities themselves.

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int N, M;
    cin >> N >> M;
    
    map<int, set<int>> knows;
    
    for(int i = 1; i <= N; i++){
        knows[i] = set<int>();
    }
    
    for(int i = 0; i < M; i++){
        int a, b;
        cin >> a >> b;
        knows[a].insert(b);
    }
    
    int Q;
    cin >> Q;
    
    int nextId = N + 1;
    
    for(int i = 0; i < Q; i++){
        int type;
        cin >> type;
        
        if(type == 1){
            int a, b;
            cin >> a >> b;
            if(knows[a].count(b)){
                cout << "YES\n";
            } else {
                cout << "NO\n";
            }
        } else {
            int a, b;
            cin >> a >> b;
            
            int c = nextId++;
            
            // New block c knows what a and b know
            knows[c] = knows[a];
            for(int x : knows[b]){
                knows[c].insert(x);
            }
            
            // Remove a and b from c's knowledge
            knows[c].erase(a);
            knows[c].erase(b);
            
            // Update all entities that knew about a or b
            for(auto& p : knows){
                if(p.first != a && p.first != b){
                    if(p.second.count(a) || p.second.count(b)){
                        p.second.erase(a);
                        p.second.erase(b);
                        p.second.insert(c);
                    }
                }
            }
        }
    }
    
    return 0;
}
```
