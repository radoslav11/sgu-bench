Looking at this problem, I need to:

1. Understand what "going through" means: two adjacent buons of different types can swap places
2. Find the minimum number of swaps to transform the first chain into the second chain

Key observations:
- We can only swap adjacent buons of different types
- This is similar to counting inversions, but with a constraint
- First, I need to check if transformation is possible at all
- If the count of '+' and '-' are different, it's impossible
- If counts match, I need to find the minimum swaps

The key insight is that we can think of this as a sorting problem where:
- We want to rearrange string A to match string B
- We can only swap adjacent elements of different types
- This means '+' and '-' can pass through each other

Let me think about the approach:
1. First check if both strings have the same counts of '+' and '-'
2. If yes, we need to count the minimum number of swaps needed
3. We can use a greedy approach: for each position in the target string, find the nearest matching character in the current arrangement and count swaps needed to bring it to that position

Here's my solution:

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    string s1, s2;
    cin >> s1 >> s2;
    
    // Check if same length
    if(s1.length() != s2.length()){
        cout << -1 << endl;
        return 0;
    }
    
    // Check if same character counts
    int count1_plus = 0, count2_plus = 0;
    for(char c : s1) if(c == '+') count1_plus++;
    for(char c : s2) if(c == '+') count2_plus++;
    
    if(count1_plus != count2_plus){
        cout << -1 << endl;
        return 0;
    }
    
    // Now find minimum swaps using greedy approach
    string current = s1;
    int swaps = 0;
    
    for(int i = 0; i < s2.length(); i++){
        if(current[i] == s2[i]){
            continue;
        }
        
        // Find the nearest character of type s2[i] starting from position i+1
        int j = i + 1;
        while(j < current.length() && current[j] != s2[i]){
            j++;
        }
        
        if(j == current.length()){
            // Should not happen if counts are equal
            cout << -1 << endl;
            return 0;
        }
        
        // Swap current[j] with current[i] by moving it left
        // Each swap with an adjacent different character costs 1
        while(j > i){
            swap(current[j], current[j-1]);
            j--;
            swaps++;
        }
    }
    
    cout << swaps << endl;
    return 0;
}
```

The algorithm:
1. Check if both strings have the same length
2. Check if both strings have the same number of '+' and '-' characters
3. Use a greedy approach: go through each position of target string from left to right
4. If current character at position i matches target character, continue
5. Otherwise, find the nearest matching character to the right and bubble it to position i by swapping with adjacent elements
6. Count all the swaps

- Start: `++-+--+`, Target: `-++--++`
