Looking at this problem, I need to find the minimal K (number of black beads) such that every necklace of 2N-1 beads with K black beads is "beautiful".

A necklace is beautiful if we can choose two black beads such that one of the two parts strictly between them contains exactly N beads.

Let me think about this:
- Total beads: 2N-1
- We need to find positions of two black beads such that the arc between them (one of the two arcs) has exactly N beads.
- Since it's a necklace (circular), if we pick two beads at distance d apart, one arc has d-1 beads and the other has (2N-1)-d beads.
- We need one arc to have exactly N beads, so: d-1 = N or (2N-1)-d = N
  - d-1 = N means d = N+1
  - (2N-1)-d = N means d = N-1

So we need two black beads that are exactly N+1 apart (or equivalently N-1 apart, which is the same on a circle of size 2N-1).

The question becomes: what's the minimum K such that ANY arrangement of K black beads on a circle of 2N-1 beads must have at least two black beads at distance N+1 (or N-1) from each other?

By pigeonhole principle, if we want to guarantee that we CANNOT avoid having two beads at distance N+1:
- Consider the positions modulo (N+1). We have N+1 residue classes (0, 1, ..., N).
- If we have more than (N-1) black beads, by pigeonhole principle, at least two must be in the same residue class modulo (N+1), meaning they're at distance that's a multiple of N+1.
- The distance must be N+1, 2(N+1), etc. Since our circle has size 2N-1 < 2(N+1), the only possibility is N+1.

Wait, let me reconsider. On a circle of 2N-1 beads:
- Distance N+1 means one arc has N beads between them
- We can have at most N positions that are each N+1 apart: positions 0, N+1, 2(N+1), ...
- Since 2(N+1) = 2N+2 > 2N-1, we can fit at most 2 such positions? Let me check: position 0 and position N+1 are N+1 apart. Position N+1 and position 2(N+1)=2N+2 ≡ 3 (mod 2N-1), so that works too but wraps.

Let me think differently using gaps. If we place K black beads, there are K gaps between consecutive black beads. The sum of gap sizes is 2N-1-K (the white beads).

For no beautiful necklace, we need all K gaps to avoid the size that would create a segment of N beads. If the gap between two black beads is of size g, then one side has g beads and the other has 2N-1-g-2 = 2N-3-g beads (excluding the two black beads).

We need: g ≠ N and 2N-3-g ≠ N, which means g ≠ N and g ≠ N-3.

The maximum K where we can arrange beads to avoid having two black beads with N beads between them:
- We need each gap to be either < N-3, or between N-2 and N-1, or > N.
- Actually simpler: gaps should be ≠ N-1 and ≠ N.

If all K gaps are in valid ranges, and sum to 2N-1-K:
- Maximum K when all gaps are of size 1 or N+1 or larger.
- If we use gaps of size 1, we get K gaps of size 1, summing to K, needing K = 2N-1-K, so K = N.

Let me verify with examples:
- N=3 (total=5): Answer should be 3. My reasoning: K=3 means we can't avoid it.
- N=4 (total=7): Answer should be 4.

Pattern: K = N+1? Let's check: N=3 gives 4, but answer is 3. N=4 gives 5, but answer is 4.

So K = N seems to be the answer!
