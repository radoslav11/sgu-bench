Let me reconsider: A simpler approach is:
- Do a DFS traversal
- On the DFS tree, assign nodes to countries alternately (parent to country 1, child to country 2, grandchild to country 1, etc.)
- This creates a bipartition where edges between countries can be handled

Actually, the easiest: since it's a tree, we can find a bipartition by coloring based on distance from a root:

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int tst;
    cin >> tst;
    
    while(tst--) {
        int n, m;
        cin >> n >> m;
        
        vector<vector<int>> adj(n + 1);
        
        for(int i = 0; i < m; i++) {
            int a, b;
            cin >> a >> b;
            adj[a].push_back(b);
            adj[b].push_back(a);
        }
        
        vector<int> country(n + 1, 0);
        queue<int> q;
        
        // BFS coloring based on depth - alternating between country 1 and 2
        q.push(1);
        country[1] = 1;
        
        while(!q.empty()) {
            int u = q.front();
            q.pop();
            
            for(int v : adj[u]) {
                if(country[v] == 0) {
                    country[v] = 3 - country[u]; // Toggle between 1 and 2
                    q.push(v);
                }
            }
        }
        
        for(int i = 1; i <= n; i++) {
            cout << country[i];
            if(i < n) cout << " ";
        }
        cout << "\n";
    }
    
    return 0;
}
```

This solution:
1. Reads the graph
2. Does a BFS from node 1
3. Colors nodes alternately: node 1 gets country 1, its neighbors get country 2, their unvisited neighbors get country 1, etc.
4. Since the graph is connected and is a tree, this creates two connected components (each being a tree)
5. Since n is even and we alternate, we get equal or near-equal split

This works because in a tree, a bipartition ensures each part remains connected.
