Looking at this problem, I need to:

1. Maintain a collection of segments on a real axis
2. For `+ L R` queries: add a segment and count how many existing segments are completely inside this new segment
3. For `- L R` queries: remove a segment if it exists

A segment [a,b] is inside segment [L,R] if `L <= a` and `b <= R` (non-strictly inside).

Let me trace through the example:
- `+ 1 2`: Add segment [1,2]. No segments exist yet, so output 0.
- `+ 1 2`: Add segment [1,2]. Check how many of {[1,2]} are inside [1,2]. The segment [1,2] is inside [1,2], so output 1.
- `+ 0 3`: Add segment [0,3]. Check how many of {[1,2], [1,2]} are inside [0,3]. Both segments [1,2] are inside, so output 2.
- `- 1 2`: Remove one segment [1,2]. Now we have {[1,2], [0,3]}.
- `+ 1 2`: Add segment [1,2]. Check how many of {[1,2], [0,3]} are inside [1,2]. Only [1,2] is inside (since [0,3] has left endpoint 0 < 1), so output 1.

The approach:
- Use a multiset or vector to store segments as pairs
- For each `+` query, count segments where `left >= L` and `right <= R`, then add the new segment
- For each `-` query, remove one occurrence of the segment

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    multiset<pair<int,int>> segments;
    string op;
    
    while(cin >> op){
        int L, R;
        cin >> L >> R;
        
        if(op == "+"){
            // Count segments inside [L, R]
            int count = 0;
            for(auto& seg : segments){
                // A segment [a,b] is inside [L,R] if L <= a and b <= R
                if(L <= seg.first && seg.second <= R){
                    count++;
                }
            }
            cout << count << "\n";
            segments.insert({L, R});
        }
        else{ // op == "-"
            // Remove one occurrence of segment [L, R]
            auto it = segments.find({L, R});
            if(it != segments.end()){
                segments.erase(it);
            }
        }
    }
    
    return 0;
}
```

The solution:
1. Uses a `multiset<pair<int,int>>` to store segments, which allows duplicate segments and keeps them ordered
2. For `+` queries: iterate through all existing segments and count those where the left endpoint is >= L and right endpoint is <= R, then insert the new segment
3. For `-` queries: find and remove one occurrence of the segment if it exists
4. Time complexity per query is O(n) where n is the number of current segments (at most 1000), which is acceptable given the constraints
