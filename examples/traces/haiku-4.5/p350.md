Looking at this problem, I need to reconstruct set A from set B, where B contains XORs of all pairs from A.

Let me think about the mathematics:
- If A = {a₁, a₂, ..., aₙ}, then B contains all aᵢ ⊕ aⱼ for i < j
- We have M = n(n-1)/2, so I can find n from M
- Key insight: If I pick any three elements from B that correspond to pairs (a₁,a₂), (a₁,a₃), (a₂,a₃), then:
  - b₁₂ = a₁ ⊕ a₂
  - b₁₃ = a₁ ⊕ a₃  
  - b₂₃ = a₂ ⊕ a₃
  
  And we have: b₁₂ ⊕ b₁₃ ⊕ b₂₃ = (a₁ ⊕ a₂) ⊕ (a₁ ⊕ a₃) ⊕ (a₂ ⊕ a₃) = 0

This gives us a constraint! Also:
- a₁ ⊕ a₂ ⊕ a₁ ⊕ a₃ = a₂ ⊕ a₃, so b₁₂ ⊕ b₁₃ = b₂₃

Strategy:
1. Calculate n from M
2. Try different approaches to recover A:
   - Assume one element of A is 0 (we can always XOR all elements to shift them)
   - If a₁ = 0, then B contains {a₂, a₃, ..., aₙ, a₂⊕a₃, a₂⊕a₄, ...}
   - So the first n-1 smallest elements of B might be a₂, a₃, ..., aₙ
   
Let me implement a solution that tries to reconstruct A:

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int M;
    cin >> M;
    
    vector<long long> B(M);
    for(int i = 0; i < M; i++){
        cin >> B[i];
    }
    
    // Find n from M = n(n-1)/2
    int n = 1;
    while(n * (n - 1) / 2 < M) n++;
    
    // Try setting A[0] = 0
    // Then B should contain A[1], A[2], ..., A[n-1] as some subset
    // and all XORs of pairs from those
    
    vector<long long> A;
    A.push_back(0);
    
    set<long long> remaining(B.begin(), B.end());
    
    // Try to greedily pick elements
    // For each potential A[i], check if all XORs with previous elements exist in B
    for(int i = 1; i < n; i++){
        for(long long candidate : remaining){
            // Check if this candidate works
            bool valid = true;
            vector<long long> new_xors;
            
            for(int j = 0; j < i; j++){
                long long xor_val = A[j] ^ candidate;
                new_xors.push_back(xor_val);
                if(remaining.find(xor_val) == remaining.end()){
                    valid = false;
                    break;
                }
            }
            
            if(valid){
                A.push_back(candidate);
                remaining.erase(candidate);
                for(long long x : new_xors){
                    remaining.erase(x);
                }
                break;
            }
        }
    }
    
    // Output the result
    for(int i = 0; i < n; i++){
        if(i > 0) cout << " ";
        cout << A[i];
    }
    cout << "\n";
    
    return 0;
}
```

The algorithm:
1. Calculate n from M using the formula M = n(n-1)/2
2. Start with A[0] = 0 (without loss of generality)
3. For each subsequent element, try candidates from B
4. For each candidate, verify that all XORs with previously selected elements exist in B
5. If valid, add the candidate to A and remove it plus the generated XORs from B
6. Continue until we have n elements

This greedy approach should work because we're guaranteed a solution exists.
