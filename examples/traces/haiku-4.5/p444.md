Looking at this problem, I need to:

1. Perform long division of x/y
2. Track the remainders we've seen during the division process
3. Detect when we enter a cycle (when we see a remainder we've seen before)
4. Report:
   - How many digits come before the cycle starts
   - How long the cycle is (0 if the division terminates)

Let me trace through an example:
- 1/3: 1÷3 = 0.333...
  - Remainder 1 → multiply by 10 → 10÷3 = 3 remainder 1
  - We see remainder 1 again, so cycle starts at position 0 with length 1

- 1/6: 1÷6 = 0.1666...
  - Remainder 1 → 10÷6 = 1 remainder 4
  - Remainder 4 → 40÷6 = 6 remainder 4
  - We see remainder 4 again, so cycle starts at position 1 with length 1

- 5/4: 5÷4 = 1.25
  - Integer part: 1, remainder 1
  - Remainder 1 → 10÷4 = 2 remainder 2
  - Remainder 2 → 20÷4 = 5 remainder 0
  - Remainder 0 means termination, 2 digits after decimal, no cycle

The algorithm:
1. First handle the integer part (x/y)
2. Start with remainder = x % y
3. Use a map to track: remainder → position where it first appeared
4. Keep computing digits by multiplying remainder by 10 and dividing by y
5. If remainder becomes 0, we have a terminating decimal
6. If we see a remainder we've seen before, we found the cycle

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    long long x, y;
    cin >> x >> y;
    
    // Map to store: remainder -> position in decimal digits
    map<long long, long long> remainderPos;
    
    long long remainder = x % y;
    long long position = 0;
    
    // If the division is exact, answer is 0 0
    if(remainder == 0){
        cout << "0 0" << endl;
        return 0;
    }
    
    // Perform long division
    while(remainder != 0){
        // If we've seen this remainder before, we found a cycle
        if(remainderPos.find(remainder) != remainderPos.end()){
            long long cycleStart = remainderPos[remainder];
            long long cycleLength = position - cycleStart;
            cout << cycleStart << " " << cycleLength << endl;
            return 0;
        }
        
        // Record this remainder's position
        remainderPos[remainder] = position;
        
        // Continue long division
        remainder = (remainder * 10) % y;
        position++;
    }
    
    // If remainder becomes 0, it's a terminating decimal
    cout << position << " 0" << endl;
    
    return 0;
}
```

Let me verify with the examples:

1. 1/3: remainder=1, pos=0. remainderPos[1]=0. remainder=(1*10)%3=1, pos=1. Found 1 at pos 0, cycle_start=0, cycle_len=1-0=1. ✓

2. 1/6: remainder=1, pos=0. remainderPos[1]=0. remainder=(1*10)%6=4, pos=1. remainder=(4*10)%6=4, pos=2. Found 4 at pos 1, cycle_start=1, cycle_len=2-1=1. ✓

3. 5/3: remainder=2, pos=0. remainderPos[2]=0. remainder=(2*10)%3=2, pos=1. Found 2 at pos 0, cycle_start=0, cycle_len=1-0=1. ✓

4. 5/4: remainder=1, pos=0. remainderPos[1]=0. remainder=(1*10)%4=2, pos=1. remainder=(2*10)%4=0, pos=2. Remainder=0, output 2 0. ✓

5. 200/2: 200%2=0, output 0 0. ✓
