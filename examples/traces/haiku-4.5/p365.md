Looking at this problem, I need to find how many S-digit numbers have digits that:
1. First go in non-descending order (can stay same or increase)
2. Then go in non-ascending order (can stay same or decrease)

The transition point between non-descending and non-ascending can happen anywhere (including at the start or end).

Let me think about this differently using dynamic programming or combinatorics.

For a valid ticket number with S digits:
- We pick digits d₁, d₂, ..., dₛ where 0 ≤ dᵢ ≤ 9
- d₁ ≤ d₂ ≤ ... ≤ dₖ ≥ dₖ₊₁ ≥ ... ≥ dₛ for some k (where k can be 0 to S)

Let me verify with the example:
- S = 1: Any single digit (0-9) is valid = 10 ✓
- S = 3: We need sequences like a ≤ b ≥ c where a,b,c are digits

For S=3, let me think of it as:
- Peak at position 0: b ≥ c (first part empty) - this is non-increasing sequences of length 3
- Peak at position 1: a ≤ b ≥ c - this has a transition
- Peak at position 2: a ≤ b ≤ c (second part empty) - this is non-decreasing sequences of length 3

The key insight: a "dromedary" ticket is a sequence that first increases (non-strictly) then decreases (non-strictly). This is equivalent to choosing:
- Which position is the peak (or plateau)
- What digit sequence

Actually, I can think of this combinatorially:
- For each possible "peak value" sequence, count valid arrangements
- Or use DP: dp[position][last_digit][phase] where phase indicates if we're in ascending or descending part

Let me use DP:
- dp[i][d][phase] = number of ways to fill first i digits, where last digit is d, and phase is 0 (ascending) or 1 (descending)
- From phase 0 (ascending), we can either stay in phase 0 or move to phase 1
- In phase 0: next digit must be ≥ d
- In phase 1: next digit must be ≤ d

```cpp
#include<bits/stdc++++.h>
using namespace std;

int main(){
    int S;
    cin >> S;
    
    // dp[position][last_digit][phase]
    // phase: 0 = ascending, 1 = descending
    long long dp[21][10][2];
    memset(dp, 0, sizeof(dp));
    
    // Base case: first digit can be 0-9
    // Start in ascending phase
    for(int d = 0; d < 10; d++){
        dp[1][d][0] = 1;
    }
    
    for(int i = 1; i < S; i++){
        for(int d = 0; d < 10; d++){
            // From ascending phase with last digit d
            if(dp[i][d][0] > 0){
                // Stay in ascending: next digit must be >= d
                for(int nd = d; nd < 10; nd++){
                    dp[i+1][nd][0] += dp[i][d][0];
                }
                // Move to descending: next digit can be anything <= d
                for(int nd = 0; nd <= d; nd++){
                    dp[i+1][nd][1] += dp[i][d][0];
                }
            }
            
            // From descending phase with last digit d
            if(dp[i][d][1] > 0){
                // Stay in descending: next digit must be <= d
                for(int nd = 0; nd <= d; nd++){
                    dp[i+1][nd][1] += dp[i][d][1];
                }
            }
        }
    }
    
    long long answer = 0;
    for(int d = 0; d < 10; d++){
        answer += dp[S][d][0];
        answer += dp[S][d][1];
    }
    
    cout << answer << endl;
    
    return 0;
}
```

Actually, the formulation seems right - we can transition at any point, and when we do, the digit at transition must work for both phases.

Let me trace through S=1: dp[1][0][0]=1, ..., dp[1][9][0]=1, so answer=10. ✓

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    int S;
    cin >> S;
    
    long long dp[21][10][2];
    memset(dp, 0, sizeof(dp));
    
    for(int d = 0; d < 10; d++){
        dp[1][d][0] = 1;
    }
    
    for(int i = 1; i < S; i++){
        for(int d = 0; d < 10; d++){
            if(dp[i][d][0] > 0){
                for(int nd = d; nd < 10; nd++){
                    dp[i+1][nd][0] += dp[i][d][0];
                }
                for(int nd = 0; nd <= d; nd++){
                    dp[i+1][nd][1] += dp[i][d][0];
                }
            }
            if(dp[i][d][1] > 0){
                for(int nd = 0; nd <= d; nd++){
                    dp[i+1][nd][1] += dp[i][d][1];
                }
            }
        }
    }
    
    long long answer = 0;
    for(int d = 0; d < 10; d++){
        answer += dp[S][d][0] + dp[S][d][1];
    }
    
    cout << answer << endl;
    
    return 0;
}
```
