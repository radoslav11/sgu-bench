Looking at this problem, I need to find a function f that maps values to [0, 50] such that:
- For the control matrix T to remain a control matrix after applying f
- The first row must remain different from all other rows after applying f
- Specifically, for each column j: f(T[0][j]) ≠ f(T[i][j]) for all i > 0

This is essentially a graph coloring problem where:
- Each unique value in the matrix is a node
- For each column j and each row i > 0, if T[0][j] == T[i][j], we have a conflict (they must map to different values)
- We need to color this graph with at most 51 colors (0-50)

Let me think about the approach:
1. Extract all unique values from the matrix
2. Build a conflict graph where edges connect values that must have different f values
3. Try to 2-color the graph if possible (using 2 colors is sufficient for bipartite graphs, and we have 51 colors available)
4. If not bipartite, use a greedy coloring approach with up to 51 colors

Actually, let me reconsider. Since we need f(T[0][j]) ≠ f(T[i][j]) for all i > 0 and all j, we can think of it as:
- For each value v that appears in the first row, it must map to something different from what values in other rows map to (when they're in the same column)

Let me use a graph-based approach:
- Create edges between values that appear in the same column in positions (0, i) where i > 0
- These values must have different function values
- Try to color this conflict graph with colors 0-50

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, m;
    cin >> n >> m;
    
    vector<vector<int>> T(n, vector<int>(m));
    set<int> unique_vals;
    
    for(int i = 0; i < n; i++){
        for(int j = 0; j < m; j++){
            cin >> T[i][j];
            unique_vals.insert(T[i][j]);
        }
    }
    
    // Build conflict graph
    map<int, set<int>> conflicts;
    for(int val : unique_vals){
        conflicts[val] = set<int>();
    }
    
    // For each column, first row value must differ from all other rows
    for(int j = 0; j < m; j++){
        for(int i = 1; i < n; i++){
            if(T[0][j] != T[i][j]){
                conflicts[T[0][j]].insert(T[i][j]);
                conflicts[T[i][j]].insert(T[0][j]);
            }
        }
    }
    
    // Try to color the graph greedily
    map<int, int> color;
    vector<int> sorted_vals(unique_vals.begin(), unique_vals.end());
    
    for(int val : sorted_vals){
        set<int> used_colors;
        for(int neighbor : conflicts[val]){
            if(color.count(neighbor)){
                used_colors.insert(color[neighbor]);
            }
        }
        
        // Find smallest available color
        int c = 0;
        while(used_colors.count(c)) c++;
        
        if(c > 50){
            cout << "No\n";
            return 0;
        }
        
        color[val] = c;
    }
    
    // Verify the solution
    for(int j = 0; j < m; j++){
        for(int i = 1; i < n; i++){
            if(color[T[0][j]] == color[T[i][j]]){
                cout << "No\n";
                return 0;
            }
        }
    }
    
    cout << "Yes\n";
    for(int val : sorted_vals){
        cout << val << " -> " << color[val] << "\n";
    }
    
    return 0;
}
```

The algorithm:
1. Extract all unique values
2. Build a conflict graph where edges connect values that must have different colors (appear in same column with first row)
3. Greedily color the graph, assigning each value the smallest available color not used by its neighbors
4. Verify that the coloring works
5. Output the result

Time complexity: O(nm + V²) where V is the number of unique values (at most 10^6)
