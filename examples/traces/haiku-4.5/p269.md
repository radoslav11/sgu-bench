Looking at this problem, I need to find the number of ways to place k non-attacking rooks on a board with irregular shape.

Key observations:
1. A "peaceful disposition" means no two rooks share the same row or column
2. The board is defined by rows of varying lengths: row i has b_i cells
3. We need to place k rooks such that no two are in the same row or column

This is a dynamic programming problem. Let me think about the approach:

- We process rows from top to bottom
- For each row, we decide how many rooks to place in it (0 or 1, since rooks can't be in the same row)
- We need to track which columns have been used

Let me define: `dp[i][j][mask]` = number of ways to place j rooks in the first i rows, where mask represents which columns are occupied.

However, with columns up to 250, a bitmask won't work. Instead, I'll use a different approach:

`dp[i][j]` = number of ways to place j rooks in the first i rows, considering only valid column assignments.

Actually, a better approach:
- Process rows sequentially
- For each row i with b_i cells, we can either:
  1. Place no rook in row i
  2. Place exactly one rook in one of the available columns (columns not used in previous rows)

Let me use: `dp[row][rooks_placed]` = map from "set of used columns" to count

But this is complex. Let me think differently using combinatorics:

For each row i, we have b_i possible positions. We need to:
1. Choose which k rows (out of n) will contain rooks
2. For each chosen row, choose which column to place the rook
3. Ensure no two rooks share a column

Let me use: `dp[i][j]` = number of ways to place j rooks in first i rows, where we also track available columns.

Since we need to track which columns are used, and columns go up to 250, I'll use a different state:

`dp[row][num_rooks][last_column_used]` won't work either.

Let me reconsider: I'll use DP where `dp[i][j]` represents placing j rooks in the first i rows. For transitions, when processing row i (with b_i cells), I need to know how many columns from all previous rows have been used.

Key insight: If we've placed j rooks in the first i rows, they occupy j distinct columns. When we move to row i+1 with b_{i+1} cells, we can place a rook in any of those b_{i+1} cells that aren't in already-occupied columns.

But tracking exactly which columns are used is hard. However, we can think about it differently:

Let's use `dp[row][rooks][used_cols]` but represent used_cols smartly. Since columns are ordered, we can think of selecting from available columns.

Actually, simpler approach: For row i with b_i cells, if we've used some columns in previous rows, the number of available columns in row i is b_i minus the count of previously-used columns that fall within [1, b_i].

Let me implement with memoization using map:

```cpp
#include<bits/stdc++.h>
using namespace std;

int n, k;
vector<int> b;
map<tuple<int,int,set<int>>, long long> memo;

long long solve(int row, int rooks_left, set<int> used_cols) {
    if (rooks_left == 0) return 1;
    if (row == n) return 0;
    
    auto key = make_tuple(row, rooks_left, used_cols);
    if (memo.count(key)) return memo[key];
    
    long long result = 0;
    
    // Option 1: Don't place a rook in this row
    result += solve(row + 1, rooks_left, used_cols);
    
    // Option 2: Place a rook in one of the available columns in this row
    if (rooks_left > 0) {
        for (int col = 1; col <= b[row]; col++) {
            if (used_cols.find(col) == used_cols.end()) {
                set<int> new_used = used_cols;
                new_used.insert(col);
                result += solve(row + 1, rooks_left - 1, new_used);
            }
        }
    }
    
    return memo[key] = result;
}

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    cin >> n >> k;
    b.resize(n);
    for (int i = 0; i < n; i++) {
        cin >> b[i];
    }
    
    set<int> empty_set;
    cout << solve(0, k, empty_set) << endl;
    
    return 0;
}
```
