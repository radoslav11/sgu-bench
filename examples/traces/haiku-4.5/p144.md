Looking at this problem, I need to find the probability that two people meet.

Let me set up the problem:
- Both arrive at random times between X and Y o'clock
- Let's say person A arrives at time `a` and person B arrives at time `b` (measured in minutes from X o'clock)
- The total time window is `T = 60*(Y-X)` minutes
- They meet if `|a - b| <= Z`

This is a classic geometric probability problem. If we plot arrival times on a 2D plane where x-axis is person A's arrival time and y-axis is person B's arrival time, both ranging from 0 to T:
- Total area = T²
- The region where they meet is: `|a - b| <= Z`, which is the area between lines `b = a - Z` and `b = a + Z`

The meeting region is bounded by:
- `b >= a - Z` (person B arrives within Z minutes after person A)
- `b <= a + Z` (person B arrives within Z minutes before person A)
- `0 <= a <= T` and `0 <= b <= T` (within the time window)

The area where they DON'T meet consists of two triangular corners:
1. Upper left corner where `b > a + Z`: This is a triangle with vertices at (0, Z), (0, T), (T-Z, T). Area = `(T-Z)²/2`
2. Lower right corner where `b < a - Z`: This is a triangle with vertices at (Z, 0), (T, 0), (T, T-Z). Area = `(T-Z)²/2`

Total area where they don't meet = `(T-Z)² `

Area where they meet = `T² - (T-Z)²`

Probability = `(T² - (T-Z)²) / T²`

Let me simplify:
- `T² - (T-Z)² = T² - (T² - 2TZ + Z²) = 2TZ - Z²`
- Probability = `(2TZ - Z²) / T² = (2TZ - Z²) / T²`

Let me verify with the sample:
- X = 11, Y = 12, Z = 20.0
- T = 60 * (12 - 11) = 60
- Probability = (2*60*20 - 20²) / 60² = (2400 - 400) / 3600 = 2000 / 3600 = 5/9 ≈ 0.555556

Perfect! This matches the expected output.

```cpp
#include<iostream>
#include<iomanip>
#include<cmath>

using namespace std;

int main(){
    int X, Y;
    double Z;
    cin >> X >> Y >> Z;
    
    double T = 60.0 * (Y - X);
    double probability = (2.0 * T * Z - Z * Z) / (T * T);
    
    cout << fixed << setprecision(7) << probability << endl;
    
    return 0;
}
```
