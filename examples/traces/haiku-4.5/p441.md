Looking at this problem, I need to find the number of ways to distribute N different photo albums to K friends where:
- Each friend gets at least 1 album
- Two distributions are the same if they differ only by order of albums in gifts or by the persons receiving gifts

The key phrase is "Two distributions are considered the same if they differ only by order of albums in the gifts or by the persons receiving gifts."

This means:
1. The albums are distinguishable (different)
2. The friends are indistinguishable (order of persons doesn't matter)
3. We only care about how to partition the N albums into K non-empty groups

This is the Stirling number of the second kind, denoted S(N, K), which counts the number of ways to partition N distinct objects into K non-empty indistinguishable subsets.

The Stirling number of the second kind can be computed using the recurrence:
- S(n, k) = k * S(n-1, k) + S(n-1, k-1)

With base cases:
- S(n, 0) = 0 for n > 0
- S(0, 0) = 1
- S(n, k) = 0 if k > n

Let me verify with the example:
- N = 3, K = 2
- We need to partition 3 albums into 2 non-empty groups
- Possible partitions: {A,B},{C}; {A,C},{B}; {A},{B,C}
- That's 3 ways, which matches the expected output!

Let me implement this:

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){
    int N, K;
    cin >> N >> K;
    
    const int MOD = 2007;
    
    // dp[n][k] = Stirling number of second kind S(n,k)
    vector<vector<long long>> dp(N+1, vector<long long>(K+1, 0));
    
    // Base case
    dp[0][0] = 1;
    
    // Fill the DP table
    for(int n = 1; n <= N; n++){
        for(int k = 1; k <= min(n, K); k++){
            dp[n][k] = (k * dp[n-1][k] + dp[n-1][k-1]) % MOD;
        }
    }
    
    cout << dp[N][K] << endl;
    
    return 0;
}
```

The algorithm works as follows:
- S(n, k) represents the number of ways to partition n items into k non-empty groups
- The recurrence relation: S(n, k) = k * S(n-1, k) + S(n-1, k-1)
  - The first term (k * S(n-1, k)) accounts for adding the nth item to one of the k existing groups
  - The second term (S(n-1, k-1)) accounts for putting the nth item in a new group by itself
- We compute this modulo 2007 as required

For the example with N=3, K=2:
- dp[3][2] = 2 * dp[2][2] + dp[2][1]
- dp[2][2] = 2 * dp[1][2] + dp[1][1] = 0 + 1 = 1
- dp[2][1] = 1 * dp[1][1] + dp[1][0] = 1 + 0 = 1
- dp[3][2] = 2 * 1 + 1 = 3 ✓
