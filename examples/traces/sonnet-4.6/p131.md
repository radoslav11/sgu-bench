Looking at this problem, I need to count the number of ways to tile an M×N grid using:
1. 2×1 rectangles (dominoes, can be placed horizontally or vertically)
2. L-shaped corners (2×2 minus one 1×1 square, in 4 orientations)

This is a profile dynamic programming problem. I'll process the grid column by column (or cell by cell), maintaining the "profile" of which cells are already filled.

Let me think about the approach: process cells left-to-right, top-to-bottom. At each cell, if it's already filled, move on. If not, try to place each valid piece that covers this cell and extends to the right/below.

I'll use bitmask DP where the state represents which cells in the current and next columns are already occupied.

Actually, let me think of a standard profile DP approach. I'll process column by column, where the profile (mask) represents which cells in the current column are already "protruding" into it from pieces placed earlier.

Let me use a recursive approach with memoization, processing cell by cell. The state is: current position (row, col) and the bitmask of already-filled cells in the current and next column beyond current position.

Actually, let me do a simpler recursive backtracking with memoization. State = (cell index, mask of filled cells from current position forward). The mask can represent which of the next M*2 or so cells are pre-filled.

Let me use profile DP processing column by column. State = bitmask of which rows in the *next* column are already occupied (by pieces placed while processing current column).

For each column, given the incoming mask (which cells are pre-filled from previous pieces), I process rows top to bottom, placing pieces.

Pieces that can be placed starting at (row, col):
- Horizontal domino: covers (row,col) and (row,col+1) → marks next column row
- Vertical domino: covers (row,col) and (row+1,col) → within same column
- 4 L-shapes (2×2 minus one corner):
  - Missing top-left: covers (r,c+1),(r+1,c),(r+1,c+1)
  - Missing top-right: covers (r,c),(r+1,c),(r+1,c+1)
  - Missing bottom-left: covers (r,c),(r,c+1),(r+1,c+1)
  - Missing bottom-right: covers (r,c),(r,c+1),(r+1,c)

I'll implement a recursive function `solve(col, row, cur_mask, next_mask)` where cur_mask is remaining cells to fill in current column, next_mask is cells pre-filled in next column.

When row == M, we move to next column with next_mask as the new cur_mask.

```cpp
#include <bits/stdc++.h>
using namespace std;

int M, N;
map<tuple<int,int,int,int>, long long> memo;

long long solve(int col, int row, int cur_mask, int next_mask) {
    if (col == N) {
        return (cur_mask == 0) ? 1 : 0;
    }
    if (row == M) {
        if (cur_mask != 0) return 0;
        return solve(col + 1, 0, next_mask, 0);
    }
    
    auto key = make_tuple(col, row, cur_mask, next_mask);
    auto it = memo.find(key);
    if (it != memo.end()) return it->second;
    
    long long res = 0;
    
    if (cur_mask & (1 << row)) {
        // already filled, move to next row
        res = solve(col, row + 1, cur_mask, next_mask);
    } else {
        // try placing pieces that cover (row, col)
        // Horizontal domino: (row,col),(row,col+1)
        if (col + 1 < N && !(next_mask & (1 << row)))
            res += solve(col, row + 1, cur_mask, next_mask | (1 << row));
        // Vertical domino: (row,col),(row+1,col)
        if (row + 1 < M && !(cur_mask & (1 << (row+1))))
            res += solve(col, row + 2, cur_mask, next_mask);
        // L-shapes - need col+1 available
        if (col + 1 < N) {
            // Missing top-right: covers (r,c),(r+1,c),(r+1,c+1)
            if (row+1<M && !(cur_mask&(1<<(row+1))) && !(next_mask&(1<<(row+1))))
                res += solve(col, row+1, cur_mask|(1<<(row+1)), next_mask|(1<<(row+1)));
            // Missing top-left: covers (r,c+1),(r+1,c),(r+1,c+1) - doesn't cover (r,c)!
            // So this can't be triggered here since (r,c) is unfilled
            // Missing bottom-left: covers (r,c),(r,c+1),(r+1,c+1)
            if (row+1<M && !(next_mask&(1<<row)) && !(next_mask&(1<<(row+1))))
                res += solve(col, row+1, cur_mask, next_mask|(1<<row)|(1<<(row+1)));
            // Missing bottom-right: covers (r,c),(r,c+1),(r+1,c)
            if (row+1<M && !(cur_mask&(1<<(row+1))) && !(next_mask&(1<<row)))
                res += solve(col, row+2, cur_mask, next_mask|(1<<row));
        }
    }
    
    memo[key] = res;
    return res;
}

int main() {
    cin >> M >> N;
    cout << solve(0, 0, 0, 0) << endl;
}
```
