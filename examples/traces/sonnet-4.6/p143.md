This is a classic tree DP problem. We need to find a connected subtree of a tree with maximum sum of node values.

The key insight: For each node, we can compute the maximum sum of a connected subtree rooted at that node (where "rooted" means this node is included and we can extend downward).

**DP formulation:**
- `dp[v]` = maximum sum of a connected subtree that includes node `v` and extends into the subtree rooted at `v`
- For each node `v` with children `c1, c2, ...`:
  - `dp[v] = profit[v] + sum of max(0, dp[ci])` for all children
  - We include a child's subtree only if it contributes positively

The answer is the maximum `dp[v]` over all nodes `v`.

Let me trace through the example:
- Tree structure: 4-1, 1-3, 1-2, 4-5
- Profits: -1, 1, 3, 1, -1 (1-indexed)
- Root at node 1:
  - Node 2: dp[2] = 1
  - Node 3: dp[3] = 3
  - Node 4 has children: 5. dp[5] = -1. dp[4] = 1 + max(0,-1) = 1
  - Node 1: dp[1] = -1 + max(0,1) + max(0,3) + max(0,1) = -1+1+3+1 = 4
- Answer = max of all dp values = 4 ✓

Since N can be up to 16000, we need to be careful about stack depth with recursion. Let's use iterative DFS or increase stack size. I'll do iterative post-order traversal.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    int n;
    cin >> n;
    
    vector<int> profit(n+1);
    for(int i = 1; i <= n; i++) cin >> profit[i];
    
    vector<vector<int>> adj(n+1);
    for(int i = 0; i < n-1; i++){
        int a, b;
        cin >> a >> b;
        adj[a].push_back(b);
        adj[b].push_back(a);
    }
    
    // Iterative post-order DFS
    vector<long long> dp(n+1, 0);
    vector<int> parent(n+1, 0);
    vector<int> order; // post-order
    
    vector<bool> visited(n+1, false);
    stack<int> stk;
    stk.push(1);
    parent[1] = -1;
    visited[1] = true;
    
    while(!stk.empty()){
        int v = stk.top();
        stk.pop();
        order.push_back(v);
        for(int u : adj[v]){
            if(!visited[u]){
                visited[u] = true;
                parent[u] = v;
                stk.push(u);
            }
        }
    }
    
    // Process in reverse order (post-order)
    reverse(order.begin(), order.end());
    
    long long ans = LLONG_MIN;
    
    for(int v : order){
        dp[v] = profit[v];
        for(int u : adj[v]){
            if(u != parent[v]){
                // u is a child of v
                if(dp[u] > 0) dp[v] += dp[u];
            }
        }
        ans = max(ans, dp[v]);
    }
    
    cout << ans << endl;
    
    return 0;
}
```

Wait, the reverse of pre-order DFS is not exactly post-order for trees unless we're careful. Let me verify: in the stack-based pre-order, children are pushed in some order and processed. Reversing gives a valid post-order where children are processed before parents. This should work correctly for trees.

Let me verify with the example and the answer is 4. The solution looks correct.
