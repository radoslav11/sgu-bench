Looking at this problem, I need to:

1. Find connected components of 'X's in the logo
2. Determine the minimum number of parts to rearrange them into a 3x3 square
3. Output the labeling and the final arrangement

Key insight: We need to find the minimum number of connected pieces such that each piece can be shifted (no rotation/flip) to fit into a 3x3 square.

Since there are 9 unit squares and we need to form a 3x3, we need to find a way to partition the logo into K connected parts, where each part when shifted fits into a 3x3 grid, and the parts together fill the 3x3 grid exactly.

The 3x3 target has cells (0,0) to (2,2). Each part occupies some cells in the logo; when shifted by some (dr, dc), they must occupy cells in [0,2]x[0,2] with no overlaps.

Approach: Try all possible assignments of the 9 'X' cells to positions in the 3x3 grid. For each assignment (a bijection from logo cells to target cells), check if we can group cells such that:
- Each group's cells in logo are connected
- Each group's offset (target_cell - logo_cell) is the same (so a single shift works)
- Minimum number of groups

This is essentially: for each logo cell, assign a target cell. Two logo cells that map to the same offset can potentially be in the same group if they're connected in the logo with the same offset.

So: find all valid bijections (logo_cells → target_cells), compute the minimum number of connected components among cells sharing the same offset.

But 9! = 362880 bijections... manageable if checking each is fast.

For each bijection: group cells by their offset (logo→target). For each group, check connectivity in the logo subgraph. Count number of connected components total. Minimize this.

Let me implement this:

1. Enumerate all permutations of 9 target cells assigned to 9 logo cells
2. For each permutation, compute offset for each logo cell
3. Group by offset, check connectivity within each group in the logo
4. Count total components, track minimum

9! = 362880, and for each we do O(9) work. Should be fast enough.
