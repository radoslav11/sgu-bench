Looking at this problem, I need to find the number of ordered triples (Shureg, Ruslan1, Ruslan2) where:
- All three holes are distinct
- The angle A at Shureg's position (between the two Ruslans) is acute (0 < A < 90°)
- floor(90/A) == floor(1000 * cos(A)) mod 10

Since the two Ruslans are indistinguishable, I need to count unordered pairs {Ruslan1, Ruslan2} for each Shureg position.

So the answer = sum over each hole S of (number of unordered pairs {B, C} from other holes such that angle at S is valid).

**Approach:**
For each hole S (N choices), compute vectors to all other holes. For each pair of other holes B, C, compute the angle at S. Check the condition.

N=800, so N^3/6 ≈ 85 million which might be tight but let's think.

Actually N=800, number of pairs of other holes per S is ~800*799/2 ≈ 320000, times 800 = 256 million operations. That's too slow naively.

Let me think differently. For each S, compute angles to all other holes (N-1 angles from S), then for each pair compute the angle between them. That's still O(N^2) pairs per S.

Wait, N=800, O(N^3) = 512 million. With simple operations might be feasible in 2.25 seconds if each operation is cheap.

For each triplet (S, B, C): compute dot product of vectors SB and SC, compute their magnitudes, then cos(A) = dot/(|SB|*|SC|). The condition involves floor(90/A) and floor(1000*cos(A)) mod 10.

Let me think about what values floor(90/A) can take for 0 < A < 90:
- floor(90/A) >= 1 when A < 90
- floor(90/A) = k means k <= 90/A < k+1, so 90/(k+1) < A <= 90/k

For k=1: 45 < A < 90 (acute part: 45 < A < 90)
For k=2: 30 < A <= 45

And the third decimal digit of cos(A) must equal k.

Let me precompute for each possible value of k (1..89), what range of cos values satisfy "floor(1000*cos(A)) mod 10 = k". That is, the digit in thousandths place equals k.

The condition is: floor(1000*cos(A)) mod 10 = k, and also k <= 90/A < k+1.

So for each pair (S, B, C), compute cos(A) from dot product, determine if angle is acute (dot > 0), compute A in degrees, check floor(90/A), check third decimal digit.

N=800: 800 * C(799,2) ≈ 800 * 318801 ≈ 255 million. Too slow perhaps.

Actually, for each S, sort other points by angle. For each pair, use dot product to get cos value directly without computing arccos... but floor(90/A) requires the actual angle.

Alternative: precompute for each S all angles to other holes, sort them, then for each pair use the angle difference.

For each S: compute angle θ_i = atan2(dy, dx) for each other hole. Then for each pair (i,j), A = |θ_i - θ_j| (normalized to [0,180]). Need 0 < A < 90 and the condition.

This is still O(N^2) per S point = O(N^3) total.

With N=800, let me just try the straightforward O(N^3) approach using precomputed angles.

For each S, compute all angles, then for each pair compute angle difference, check condition using acos or angle arithmetic.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int n; cin>>n;
    vector<double>x(n),y(n);
    for(int i=0;i<n;i++) cin>>x[i]>>y[i];
    long long ans=0;
    for(int s=0;s<n;s++){
        // for each pair b,c
        for(int b=0;b<n;b++) if(b!=s){
            for(int c=b+1;c<n;c++) if(c!=s){
                double bx=x[b]-x[s],by=y[b]-y[s];
                double cx=x[c]-x[s],cy=y[c]-y[s];
                double dot=bx*cx+by*cy;
                if(dot<=0) continue;
                double lb=sqrt(bx*bx+by*by),lc=sqrt(cx*cx+cy*cy);
                double cosA=dot/(lb*lc);
                if(cosA>1) cosA=1;
                double A=acos(cosA)*180.0/M_PI;
                int k=(int)(90.0/A);
                int d=(int)(1000*cosA)%10;
                if(k>=1 && k==d) ans++;
            }
        }
    }
    cout<<ans<<endl;
}
```
