Looking at this problem, I need to find the rotation angle between two constellations (same shape, different orientation).

Key observations:
1. The constellations are the same shape but rotated (and possibly translated)
2. I need to find the rotation angle between 0 and π

Approach:
- Normalize both constellations by moving centroid to origin
- The constellations are related by a pure rotation (since they're the same shape)
- Find the rotation angle by matching points

Since ordering doesn't matter, I need to figure out correspondence. But given that the shape is the same under rotation only, I can:

1. Compute centroids and center both constellations
2. Find the rotation angle that maps one to the other

To find the rotation angle:
- Pick one point from constellation 1, find its angle
- The corresponding point in constellation 2 should be at the same magnitude (since it's a rotation)
- For each star in photo 1, compute its angle. For each star in photo 2, compute its angle.
- The rotation angle = angle2 - angle1 (mod 2π) for corresponding points

Since stars are at the same distance from centroid but in different angles, I can:
1. Sort both sets by their distance from centroid
2. Points at the same distance are potential matches
3. The rotation angle should be consistent across all pairs

Let me think more carefully. After centering:
- Each star i in photo1 maps to some star j in photo2 via rotation by θ
- So angle(star_j) - angle(star_i) = θ (mod 2π)

Algorithm:
1. Center both constellations
2. For each point in photo1 (sorted by distance), find corresponding point in photo2 (same distance)
3. Compute candidate θ values
4. Find the θ that's consistent with all pairs

Since n ≤ 1000, I can try all pairs between photo1 and photo2 as potential correspondences for one star, get candidate angles, then verify.

More precisely:
- Sort photo1 points by distance from centroid
- Sort photo2 points by distance from centroid  
- Use the first point of photo1 and try matching it with each point of photo2 at the same distance
- For each candidate rotation θ, verify if it maps all photo1 points to photo2 points

The answer should be in [0, π], so if θ is a solution, so is -θ (= 2π - θ), and I take the minimum absolute value.

#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<double> x1(n), y1(n), x2(n), y2(n);
    for (int i = 0; i < n; i++) cin >> x1[i] >> y1[i];
    for (int i = 0; i < n; i++) cin >> x2[i] >> y2[i];
    
    // Center
    double cx1=0,cy1=0,cx2=0,cy2=0;
    for(int i=0;i<n;i++){cx1+=x1[i];cy1+=y1[i];cx2+=x2[i];cy2+=y2[i];}
    cx1/=n;cy1/=n;cx2/=n;cy2/=n;
    for(int i=0;i<n;i++){x1[i]-=cx1;y1[i]-=cy1;x2[i]-=cx2;y2[i]-=cy2;}
    
    // Sort photo2 by angle for quick lookup
    // Try each candidate theta from matching first point of photo1 with each point of photo2
    auto verify = [&](double theta) -> bool {
        const double eps = 1e-4;
        vector<bool> used(n, false);
        for(int i=0;i<n;i++){
            double rx = x1[i]*cos(theta) - y1[i]*sin(theta);
            double ry = x1[i]*sin(theta) + y1[i]*cos(theta);
            bool found = false;
            for(int j=0;j<n;j++){
                if(!used[j] && fabs(x2[j]-rx)<eps && fabs(y2[j]-ry)<eps){
                    used[j]=true; found=true; break;
                }
            }
            if(!found) return false;
        }
        return true;
    };
    
    double r1 = sqrt(x1[0]*x1[0]+y1[0]*y1[0]);
    double best = 1e18;
    for(int j=0;j<n;j++){
        double r2 = sqrt(x2[j]*x2[j]+y2[j]*y2[j]);
        if(fabs(r1-r2)>1e-4) continue;
        double theta = atan2(x1[0]*y2[j]-y1[0]*x2[j], x1[0]*x2[j]+y1[0]*y2[j]);
        if(verify(theta)){
            double ang = fabs(theta);
            if(ang > M_PI) ang = 2*M_PI - ang;
            best = min(best, ang);
        }
    }
    if(best==1e18) best=0;
    printf("%.9f\n", best);
}
