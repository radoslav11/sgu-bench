Looking at this problem, I need to find minimum flow from node 1 to node N such that certain edges are fully saturated (flow = capacity).

This is a classic "flow with lower bounds" problem where some edges have lower bound = capacity (must be fully filled) and upper bound = capacity.

**Approach: Minimum flow with lower bounds**

For edges with `Ci=1`: lower bound = `Zi`, upper bound = `Zi`
For edges with `Ci=0`: lower bound = `0`, upper bound = `Zi`

**Standard technique for lower bounds:**
For each edge (u,v) with lower bound `l` and upper bound `c`:
- Add edge (u,v) with capacity `c-l`
- Add `l` to excess[v] and subtract `l` from excess[u]

Then add a super-source S and super-sink T:
- For each node with excess[v] > 0: add edge S->v with capacity excess[v]
- For each node with excess[v] < 0: add edge v->T with capacity -excess[v]

Also add edge T(=N) -> S(=1) with capacity infinity to allow the main flow.

First find max flow from S to T. If it saturates all S-edges, feasible flow exists.

Then to find **minimum** flow from 1 to N:
After finding feasibility, remove the edge N->1 (or set its capacity to 0), then find max flow from N to 1 (which reduces the flow on 1->N edge). The minimum flow = initial flow on N->1 edge - max flow from N to 1.
