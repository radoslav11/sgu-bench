Let me define the state:
- Position in the ticket (0 to S-1)
- Last digit placed
- Phase: 0 = ascending phase, 1 = descending phase

DP[pos][last_digit][phase] = number of ways to fill positions from `pos` to `S-1` given current state.

Let me think about counting:
- `dp[i][d][p]` = number of sequences of length `i` starting with digit `d` in phase `p`
- Phase 0: next digit can be >= d (stay phase 0) or <= d (switch to phase 1)
- Phase 1: next digit must be <= d

Actually let me do forward DP.

`dp[pos][last][phase]` = count of valid sequences of length `pos` where last digit is `last` and we're in `phase`.

Base case: `dp[1][d][0] = 1` for all d from 0 to 9.

Transitions from position `pos` to `pos+1`:
- If phase == 0: 
  - Next digit `nd >= last`: go to phase 0
  - Next digit `nd <= last`: go to phase 1
  - But wait, if nd >= last, we stay in 0. If nd <= last, we go to 1. But nd could equal last, which satisfies both conditions. We need to avoid double counting.
  - So: nd >= last → phase 0; nd < last → phase 1. (equal stays in phase 0)
- If phase == 1:
  - Next digit `nd <= last`: stay in phase 1
