Looking at this problem, I need to make two trees identical by performing swap operations (add one edge creating a cycle, remove one edge from the cycle).

Key insight: Each operation on a tree replaces one edge with another. To make tree T1 equal to tree T2, I need to find edges in T1 not in T2 and edges in T2 not in T1, and fix them.

Let me think about this. Let E1 = edges of Berland, E2 = edges of Beerland. Both are trees on N nodes.

Edges in E1 but not E2 are "wrong" edges in Berland. Edges in E2 but not E1 are "wrong" edges in Beerland.

For each edge `e` in E2 but not E1: adding `e` to T1 creates a cycle. I need to remove some edge from that cycle that's in E1 but not E2.

The key observation: if I add edge (a,b) from T2 to T1, it creates a cycle. The path from a to b in T1 contains edges. If any of those path edges is not in T2, I can remove it. This gives me a valid operation.

**Strategy**: Repeatedly find an edge `e = (a,b)` in T2 but not in T1. Add it to T1, find the path from a to b in T1, find an edge on this path that's not in T2, remove it. This modifies T1 toward T2.

Each such operation reduces the symmetric difference by 2 (we add one edge that T2 has, remove one edge that T1 has but T2 doesn't).

Number of operations = |E1 △ E2| / 2.

Wait, but we can also work on Beerland (country 2). However, working on Berland moving it toward Beerland seems optimal — each step reduces the difference by 2, so we need exactly `k = |E1 △ E2| / 2` operations.

Can we always find an edge on the path that's not in T2? Yes! Because if all edges on the path from a to b in T1 were in T2, then T2 would have a cycle (the path edges plus edge (a,b)), contradiction.

So the algorithm is:
1. Find all edges in T2 not in T1.
2. For each such edge (a,b), find path in T1 from a to b, find an edge (u,v) on path not in T2.
3. Perform operation: in Berland (p=1), add (a,b), remove (u,v).
4. Update T1 accordingly.

Let me implement this. I'll represent edges as sets of pairs (normalized so first < second).

For finding path in T1: BFS/DFS from a to b.

Let me code this up:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    int n;
    cin>>n;
    set<pair<int,int>> e1,e2;
    vector<set<int>> adj1(n+1), adj2(n+1);
    for(int i=0;i<n-1;i++){int a,b;cin>>a>>b;e1.insert({min(a,b),max(a,b)});adj1[a].insert(b);adj1[b].insert(a);}
    for(int i=0;i<n-1;i++){int a,b;cin>>a>>b;e2.insert({min(a,b),max(a,b)});adj2[a].insert(b);adj2[b].insert(a);}
    
    vector<tuple<int,int,int,int,int>> ops;
    
    while(e1!=e2){
        // find edge in e2 not in e1
        pair<int,int> target={-1,-1};
        for(auto&p:e2) if(!e1.count(p)){target=p;break;}
        int a=target.first,b=target.second;
        // find path from a to b in T1
        vector<int> par(n+1,-1);
        vector<bool> vis(n+1,false);
        queue<int> q;
        q.push(a);vis[a]=true;
        while(!q.empty()){int u=q.front();q.pop();for(int v:adj1[u])if(!vis[v]){vis[v]=true;par[v]=u;q.push(v);}}
        // trace path
        int cur=b;
        pair<int,int> rem={-1,-1};
        while(cur!=a){
            int p=par[cur];
            pair<int,int> ed={min(p,cur),max(p,cur)};
            if(!e2.count(ed)){rem=ed;break;}
            cur=p;
        }
        // operation: p=1, add (a,b), remove rem
        ops.push_back({1,a,b,rem.first,rem.second});
        e1.insert(target);adj1[a].insert(b);adj1[b].insert(a);
        e1.erase(rem);adj1[rem.first].erase(rem.second);adj1[rem.second].erase(rem.first);
    }
    
    cout<<ops.size()<<"\n";
    for(auto&[p,a1,b1,a2,b2]:ops) cout<<p<<" "<<a1<<" "<<b1<<" "<<a2<<" "<<b2<<"\n";
}
```

This looks correct. Each iteration removes one "wrong" edge from e2\e1 and one from e1\e2, reducing the difference by 2. The number of operations equals |e1△e2|/2 which is minimal.
