Looking at this problem, I need to simulate a text editor with cursor movements and character insertions.

The key insight is that I need an efficient data structure. A simple approach using a linked list or a rope would work, but let me think about using a simpler simulation.

Since the string can be up to 10^6 characters, and each operation is O(1) amortized with the right data structure, I'll use two stacks (or vectors) representing the text to the left and right of the cursor.

- Left stack: characters to the left of cursor (top = rightmost character before cursor)
- Right stack: characters to the right of cursor (top = leftmost character after cursor)

Operations:
- Insert 'a'-'z': push to left stack
- 'L': if left stack not empty, pop from left and push to right
- 'R': if right stack not empty, pop from right and push to left

At the end, the result is: all characters in left stack (bottom to top) + all characters in right stack (top to bottom).

Let me verify with the example:
- `abLcd`
- Start: left=[], right=[]
- 'a': left=[a], right=[]
- 'b': left=[a,b], right=[]
- 'L': left=[a], right=[b]
- 'c': left=[a,c], right=[b]
- 'd': left=[a,c,d], right=[b]
- Result: a,c,d + b (right stack top to bottom) = "acdb" ✓

Second example: `icpLLLLLacmRRRRRRRRRRRRc`
- 'i','c','p': left=[i,c,p], right=[]
- 'L','L','L','L','L': move left 5 times, but only 3 chars
  - After 3 L's: left=[], right=[p,c,i]
  - Next 2 L's ignored
- 'a','c','m': left=[a,c,m], right=[p,c,i]
- 'R' x12: move right, but only 3 chars on right
  - After 3 R's: left=[a,c,m,p,c,i], right=[]
  - Next 9 R's ignored
- 'c': left=[a,c,m,p,c,i,c], right=[]
- Result: "acmpcic"... wait that doesn't match.

Let me re-check. Expected output: `acmicpc`

Let me redo: after 'i','c','p': left=[i,c,p]
After L,L,L: left=[], right=[p,c,i] (right stack: top is p, then c, then i from top)

Wait, right stack top = closest to cursor. So:
- L: pop p from left... no wait left=[i,c,p], pop p, push to right. right=[p]
- L: pop c from left=[i,c], push to right=[p,c]
- L: pop i from left=[i], push to right=[p,c,i]
- left=[], right=[p,c,i] (right[top]=i, meaning i is just to the right of cursor)

Hmm, that means right stack: top is i, then c, then p.

After 'a','c','m': left=[a,c,m]
After R x3: 
- R: pop i from right, push to left. left=[a,c,m,i]
- R: pop c, left=[a,c,m,i,c]
- R: pop p, left=[a,c,m,i,c,p]
After 'c': left=[a,c,m,i,c,p,c]
Result: "acmicpc" ✓

Great, the algorithm is correct. Let me implement it:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    string input;
    cin >> input;
    
    vector<char> left_stack, right_stack;
    left_stack.reserve(input.size());
    right_stack.reserve(input.size());
    
    for(char c : input){
        if(c == 'L'){
            if(!left_stack.empty()){
                right_stack.push_back(left_stack.back());
                left_stack.pop_back();
            }
        } else if(c == 'R'){
            if(!right_stack.empty()){
                left_stack.push_back(right_stack.back());
                right_stack.pop_back();
            }
        } else {
            left_stack.push_back(c);
        }
    }
    
    string result;
    result.reserve(left_stack.size() + right_stack.size());
    for(char c : left_stack) result += c;
    for(int i = right_stack.size()-1; i >= 0; i--) result += right_stack[i];
    
    cout << result << "\n";
    
    return 0;
}
```
