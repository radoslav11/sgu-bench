Looking at this problem, I need to find the largest palindrome square in a given rectangle.

A palindrome square of size n has every row and every column being a palindrome.

Key insight: If every row is a palindrome AND every column is a palindrome, then the square has a special structure. 

Let me think about what constraints this places:
- Each row is a palindrome: `grid[r][c] = grid[r][n-1-c]`
- Each column is a palindrome: `grid[r][c] = grid[n-1-r][c]`

So the square is determined by its top-left quadrant essentially.

**Approach:** For each possible top-left corner `(r, c)` and size `n`, check if it's a palindrome square. But O(h*w*min(h,w)) could be up to 700*700*700 = 343M which is too slow naively.

**Better approach:** 
- Precompute for each cell `(r, c)`, the maximum horizontal palindrome length centered there.

Actually, let me think differently. A palindrome square of size n starting at (r, c):
- Every row from r to r+n-1 must be a palindrome of length n starting at column c.
- Every column from c to c+n-1 must be a palindrome of length n starting at row r.

For a row to be a palindrome of length n starting at col c: characters at positions c..c+n-1 form a palindrome.

Let me precompute: `hpal[r][c]` = largest odd/even... actually, let me precompute for each (r, c) and each possible half-size, whether the row palindrome holds.

Alternative: For each pair of rows `(r1, r2)` where r2 >= r1, compute the set of columns where both rows are mirrors of each other (i.e., grid[r1][c] = grid[r2][c] for all c in the range). But this is complex.

Let me try: precompute `row_pal[r][c][len]` = 1 if row r, starting at col c, length len is palindrome. This is O(h*w^2) space which is too much.

Instead: For each row r, use Manacher's to find all palindromic substrings. For each center, store the max radius.

For a square of size n at (r,c): all rows r..r+n-1 must have a palindrome of length n at column c. 

Let `rp[r][c]` = max radius of palindrome in row r centered at column c (0-indexed, for odd length palindromes centered at c, length = 2*rp+1). For even: center between c and c+1.

For odd-sized squares of size (2k+1): center row = r+k, center col = c+k. All rows must have palindrome radius >= k at center col c+k. All cols must have palindrome radius >= k at center row r+k.

For each center (cr, cc) and odd size 2k+1: need all rows in [cr-k, cr+k] to have row_radius[row][cc] >= k, and all cols in [cc-k, cc+k] to have col_radius[cr][col] >= k.

Let me precompute `min_row_rad[cr][cc]` = for center column cc, the maximum k such that all rows within k of cr have row palindrome radius >= k. This seems complex.

Let me try a simpler O(h*w*min(h,w)) approach but optimized. With h,w=700, min=700, that's 343M - might be tight at 0.25s. But maybe with simple operations it works.

Precompute `rp[r][c]` = palindrome radius for row r at col c (Manacher's, O(h*w)).
Precompute `cp[r][c]` = palindrome radius for col c at row r (Manacher's, O(h*w)).

For each possible center (cr, cc), find max k such that:
- For all rows in [cr-k, cr+k]: rp[row][cc] >= k
- For all cols in [cc-k, cc+k]: cp[cr][col] >= k

Max k is bounded by min(rp[cr][cc], cp[cr][cc], cr, h-1-cr, cc, w-1-cc).

I'll iterate k from max down, checking incrementally. For each center, binary search on k using precomputed prefix minimums.

#include <bits/stdc++.h>
using namespace std;
int main(){
    int h,w; cin>>h>>w;
    vector<string> g(h); for(auto&s:g)cin>>s;
    vector<vector<int>> rp(h,vector<int>(w,0)),cp(h,vector<int>(w,0));
    for(int r=0;r<h;r++){int c=0;while(c<w){int k=1;while(c-k>=0&&c+k<w&&g[r][c-k]==g[r][c+k])k++;rp[r][c]=k-1;c++;}}
    for(int c=0;c<w;c++){int r=0;while(r<h){int k=1;while(r-k>=0&&r+k<h&&g[r-k][c]==g[r+k][c])k++;cp[r][c]=k-1;r++;}}
    // prefix min of rp along rows for fixed cc, and cp along cols for fixed cr
    // mrp[cc][r] = rp[r][cc], build prefix min in both directions from each center
    // For center (cr,cc), max k where all rows in [cr-k,cr+k] have rp[row][cc]>=k
    // Use: for fixed cc, array A[r]=rp[r][cc]. For each cr, find max k<=min(cr,h-1-cr) s.t. min(A[cr-k..cr+k])>=k
    // Precompute for fixed cc: left_min[r] = min of A[r..cr] going up, but k-dependent - hard
    // Just do O(h*w*min(h,w)) naively
    int br=0,bc=0,bsize=1;
    for(int cr=0;cr<h;cr++)for(int cc=0;cc<w;cc++){
        int maxk=min({rp[cr][cc],cp[cr][cc],cr,h-1-cr,cc,w-1-cc});
        for(int k=maxk;k>=bsize/2;k--){ // only if 2k+1>bsize
            if(2*k+1<=bsize)break;
            bool ok=true;
            for(int dr=1;dr<=k&&ok;dr++)if(rp[cr-dr][cc]<k||rp[cr+dr][cc]<k)ok=false;
            for(int dc=1;dc<=k&&ok;dc++)if(cp[cr][cc-dc]<k||cp[cr][cc+dc]<k)ok=false;
            if(ok){bsize=2*k+1;br=cr-k;bc=cc-k;break;}
        }
    }
    cout<<br+1<<" "<<bc+1<<" "<<br+bsize<<" "<<bc+bsize<<endl;
}
