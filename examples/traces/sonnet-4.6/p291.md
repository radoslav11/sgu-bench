Looking at this problem, I need to simulate bacterial growth where cultures spread like BFS waves, but with priorities based on culture order.

Key observations:
1. Each second, cultures divide in order 1, 2, ..., n
2. When a culture divides, it spreads to adjacent cells
3. A newer bacterium kills an older one of the same culture
4. Bacteria die if the cell is occupied by another culture

The spread pattern is essentially BFS - after t seconds, culture i can reach cells within Manhattan distance t from its start (but blocked by other cultures).

Since t can be huge (up to 10^9), but the grid is only 1000x1000 = 10^6 cells, the simulation converges once all cells are filled or no changes occur.

The key insight: The actual spread is like a BFS/Voronoi diagram but with tie-breaking based on which culture processes first within a second. Since all cultures spread simultaneously each second (but in order within a second), this is complex.

However, with t up to 10^9 and grid 1000x1000, the simulation will stabilize in at most 1000+1000=2000 steps (diameter of grid). So we can simulate second by second until stable or t steps reached.

Each second: For each culture in order 1..n, expand its territory. When culture i expands, it can overwrite cells of culture j where j < i (since i processes after j within this second, i's new bacteria are "younger" relative to same-culture but what about different cultures?).

Wait, re-reading: "if the required cell is occupied by bacterium of the other culture, the corresponding bacterium dies." So bacteria cannot spread into cells occupied by other cultures. Only same-culture cells can be overwritten (younger kills older).

So different cultures block each other! The boundaries are fixed once cultures meet. The spread is like BFS per culture, stopped by other cultures' territories.

So the simulation: each culture does a BFS expansion each second, but can't cross into other cultures' cells. The order within a second doesn't matter for inter-culture blocking (since they can't enter each other's cells anyway).

This simplifies things: it's just parallel BFS expansion, one step per second, where cells are permanently claimed once occupied.

Better: Run multi-source BFS simultaneously. Each cell gets assigned to the culture whose BFS front reaches it first. Ties broken by... culture order? Since all expand simultaneously and can't enter each other's cells, ties just mean one wins (random per problem, but we need deterministic output).

Actually since bacteria can't enter cells of other cultures, once a cell is claimed, it stays. So it's exactly a Voronoi diagram on grid with Manhattan distance, where ties could go either way. But the sample output suggests a deterministic answer.

Let me just do multi-source BFS: all cultures expand simultaneously. When a cell is reached at distance d by culture i, it's claimed. If two cultures reach at same distance, we need a tie-break - perhaps by culture index (lower index wins since it processes first in a second).

Let me implement multi-source BFS with priority: process by distance, then by culture index for ties.

#include <bits/stdc++.h>
using namespace std;
int main(){
    int q,c,n,t;
    scanf("%d%d%d%d",&q,&c,&n,&t);
    vector<int> owner(q*c,-1);
    // BFS
    vector<deque<int>> bfs(1);
    // Use standard BFS with dist tracking
    vector<int> dist(q*c,INT_MAX);
    // multi-source, tie-break by culture index (lower=better)
    // Use priority queue: (dist, culture_id, cell)
    priority_queue<tuple<int,int,int>,vector<tuple<int,int,int>>,greater<>> pq;
    for(int i=0;i<n;i++){
        int x,y; scanf("%d%d",&x,&y); x--;y--;
        int cell=x*c+y;
        pq.push({0,i,cell});
    }
    int dx[]={0,0,1,-1},dy[]={1,-1,0,0};
    while(!pq.empty()){
        auto[d,cu,cell]=pq.top();pq.pop();
        if(dist[cell]!=INT_MAX)continue;
        if(d>t)break;
        dist[cell]=d;owner[cell]=cu;
        int x=cell/c,y=cell%c;

    }
    vector<int> cnt(n,0);
    for(int i=0;i<q*c;i++)if(owner[i]>=0)cnt[owner[i]]++;
