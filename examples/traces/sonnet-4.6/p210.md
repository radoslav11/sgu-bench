Looking at this problem, I need to find a maximum weight bipartite matching where:
- Left nodes are sons (1..N)
- Right nodes are girls (1..N)
- Each son i can be matched to girls he likes
- Weight of matching son i is Ai²
- I want to maximize the sum of Ai² for matched sons

Since we want to maximize sum of squares, and each son has weight Ai², I should use maximum weight bipartite matching.

The key insight: we want to maximize Σ Ai² for sons who get matched to a girl they like. This is a maximum weight matching problem.

With N ≤ 400, I can use the Hungarian algorithm or a max-weight matching approach.

Let me think about the approach:
- We have a bipartite graph with sons on left and girls on right
- Edge (i, j) exists if son i likes girl j, with weight Ai²
- We want maximum weight matching

Since all weights are on the left side (weight of son i regardless of which girl), I can use a greedy approach combined with augmenting paths: sort sons by Ai in descending order, and try to match them using augmenting paths.

Actually, since we want to maximize sum of squares, and the weights are per-son (not per-edge), we should try to match sons with higher Ai² first. A greedy approach: sort sons by Ai descending, try to match each one using augmenting paths from standard bipartite matching.

Wait, but this greedy might not be optimal. For example, son with weight 3 and son with weight 2 both like girl 1, but son with weight 2 also likes girl 2. The greedy would correctly match son 3 to girl 1 and son 2 to girl 2.

Actually, this greedy IS optimal for this case: process sons in decreasing order of Ai, and use augmenting paths. Since we always prefer higher-weight sons, any augmenting path that helps a higher-weight son won't displace them later (lower-weight sons process after).

This is the standard approach for maximum weight bipartite matching when weights are on one side only - sort by weight descending and greedily augment.

Let me verify: we want to maximize Σ Ai². If we process sons in decreasing order of Ai², for each son we try to find an augmenting path. If we can match him, we do. This greedy is optimal because matching a higher-valued son can only displace lower-valued sons, and the gain (Ai²) is always larger.

This is correct! The greedy works here because the weight function has a "substitution" property - replacing a lower-weight matched son with a higher-weight one always improves the objective.

Let me implement this:
1. Sort sons by Ai descending
2. For each son in order, try to find augmenting path in bipartite graph
3. Output the matching

```cpp
#include <bits/stdc++.h>
using namespace std;

int N;
vector<int> adj[405]; // adj[son] = list of girls
int matchSon[405]; // matchSon[son] = girl (0 if unmatched)
int matchGirl[405]; // matchGirl[girl] = son (0 if unmatched)
bool visited[405];

bool augment(int son) {
    for (int girl : adj[son]) {
        if (!visited[girl]) {
            visited[girl] = true;
            if (matchGirl[girl] == 0 || augment(matchGirl[girl])) {
                matchSon[son] = girl;
                matchGirl[girl] = son;
                return true;
            }
        }
    }
    return false;
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    cin >> N;
    vector<int> A(N+1);
    for (int i = 1; i <= N; i++) cin >> A[i];
    
    for (int i = 1; i <= N; i++) {
        int k; cin >> k;
        for (int j = 0; j < k; j++) {
            int g; cin >> g;
            adj[i].push_back(g);
        }
    }
    
    // Sort sons by A[i] descending
    vector<int> order(N);
    iota(order.begin(), order.end(), 1);
    sort(order.begin(), order.end(), [&](int a, int b){ return A[a] > A[b]; });
    
    memset(matchSon, 0, sizeof(matchSon));
    memset(matchGirl, 0, sizeof(matchGirl));
    
    for (int son : order) {
        memset(visited, false, sizeof(visited));
        augment(son);
    }
    
    for (int i = 1; i <= N; i++) {
        cout << matchSon[i];
        if (i < N) cout << " ";
    }
    cout << endl;
    
    return 0;
}
```
