Looking at this problem:

- We have sons with mental potentials (integers from -3 to 3)
- We choose a subset of sons to maximize the sum of their potentials raised to power p (p is 1, 2, or 3)
- We need to find the maximum possible sum (if all values are negative, we can choose empty subset giving 0)

Key insight: For each son, we compute value^p and include him if that value is positive. If no values are positive, the answer is 0.

Wait, but we can choose any subset including the empty subset (sum = 0).

So the strategy is: include son i if and only if (potential_i)^p > 0.

- If p=1: include sons with positive potential
- If p=2: squares are always non-negative; include if square > 0 (i.e., potential ≠ 0). But wait, all squares are positive for non-zero, so we'd include all non-zero sons. But that always gives a non-negative sum. We want to maximize, so include all sons with potential ≠ 0 (their contribution is positive).
- If p=3: cube has same sign as base; include sons with positive potential (cube > 0)

For p=2: all contributions are ≥ 0, so include all non-zero sons (contributions are positive). The sum will be ≥ 0.

The answer is max(0, sum of (v^p) for all sons where v^p > 0).

Actually just: sum all v^p where v^p > 0, that gives the maximum (since adding a positive term increases sum, adding zero or negative term decreases or keeps same).

Let me verify with examples:
- Example 1: sons = [2, -1, 1], p=3. Values: 8, -1, 1. Include 8 and 1 → sum = 9. ✓
- Example 2: sons = [-1], p=1. Values: [-1]. No positive values, answer = 0. ✓

So the algorithm is simply: sum up v^p for each son where v^p > 0.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, p;
    cin >> n >> p;
    
    long long total = 0;
    for (int i = 0; i < n; i++) {
        int v;
        cin >> v;
        long long vp = 1;
        for (int j = 0; j < p; j++) vp *= v;
        if (vp > 0) total += vp;
    }
    
    cout << total << endl;
    return 0;
}
```
