Looking at this problem, I need to find the minimum number of user groups.

Key constraints:
1. No two groups share a resource (each resource belongs to at most one group)
2. A user accesses a resource iff they're in the group containing that resource
3. Each user gets exactly the resources they need

Since each resource belongs to exactly one group, and a user accesses exactly the resources of groups they belong to, the set of resources a user can access is the union of resources of all groups they belong to.

For user i to access exactly set S_i, the groups containing resources must partition S_i for each user.

Since resources can't be shared between groups, each resource r belongs to exactly one group g(r). User i must belong to group g(r) for every r in S_i, and must NOT belong to any group containing resources outside S_i.

Two resources r1 and r2 can be in the same group only if every user who needs r1 also needs r2, and vice versa - i.e., they have exactly the same set of users who need them.

Wait, let me think more carefully. If r1 and r2 are in the same group g, then any user in group g gets both r1 and r2. So if user i needs r1 but not r2, they can't be in group g, but then they can't access r1 either. So r1 and r2 can only be in the same group if they have exactly the same set of requiring users.

So resources with the same "demand set" (set of users who need them) can be grouped together, and resources with different demand sets must be in different groups.

The minimum number of groups = number of distinct non-empty demand sets across all resources.

Resources that no one needs don't need to be in any group (they'd just sit there, but we need to make sure no user accidentally gets access to them - actually resources with empty demand set just aren't included in any group, or we could include them in a group with no users, but that would add a group unnecessarily... actually if a resource has no users needing it, we simply don't put it in any group, contributing 0 groups).

Wait, but do we need to assign every resource to a group? The problem says "each resource can belong to at most one user group" - so resources can be unassigned. And since no user needs unneeded resources, we just don't assign them.

So the answer is: count the number of distinct non-empty "user sets" across all resources, where the user set of resource r is {i : r ∈ S_i}.

Let me verify with the example:
- 5 resources, 2 users
- User 1 needs: {1, 3}
- User 2 needs: {3, 4, 5}

Resource 1: needed by {user 1} → set {1}
Resource 2: needed by {} → empty, skip
Resource 3: needed by {user 1, user 2} → set {1,2}
Resource 4: needed by {user 2} → set {2}
Resource 5: needed by {user 2} → set {2}

Distinct non-empty sets: {1}, {1,2}, {2} → 3 groups. ✓

So the solution is to compute for each resource which users need it, then count distinct non-empty sets.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int N, M;
    cin >> N >> M;
    
    // For each resource, which users need it
    vector<set<int>> demand(N + 1);
    
    for (int i = 1; i <= M; i++) {
        int sz;
        cin >> sz;
        for (int j = 0; j < sz; j++) {
            int r;
            cin >> r;
            demand[r].insert(i);
        }
    }
    
    set<set<int>> distinct_sets;
    for (int r = 1; r <= N; r++) {
        if (!demand[r].empty()) {
            distinct_sets.insert(demand[r]);
        }
    }
    
    cout << distinct_sets.size() << endl;
    
    return 0;
}
```
