Looking at this problem, I need to find the minimal expected time to type n characters correctly.

Let me think about the structure. After typing some characters, Vasya can look at the screen (costs t time). If there's a typo, he erases back to the first typo and retypes. If no typo, he continues.

Let me define `dp[i]` = minimum expected time to go from having `i` correct characters typed (and verified) to having all `n` characters typed and verified.

From position `i`, Vasya needs to type characters `i+1, i+2, ..., n`. He can choose to look at the screen after typing `k` more characters (for k = 1, 2, ..., n-i).

When he types characters `i+1` through `i+k`:
- Let `p_j` = probability that character `i+j` is correct = `1 - a[i+j]`
- Let `P(j)` = probability that all characters `i+1` through `i+j` are correct = product of `(1-a[i+1])...(1-a[i+j])`
- `P(0) = 1`

When he looks after typing k characters:
- Probability all k are correct: `P(k)` → pays k (typing) + t (looking) + dp[i+k]
- Probability first error is at position j (1-indexed within the k chars): `P(j-1) * a[i+j]`
  - He typed k characters + t (looking) + then erases back to position i (but actually erases to just after position i), which costs `k - j + 1` backspaces... wait, he erases from end to first typo inclusive.
  - If first typo is at relative position j, he erases `k - j + 1` characters, returning to position i+j-1... but position i+j-1 might not be "verified". Actually wait - he's at position i which is already verified. The typed characters i+1..i+k are unverified. If first typo is at j, he erases back to position i+j-1 (which is after the typo, inclusive). So he ends up at position i+j-1... but those characters aren't verified either.

Hmm, actually he ends up at position i+j-1 with i+j-1 characters on screen, but only i are verified correct. So he's back to having typed i+j-1 characters with i verified... but that's not quite dp[i].

Wait, re-reading: he "leaves only the correct part intact". So after erasing the typo and everything after, he has exactly i+j-1 characters on screen, which are all correct (since the first j-1 were typed correctly). But they're not yet verified by looking. However, since we know they're correct (from our probability calculation), when he looks next time, they'll be fine.

Actually, the state should just be "verified correct characters = i". After a typo at position j, he's back to having i+j-1 characters typed (unverified but correct), which means he's effectively at state i+j-1 if we allow unverified states... 

Let me redefine: `dp[i]` = expected time from having exactly `i` characters correctly on screen (not necessarily verified) to finish. The strategy is: type k more characters, look, handle result.

Actually if we're at position i (correctly typed, may be unverified), and we type k more: cost is k + t + (probability issues lead back somewhere).

If first error at position j among the k: cost k + t + (k-j+1 backspaces) + dp[i+j-1]
If all correct: cost k + t + dp[i+k]

So: `dp[i] = min over k of { k + t + sum_{j=1}^{k} P(j-1)*a[i+j]*(k-j+1 + dp[i+j-1]) + P(k)*dp[i+k] }`

where `P(j) = prod_{m=1}^{j} (1-a[i+m])`, `dp[n] = 0`.

This is O(n^2) states and transitions, so O(n^2) total which is fine for n=3000.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    int n, t;
    cin >> n >> t;
    vector<double> a(n+1);
    for(int i = 1; i <= n; i++) cin >> a[i];
    
    vector<double> dp(n+1, 0.0);
    // dp[n] = 0
    // dp[i] = min over k=1..n-i of { k + t + sum_{j=1}^{k} P(j-1)*a[i+j]*(k-j+1+dp[i+j-1]) + P(k)*dp[i+k] }
    
    for(int i = n-1; i >= 0; i--){
        double best = 1e18;
        double Pprod = 1.0; // P(j-1) as j goes 1..k, so initially P(0)=1
        double sumErr = 0.0; // sum_{j=1}^{k} P(j-1)*a[i+j]*(dp[i+j-1])
        double sumErrCoeff = 0.0; // sum_{j=1}^{k} P(j-1)*a[i+j]*(-j)  [for the (k-j+1) part]
        double sumErrProb = 0.0; // sum_{j=1}^{k} P(j-1)*a[i+j]  [coefficient of k+1]
        
        for(int k = 1; k <= n-i; k++){
            int j = k;
            double pj1 = Pprod; // P(j-1) = P(k-1)
            double contrib = pj1 * a[i+j];
            sumErr += contrib * dp[i+j-1];
            sumErrCoeff += contrib * (-j);
            sumErrProb += contrib;
            Pprod *= (1.0 - a[i+j]);
            
            double val = (double)k + t + (k+1)*sumErrProb + sumErrCoeff + sumErr + Pprod * dp[i+k];
            if(val < best) best = val;
        }
        dp[i] = best;
    }
    
    cout << fixed << setprecision(15) << dp[0] << endl;
    return 0;
}
```
