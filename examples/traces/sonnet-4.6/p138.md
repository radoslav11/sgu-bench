Looking at this problem, I need to schedule chess games where:
1. The first game has two players
2. Each subsequent game involves the winner of the previous game and someone new (or the previous loser)
3. Each player plays exactly the number of games specified

Let me think about this. The total number of games G = sum of all games / 2 (since each game involves exactly 2 players).

The player with the most games must be involved in almost every game. The key insight: the player who plays the most games will likely be the "champion" who stays on the longest.

Let me think of a greedy approach:
- Sort players by number of games (descending)
- The player with the most games should be the one who keeps winning/staying

Actually, let me think differently. Consider that in G games, the "current champion" plays in many games. 

The player with the maximum games played - let's call them the "dominant" player. They need to appear in their count of games consecutively (or spread out).

Let me think of a simulation: maintain a priority queue. At each step, the current winner plays against someone. 

Key observation: The player with the most games must play consecutively (as champion). 

Let me think of it as: sort by frequency descending. The most frequent player should win most games. 

Algorithm idea:
1. Find the player with max games (say player M with count m)
2. They will be champion for stretches. 
3. Use a greedy: at each game, the current champion plays against the person with the highest remaining game count (who isn't the current champion).

Let me simulate: Use a priority queue of (remaining_games, player_id). 

- Pick the two players with highest remaining counts as first game participants
- The winner is whoever has more games remaining
- For each subsequent game: current winner plays against the person with highest remaining games count (excluding current winner)
- Decrement both players' counts
- Continue until all counts are 0

Actually when tied, the loser should be the current champion if the other player needs to continue (has remaining games). Let me reconsider: winner should be whoever has MORE remaining games after this game.

Actually the winner is the one who needs to play more future games. So compare remaining-1 (after this game). If player A has rA and player B has rB, winner should be whoever has higher remaining after decrement, i.e., max(rA-1, rB-1). So winner = player with higher count. If equal, try both... 

Let me just implement: winner = the one with more games remaining. Break ties by choosing the current champion (they stay).

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int n; cin>>n;
    vector<int> cnt(n+1);
    for(int i=1;i<=n;i++) cin>>cnt[i];
    int G=0; for(int i=1;i<=n;i++) G+=cnt[i]; G/=2;
    cout<<G<<"\n";
     // priority queue: (remaining, id)
    // greedy: winner is who has more remaining
    auto cmp=[](pair<int,int>a,pair<int,int>b){return a.first<b.first;};
    priority_queue<pair<int,int>,vector<pair<int,int>>,decltype(cmp)> pq(cmp);
    for(int i=1;i<=n;i++) if(cnt[i]>0) pq.push({cnt[i],i});
    auto [r1,p1]=pq.top(); pq.pop();
    auto [r2,p2]=pq.top(); pq.pop();
    // winner is p1 (higher count), loser p2
    cout<<p1<<" "<<p2<<"\n";
    r1--; r2--;
    if(r2>0) pq.push({r2,p2});
    int champ=p1; int champ_r=r1;
    for(int g=1;g<G;g++){
        auto [rb,pb]=pq.top(); pq.pop();
        int winner,loser,wr,lr;
        if(champ_r>=rb){winner=champ;loser=pb;wr=champ_r-1;lr=rb-1;}
        else{winner=pb;loser=champ;wr=rb-1;lr=champ_r-1;}
        cout<<winner<<" "<<loser<<"\n";
        if(lr>0) pq.push({lr,loser});
        champ=winner; champ_r=wr;
    }
}
```
