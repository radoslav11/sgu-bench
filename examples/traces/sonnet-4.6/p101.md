Looking at this problem, I need to arrange domino pieces in a line where adjacent pieces share equal values at their touching ends.

This is essentially finding an Eulerian path in a graph where:
- Nodes are digit values (0-6)
- Each domino piece is an edge between its two values

An Eulerian path exists if:
- The graph is connected (considering only vertices with edges)
- Either all vertices have even degree (Eulerian circuit) or exactly 2 vertices have odd degree (Eulerian path)

Algorithm:
1. Build the graph
2. Check Eulerian path conditions
3. Find the path using Hierholzer's algorithm
4. Output the pieces in order with orientation

Let me implement this:

For the Eulerian path:
- If exactly 2 odd-degree vertices exist, start from one of them
- If all even, start from any vertex with edges

For output:
- Each edge (domino) needs to be labeled with its original index and whether it was flipped
- When traversing edge from u to v, if original piece is (u, v) → "+" (not rotated), if (v, u) → "-" (rotated)

I need to handle multigraph (multiple edges between same nodes).
