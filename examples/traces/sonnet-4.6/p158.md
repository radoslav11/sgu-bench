Looking at this problem, I need to find train position S that maximizes the sum of minimum distances from each passenger to the nearest door.

Key constraints:
- Platform length L (0 < L ≤ 5000)
- M passengers (≤ 300) at positions P[i]
- N doors (≤ 300) at positions D[j] (D[1]=0)
- Train position S: the first door is at position S, so door j is at D[j] + S
- S must be valid: 0 ≤ S and D[N] + S ≤ L, so 0 ≤ S ≤ L - D[N]

For a given S, the score is: sum over all passengers i of min over all doors j of |D[j] + S - P[i]|

I need to maximize this over valid S.

The function f(S) = sum_i min_j |D[j] + S - P[i]| is piecewise linear in S.

The breakpoints occur when a passenger is equidistant between two doors, i.e., when (D[j] + S + D[k] + S)/2 = P[i], meaning S = P[i] - (D[j]+D[k])/2, or when a passenger is exactly at a door position S = P[i] - D[j].

The number of breakpoints is O(M*N^2) which could be 300*300*300 = 27 million... that might be too many.

Actually, let me think differently. For each passenger i, min_j |D[j] + S - P[i]| = min_j |S - (P[i] - D[j])|. So passenger i contributes min_j |S - c_{i,j}| where c_{i,j} = P[i] - D[j].

This is the distance from S to the nearest point in set {P[i] - D[j] : j=1..N} for passenger i.

The function for each passenger is piecewise linear with breakpoints at midpoints between consecutive c_{i,j} values and at the c_{i,j} values themselves.

Total breakpoints: O(M*N) = 90000. That's manageable.

I'll collect all critical S values (breakpoints), evaluate f(S) at each, and find the maximum. But I also need to check midpoints between breakpoints since f is piecewise linear - actually the maximum of a piecewise linear function occurs at a breakpoint.

Let me collect all S values where the assignment of passengers to doors can change. These are: midpoints (c_{i,j} + c_{i,k})/2 for each passenger i and adjacent doors j,k in sorted order of c_{i,j}.

Also the endpoints 0 and L - D[N].

For each candidate S in [0, L-D[N]], evaluate f(S).

The number of candidates: for each passenger, there are N-1 midpoints, so M*(N-1) + 2 ≈ 90000 candidates. For each, evaluation takes O(M*N). Total: 90000 * 300 * 300 = 8.1 billion... too slow.

Actually evaluation is O(M*N): for each passenger, find min over N doors. That's 300*300 = 90000 per candidate S. With 90000 candidates: 90000 * 90000 = 8.1 billion. Too slow.

Wait, evaluation is O(M*N) = 90000, candidates = 90000, total = 8.1 billion operations. Still too slow.

Let me think about scanning S continuously. L ≤ 5000 but S can be fractional (see sample 2 with 0.5).

Actually L ≤ 5000, M,N ≤ 300. Let me try: candidates are O(M*N) = 90000, each evaluation O(M*N) = 90000. That's 8.1e9... too slow for 0.5s.

But wait - for each passenger i, the midpoints are between sorted {P[i]-D[j]}. Since D is sorted, P[i]-D[j] is decreasing in j. So for passenger i, sort the c values and compute midpoints: N-1 midpoints per passenger. Total candidates: M*(N-1) ≈ 90000. Evaluation per candidate: O(M*N) = 90000. Total = 8.1e9. Still too slow.

But M,N ≤ 300 means M*N = 90000 and candidates ≈ 90000. With simple operations, maybe it fits? Let me just try it and see.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int L,M,N;
    cin>>L>>M;
    vector<double>P(M);
    for(auto&x:P)cin>>x;
    cin>>N;
    vector<double>D(N);D[0]=0;
    for(int i=1;i<N;i++)cin>>D[i];
    double Smax=L-D[N-1];
    vector<double>cands;
    cands.push_back(0);cands.push_back(Smax);
    for(int i=0;i<M;i++){
        vector<double>c(N);
        for(int j=0;j<N;j++)c[j]=P[i]-D[j];
        sort(c.begin(),c.end());
        for(int j=0;j+1<N;j++)cands.push_back((c[j]+c[j+1])/2.0);
        for(int j=0;j<N;j++)cands.push_back(c[j]);
    }
    double bestS=0,bestVal=-1e18;
    for(double s:cands){
        if(s<0||s>Smax)continue;
        double val=0;
        for(int i=0;i<M;i++){
            double mn=1e18;
            for(int j=0;j<N;j++)mn=min(mn,fabs(D[j]+s-P[i]));
            val+=mn;
        }
        if(val>bestVal){bestVal=val;bestS=s;}
    }
    cout<<fixed<<setprecision(1)<<bestS<<" "<<bestVal<<endl;
}
```
