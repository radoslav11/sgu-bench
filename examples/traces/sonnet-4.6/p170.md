Looking at this problem, I need to find the minimum number of swaps (of adjacent different elements) to transform one string into another.

First, check if transformation is possible: both strings must have the same number of '+' and '-' characters.

If possible, find the minimum number of adjacent swaps needed.

Key insight: Since we can only swap adjacent elements of different types, this is essentially counting inversions. 

Let me think about this more carefully. The '+' characters need to go to specific positions, and same for '-'. 

Since swapping only happens between different adjacent characters, the relative order of '+' characters among themselves is preserved, and similarly for '-' characters. This is because two '+' characters can never swap (they're the same type), and two '-' characters can never swap.

So if we label the '+' positions in string 1 as p1[0], p1[1], ... and in string 2 as p2[0], p2[1], ..., the i-th '+' in string 1 must go to p2[i] (since relative order is preserved). Similarly for '-'.

The minimum number of swaps equals the sum of |p1[i] - p2[i]| for all '+' positions... wait, that's not quite right either.

Actually, when moving elements with preserved relative order, the minimum swaps is sum of |p1[i] - p2[i]| for each element... No, that's for a different problem.

Let me reconsider. Since relative order of same-type particles is preserved, we can match the i-th '+' in s1 to the i-th '+' in s2. Similarly for '-'.

The minimum number of adjacent swaps to sort is equivalent to counting inversions in a certain permutation. But here we have two types, not a full permutation.

Actually, I recall that when we have two types of elements and can only swap adjacent different elements, the minimum swaps is equivalent to: for each '+' at position i in s1, it maps to position p2[j] in s2 (where j is its rank among '+'s), and we need to compute the total "work" done.

Let me think of it as: create a sequence where position i gets the value (rank of that position's character type in s2). Then count inversions? 

Actually the answer is simply: sum over all i of |pos_plus1[i] - pos_plus2[i]|? No...

Let me think again. The minimum number of adjacent swaps = number of inversions in the permutation that maps s1 to s2, considering only the relative movements.

For two-element (binary) sequences: since '+' particles maintain relative order, and '-' particles maintain relative order, we match them by rank. The minimum number of swaps is actually: sum |p1_plus[i] - p2_plus[i]| for all i... 

Actually I think the answer is: consider all '+' positions in s1 (call them a[0..k-1]) and in s2 (call them b[0..k-1]). The minimum swaps = sum of |a[i] - b[i]|? Let me verify with the example.

s1 = ++-+--+  positions of '+': 0,1,3,6; positions of '-': 2,4,5
s2 = -++--++  positions of '+': 1,2,5,6; positions of '-': 0,3,4

Sum for '+': |0-1|+|1-2|+|3-5|+|6-6| = 1+1+2+0 = 4
Sum for '-': |2-0|+|4-3|+|5-4| = 2+1+1 = 4

But the answer is 4, not 8. So I'm double-counting. Each swap moves one '+' and one '-' simultaneously. So the total "displacement" is shared. Actually sum for '+' should equal sum for '-' (conservation), and the answer is that sum.

Indeed 4 = 4, and the answer is 4. So the answer is sum of |a[i] - b[i]| for '+' positions (which equals the same for '-').

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    string s1, s2;
    cin >> s1 >> s2;
    
    vector<int> plus1, minus1, plus2, minus2;
    for (int i = 0; i < (int)s1.size(); i++) {
        if (s1[i] == '+') plus1.push_back(i);
        else minus1.push_back(i);
    }
    for (int i = 0; i < (int)s2.size(); i++) {
        if (s2[i] == '+') plus2.push_back(i);
        else minus2.push_back(i);
    }
    
    if (plus1.size() != plus2.size() || minus1.size() != minus2.size()) {
        cout << -1 << endl;
        return 0;
    }
    
    long long ans = 0;
    for (int i = 0; i < (int)plus1.size(); i++) {
        ans += abs(plus1[i] - plus2[i]);
    }
    
    cout << ans << endl;
    return 0;
}
```
