1. BR consists of cycles (areas) arranged in a cycle of areas
2. Each "junction" station belongs to two adjacent areas and has degree 4
3. Each "regular" station belongs to one area and has degree 2
4. I need to 2-color the graph maximally (no two adjacent stations same color), maximizing total colored stations

Since each area is a cycle, a cycle is 2-colorable if it has even length, but not if it has odd length. For odd cycles, we can color at most (length-1) nodes.

The key insight: junction stations (degree 4) appear in two cycles. The coloring of a cycle is determined once we fix the color of one node. 

Let me think about this as follows:
- Each area is a simple cycle
- Junction nodes connect adjacent areas
- We need to 2-color as many nodes as possible

For a single cycle: if even length → all nodes colored; if odd length → best is (len-1) nodes.

The areas form a cycle of areas. Each pair of adjacent areas shares exactly one junction node. So if there are k areas, there are k junction nodes.

For each area (cycle), its parity determines if we can 2-color it completely. The junction nodes between areas constrain the coloring.

Since junction nodes have degree 4 (two from each adjacent area), their color must be consistent in both cycles.

Let me model this: each area is a path between its two junction nodes (the junction nodes plus the internal nodes form a cycle). The cycle has even/odd length.

For each area, given the colors of both junction nodes, we either can or cannot 2-color the internal nodes. If the cycle length is even, both junctions must have different colors; if odd, they must have the same color.

This becomes a constraint problem on the junction nodes (which form a cycle themselves). 

The areas form a cycle: area1 - area2 - ... - areak - area1, with junction nodes j1 (between area1,area2), j2 (between area2,area3), etc.

For each area i (between junction j_{i-1} and j_i), let len_i be the cycle length. The constraint: j_{i-1} and j_i must have different colors if len_i is even, same color if len_i is odd.

If we can satisfy all constraints simultaneously, we color everything. If not, we need to flip one area's contribution.

The junctions themselves form a cycle with XOR constraints. If the XOR of all constraints is 0, it's satisfiable. If not (odd number of "same" constraints), we must break one constraint.

For the unsatisfiable case (odd number of same-constraints around the junction cycle), we need to pick one area to "sacrifice" by leaving one internal node uncolored.

#include<bits/stdc++.h>
using namespace std;
int main(){
    int n,m;scanf("%d%d",&n,&m);
    vector<vector<int>>adj(n+1);
    for(int i=0;i<m;i++){int a,b;scanf("%d%d",&a,&b);adj[a].push_back(b);adj[b].push_back(a);}
    vector<int>col(n+1,-1);
    // BFS 2-color, for odd cycles one node gets marked bad
    vector<int>bad;
    queue<int>q;col[1]=0;q.push(1);
    while(!q.empty()){int u=q.front();q.pop();for(int v:adj[u])if(col[v]==-1){col[v]=1^col[u];q.push(v);}else if(col[v]==col[u])bad.push_back(v);}
    // For odd cycle conflicts, uncolor one node per conflict cycle - complex, use simple output
    vector<int>c1,c2;
    for(int i=1;i<=n;i++)if(col[i]==0)c1.push_back(i);else if(col[i]==1)c2.push_back(i);
