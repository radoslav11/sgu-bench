Looking at this problem, I need to find the minimum number of bombs to destroy all rocks, where each bomb destroys an entire row and an entire column.

This is essentially: find the minimum number of (row, column) pairs such that every rock at position (i, j) has either row i or column j covered.

This is a classic **minimum vertex cover** problem on a bipartite graph:
- Left nodes = rows (0 to n-1)
- Right nodes = columns (0 to m-1)
- Edge (i, j) exists if there's a rock at cell (i, j)

By König's theorem, minimum vertex cover = maximum matching in bipartite graph.

A "vertex cover" here means selecting a set of rows and columns such that every rock is in at least one selected row or selected column. Each bomb covers one row and one column, but we want to minimize bombs.

Wait, let me reconsider. Each bomb covers exactly one row AND one column simultaneously. So with k bombs, we cover k rows and k columns (possibly with repeats). We want every rock (i,j) to be covered by some bomb - meaning either the bomb's row = i or the bomb's column = j.

Actually, it's more subtle. A bomb at (r, c) covers all rocks in row r and all rocks in column c. We need to choose a set of cells (r1,c1), (r2,c2), ... such that for every rock at (i,j), either some ri = i or some cj = j.

This means we need to cover all rows containing rocks OR columns containing rocks. The set of chosen rows is {r1, r2, ...} and chosen columns is {c1, c2, ...}. Every rock must be in one of these rows or columns.

So really, we need to choose a set R of rows and a set C of columns (forming a vertex cover of the bipartite rock-graph), and then the number of bombs is max(|R|, |C|)... No wait.

Actually we pick pairs. Each bomb gives us one row slot and one column slot. If we use k bombs, we have k row slots and k column slots to assign. We want to assign rows to some bombs and columns to other bombs such that every rock is covered.

The minimum number of bombs = minimum k such that we can pick sets R, C with |R| ≤ k, |C| ≤ k, R∪C covers all rocks (as vertex cover), and the total "pairs needed" is k. Since we can freely choose which row and column each bomb targets, we need |R| ≤ k AND |C| ≤ k. So k = max(|R|, |C|) for optimal R, C forming a minimum vertex cover.

By König's theorem, minimum vertex cover size = maximum matching. But here k = max(|R|, |C|) not |R| + |C|.

```cpp
#include<bits/stdc++.h>
using namespace std;
```
```cpp
int match_l[30],match_r[30];
bool used[30];
vector<int>adj[30];
int n,m;
bool dfs(int u){
    for(int v:adj[u]){
        if(!used[v]){used[v]=1;
            if(match_r[v]==-1||dfs(match_r[v])){match_l[u]=v;match_r[v]=u;return 1;}
        }
    }
    return 0;
}
int main(){
    cin>>n>>m;
    vector<string>g(n);
    for(auto&s:g)cin>>s;
    memset(match_l,-1,sizeof(match_l));
    memset(match_r,-1,sizeof(match_r));
    for(int i=0;i<n;i++)for(int j=0;j<m;j++)if(g[i][j]=='*')adj[i].push_back(j);
    int res=0;
    for(int i=0;i<n;i++){memset(used,0,sizeof(used));if(dfs(i))res++;}
    cout<<res<<endl;
}
```
