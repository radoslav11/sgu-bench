Looking at this problem, I need to:

1. Find dowry values A[1..N] such that:
   - For each promise (Xi, Yi, Ci): A[Xi] >= A[Yi] + Ci
   - -10000 <= A[i] <= 10000
   - A[N] - A[1] is minimized

The constraint A[Xi] >= A[Yi] + Ci is a difference constraint: A[Xi] - A[Yi] >= Ci.

This is a classic difference constraints / shortest path problem.

To minimize A[N] - A[1], we want to find the shortest path from node 1 to node N in the constraint graph.

**Setting up the graph:**
- For each constraint A[Xi] - A[Yi] >= Ci, add edge Yi -> Xi with weight Ci (meaning A[Xi] >= A[Yi] + Ci).
- Also need -10000 <= A[i] <= 10000, meaning:
  - A[i] >= -10000: add edge from source (node 0) to i with weight -10000
  - A[i] <= 10000: add edge from i to source with weight -10000 (A[0] >= A[i] - 10000, i.e., A[i] - A[0] <= 10000)

Actually, let me use a standard approach. Add a super-source node 0. For each node i (1..N):
- Edge 0 -> i with weight -10000 (A[i] >= -10000)
- Edge i -> 0 with weight 10000 (A[i] <= 10000, meaning A[0] >= A[i] - 10000, so A[i] - A[0] <= 10000, edge weight -(-10000)... 

Let me think differently. The solution is: run Bellman-Ford from a super-source connected to all nodes with weight 0. Then A[i] = dist[i]. But we need to incorporate the box constraints.

For box constraints: -10000 <= A[i] <= 10000:
- A[i] >= -10000: equivalent to A[i] - A[src] >= -10000, add edge src->i weight -10000
- A[i] <= 10000: equivalent to A[src] - A[i] >= -10000, add edge i->src weight -10000 (but src is virtual with value 0... this gets complicated)

Let me just set A[i] = dist[1->i] using Bellman-Ford (with the difference constraints edges), then shift everything so A[1] is at some value that satisfies the box constraints and minimizes A[N]-A[1].

Actually since we want to minimize A[N]-A[1], and the constraint graph gives us A[i] relative differences, let's:
1. Find shortest paths d[i] from node 1 to all nodes (using Bellman-Ford with edges from constraints).
2. Check feasibility (no negative cycles reachable between 1 and N path).
3. Set A[i] = d[i] + offset, choose offset to satisfy -10000 <= A[i] <= 10000.

The minimum required d[N]-d[1] is the answer. d[1]=0 always. We need to find valid offset.

For each node reachable: A[i] = d[i] + offset. Need -10000 <= d[i]+offset <= 10000 for all i.
So offset >= -10000 - d[i] and offset <= 10000 - d[i].
offset_min = max(-10000 - d[i]) = -10000 - min(d[i])
offset_max = min(10000 - d[i]) = 10000 - max(d[i])

If offset_min > offset_max, no solution.

For unreachable nodes, we can set them freely within [-10000, 10000].

Let me implement this. Need to handle negative cycles (output -1).

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int N,M;
    cin>>N>>M;
    vector<tuple<int,int,int>> edges;
    for(int i=0;i<M;i++){
        int x,y,c; cin>>x>>y>>c;
        // A[x] >= A[y] + c => edge y->x weight c
        edges.push_back({y,x,c});
    }
    const int INF=1e9;
    vector<int> dist(N+1,INF);
    dist[1]=0;
    for(int iter=0;iter<N;iter++){
        for(auto&[u,v,w]:edges){
            if(dist[u]!=INF && dist[u]+w>dist[v]) dist[v]=dist[u]+w;
        }
    }
    for(auto&[u,v,w]:edges){
        if(dist[u]!=INF && dist[u]+w>dist[v]){cout<<-1;return 0;}
    }
    int mn=0,mx=0;
    for(int i=1;i<=N;i++) if(dist[i]!=INF){mn=min(mn,dist[i]);mx=max(mx,dist[i]);}
    int omin=-10000-mn, omax=10000-mx;
    if(omin>omax){cout<<-1;return 0;}
    int offset=omin;
    for(int i=1;i<=N;i++){
        if(dist[i]==INF) cout<<-10000;
        else cout<<dist[i]+offset;
        if(i<N)cout<<" ";
    }
}
```
