Looking at this problem, I need to find a maximum number of "crystal types" where each crystal type corresponds to a set of edges (a cut) that blocks all paths from S to T, and no edge is used by more than one crystal type.

This is essentially finding the maximum number of **edge-disjoint cuts** between S and T.

Key insight: The minimum cut value equals the maximum flow. The maximum number of edge-disjoint cuts equals the minimum cut value (by a duality argument). Each unit of flow corresponds to one crystal type.

Wait, let me think more carefully. Each crystal type needs to be an S-T cut by itself. We want maximum number of such cuts where edges are disjoint across cuts.

This is related to the concept that the maximum number of edge-disjoint S-T cuts equals the minimum number of edges in any S-T path (i.e., the shortest path in terms of edge count). 

Actually, by Menger's theorem duality: max edge-disjoint cuts = min path length (number of edges in shortest S-T path).

Each cut must contain exactly one edge from each S-T path. If the shortest path has length `d`, we can create `d` edge-disjoint cuts by partitioning edges by their "level" in a BFS layered graph.

So the algorithm:
1. Run BFS from S to find shortest path distance `d` to T.
2. The answer is `d` crystal types.
3. For each level i (0 to d-1), the cut consists of all edges going from BFS-level i to BFS-level i+1... but we need to be careful to use each edge in exactly one cut.

Actually, each edge (u,v) where dist[u]=i and dist[v]=i+1 belongs to cut level i. This partitions all "forward" edges. But we only need edges that are actually on some S-T path in the min-cut structure.

Let me verify with the sample: 4 nodes, edges: 1-2, 1-3, 2-4, 3-4. S=1, T=4. BFS from 1: dist[1]=0, dist[2]=1, dist[3]=1, dist[4]=2. So d=2.

Cut 0 (level 0→1): edges from dist=0 to dist=1: edges 1-2 (edge 1), 1-3 (edge 2). That's crystal type 1: {1,2}.
Cut 1 (level 1→2): edges from dist=1 to dist=2: edges 2-4 (edge 3), 3-4 (edge 4). Crystal type 2: {3,4}.

This matches the expected output!

So the algorithm is:
1. BFS from S to get distances.
2. Number of crystals = dist[T].
3. For crystal i (1-indexed), output all edges where dist[u]=i-1 and dist[v]=i (or vice versa).

But wait - do we need ALL such edges, or just a subset? The condition is that for each crystal type, those edges alone cut all paths. Using all forward edges at level i guarantees this because any path from S to T must cross from level i-1 to level i.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    int N, M, S, T;
    cin >> N >> M >> S >> T;
    
    vector<pair<int,int>> edges(M);
    for(int i = 0; i < M; i++){
        cin >> edges[i].first >> edges[i].second;
    }
    
    // BFS from S
    vector<int> dist(N+1, -1);
    queue<int> q;
    dist[S] = 0;
    q.push(S);
    while(!q.empty()){
        int u = q.front(); q.pop();
        for(int i = 0; i < M; i++){
            int a = edges[i].first, b = edges[i].second;
            if(a == u && dist[b] == -1){ dist[b] = dist[u]+1; q.push(b); }
            if(b == u && dist[a] == -1){ dist[a] = dist[u]+1; q.push(a); }
        }
    }
    
    int d = dist[T];
    cout << d << "\n";
    
    for(int level = 0; level < d; level++){
        vector<int> cut;
        for(int i = 0; i < M; i++){
            int a = edges[i].first, b = edges[i].second;
            if((dist[a] == level && dist[b] == level+1) ||
               (dist[b] == level && dist[a] == level+1)){
                cut.push_back(i+1);
            }
        }
        cout << cut.size();
        for(int x : cut) cout << " " << x;
        cout << "\n";
    }
    
    return 0;
}
```
