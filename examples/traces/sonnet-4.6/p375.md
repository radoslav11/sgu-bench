Looking at this problem, I need to find the minimum number of amplifiers to go from voltage 1 to voltage N.

Type 1: X → 2X-1
Type 2: X → 2X+1

Starting from 1, I need to reach N.

Let me think backwards. Starting from N, I want to reach 1.

Reverse operations:
- If current value Y came from type 1 (Y = 2X-1), then X = (Y+1)/2. This is valid when Y is odd and (Y+1)/2 ≥ 1, i.e., Y ≥ 1.
- If current value Y came from type 2 (Y = 2X+1), then X = (Y-1)/2. This is valid when Y is odd and (Y-1)/2 ≥ 1, i.e., Y ≥ 3.

So both reverse operations only work when Y is odd.

What if Y is even? Then we can't directly reverse. But wait, can we reach even numbers?

Starting from 1 (odd):
- Type 1: 2*1-1 = 1 (odd)
- Type 2: 2*1+1 = 3 (odd)

From any odd X:
- 2X-1 is odd
- 2X+1 is odd

So all reachable values are odd! If N is even (and N > 1), there's no solution.

Special case: N = 1, no amplifiers needed (0 amplifiers).

Now working backwards from N (which must be odd):
- At each step, if current = 1, we're done.
- If current is odd > 1: we can apply reverse of type 1 (gives (current+1)/2) or reverse of type 2 (gives (current-1)/2).
  - We want to minimize steps, so we use BFS or greedy.

Since both operations divide by ~2, this is like binary representation. Let me think greedily working backwards:

At each step with value Y (odd, >1):
- Reverse type 1: Y → (Y+1)/2
- Reverse type 2: Y → (Y-1)/2

We want to reach 1 in minimum steps.

Note (Y+1)/2 and (Y-1)/2 differ by 1. One of them might be even, allowing further halving without cost... wait no, each reversal counts as one amplifier.

Actually since Y is always odd in our forward path, both (Y+1)/2 and (Y-1)/2 are integers. But are they odd?
- Y odd: Y+1 is even, (Y+1)/2 could be odd or even.
- Similarly (Y-1)/2.

If (Y+1)/2 is even, then from forward perspective we'd be at an even number which can't be reached. So we must choose the path that stays odd, OR we need to continue reducing.

Wait, but working backwards we just need to track what value we'd be at going forward. Let me reconsider.

Working backwards, at each step we reduce. The greedy approach: always pick the path that leads to 1 fastest.

If Y ≡ 1 (mod 4): (Y-1)/2 is even... hmm but that's a backward step, meaning in forward we'd go from (Y-1)/2 to Y. But (Y-1)/2 might be even, which is unreachable from 1.

So we must pick the backward operation that keeps values reachable (odd or equal to... hmm).

Let me just use BFS for small N or think about the greedy more carefully.

Since N can be up to 2*10^9, we need O(log N) approach. Working backwards:
- Y is odd. If Y == 1, done.
- Option A: (Y+1)/2 — reverse type1
- Option B: (Y-1)/2 — reverse type2

We need to keep going until we hit 1. Greedily, we want to minimize steps. Both options roughly halve Y, so total steps ≈ log2(N). We just try both and pick minimum (but this could branch exponentially... actually no, because we only have ~31 steps max and can use memoization/recursion with memo).

Let me implement recursive search with memoization.

```cpp
#include <bits/stdc++.h>
using namespace std;

map<long long, pair<int,vector<int>>> memo;

pair<int,vector<int>> solve(long long n) {
    if (n == 1) return {0, {}};
    if (n % 2 == 0) return {INT_MAX, {}};
    if (memo.count(n)) return memo[n];
    
    // reverse type1: n = 2x-1 => x = (n+1)/2, then forward was type1
    // reverse type2: n = 2x+1 => x = (n-1)/2, then forward was type2
    auto r1 = solve((n+1)/2);
    auto r2 = solve((n-1)/2);
    
    pair<int,vector<int>> res;
    if (r1.first == INT_MAX && r2.first == INT_MAX) res = {INT_MAX, {}};
    else if (r1.first <= r2.first && r1.first != INT_MAX) {
        res = r1; res.first++; res.second.push_back(1);
    } else {
        res = r2; res.first++; res.second.push_back(2);
    }
    return memo[n] = res;
}

int main(){
    long long n; cin >> n;
    if(n==1){cout<<0<<endl<<endl;return 0;}
    auto [cnt, seq] = solve(n);
    if(cnt == INT_MAX){cout<<"No solution"<<endl;return 0;}
    cout << cnt << "\n";
    for(int x: seq) cout << x << " ";
    cout << "\n";
}
```
