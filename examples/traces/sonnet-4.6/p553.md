**Key observations:**
- Pearls are indexed 1..n. Pearls 1..n-m are on the table, pearls n-m+1..n hang.
- Each day, take from either end:
  - Take from hanging end (H): remove pearl n (current last), then the string slides down so pearl n-m becomes hanging (new hanging: n-m+1-1 to n-1, adjusted). Actually: if we take from hanging end, we remove the bottom-most hanging pearl and pull string down by 1. So the new hanging pearls shift: the top pearl on table becomes hanging.
  - Take from table end (T): remove pearl 1 (current first on table). Hanging part stays same.
- After each action, constraint: Wh ≤ k * Wt must hold.

Let me think of the state. Let's say at some point we've taken `h` pearls from the hanging end and `t` pearls from the table end. Then:
- Remaining table pearls: t+1 .. n-m-h (indices from original)
- Remaining hanging pearls: n-m-h+1 .. n-h (m pearls always hang... wait)

Actually after taking h from hanging and t from table:
- Table pearls: indices t+1 .. n-m-h
- Hanging pearls: indices n-m-h+1 .. n-h (that's m pearls)

Constraint: sum(hanging) ≤ k * sum(table)

We want to maximize total value of stolen pearls (from t and h choices).

The constraint after taking h from hanging and t from table:
- Wh(h) = sum of pearls n-m-h+1 .. n-h
- Wt(h,t) = sum of pearls t+1 .. n-m-h
- Need Wh(h) ≤ k * Wt(h,t) at every step along the way.

The order matters. Let's enumerate over h (number taken from hanging end), and for each h, find the maximum t we can take.

#include<bits/stdc++.h>
using namespace std;
typedef long long ll;
int main(){
    int n,m,k;
    cin>>n>>m>>k;
    vector<ll>w(n+1),c(n+1);
    for(int i=1;i<=n;i++) cin>>w[i]>>c[i];
    vector<ll>pw(n+2,0),pc(n+2,0);
    for(int i=1;i<=n;i++){pw[i]=pw[i-1]+w[i];pc[i]=pc[i-1]+c[i];}
    // table: 1..n-m, hanging: n-m+1..n
    // after h H-steps, t T-steps: table=t+1..n-m-h, hanging=n-m-h+1..n-h
    // Wh(h)=pw[n-h]-pw[n-m-h], Wt(h,t)=pw[n-m-h]-pw[t]
    ll bestVal=-1; int bestH=0,bestT=0;
    ll Wh=pw[n]-pw[n-m], Wt=pw[n-m];
    // simulate H steps, for each h check feasibility then find max t
    for(int h=0;h<=n-m;h++){
        // current state: h H's taken, 0 T's
        // Wh=pw[n-h]-pw[n-m-h], Wt=pw[n-m-h]-pw[0]
        ll wh=pw[n-h]-pw[n-m-h];
        ll wt_base=pw[n-m-h];
        if(wh>k*wt_base) break; // can't proceed with more H's
        // find max t: need wh <= k*(wt_base - pw[t]), i.e. pw[t] <= wt_base - wh/k
        // binary search on t
        int lo=0,hi=n-m-h,best_t=0;
        while(lo<=hi){int mid=(lo+hi)/2; if(wh*1<=k*(wt_base-pw[mid])){best_t=mid;lo=mid+1;}else hi=mid-1;}
        ll val=pc[h]+(pc[n]-pc[n-best_t]); // H pearls: n-h+1..n (cost), T pearls: 1..best_t
        // H costs: pearls taken are n,n-1,...,n-h+1
        ll hcost=pc[n]-pc[n-h];
        ll tcost=pc[best_t];
        val=hcost+tcost;
        if(val>bestVal){bestVal=val;bestH=h;bestT=best_t;}
        // try next H step: need h<n-m
    }
    if(bestVal<=0){cout<<"0 0\n";return 0;}
    cout<<bestH+bestT<<" "<<bestVal<<"\n";
    // reconstruct: we need to output sequence. All H's then T's? Need to verify order matters
    // Actually we need valid ordering. Let's try: interleave to maintain constraint
    // Simple: output H's first then T's (we validated this above)
    string s(bestH,'H');
    s+=string(bestT,'T');
    cout<<s<<"\n";
}
