Looking at this problem, I need to find:
- List A: cities that **might** be visited in some longest increasing subsequence (LIS) where both x and y coordinates are strictly increasing
- List B: cities that **must** be visited in every longest increasing subsequence

First, let me understand the structure:
1. Find the length of the LIS (longest chain where both x and y are strictly increasing)
2. List A = cities that appear in at least one such longest chain
3. List B = cities that appear in every such longest chain

**Approach:**

Sort cities by x (and by y if x is equal, but since we need strictly increasing in both coordinates, ties in x can't both be in the sequence).

For each city i, compute:
- `dp[i]` = length of longest chain ending at city i
- `dp2[i]` = length of longest chain starting at city i

The LIS length L = max(dp[i]).

A city i is in list A if `dp[i] + dp2[i] - 1 == L`.

A city i is in list B if it's in A AND it's the **unique** city with `dp[i] = k` for its particular value k among all A-cities.

For LIS in 2D (both x and y strictly increasing): sort by x, then find LIS on y values (strictly increasing).

Since we need both `dp` (forward) and `dp2` (backward), let me:
- Sort by x ascending, compute LIS lengths ending at each element (on y-values, strictly increasing)
- Sort by x descending, compute LIS lengths starting at each element (on -y values, strictly increasing, i.e., y strictly decreasing)

For the "must visit" cities: among all cities in A, those where for their dp value k, exactly one city in A has dp[i] = k.

Let me implement this carefully.

For computing `dp[i]` (forward LIS on y after sorting by x):
- Sort cities by x ascending; if x equal, sort by y descending (so equal x don't contribute to each other in strictly increasing)
- Use patience sorting / BIT on compressed y values

For `dp2[i]` (backward):
- Sort by x descending; if x equal, sort by y ascending
- Find LIS on (-y) values strictly increasing = y strictly decreasing

Let me code this up:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    int n; cin>>n;
    vector<int> x(n),y(n);
    for(int i=0;i<n;i++) cin>>x[i]>>y[i];
    
    // Compute dp1[i] = LIS length ending at i (strictly increasing x and y)
    // Sort by x asc, y desc for strict
    vector<int> idx(n); iota(idx.begin(),idx.end(),0);
    sort(idx.begin(),idx.end(),[&](int a,int b){ return x[a]!=x[b]?x[a]<x[b]:y[a]>y[b]; });
    
    // Compress y
    vector<int> ys=y; sort(ys.begin(),ys.end()); ys.erase(unique(ys.begin(),ys.end()),ys.end());
    auto cy=[&](int v){ return lower_bound(ys.begin(),ys.end(),v)-ys.begin()+1; };
    int M=ys.size();
    vector<int> bit(M+2,0);
    auto upd=[&](int p,int v){ for(;p<=M;p+=p&-p) bit[p]=max(bit[p],v); };
    auto qry=[&](int p){ int r=0; for(;p>0;p-=p&-p) r=max(r,bit[p]); return r; };
    
    vector<int> dp1(n),dp2(n);
    for(int i:idx){ int c=cy(y[i]); dp1[i]=qry(c-1)+1; upd(c,dp1[i]); }
    
    fill(bit.begin(),bit.end(),0);
    sort(idx.begin(),idx.end(),[&](int a,int b){ return x[a]!=x[b]?x[a]>x[b]:y[a]<y[b]; });
    // For dp2, we want strictly decreasing y, so use M+1-cy(y[i])
    for(int i:idx){ int c=M+1-cy(y[i]); dp2[i]=qry(c-1)+1; upd(c,dp2[i]); }
    
    int L=0; for(int i=0;i<n;i++) L=max(L,dp1[i]+dp2[i]-1);
    
    vector<int> A,B;
    for(int i=0;i<n;i++) if(dp1[i]+dp2[i]-1==L) A.push_back(i+1);
    
    // B: for each level k in [1..L], count cities in A with dp1[i]==k; if exactly 1, add to B
    map<int,vector<int>> byLevel;
    for(int v:A) byLevel[dp1[v-1]].push_back(v);
    for(auto&[k,vec]:byLevel) if(vec.size()==1) B.push_back(vec[0]);
    sort(B.begin(),B.end());
    
    cout<<A.size(); for(int v:A) cout<<" "<<v; cout<<"\n";
    cout<<B.size(); for(int v:B) cout<<" "<<v; cout<<"\n";
}
```
