Looking at this problem, I need to construct an N²×N² grid where:
1. Every row has exactly K blue cells ('*')
2. Every column has exactly K blue cells
3. Every N×N block has exactly K blue cells

Let me think about a construction approach.

The grid is divided into N×N blocks, each of size N×N. There are N² blocks total (arranged in an N×N grid of blocks).

Looking at the examples:
- N=2, K=2: Each row of 4 cells has 2 stars, each column has 2 stars, each 2×2 block has 2 stars.
- N=3, K=1: Each row of 9 cells has 1 star, each 3×3 block has 1 star.

For N=3, K=1: The pattern looks like a Latin square / permutation matrix structure. In each block row, each column of blocks has exactly one star per block row.

Let me think of a construction based on permutations.

Consider placing stars such that within each N×N block, exactly K stars appear. 

A key observation: if we can define for each block (br, bc) (block row, block column) a set of K positions (local row, local col), we need:
- In each global row: exactly K stars total (one per block? No, K per block-row's contribution)
- Actually, each global row belongs to one block-row. In that global row, we look across all N blocks in that block-row. Each block must contribute cells in that global row, and the total across all N blocks must be K.

Wait, each block has K stars total distributed across N rows. Let's say each block-row of a block gets K/N stars... but K might not be divisible by N.

Let me try a different approach: use a pattern based on modular arithmetic.

For cell (i, j), mark it blue if `(i/N + j/N + i%N ... )` satisfies some condition.

Looking at sample 2 (N=3, K=1): The star in row i is at column (i * 4) % 9? Row 0→col 0, row 1→col 3, row 2→col 6, row 3→col 1, row 4→col 4, row 5→col 7, row 6→col 2, row 7→col 5, row 8→col 8. So column = (i * (N+1)) % N²? i*(N+1) mod N²: 0,4,8,3,7,2,6,1,5... no. Actually it's (i*3)%9 for col: 0,3,6,0... no. It's col[i] = (3*(i%3) + i/3) % 9? For i=0:0, i=1:3, i=2:6, i=3:1, i=4:4, i=5:7, i=6:2, i=7:5, i=8:8. Yes! col = (N*(i%N) + i/N).

This is a permutation. For K=1 with N=3, each row has 1 star and each column has 1 star (it's a permutation matrix), and each block has 1 star.

For general K, we can take K such permutations (with offsets) and OR them together, ensuring no overlaps.

The permutation is: `col(i) = (N*(i%N) + i/N + offset) % N²`

For offset 0,1,...,K-1, we get K permutations. We need them to be disjoint.

Two permutations with offsets o1, o2 collide if `(N*(i%N) + i/N + o1) % N² == (N*(i%N) + i/N + o2) % N²`, which means `(o1-o2) % N² == 0`. So all N² offsets 0..N²-1 give disjoint permutations! 

So for K ≤ N², we can use offsets 0..K-1 and union the K permutation matrices.

But wait, K can be up to N². If K=N², all cells are blue - valid. If K=0, no cells - valid.

Each permutation contributes exactly 1 star per row, 1 per column, and 1 per block. So K permutations give K stars per row, column, and block. This works as long as K ≤ N² and the permutations are disjoint (which they are for offsets 0..K-1 since N²×N² grid).

Let me verify with sample 1 (N=2, K=2): offsets 0 and 1.
- Offset 0: col(i) = (2*(i%2) + i/2) % 4: i=0→0, i=1→2, i=2→1, i=3→3
- Offset 1: col(i) = (2*(i%2) + i/2 + 1) % 4: i=0→1, i=1→3, i=2→2, i=3→0

Row 0: cols 0,1 → `**..` but sample shows `*..*`. Different but still valid!

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int N, K;
    cin >> N >> K;
    int N2 = N * N;
    if(K < 0 || K > N2){
        cout << "NO SOLUTION\n";
        return 0;
    }
    vector<string> grid(N2, string(N2, '.'));
    for(int off = 0; off < K; off++){
        for(int i = 0; i < N2; i++){
            int col = (N*(i%N) + i/N + off) % N2;
            grid[i][col] = '*';
        }
    }
    for(auto& row : grid) cout << row << "\n";
}
```
