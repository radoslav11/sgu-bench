Looking at this problem, I need to:

1. Maintain a collection of ice cream pieces with prices
2. For BUY queries, check if the n cheapest pieces cost ≤ t total
3. If yes, sell them (remove from collection); if no, output UNHAPPY

Key operations needed:
- Add n pieces at price c
- Find sum of n cheapest pieces
- Remove n cheapest pieces if affordable

I'll use an ordered map (price → count) to store ice cream inventory, plus a Fenwick tree or just iterate through the sorted map to find the n cheapest sum.

Since prices can be up to 10^6 and there are up to 10^5 queries each with n up to 10^6, I could have up to 10^11 pieces total - too many to store individually.

I'll use a `std::map<int, long long>` mapping price to count, and for BUY queries, iterate through the map in order (cheapest first) accumulating sum until I have n pieces.

With up to 10^5 queries, and each BUY potentially iterating through distinct price levels - in worst case 10^5 distinct prices - this could be 10^10 operations total. But actually each BUY iterates at most O(distinct prices) = O(10^5), so total is O(10^10) which is too slow.

Wait, but realistically there are at most 10^5 queries, so at most 10^5 distinct prices. Each BUY iterates through at most 10^5 entries. With 10^5 BUY queries that's 10^10... but with the 0.5 second limit that's too slow.

However, in practice the number of distinct price levels encountered in a BUY is bounded by the total number of ARRIVE queries before it, which is at most 10^5. But iterating through all of them each BUY is O(10^5) per query, O(10^10) total - too slow.

Let me think of a better approach. I could use a Fenwick tree (BIT) indexed by price (1 to 10^6) storing:
- count of items at each price
- sum of prices at each price

Then for a BUY n query, I binary search for the smallest price P such that count[1..P] >= n, then compute the cost.

With a BIT over 10^6 elements, each query is O(log(10^6)) ≈ 20 operations. Binary search over the BIT is O(log^2) or O(log) with the "walk down BIT" technique.

I'll maintain two BITs:
- `cnt[i]` = total count of items with price ≤ i
- `sum[i]` = total sum of prices of items with price ≤ i

For BUY n t:
1. Binary search (or walk BIT) to find minimum price level where cumulative count ≥ n
2. Compute cost = sum of (n - count up to price-1) items at that price + sum up to price-1
3. If cost ≤ t: output HAPPY, remove those items from BIT
4. Else: UNHAPPY

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXP = 1000001;
long long cnt[MAXP+1], sm[MAXP+1];

void update(int i, long long dc, long long ds) {
    for (int x = i; x <= MAXP; x += x & (-x)) {
        cnt[x] += dc;
        sm[x] += ds;
    }
}

pair<long long,long long> query(int i) {
    long long c = 0, s = 0;
    for (int x = i; x > 0; x -= x & (-x)) {
        c += cnt[x];
        s += sm[x];
    }
    return {c, s};
}

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    string op;
    while(cin >> op) {
        if(op == "ARRIVE") {
            long long n, c;
            cin >> n >> c;
            update(c, n, n*c);
        } else {
            long long n, t;
            cin >> n >> t;
            auto [tc, ts] = query(MAXP);
            if(tc < n) { cout << "UNHAPPY\n"; continue; }
            // binary search for smallest price where cumulative count >= n
            int lo = 1, hi = MAXP, ans = MAXP;
            while(lo <= hi) {
                int mid = (lo+hi)/2;
                if(query(mid).first >= n) { ans = mid; hi = mid-1; }
                else lo = mid+1;
            }
            auto [c1, s1] = query(ans-1);
            long long need = n - c1;
            long long cost = s1 + need * ans;
            if(cost <= t) {
                cout << "HAPPY\n";
                update(ans, -need, -need*ans);
            } else {
                cout << "UNHAPPY\n";
            }
        }
    }
}
```
