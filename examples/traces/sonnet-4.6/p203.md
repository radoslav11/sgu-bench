The classic Huffman encoding problem. I need to efficiently build the Huffman tree and compute the total encoded length.

Key insight: Use two queues approach. Since the input is sorted, we can use:
1. Queue 1: original frequencies (sorted)
2. Queue 2: merged node weights (these come out in non-decreasing order)

Always pick the two smallest from the fronts of the two queues.

The answer is the sum of all internal node weights (each merge adds to the total encoded length).

Let me trace through the example:
- N=3, frequencies: 1 1 4
- Queue1: [1, 1, 4], Queue2: []
- Pick 1, 1 → merge = 2, Queue2: [2], total += 2
- Pick 2 (from Q2), 4 (from Q1) → merge = 6, Queue2: [2, 6], total += 6
- Total = 8 ✓

The answer needs to be a 64-bit integer since Pi can be up to 1e9 and N up to 500000, and internal nodes can sum up large values. Maximum possible: 500000 * 1e9 * log2(500000) ≈ 5e14 * 19 ≈ 9.5e15, fits in long long.

Special case: N=1? Problem says N≥2, so no issue. But what if N=2? Then we pick both, merge, total += sum. Each character gets code length 1.

Let me implement this:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    int n;
    cin >> n;
    
    deque<long long> q1, q2;
    
    for (int i = 0; i < n; i++) {
        long long x;
        cin >> x;
        q1.push_back(x);
    }
    
    long long total = 0;
    
    // Special case: if only 1 character... but N>=2
    
    while (q1.size() + q2.size() > 1) {
        // Pick two smallest
        long long a, b;
        
        // Pick first
        if (q2.empty() || (!q1.empty() && q1.front() <= q2.front())) {
            a = q1.front(); q1.pop_front();
        } else {
            a = q2.front(); q2.pop_front();
        }
        
        // Pick second
        if (q2.empty() || (!q1.empty() && q1.front() <= q2.front())) {
            b = q1.front(); q1.pop_front();
        } else {
            b = q2.front(); q2.pop_front();
        }
        
        long long merged = a + b;
        total += merged;
        q2.push_back(merged);
    }
    
    cout << total << endl;
    
    return 0;
}
```

- N=3, q1=[1,1,4], q2=[]
- Iteration 1: a=1(q1), b=1(q1), merged=2, total=2, q2=[2]
- Iteration 2: q1=[4], q2=[2]. Compare 4 vs 2 → a=2(q2). q1=[4], q2=[]. b=4(q1). merged=6, total=8, q2=[6]
- q1=[], q2=[6], size=1 → stop
- Output: 8 ✓
