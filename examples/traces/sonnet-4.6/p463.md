Looking at this problem, I need to simulate a pedestrian walking through a grid of blocks.

Key points:
- Grid is n rows × m columns of blocks
- Streets (n+1 horizontal lines) and avenues (m+1 vertical lines)
- Pedestrian starts at north-west corner facing east
- L = turn left 90°, R = turn right 90°, M = move one block forward
- When passing a block side, satisfaction increases by b[i][j] first time, b[i][j]/2 (floor) subsequent times

I need to track how many times each block has been passed (each of its 4 sides).

Let me think about coordinates:
- Position is at intersection (row, col) where row ∈ [0..n], col ∈ [0..m]
- Start: (row=0, col=0), facing East
- Directions: East=(0,+1), South=(+1,0), West=(0,-1), North=(-1,0)

When moving from one intersection to another, we pass a block:
- Moving East from (r,c) to (r,c+1): we pass the south side of block (r-1, c) if r>0, and north side of block (r, c) if r<n. Wait, actually moving along a street passes the blocks above and below the street.

Actually, when moving along a street segment between (r,c) and (r,c+1), we pass the **north side** of block (r, c) (block at row r, col c) and the **south side** of block (r-1, c).

Hmm, but the problem says "passes a block" when walking along any of the four sides. So each block side can be passed multiple times.

For each block (i,j) (0-indexed, i∈[0..n-1], j∈[0..m-1]), I'll track a pass count. First pass gives b[i][j], subsequent passes give floor(b[i][j]/2).

When moving East along row r from col c to c+1:
- Block (r, c) has its north side traversed (if r < n, i.e., block exists below)
- Block (r-1, c) has its south side traversed (if r > 0, i.e., block exists above)

When moving South along col c from row r to r+1:
- Block (r, c) has its west side traversed (if c < m)
- Block (r, c-1) has its east side traversed (if c > 0)

Each "side" of a block is separate - I need to track passes per side, not per block. Wait, re-reading: "every time the pedestrian passes a block his satisfaction is increased." So it's per block, not per side. If you pass the same side twice or two different sides, the second pass is halved.

So I need pass_count[i][j] = total number of times block (i,j) has been passed (any side).

Let me implement this:

- Moving East at intersection row r, col c→c+1: passes block (r,c) [north side] and block (r-1,c) [south side]
- Moving West at intersection row r, col c→c-1: passes block (r,c-1) [north side] and block (r-1,c-1) [south side]  
- Moving South at intersection col c, row r→r+1: passes block (r,c) [west side] and block (r,c-1) [east side]
- Moving North at intersection col c, row r→r-1: passes block (r-1,c) [west side] and block (r-1,c-1) [east side]

For each block passed, add b[i][j] if first time, else floor(b[i][j]/2).

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    int n, m;
    cin >> n >> m;
    int b[100][100];
    for(int i=0;i<n;i++){
        string s; cin>>s;
        for(int j=0;j<m;j++) b[i][j]=s[j]-'0';
    }
    string path; cin>>path;
    
    int pass[100][100]={};
    int total=0;
    
    auto passBlock=[&](int i,int j){
        if(i<0||i>=n||j<0||j>=m) return;
        if(pass[i][j]==0) total+=b[i][j];
        else total+=b[i][j]/2;
        pass[i][j]++;
    };
    
    // directions: 0=East,1=South,2=West,3=North
    int dr[]={0,1,0,-1};
    int dc[]={1,0,-1,0};
    int r=0,c=0,dir=0;
    
    for(char ch:path){
        if(ch=='L') dir=(dir+3)%4;
        else if(ch=='R') dir=(dir+1)%4;
        else { // M
            int nr=r+dr[dir], nc=c+dc[dir];
            if(dir==0){passBlock(r,c);passBlock(r-1,c);}
            else if(dir==2){passBlock(r,nc);passBlock(r-1,nc);}
            else if(dir==1){passBlock(r,c);passBlock(r,c-1);}
            else{passBlock(nr,c);passBlock(nr,c-1);}
            r=nr;c=nc;
        }
    }
    cout<<total<<endl;
}
```
