Looking at this problem, Petya is at position (0,0) (bottom-left corner), and enemies are at all other cells of an N×M grid.

The enemies are at positions (i,j) where 0≤i<N, 0≤j<M, except (0,0).

Each shot destroys all enemies along a straight line through (0,0). Two enemies (i1,j1) and (i2,j2) are on the same line through (0,0) if they have the same direction, i.e., (i1,j1) = k*(i2,j2) for some scalar k, which means i1/j1 = i2/j2, i.e., the ratio i:j is the same (in reduced form).

So the number of shots = number of distinct directions from (0,0) to any enemy cell.

Each direction is characterized by (i,j) in reduced form (divide by gcd). But we need to count distinct pairs (a,b) where a = i/gcd(i,j), b = j/gcd(i,j), for all (i,j) ≠ (0,0) with 0≤i<N, 0≤j<M.

Note: i and j are both non-negative (since grid indices 0 to N-1 and 0 to M-1). Also i≥0, j≥0. The direction (0,0) is excluded (that's Petya's cell).

Possible cases:
- (i=0, j>0): direction (0,1) — only one such direction
- (i>0, j=0): direction (1,0) — only one such direction  
- (i>0, j>0): direction (a,b) where gcd(a,b)=1, a≥1, b≥1, a<N, b<M... wait, not exactly. We need some multiple of (a,b) to exist in the grid.

For direction (a,b) with gcd(a,b)=1, a≥1, b≥1: this direction is achievable if there exists k≥1 such that k*a < N and k*b < M. The minimum such k is 1, so we just need a < N and b < M. Wait, k=1 means a<N and b<M. But actually k could be larger. The direction (a,b) is achievable if ∃k≥1: k*a ≤ N-1 and k*b ≤ M-1, i.e., k ≤ (N-1)/a and k ≤ (M-1)/b. So we need floor((N-1)/a) ≥ 1 and floor((M-1)/b) ≥ 1, i.e., a ≤ N-1 and b ≤ M-1.

So the number of shots = (N≥2 ? 1 : 0) + (M≥2 ? 1 : 0) + number of pairs (a,b) with a≥1, b≥1, gcd(a,b)=1, a≤N-1, b≤M-1.

Wait, but when N=1: no enemies with i>0 except those on the bottom row. When N=1, all enemies are at (0,j) for j=1..M-1, so just 1 shot (if M≥2). Similarly M=1.

Let me verify with examples:
- N=3, M=4: directions: (0,1)=1, (1,0)=1, plus pairs (a,b) with a∈[1,2], b∈[1,3], gcd=1.
  - (1,1),(1,2),(1,3),(2,1),(2,3) = 5 pairs. Total = 1+1+5=7. ✓
- N=2, M=10: (0,1)=1, (1,0)=1, pairs (a,b): a∈[1,1], b∈[1,9], gcd(1,b)=1 always: 9 pairs. Total=1+1+9=11. ✓

So the answer is: (N≥2) + (M≥2) + #{(a,b): 1≤a≤N-1, 1≤b≤M-1, gcd(a,b)=1}.

The last count is sum over a=1 to N-1 of #{b: 1≤b≤M-1, gcd(a,b)=1} = sum_{a=1}^{N-1} sum_{b=1}^{M-1} [gcd(a,b)=1].

This equals sum_{d=1}^{min(N-1,M-1)} μ(d) * floor((N-1)/d) * floor((M-1)/d).

N,M up to 10^6, so we need Möbius function up to 10^6 and then iterate. This is O(min(N,M)) which is fine.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    long long N,M;
    cin>>N>>M;
    if(N==1&&M==1){cout<<0;return 0;}
    long long A=N-1,B=M-1;
    int lim=max(A,B);
    vector<int> mu(lim+1,0);
    vector<bool> notprime(lim+1,false);
    vector<int> primes;
    mu[1]=1;
    for(int i=2;i<=lim;i++){
        if(!notprime[i]){primes.push_back(i);mu[i]=-1;}
        for(int j=0;j<(int)primes.size()&&(long long)i*primes[j]<=lim;j++){
            notprime[i*primes[j]]=true;
            if(i%primes[j]==0){mu[i*primes[j]]=0;break;}
            else mu[i*primes[j]]=-mu[i];
        }
    }
    long long ans=0;
    long long mn=min(A,B);
    for(long long d=1;d<=mn;d++){
        if(mu[d]!=0) ans+=mu[d]*(A/d)*(B/d);
    }
    ans+=(N>=2?1:0)+(M>=2?1:0);
    cout<<ans<<endl;
}
```
