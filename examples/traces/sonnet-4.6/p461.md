Looking at this problem, I need to parse wiki-text with list markers (`*` and `#`) and convert to HTML.

Key observations:
1. Each line starts with some prefix of `*` and `#` characters, followed by content
2. Lines need to be grouped into nested lists based on their prefixes
3. The prefix determines the nesting structure

Let me think about the algorithm:

For each line, I parse the leading list markers as a "path" (e.g., `**#` means nested ul > ul > ol), and the rest is content.

I need to maintain a current "stack" of open tags. When processing each line:
1. Compare the current line's prefix with the current stack state
2. Close tags that differ, open new ones as needed

Let me think more carefully. The prefix characters define the list structure. For example:
- `*Item` → depth 1, type `ul`
- `**Item` → depth 2, types `ul, ul`
- `*#Item` → depth 2, types `ul, ol`

The current stack tracks which list types are open. When moving from one line to next:
- Find the common prefix length
- Close all open lists deeper than common prefix
- If at the common prefix level the type changed, close that level too and reopen
- Open new levels as needed
- Each list item needs `<li>...</li>` wrapping

Actually, each line is a list item at its depth level. The content of the `<li>` is either plain text or includes nested lists.

Let me think recursively: group consecutive lines by their first marker character, then recursively process the rest of each line's prefix.

Algorithm:
1. Parse all lines into (prefix_string, content) pairs
2. Process groups of lines

For a set of lines starting at some position, if they all have no prefix, just print them. Otherwise, group consecutive lines by their first prefix character, and for each group, emit `<ul>` or `<ol>`, then for each line in the group, emit `<li>`, recursively process the line with prefix stripped by 1, emit `</li>`.

Wait, but "a group of two or more consecutive lines" - what about a single line? Looking at sample: `*ItemZ` is a single line and it's printed as just `*ItemZ` (raw). And `#3` is also printed raw.

So a list only forms when there are **2 or more** consecutive lines with the same first marker!

Let me re-read: "A group of two or more consecutive lines should be treated as elements of the same list"

So single lines with markers are NOT converted - they're printed as-is.

And for nested lists: "A group of two or more consecutive list elements should be considered as a nested list if these elements start with the same list marker symbol." - also requires 2 or more.

So the rule applies at every level recursively: only 2+ consecutive same-marker lines form a list.

Algorithm: Given a list of lines (strings), process them by grouping consecutive lines that start with the same marker character, but only if the group has size >= 2.

```cpp
#include <bits/stdc++.h>
using namespace std;

void process(vector<string>& lines) {
    int n = lines.size();
    int i = 0;
    while (i < n) {
        char c = lines[i][0];
        if (c == '*' || c == '#') {
            int j = i;
            while (j < n && lines[j][0] == c) j++;
            if (j - i >= 2) {
                string tag = (c == '*') ? "ul" : "ol";
                cout << "<" << tag << ">\n";
                for (int k = i; k < j; k++) {
                    cout << "<li>\n";
                    vector<string> sub = {lines[k].substr(1)};
                    process(sub);
                    cout << "</li>\n";
                }
                cout << "</" << tag << ">\n";
            } else {
                cout << lines[i] << "\n";
            }
            i = j;
        } else {
            cout << lines[i] << "\n";
            i++;
        }
    }
}

int main() {
    vector<string> lines;
    string s;
    while (getline(cin, s)) lines.push_back(s);
    process(lines);
}
```
