Looking at this problem, I need to find which dragons to kill to maximize gold while ensuring:
1. The knight doesn't accidentally propose to any intermediate princess (i.e., at each princess cell before cell n, the number of dragons killed so far must be strictly less than her beauty)
2. At cell n (which is a princess), the number of dragons killed must be >= her beauty

Let me think about this:

- The knight visits all cells in order
- At each princess (except the last), if killed_count >= beauty, the journey ends early (bad)
- At the last princess, we need killed_count >= beauty_n

So the constraints are:
- For each intermediate princess at cell i with beauty b_i: dragons killed among cells 2..i-1 (dragons only) must be < b_i
- For last princess at cell n with beauty b_n: dragons killed must be >= b_n

We want to maximize total gold collected while satisfying these constraints.

Let's denote dragons as d_1, d_2, ..., d_m with positions and gold values, appearing in order. For each princess at position p_j (intermediate), let c_j = number of dragons before position p_j. The constraint is: among the first c_j dragons (in order of position), we kill fewer than b_j of them.

For the last princess, total dragons killed >= b_n.

Strategy: We need to select a subset S of dragons to kill such that:
- For each intermediate princess j: |S ∩ {dragons before position j}| < b_j
- |S| >= b_n (total)
- Sum of gold in S is maximized

To maximize gold while respecting the prefix constraints: at each intermediate princess with beauty b_j, among the first c_j dragons, we can kill at most b_j - 1 of them.

This is a greedy problem. We want to pick dragons greedily by value, but subject to prefix capacity constraints.

The constraints form: for each princess at position p_j, prefix of dragons up to p_j can have at most b_j - 1 kills.

We also need total kills >= b_n.

Approach: 
- Process dragons in order. For each "segment" ending at a princess, we have a capacity constraint.
- We want to maximize total gold with these prefix constraints, and total >= b_n.

This looks like we can use a greedy with a min-heap: 

Go through all cells. Maintain a min-heap of gold values of killed dragons. When we encounter a dragon, add it to a "candidates" pool. When we encounter an intermediate princess with beauty b, we need kills so far < b. So if current kills >= b, we must drop the minimum gold dragon until kills < b.

After processing all cells up to (but not including) the last princess, we need total kills >= b_n. We should maximize gold, so we greedily kill the highest value dragons, subject to prefix constraints.

Let me reconsider: we maintain a set of "killed" dragons and "available (unkilled)" dragons. At each intermediate princess, if killed_count >= b, we swap the minimum-gold killed dragon with nothing (just unkill it).

After all intermediate princesses, we need killed_count >= b_n. If current killed_count < b_n, we need to kill more dragons from the available ones. We should pick the highest gold ones.

Let me implement this:
1. Keep a min-heap of killed dragons' gold values
2. Keep a max-heap of unkilled dragons' gold values  
3. At each intermediate princess with beauty b: while killed_count >= b, remove minimum from killed heap (move to unkilled)
4. After processing all, if killed_count < b_n: greedily add from max-heap of unkilled until killed_count >= b_n or no more available
5. If killed_count < b_n after that, output -1

Need to track which cells are killed, so store (gold, cell_index) in heaps.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    int n;
    cin >> n;
    vector<char> type(n+1);
    vector<int> val(n+1);
    for(int i = 2; i <= n; i++){
        cin >> type[i] >> val[i];
    }
    
    // min-heap of killed: (gold, cell)
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> killed;
    // max-heap of unkilled dragons seen so far
    priority_queue<pair<int,int>> unkilled;
    
    for(int i = 2; i < n; i++){
        if(type[i] == 'd'){
            killed.push({val[i], i});
        } else {
            int b = val[i];
            while((int)killed.size() >= b){
                auto mn = killed.top(); killed.pop();
                unkilled.push(mn);
            }
        }
    }
    int bn = val[n];
    while((int)killed.size() < bn && !unkilled.empty()){
        killed.push(unkilled.top()); unkilled.pop();
    }
    if((int)killed.size() < bn){ cout << -1; return 0; }
    long long gold = 0;
    vector<int> cells;
    while(!killed.empty()){ auto [g,c]=killed.top();killed.pop();gold+=g;cells.push_back(c);}
    sort(cells.begin(),cells.end());
    cout<<gold<<"\n"<<cells.size()<<"\n";
    for(int c:cells) cout<<c<<" ";
}
```
