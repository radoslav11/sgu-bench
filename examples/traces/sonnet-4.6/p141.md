Looking at this problem, I need to find non-negative integers P1, N1, P2, N2 such that:
- P1*x1 - N1*x1 + P2*x2 - N2*x2 = P
- P1 + N1 + P2 + N2 = K

Let me reformulate. Let a = P1 - N1 (net jumps of x1) and b = P2 - N2 (net jumps of x2).
Then: a*x1 + b*x2 = P

Also, P1 + N1 = some value k1, P2 + N2 = k2, where k1 + k2 = K.
Given a = P1 - N1 and k1 = P1 + N1, we need k1 >= |a| and k1 ≡ a (mod 2).
Similarly k2 >= |b| and k2 ≡ b (mod 2).

So for each valid (a, b) with a*x1 + b*x2 = P, I need to find k1, k2 >= 0 such that:
- k1 + k2 = K
- k1 >= |a|, k1 ≡ a (mod 2)
- k2 >= |b|, k2 ≡ b (mod 2)

Strategy: iterate over a (number of net x1 jumps), compute b = (P - a*x1) / x2 if divisible.

Range of a: since |a*x1| can be at most K*x1, but K can be up to 2e9. However, P is bounded by 40000, so a*x1 + b*x2 = P. Since x1, x2 > 0, |a| <= (|P| + |b|*x2)/x1. 

Actually, let's think differently. We need a*x1 + b*x2 = P where a ranges... but a and b can be large. However, we can iterate over a in range that covers all useful values. The key insight: we only need to try a from some bounded range, because if we have a solution (a,b), we can adjust: replace a by a+x2, b by b-x1 (this keeps the equation valid). So solutions repeat with period related to lcm.

Let me iterate a from -K to K would be too slow. Instead, iterate a from a small range. Since a*x1 + b*x2 = P, and we can vary a by multiples of x2/gcd and b by x1/gcd, we can find a base solution and then enumerate within feasible range.

The feasible condition on k1: k1 >= |a|, k1 ≡ a (mod 2), and k2 = K - k1 >= |b|, k2 ≡ b (mod 2).

So k1 is in [|a|, K-|b|] and k1 ≡ a (mod 2) and (K-k1) ≡ b (mod 2), meaning k1 ≡ K-b (mod 2)... so a ≡ K-b (mod 2), i.e., a+b ≡ K (mod 2).

Also a+b ≡ K (mod 2) must hold.

Let me iterate a over a range of size ~2*x2 (to cover one period), find corresponding b, check feasibility.

Range for a: at most |a| + |b| <= K (since k1+k2=K), so not directly helpful. But the minimum k1+k2 = |a|+|b| <= K. Let me just iterate a from -2*40000 to 2*40000 (since P < 40000 and if |a|>some bound, |b| would need to be negative large making |a|+|b| > K unless K is huge).

Actually I'll iterate a from roughly -K to K but that's too big. Let me limit: |a| <= K and |b| <= K, but also |a*x1| <= K*x1. The range for a is at most about (P + K*x2)/x1 which could be huge.

Better: iterate a from -40000*2/x1 range... Let me just try a from -80000 to 80000.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    long long x1,x2,P,K;
    cin>>x1>>x2>>P>>K;
    for(long long a=-80000;a<=80000;a++){
        long long rem=P-a*x1;
        if(rem%x2!=0) continue;
        long long b=rem/x2;
        long long mink=abs(a)+abs(b);
        if(mink>K) continue;
        if((K-mink)%2!=0) continue;
        if((a+b)%2!=0 && K%2!=(a+b)%2) continue;
        // need k1>=|a|, k2>=|b|, k1+k2=K, k1≡a mod2, k2≡b mod2
        // k1=|a|+2t, k2=K-k1>=|b| => t<=(K-|a|-|b|)/2
        long long k1=abs(a), k2=K-k1;
        if(k2<abs(b)||(k2-abs(b))%2!=0){ k1=abs(a)+1; k2=K-k1; }
        if(k2<abs(b)||(k2-abs(b))%2!=0) continue;
        long long P1=(k1+a)/2, N1=(k1-a)/2, P2=(k2+b)/2, N2=(k2-b)/2;
        if(P1<0||N1<0||P2<0||N2<0) continue;
        cout<<"YES\n"<<P1<<" "<<N1<<" "<<P2<<" "<<N2<<"\n";
        return 0;
    }
    cout<<"NO\n";
}
```
