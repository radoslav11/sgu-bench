Looking at this problem, I need to maximize total intensity subject to:
1. Total volume ≤ A
2. Total cost ≤ B

For fuel type i, if I buy m_i kg:
- Volume: m_i * a_i
- Cost: m_i * b_i  
- Intensity: m_i * c_i

This is a linear program. I want to maximize Σ m_i * c_i subject to:
- Σ m_i * a_i ≤ A
- Σ m_i * b_i ≤ B
- m_i ≥ 0

This is a 2-constraint LP. The optimal solution will use at most 2 variables (basic feasible solution), or we might be able to identify the optimal by looking at ratios.

For a 2-constraint LP with non-negativity, the optimal BFS has at most 2 non-zero variables. So I need to check:
1. All pairs of fuel types (i, j) that form a basis
2. Single fuel types using one constraint binding
3. Cases where both constraints bind simultaneously

But N can be 75000, so O(N²) pairs would be ~5.6 billion - too slow.

Wait, let me think differently. We can parameterize: suppose we spend fraction λ of our "budget" on volume and (1-λ) on cost. For each fuel type, the "effective rate" is c_i / (α*a_i + β*b_i) where α and β are the dual variables.

Actually, for a fixed ratio of shadow prices (u for volume, v for cost), we'd pick the fuel with highest (c_i - u*a_i - v*b_i). 

The LP has 2 constraints, so optimal is achieved at a vertex of the feasible polytope, which means at most 2 fuels are used. We need to find the best pair.

For each pair (i,j), solve:
- m_i * a_i + m_j * a_j = A (or ≤)
- m_i * b_i + m_j * b_j = B (or ≤)

With N=75000 and 0.5s time limit, O(N²) is too slow.

But wait: a_i, b_i, c_i are integers 0-100. So there are at most 101³ ≈ 10^6 distinct fuel types. But N=75000 so many are distinct still...

Actually the key insight: since a_i, b_i ∈ {0,...,100}, there are at most 101*101 = ~10201 distinct (a,b) pairs. For each distinct (a,b) pair, we want the maximum c. So we reduce to at most 10201 distinct fuels!

Then O(10201²) ≈ 10^8 pairs... might be tight but let's try.

Actually with deduplication we get at most 10201 fuels, and O(n²) with n=10201 is ~10^8 which might be too slow for 0.5s.

But we also need to check single-fuel solutions and degenerate cases (a_i=0 or b_i=0).

Let me implement:
1. Deduplicate: keep best c for each (a,b) pair
2. For single fuels: max intensity using all A volume or all B cost
3. For pairs: solve the 2x2 system and check feasibility

For the pairs loop with n up to ~10201, O(n²) is ~10^8 which is borderline. Let me try to optimize.

Actually, let me also handle edge cases: a_i=0 means no volume cost; b_i=0 means free fuel.

```cpp
#include <bits/stdc++.h>
using default_type = double;
using namespace std;
int main(){
    int N,A,B;
    scanf("%d%d%d",&N,&A,&B);
    // deduplicate by (a,b), keep max c
    map<pair<int,int>,int> best;
    for(int i=0;i<N;i++){
        int a,b,c; scanf("%d%d%d",&a,&b,&c);
        auto key=make_pair(a,b);
        best[key]=max(best[key],c);
    }
    vector<tuple<int,int,int>> fuels;
    for(auto&p:best) fuels.push_back({p.first.first,p.first.second,p.second});
    int n=fuels.size();
    double ans=0;
    for(int i=0;i<n;i++){
        auto[ai,bi,ci]=fuels[i];
        // use only fuel i
        double m=1e18;
        if(ai>0) m=min(m,(double)A/ai);
        if(bi>0) m=min(m,(double)B/bi);
        if(ai==0&&bi==0) m=0; // can't get intensity if both 0? c might be 0 too
        ans=max(ans,m*ci);
        for(int j=i+1;j<n;j++){
            auto[aj,bj,cj]=fuels[j];
            double det=(double)ai*bj-(double)aj*bi;
            if(abs(det)>1e-9){
                double mi=((double)A*bj-(double)aj*B)/det;
                double mj=((double)ai*B-(double)A*bi)/det;
                if(mi>=0&&mj>=0) ans=max(ans,mi*ci+mj*cj);
            }
        }
    }
    printf("%.6f\n",ans);
}
```
