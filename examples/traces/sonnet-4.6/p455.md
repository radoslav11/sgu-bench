Looking at this problem, I need to:

1. Generate the sequence x[0] = 1, x[i] = (A * x[i-1] + B) % C
2. Find the first repeatable element M (the earliest element that appears again later)
3. Find the index of M's second occurrence
4. Output that index if ≤ 2·10^6, else -1

The key challenge: find the *first* element that repeats (not just any repeated element, but the one with the smallest first index that also appears again).

With the limit of 2·10^6, I can store up to 2·10^6 + 1 elements and check.

Approach:
- Generate sequence up to 2·10^6 elements
- For each element, track when it first appeared
- Find the cycle: eventually the sequence must repeat a value (since values are bounded by C, or we hit the limit)
- Once we detect a repeated value (xq = xp for q > p), we need to find the minimum p such that xp appears again somewhere after p

Let me think more carefully. The sequence will eventually cycle. Once we find some repeated value (first time xq = xp), the cycle is from p to q-1. The first repeatable element is the one with the smallest index p that is part of this cycle or appears again.

Actually, I'll use Floyd's or just direct tracking:
- Store all values and their first-seen indices in a map
- Generate elements one by one up to 2·10^6
- When I see a value that was seen before at index `first`, I've found that x[first] repeats at current index `i`
- But I need the *minimum* first index among all such repeating elements

Wait - the sequence is deterministic. Once a value repeats, we're in a cycle. The first time we see a duplicate (say x[i] = x[j] with j < i), x[j] is repeatable. But could there be an earlier element x[k] (k < j) that is also repeatable?

The first repeated value we encounter (first collision at index i where x[i] = x[j], j < i) - x[j] is repeatable. But elements before j might also be repeatable if they appear in the cycle.

Hmm, let me think: the sequence has a "rho" shape - a tail leading into a cycle. Elements in the tail that are not in the cycle are NOT repeatable. Elements in the cycle ARE repeatable. The first repeatable element is the first element of the cycle encountered in the sequence, i.e., the element where the tail meets the cycle (index = start of cycle).

So I need to find the start of the cycle. That's exactly the element x[p] where p is the smallest index such that x[p] appears again later.

Algorithm:
- Use a hash map to store first occurrence of each value
- Generate sequence; when x[i] collides with x[j] (j < i), the cycle start is x[j], and it repeats at index i... but wait, j might not be the cycle start.

Actually when x[i] = x[j] with j < i and i is the first collision ever detected, then x[j] is indeed the entry point of the cycle (since all previous elements were distinct). So j is the cycle start, and the second occurrence is at index i.

Wait - is it guaranteed that the first collision gives us the cycle entry? Yes! Because we process in order. The first time we see a duplicate, say x[i] = x[j], j < i, and all x[0]...x[i-1] were distinct. Then x[j] is the first repeatable element (it appears at i > j), and nothing before j repeats (since all were distinct before this collision). So the answer is i.

So the answer is simply the index of the first collision!

Let me verify with examples:
- Example 1: 1,3,7,6,3 → x[4]=3=x[1], answer=4 ✓
- Example 2: 1,...,0,0 → x[5]=0=x[4], answer=5 ✓  
- Example 3: 1,-2,4,-3,1 → x[4]=1=x[0], answer=4 ✓

So just find the first i where x[i] was seen before, output i (if ≤ 2·10^6), else -1.

#include <bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    long long A, B, C;
    cin >> A >> B >> C;
    
    unordered_map<long long, int> seen;
    seen.reserve(1 << 22);
    seen.max_load_factor(0.25);
    
    long long x = 1;
    int limit = 2000000;
    
    for(int i = 0; i <= limit; i++){
        if(seen.count(x)){
            cout << i << endl;
            return 0;
        }
        seen[x] = i;
        x = (A * x + B) % C;
    }
    
    cout << -1 << endl;
    return 0;
}
