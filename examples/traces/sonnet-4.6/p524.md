Looking at this problem, I need to place n buoys at equally spaced positions while minimizing the total distance moved.

Let me think about this. The final positions form an arithmetic sequence: `a, a+d, a+2d, ..., a+(n-1)*d` where `a` is the starting position and `d` is the spacing.

The total shift is `sum |x_i - (a + (i-1)*d)|` for i=1..n.

Since the order must be preserved and coordinates strictly increasing, we need `d > 0`.

The constraints: the endpoints. Actually we need to figure out `a` and `d`. 

Key insight: The first and last buoys' positions are `a` and `a+(n-1)*d`. These aren't necessarily fixed.

Let me think about this differently. Let `y_i = a + (i-1)*d` be the new position of buoy i. We minimize `sum |x_i - y_i|`.

Let's substitute: let `z_i = x_i - (i-1)*d`. Then `y_i = a + (i-1)*d`, so `x_i - y_i = x_i - a - (i-1)*d = z_i - a`. So we minimize `sum |z_i - a|` which is minimized when `a` is the median of `z_i`.

But `d` is also a variable! So we need to minimize over both `a` and `d`.

For fixed `d`, define `z_i(d) = x_i - (i-1)*d`. Then the optimal `a` is the median of `z_i(d)`, and the cost is `sum |z_i(d) - median(z_i(d))|`.

So I need to minimize over `d > 0`: `f(d) = sum |x_i - (i-1)*d - median_d|` where `median_d` is the median of `{x_i - (i-1)*d}`.

The function `f(d)` is piecewise linear and convex. The breakpoints occur when two `z_i(d)` values are equal, i.e., `x_i - (i-1)*d = x_j - (j-1)*d` → `d = (x_i - x_j)/((i-1)-(j-1)) = (x_i-x_j)/(i-j)`.

So I can enumerate all O(n²) candidate values of `d`, plus do ternary search since `f(d)` is convex.

Let me use ternary search on `d`. The range of `d`: minimum is 0 (but must be > 0), maximum is `(x_n - x_1)/(n-1)` perhaps... actually d could be larger if we compress, but that would increase shifts. The answer's d is at most `(x_n - x_1)/(n-1)` since that's fixed endpoints. Actually d can range quite freely.

Given `d` ranges, since `x_i` are between -10000 and 10000, and n up to 400, reasonable range for d is [0, 20000].

Let me do ternary search on d in [0, 20000] with many iterations (say 200), then for each d compute median of z_i and the cost.

But wait - we also need d > 0 to maintain strict order. The ternary search should be fine.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<double> x(n);
    for (int i = 0; i < n; i++) cin >> x[i];
    
    auto cost = [&](double d) -> pair<double,double> {
        vector<double> z(n);
        for (int i = 0; i < n; i++) z[i] = x[i] - i * d;
        vector<double> sz = z;
        sort(sz.begin(), sz.end());
        double a = sz[n/2]; // median (for odd, middle; for even, either works)
        // Try both medians for even n
        double best = 1e18;
        double ba = a;
        for (int m = (n-1)/2; m <= n/2; m++) {
            double aa = sz[m];
            double c = 0;
            for (int i = 0; i < n; i++) c += fabs(z[i] - aa);
            if (c < best) { best = c; ba = aa; }
        }
        return {best, ba};
    };
    
    double lo = 0, hi = 20000;
    for (int iter = 0; iter < 300; iter++) {
        double m1 = lo + (hi-lo)/3, m2 = hi - (hi-lo)/3;
        if (cost(m1).first < cost(m2).first) hi = m2;
        else lo = m1;
    }
    double d = (lo+hi)/2;
    auto [c, a] = cost(d);
    
    printf("%.4f\n", c);
    for (int i = 0; i < n; i++) printf("%.10f%c", a + i*d, " \n"[i==n-1]);
}
```
