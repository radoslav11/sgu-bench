Looking at this problem:

- We maintain a multiset of segments
- For `+ L R`: count how many existing segments [l,r] satisfy L≤l and r≤R, then add [L,R]
- For `- L R`: remove one occurrence of [L,R] if it exists

Since at most 1000 segments are present simultaneously, and up to 250k queries, we can afford O(n) scan for each query.

I'll use a multiset (or just a vector/list) to store segments, and for each `+` query, iterate through all current segments counting those that are non-strictly inside [L,R].

For the multiset approach, I'll store pairs and use a multiset to handle duplicates.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    multiset<pair<int,int>> segs;
    
    char op;
    int L, R;
    while(cin >> op >> L >> R){
        if(op == '+'){
            int cnt = 0;
            for(auto& s : segs){
                if(s.first >= L && s.second <= R) cnt++;
            }
            cout << cnt << "\n";
            segs.insert({L, R});
        } else {
            auto it = segs.find({L, R});
            if(it != segs.end()) segs.erase(it);
        }
    }
    return 0;
}
```
