Looking at this problem, I need to find the minimum time to fill M balloons using N volunteers with a single device.

Key observations:
- Each minute, exactly one volunteer uses the device (or no one, but that would be suboptimal)
- Volunteer i fills Ai balloons per minute they operate, but after operating must rest Bi minutes before operating again
- I need to fill at least M balloons total

This is a scheduling problem. Let me think about it with BFS/DP over states.

State: (time, balloons_filled, rest_timers_for_each_volunteer)

The rest timer for volunteer i: how many more minutes they must rest before being available.
- When volunteer i works at minute t, they fill Ai balloons and then must rest Bi minutes, so they're available again at t + 1 + Bi (i.e., rest_timer = Bi after working).
- Each minute that passes, rest timers decrease by 1.

State = (balloons_so_far, rest_timers[N]) → minimum time to reach this state.

Since M ≤ 100, Bi ≤ 4, N ≤ 10: the state space is manageable.

Rest timers for each volunteer: 0 to 4 (0 means available). Each volunteer has 5 possible states.
State space: 101 * 5^10 ≈ 101 * ~10M = too large.

Wait, 5^10 ≈ 9.7 million, times 101 ≈ ~1 billion. Too large.

Let me think differently. Actually the rest timers are bounded by Bi ≤ 4, so state is (balloons, rest_timers) where each rest_timer is 0..4.

But maybe we can do BFS where state is (balloons, rest_timers) and we want minimum time (steps). Since we want minimum time and each step is 1 minute, BFS gives shortest path.

5^10 * 101 is about 1 billion - too large. But N ≤ 10 and Bi ≤ 4, so maybe in practice we can use a map or note that many states are unreachable.

Alternative: since M is small (≤ 100), and Ai ≤ 10, we'd need at least ceil(M / max_Ai) minutes. The answer is at most ~100 minutes (if one person fills 1 per minute). 

Let me use DP with state = (time, rest_timers[]) minimizing time to reach M balloons. Since time is at most say 500, and 5^10 ≈ 10M states... 500 * 10M = 5 billion, still too large.

But wait: rest_timer[i] ≤ Bi ≤ 4, so actually the product is product of (Bi+1) for each volunteer, which for N=10 and all Bi=4 gives 5^10 ≈ 10M. With time up to 500... hmm.

Let me try BFS with state (balloons, rest_timers) and track minimum time. Since once balloons >= M we stop, and balloons only increases, the BFS explores at most 101 * 5^10 ≈ 1 billion states. That's too much.

However, in practice with N=10 and Bi all ≤ 4, let me try using a map from state to min_time, and BFS layer by layer (each layer = one minute).

Actually, let's cap at time=500 and use unordered_map. Or note that really most states won't be reached.

Let me just implement BFS with states encoded as integers and use arrays, keeping it within 5^10 * 101.

5^10 = 9,765,625. Times 101 ≈ 986 million - too large for memory.

Let me separate: track dp[rest_state] = {min_time, max_balloons_at_that_time}. Since we want min time to fill M balloons, at each time step, for each reachable (rest_state, balloons) pick a volunteer or idle.

Let me do BFS per time step: dp[t][rest_state] = max balloons achievable. Size: T * 5^10. With T=200, that's 2 billion. Still too large.

But actually: dp[rest_state] = max balloons, updated each minute. 5^10 ≈ 10M entries, feasible!

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int M,N; cin>>M>>N;
    vector<int> A(N),B(N);
    for(int i=0;i<N;i++) cin>>A[i]>>B[i];
    // state: rest timers encoded, dp[state]=max balloons
    int S=1; for(int i=0;i<N;i++) S*=(B[i]+1);
    vector<int> mul(N); mul[0]=1; for(int i=1;i<N;i++) mul[i]=mul[i-1]*(B[i-1]+1);
    vector<int> dp(S,-1); dp[0]=0;
    for(int t=1;t<=500;t++){
        vector<int> ndp(S,-1);
        for(int s=0;s<S;s++){
            if(dp[s]<0) continue;
            int balls=dp[s];
            // decode timers
            vector<int> tim(N); int tmp=s;
            for(int i=0;i<N;i++){tim[i]=tmp%(B[i]+1);tmp/=(B[i]+1);}
            // advance timers by 1 (rest)
            vector<int> ntim(N); for(int i=0;i<N;i++) ntim[i]=max(0,tim[i]-1);
            // option: idle
            int ns=0; for(int i=0;i<N;i++) ns+=ntim[i]*mul[i];
            if(ndp[ns]<balls) ndp[ns]=balls;
            // option: volunteer i works (tim[i]==0)
            for(int i=0;i<N;i++){
                if(tim[i]==0){
                    vector<int> wt=ntim; wt[i]=B[i];
                    int ws=0; for(int j=0;j<N;j++) ws+=wt[j]*mul[j];
                    int nb=min(balls+A[i],M);
                    if(ndp[ws]<nb) ndp[ws]=nb;
                }
            }
        }
        dp=ndp;
        for(int s=0;s<S;s++) if(dp[s]>=M){cout<<t;return 0;}
    }
}
```
