Looking at this problem, I need to find a "funny string" (Euclid string) of N elements summing to K.

A funny string S is one where rotating it gives S with S[0]+1 and S[N-1]-1. This is related to the Euclidean algorithm / Bresenham's line algorithm.

Let me think about Euclid strings. They're constructed similarly to how Euclidean rhythm/Bresenham works.

The key insight: A funny string of length N with sum K can be constructed using the Euclidean algorithm approach. Each element is either floor(K/N) or ceil(K/N). Since GCD(N,K)=1, K/N is never an integer, so elements are either floor(K/N) or floor(K/N)+1.

Let me think about the construction. The Bresenham/Euclidean string construction:

For N elements summing to K with GCD(N,K)=1:
- Each element is either `q = K/N` (integer division) or `q+1`
- The pattern of which positions get `q+1` follows a Bresenham-like distribution

The standard Euclidean string construction: position i gets value `floor((K*(i+1))/N) - floor((K*i)/N)`.

Let me verify with the example: N=9, K=16, q=1, r=7 (so 7 ones get +1, making them 2, and 2 stay as 1).

For i=0..8: floor(16*(i+1)/9) - floor(16*i/9)
- i=0: floor(16/9)-0 = 1
- i=1: floor(32/9)-floor(16/9) = 3-1=2
- i=2: floor(48/9)-floor(32/9) = 5-3=2
- i=3: floor(64/9)-floor(48/9) = 7-5=2
- i=4: floor(80/9)-floor(64/9) = 8-7=1
- i=5: floor(96/9)-floor(80/9) = 10-8=2
- i=6: floor(112/9)-floor(96/9) = 12-10=2
- i=7: floor(128/9)-floor(112/9) = 14-12=2
- i=8: floor(144/9)-floor(128/9) = 16-14=2

Result: 1 2 2 2 1 2 2 2 2 — this matches the sample output!

Now I need to verify this is indeed a funny string. The property says S1+1, S2, ..., SN-1, SN-1 is a rotation of the original. This is related to the three-distance theorem / Euclidean strings property, and since this is defined as "Euclid strings" in the problem, using this Bresenham construction should work.

#include <bits/stdc++.h>
using namespace std;

int main() {
    long long N, K;
    cin >> N >> K;
    
    for (int i = 0; i < N; i++) {
        long long val = (K * (long long)(i + 1)) / N - (K * (long long)i) / N;
        if (i > 0) cout << " ";
        cout << val;
    }
    cout << endl;
    
    return 0;
}
