Looking at this problem, I need to find pairs of segments that form an L-shape: they meet at exactly one endpoint (one endpoint of one segment coincides with one endpoint of the other) and they are perpendicular (90 degree angle).

Let me think about the approach:

1. For each pair of segments (i, j), check if they form an L-shape.
2. Two segments form an L-shape if:
   - One endpoint of segment i equals one endpoint of segment j
   - The segments are perpendicular (dot product of direction vectors = 0)

With N ≤ 5000, there are up to ~12.5 million pairs, which might be tight but should work within 0.75 seconds if the per-pair check is simple.

For perpendicularity: if segment i has direction (dx1, dy1) and segment j has direction (dx2, dy2), they are perpendicular if dx1*dx2 + dy1*dy2 = 0.

For endpoint matching: segment i has endpoints (x1,y1) and (x2,y2); segment j has endpoints (x3,y3) and (x4,y4). We need at least one pair of endpoints to match.

But wait - the condition is "one endpoint of one segment coincides with one endpoint of the other". So exactly one endpoint of one coincides with one endpoint of the other? Or at least one?

Looking at the example: segments 4 and 7 coincide (both are along the same line from (0,4) to (4,4)), but they're counted as separate segments. Pair (1,7) forms an L-shape. Segment 1 is (0,4)-(0,7), segment 7 is (0,4)-(4,4). They share endpoint (0,4) and are perpendicular. That works.

Now, what if multiple endpoints coincide? I think the condition is: at least one endpoint of segment i equals at least one endpoint of segment j, AND the segments are perpendicular.

But could both endpoints of one segment coincide with endpoints of the other? That would mean the segments are the same or one is contained in the other — but they'd need to be perpendicular too, which would require zero-length, so it can't happen with non-zero segments.

Actually, if all 4 endpoints are the same, the segments are identical, but they could share both endpoints. If segments share both endpoints but go in opposite directions — they're still the same segment geometrically. The direction vectors would be negatives of each other, so dot product = -(dx²+dy²) ≠ 0. So perpendicular segments can share at most... well, two distinct points defining perpendicular directions would be different.

I'll just check: for each pair (i,j), check if any endpoint of i equals any endpoint of j, AND the segments are perpendicular.

But if a pair shares multiple endpoint pairs, I should count the pair only once (not multiple times per shared endpoint).

So the algorithm:
- For each pair (i,j) where i < j:
  - Check if (p1 of i == p1 of j) OR (p1 of i == p2 of j) OR (p2 of i == p1 of j) OR (p2 of i == p2 of j)
  - Check if perpendicular: dot product of direction vectors = 0
  - If both, count it

With N=5000, that's 12.5M pairs, each with simple integer arithmetic. Should be fine.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    int n;
    cin >> n;
    
    vector<long long> x1(n), y1(n), x2(n), y2(n);
    vector<long long> dx(n), dy(n);
    
    for(int i = 0; i < n; i++){
        cin >> x1[i] >> y1[i] >> x2[i] >> y2[i];
        dx[i] = x2[i] - x1[i];
        dy[i] = y2[i] - y1[i];
    }
    
    long long count = 0;
    
    for(int i = 0; i < n; i++){
        for(int j = i+1; j < n; j++){
            // Check if they share an endpoint
            bool share = false;
            if(x1[i]==x1[j] && y1[i]==y1[j]) share = true;
            else if(x1[i]==x2[j] && y1[i]==y2[j]) share = true;
            else if(x2[i]==x1[j] && y2[i]==y1[j]) share = true;
            else if(x2[i]==x2[j] && y2[i]==y2[j]) share = true;
            
            if(!share) continue;
            
            // Check perpendicularity
            long long dot = dx[i]*dx[j] + dy[i]*dy[j];
            if(dot == 0){
                count++;
            }
        }
    }
    
    cout << count << endl;
    
    return 0;
}
```
