Looking at this problem, I need to analyze a combinatorial game theory problem on a chain graph.

Key observations:
1. The game is an impartial-like game but actually it's a partizan game since both players can use either color... wait, actually both players have the same moves available (can color any uncolored vertex with any valid color). So this IS an impartial game, and we can use Sprague-Grundy theory.

2. When a vertex is colored, it constrains its neighbors. The chain gets split into independent segments by the colored vertices.

3. Each segment of consecutive uncolored vertices between colored vertices (or endpoints) forms an independent game. We need the XOR of Grundy values.

Let me think about segments:
- A segment of uncolored vertices can have constraints on its ends:
  - Free end (at the boundary of the chain)
  - Constrained end (adjacent to a colored vertex, meaning one color is forbidden)

When you color a vertex in a segment, you split it into two subsegments. The color you choose affects the constraints on neighbors.

Let me think about the types of segments:
- Type FF: both ends free (no colored neighbors)
- Type FC: one end free, one end constrained (one color forbidden)
- Type CC: both ends constrained (both colors forbidden at respective ends)

Wait, but CC could mean same color forbidden on both ends, or different colors forbidden on both ends - same or different constraint matters.

Actually when both ends are constrained:
- If both ends have the same color constraint (same color forbidden), call it CC-same
- If both ends have different color constraints (different colors forbidden), call it CC-diff

Let me compute Grundy values for each type and length via game tree search.

For a segment of length n, when you color position i (1-indexed within segment), you pick a valid color and split into two sub-segments. I need to enumerate all possible moves.

Let me code up a Grundy value computation with memoization.

State: (left_constraint, right_constraint, length)
- left/right constraint: 0=free, 1=black forbidden, 2=white forbidden

For each position i in [1..n] and color c in {1,2}:
- Check if c is valid (not equal to left neighbor color if i==1, not equal to right neighbor color if n==1, etc.)
- Actually constraint means: if left_constraint=1, then color 1 is forbidden at position 1
- When coloring position i with color c: left segment has (left_constraint, c, i-1), right segment has (c, right_constraint, n-i)
- The Grundy value of a move is XOR of Grundy values of two resulting segments

Let me implement this with memoization.

After parsing the input, I identify the free segments and their boundary constraints, compute Grundy values, XOR them, and output FIRST if nonzero, SECOND otherwise.

```cpp
#include <bits/stdc++.h>
using default_random_engine = mt19937;
using namespace std;

map<tuple<int,int,int>, int> memo;

int grundy(int lc, int rc, int n) {
    // lc: 0=free,1=black forbidden,2=white forbidden on left end
    // rc: similar on right end
    if (n == 0) return 0;
    auto key = make_tuple(lc, rc, n);
    auto it = memo.find(key);
    if (it != memo.end()) return it->second;
    
    set<int> reachable;
    for (int i = 1; i <= n; i++) {
        for (int c = 1; c <= 2; c++) {
            // Check if coloring position i with color c is valid
            bool ok = true;
            if (i == 1 && lc == c) ok = false;
            if (i == n && rc == c) ok = false;
            if (!ok) continue;
            // left segment: positions 1..i-1, left constraint lc, right constraint c
            int gl = grundy(lc, c, i-1);
            // right segment: positions i+1..n, left constraint c, right constraint rc
            int gr = grundy(c, rc, n-i);
            reachable.insert(gl ^ gr);
        }
    }
    int mex = 0;
    while (reachable.count(mex)) mex++;
    memo[key] = mex;
    return mex;
}

int main(){
    int n;
    cin >> n;
    string s;
    cin >> s;
    
    int xorSum = 0;
    int i = 0;
    while (i < n) {
        if (s[i] != '0') { i++; continue; }
        // find segment of zeros
        int lc = (i > 0) ? (s[i-1]-'0') : 0;
        int j = i;
        while (j < n && s[j] == '0') j++;
        int rc = (j < n) ? (s[j]-'0') : 0;
        int len = j - i;
        xorSum ^= grundy(lc, rc, len);
        i = j;
    }
    
    cout << (xorSum != 0 ? "FIRST" : "SECOND") << endl;
    return 0;
}
```
