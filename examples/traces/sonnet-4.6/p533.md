Let me analyze this problem.

For a tower of k dice:
- The total sum of all faces of k dice = k * 21 (since each die has faces 1+2+3+4+5+6=21)
- We need to subtract the hidden faces (internal faces between dice, and the contact faces between adjacent dice)

For a tower of k dice:
- Between adjacent dice, there are (k-1) pairs of touching faces. Each pair consists of two faces that sum to some value. But we can choose how dice are oriented!
- The bottom face of the tower (touching the table) - wait, the problem says "outer surface" includes bottom face
- Actually, the outer surface = all faces except the internal touching faces between dice

Wait, let me reconsider. The dice are stacked. Between two adjacent dice, the top face of the lower die and the bottom face of the upper die are hidden (not on outer surface). There are (k-1) such pairs.

For k dice:
- Total faces = 6k
- Hidden faces = 2*(k-1) (the internal contact faces)
- Visible faces = 6k - 2*(k-1) = 4k + 2

Now, what's the sum of visible faces?
- Total sum of all faces = 21k
- We subtract the hidden faces' sum

The hidden faces: between adjacent dice, the top of lower die and bottom of upper die. On a standard die, opposite faces sum to 7. So if the top face shows value `t`, the bottom face shows `7-t`. 

For the interface between two dice: the top of lower die (value `a`) and bottom of upper die (value `b`). These can be chosen independently! `a` can be 1-6, `b` can be 1-6.

So the hidden sum ranges:
- For each of (k-1) interfaces: we hide `a` (top of lower) + `b` (bottom of upper), where a ∈ [1,6], b ∈ [1,6]
- Min hidden per interface = 1+1 = 2
- Max hidden per interface = 6+6 = 12

So for k dice:
- Sum of visible = 21k - hidden_sum
- hidden_sum ∈ [2*(k-1), 12*(k-1)]
- Visible sum ∈ [21k - 12*(k-1), 21k - 2*(k-1)]
- = [21k - 12k + 12, 21k - 2k + 2]
- = [9k + 12, 19k + 2]

For k=1: no hidden faces, sum = 21 (fixed). Range [21, 21].

Wait for k=1: [9+12, 19+2] = [21, 21]. Correct!

For k=2: [18+12, 38+2] = [30, 40]
Sample says n=32 → answer 2. 32 is in [30,40]. ✓

For k=3: [27+12, 57+2] = [39, 59]
Sample says n=50 → answer 3. 50 is in [39,59]. ✓

For n=7: Check k=1: only 21. k=2: [30,40]. k=3: [39,59]... 7 < 21, so no tower works → -1. ✓

So algorithm: find minimum k ≥ 1 such that n ∈ [9k+12, 19k+2].

Wait, for k=1: [21,21]. Let me verify: 9(1)+12=21, 19(1)+2=21. Yes.

For k≥2: check if 9k+12 ≤ n ≤ 19k+2.

We want minimum k. 

From 9k+12 ≤ n: k ≤ (n-12)/9
From n ≤ 19k+2: k ≥ (n-2)/19

So minimum k = ceil((n-2)/19), and we need to verify it satisfies 9k+12 ≤ n.

Special case k=1: n must equal 21.

Let me code this:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;
    
    // k=1: sum must be exactly 21
    if (n == 21) {
        cout << 1 << endl;
        return 0;
    }
    
    // For k >= 2: n in [9k+12, 19k+2]
    // minimum k >= ceil((n-2)/19)
    // need 9k+12 <= n => k <= (n-12)/9
    
    // find min k >= 2 such that 9k+12 <= n <= 19k+2
    if (n < 30) { // min for k=2 is 30
        cout << -1 << endl;
        return 0;
    }
    
    int k = max(2, (int)ceil((n - 2.0) / 19.0));
    
    if (9*k + 12 <= n && n <= 19*k + 2) {
        cout << k << endl;
    } else {
        cout << -1 << endl;
    }
    
    return 0;
}
```
