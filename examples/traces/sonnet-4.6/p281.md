Looking at this problem, I need to find the final rating based on two contest ratings.

The rule is:
- For each M from 1 to N, if the set of first M teams in rating 1 equals the set of first M teams in rating 2, then those M teams form the first M teams in the final rating.
- Within each such "block", teams are ordered alphabetically (since their relative order can't be determined by the rule).

Wait, let me re-read. "If for some M (1≤M≤N) the sets of first M teams in both ratings are the same, then this set is the set of the first M teams in the final rating."

So I need to find all valid M values where both top-M sets match. The final rating is built from these blocks.

The blocks are formed by consecutive valid M values. For example, if valid M values are 2, 5, 6 (and N=6), then:
- Block 1: positions 1-2 (the 2 teams in set M=2), sorted alphabetically
- Block 2: positions 3-5 (the teams in set M=5 minus set M=2), sorted alphabetically  
- Block 3: position 6 (team in set M=6 minus set M=5), sorted alphabetically

Wait, but M=6 is always valid since both ratings contain all N teams. So N is always a valid M.

Algorithm:
1. Read both ratings.
2. Find all M where top-M of rating1 == top-M of rating2.
3. These valid M values partition into blocks (consecutive differences).
4. For each block, output the teams in alphabetical order.

To efficiently find matching M values:
- Maintain a set of teams seen in rating1 so far and rating2 so far.
- As we process positions 1..N, keep track of teams added to each set.
- A position M is valid if the two sets are equal.

I can track the "difference" between the two sets: how many teams are in set1 but not set2, and vice versa. When both counts are 0, the sets are equal.
