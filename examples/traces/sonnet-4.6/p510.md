The number of distinct substrings of a string can be computed using suffix automaton or suffix array. Let me think about the approach.

For a string of length L, the number of distinct substrings is sum over all suffixes of (L - suffix_array[i] - lcp[i]) using suffix arrays, or equivalently using suffix automaton: sum of (len[v] - len[link[v]]) for all states v except the initial state.

Given n ≤ 300, the string won't be very long. Let me think about bounds: a string of length L has at most L*(L+1)/2 distinct substrings, so L*(L+1)/2 ≥ n means L ≈ 25 for n=300.

Approach: BFS/iterative deepening over string length, and for each length, try to find the lexicographically smallest string with exactly n distinct substrings.

But iterating over all strings is too slow. Instead, I'll do a greedy approach: build the string character by character, at each position try the smallest character ('a' first) and check if a valid completion exists.

For each partial string prefix, I need to know: can we extend it to some string of total length L with exactly n distinct substrings?

Strategy:

#include <bits/stdc++.h>
using namespace std;

int countDistinct(const string &s) {
    int n = s.size();
    // suffix automaton
    vector<int> len(2*n+5,0), link(2*n+5,-1), cnt(2*n+5,0);
    vector<array<int,26>> ch(2*n+5);
    for(auto &x:ch) x.fill(-1);
    int last=0,sz=1;
    auto extend=[&](int c){
        int cur=sz++;len[cur]=len[last]+1;
        int p=last;
        while(p!=-1&&ch[p][c]==-1){ch[p][c]=cur;p=link[p];}
        if(p==-1)link[cur]=0;
        else{int q=ch[p][c];if(len[q]==len[p]+1)link[cur]=q;else{int clone=sz++;len[clone]=len[p]+1;link[clone]=link[q];ch[clone]=ch[q];while(p!=-1&&ch[p][c]==q){ch[p][c]=clone;p=link[p];}link[q]=link[cur]=clone;}}
        last=cur;
    };
    for(char c:s)extend(c-'a');
    int res=0;for(int i=1;i<sz;i++)res+=len[i]-len[link[i]];
    return res;
}
