Looking at this problem, I need to:

1. Read a convex polygon (the middle son's territory - it's convex because "moving straight from any place to any other place won't cross the boundary")
2. For each line segment, find the length of the part strictly inside the polygon

Since the polygon is convex, I can clip each segment against each edge of the polygon using the Sutherland-Hodgman line clipping approach.

For a convex polygon, I'll clip the segment against each half-plane defined by each edge.

**Approach:**
- The polygon vertices are given in random order, so I need to sort them to form a proper convex polygon.
- For each segment, clip it against all half-planes of the convex polygon.
- The interior is the intersection of all half-planes.

**Steps:**
1. Sort polygon vertices to form convex hull order (since they're given in random order but form a convex polygon with no 3 collinear points).
2. Compute convex hull to get proper order.
3. For each mineral deposit segment, clip against the convex polygon's interior (strict interior - not including boundaries).

For clipping a segment against a convex polygon:
- Represent the segment parametrically: P(t) = A + t*(B-A), t in [0,1]
- For each edge of the polygon, determine the valid range of t that keeps the point on the interior side (strictly).
- The answer is max(0, t_max - t_min) * |B-A|

For strict interior (not including boundary), when a point is exactly on an edge, it should be excluded. But for length calculation with precision 0.01, the boundary has measure zero, so it doesn't matter for length.

Let me implement the segment clipping:

For each edge (P_i, P_{i+1}) with inward normal, the constraint is:
`dot(normal, P(t) - P_i) > 0` (strictly inside, but for length it's same as >= 0)

The half-plane: `dot(N, A + t*(B-A) - P_i) >= 0`
= `dot(N, A - P_i) + t * dot(N, B-A) >= 0`

Let `d0 = dot(N, A - P_i)`, `dd = dot(N, B - A)`

- If dd > 0: t >= -d0/dd → t_lo = max(t_lo, -d0/dd)
- If dd < 0: t <= -d0/dd → t_hi = min(t_hi, -d0/dd)  
- If dd == 0: if d0 < 0, no intersection (segment outside)

For the polygon orientation: I'll compute the convex hull and ensure CCW order. The inward normal for edge (P_i, P_{i+1}) in CCW polygon is the left normal of the edge direction.

Edge direction: (dx, dy) = P_{i+1} - P_i
Left normal (inward for CCW): (-dy, dx)

Let me code this up:

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef double ld;

struct P { ld x, y; };

vector<P> convexHull(vector<P> pts) {
    int n = pts.size();
    sort(pts.begin(), pts.end(), [](const P& a, const P& b){ return a.x < b.x || (a.x == b.x && a.y < b.y); });
    vector<P> h;
    // lower
    for(int i = 0; i < n; i++) { while(h.size()>=2 && ((h.back().x-h[h.size()-2].x)*(pts[i].y-h[h.size()-2].y)-(h.back().y-h[h.size()-2].y)*(pts[i].x-h[h.size()-2].x)) <= 0) h.pop_back(); h.push_back(pts[i]); }
    int lo = h.size();
    for(int i = n-2; i >= 0; i--) { while(h.size()>lo && ((h.back().x-h[h.size()-2].x)*(pts[i].y-h[h.size()-2].y)-(h.back().y-h[h.size()-2].y)*(pts[i].x-h[h.size()-2].x)) <= 0) h.pop_back(); h.push_back(pts[i]); }
    h.pop_back();
    return h;
}

int main(){
    int n; scanf("%d",&n);
    vector<P> pts(n);
    for(int i=0;i<n;i++) scanf("%lf%lf",&pts[i].x,&pts[i].y);
    vector<P> poly = convexHull(pts);
    int m; scanf("%d",&m);
    while(m--){
        ld ax,ay,bx,by; scanf("%lf%lf%lf%lf",&ax,&ay,&bx,&by);
        ld tlo=0, thi=1;
        int sz=poly.size();
        for(int i=0;i<sz;i++){
            P pi=poly[i], pj=poly[(i+1)%sz];
            ld nx=-(pj.y-pi.y), ny=(pj.x-pi.x);
            ld d0=nx*(ax-pi.x)+ny*(ay-pi.y);
            ld dd=nx*(bx-ax)+ny*(by-ay);
            if(abs(dd)<1e-12){ if(d0<0){tlo=2;break;} }
            else { ld t=-d0/dd; if(dd>0) tlo=max(tlo,t); else thi=min(thi,t); }
        }
        ld len=0;
        if(tlo<thi) len=sqrt((bx-ax)*(bx-ax)+(by-ay)*(by-ay))*(thi-tlo);
        printf("%.2f\n",len);
    }
}
```
