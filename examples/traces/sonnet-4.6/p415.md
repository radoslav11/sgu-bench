Looking at this problem, I need to find which coins appear in **every** subset that sums to x.

A coin is "necessary" if it appears in all subsets that sum to x.

Approach:
- Find all subsets that sum to x
- A coin is necessary if removing it makes it impossible to reach x with the remaining coins

So for each coin `a[i]`, check: can we form sum x using the other coins (excluding `a[i]`)? If NO, then `a[i]` is necessary.

Since n ≤ 200 and x ≤ 10^4, this is feasible with a standard subset-sum DP.

For each coin i:
- Run DP on all coins except coin i
- Check if sum x is achievable
- If not achievable, coin i is necessary

The DP is O(n * x) for each coin, and we do this n times, so total O(n² * x) = 200 * 200 * 10000 = 400,000,000. That might be tight at 1.25 seconds.

Let me think of a better approach. Actually, 4*10^8 simple boolean operations might be too slow.

Alternative: Use bitset DP. The DP state is a bitset of size x+1. For each subset of coins, we can represent reachable sums. 

Actually, let's think differently. 

A coin a[i] is necessary iff it appears in every solution. Equivalently, a[i] is necessary iff we cannot form x without a[i].

With bitset optimization: dp is a bitset<10001>. For all coins except i, compute the DP. Check if bit x is set.

With bitset, each DP step (adding a coin) is O(x/64). Total: O(n * n * x/64) = 200 * 200 * 10000/64 ≈ 6.25 * 10^6. That's fine.

Let me implement this:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    int n, x;
    cin >> n >> x;
    vector<int> a(n);
    for(int i = 0; i < n; i++) cin >> a[i];
    
    vector<int> necessary;
    
    for(int i = 0; i < n; i++){
        // Check if x is reachable without coin i
        bitset<10001> dp;
        dp[0] = 1;
        for(int j = 0; j < n; j++){
            if(j == i) continue;
            // dp |= dp << a[j], but we need to do it carefully
            dp |= (dp << a[j]);
        }
        if(!dp[x]){
            // Cannot reach x without coin i, so it's necessary
            necessary.push_back(a[i]);
        }
    }
    
    cout << necessary.size() << "\n";
    for(int i = 0; i < (int)necessary.size(); i++){
        if(i) cout << " ";
        cout << necessary[i];
    }
    cout << "\n";
    
    return 0;
}
```

Wait, but the bitset shift `dp << a[j]` when a[j] can be up to x=10^4 and bitset size is 10001, this should work correctly since bits beyond 10000 are discarded.

Let me verify with the example:
- Coins: 1 2 3 5 10, x=18
- Without coin 5 (value 5): can we make 18 from {1,2,3,10}? 1+2+3+10=16≠18, 10+3+2+1=16... max is 16. No! So 5 is necessary.
- Without coin 10 (value 10): can we make 18 from {1,2,3,5}? max=11. No! So 10 is necessary.
- Without coin 1: {2,3,5,10}: 2+3+5+10=20, 3+5+10=18. Yes! So 1 is not necessary.
- Without coin 2: {1,3,5,10}: 1+2... wait 1+3+4? 1+2+5+10=18... no, without 2: 3+5+10=18. Yes! Not necessary.
- Without coin 3: {1,2,5,10}: 1+2+5+10=18. Yes! Not necessary.

Output: 2 coins: 5 10. Matches!
