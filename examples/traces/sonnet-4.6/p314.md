Looking at this problem, I need to find the k shortest paths (not necessarily simple) from s to t.

This is the classic k-shortest paths problem, solvable with Yen's algorithm for simple paths, but since paths can be non-simple, I can use the simpler approach with a modified Dijkstra using a priority queue where each node can be visited up to k times.

**Algorithm:** 
- Use a priority queue (min-heap) with (distance, node).
- Each time we pop a node, that's the next shortest path to that node.
- When we pop t for the i-th time, that's the i-th shortest path to t.
- A node can be popped at most k times (since we need at most k paths to t).

But we need to be careful: if t is unreachable or there are fewer than k paths, output "NO".

However, for non-simple paths, can we always get more paths? Yes, if there's a cycle reachable from s that can reach t, we can get infinitely many paths. But if t is only reachable through a finite number of distinct path lengths... wait, since paths can repeat edges/vertices, there are potentially infinite paths as long as t is reachable and there's any cycle on any s-t path.

The approach:
- Priority queue with (cost, node)
- `count[v]` = number of times node v has been popped
- When we pop (cost, v), increment count[v]. If count[v] > k, skip.
- If v == t and count[v] <= k, record the path length.
- For each neighbor u of v, if count[u] < k, push (cost + edge_weight, u).

This works because each node needs to be settled at most k times.

For "NO": if after exhausting the queue we haven't found k paths to t, output NO for remaining.

#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m, k;
    cin >> n >> m >> k;
    int s, t;
    cin >> s >> t;
    
    vector<vector<pair<int,int>>> adj(n+1);
    for (int i = 0; i < m; i++) {
        int a, b, c;
        cin >> a >> b >> c;
        adj[a].push_back({b, c});
    }
    
    vector<int> count(n+1, 0);
    priority_queue<pair<long long,int>, vector<pair<long long,int>>, greater<>> pq;
    pq.push({0, s});
    
    vector<long long> results;
    
    while (!pq.empty() && (int)results.size() < k) {
        auto [d, u] = pq.top(); pq.pop();
        count[u]++;
        if (count[u] > k) continue;
        if (u == t) results.push_back(d);
        for (auto [v, w] : adj[u]) {
            if (count[v] < k) {
                pq.push({d + w, v});
            }
        }
    }
    
    for (int i = 0; i < k; i++) {
        if (i < (int)results.size()) cout << results[i] << "\n";
        else cout << "NO\n";
    }
}
