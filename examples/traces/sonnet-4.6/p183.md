Looking at this problem, I need to paint balls such that in any M successive balls, at least 2 are black. I need to minimize the total cost.

Let me think about this with dynamic programming.

**State**: Let `dp[i]` = minimum cost to satisfy the constraint for all windows ending at or before position `i`, where the last black ball is at position `i`.

Actually, let me think more carefully. The constraint is: for every window of M consecutive balls `[j, j+M-1]`, at least 2 are black.

Let me define `dp[i][j]` = minimum cost considering the first `i` balls, where the last black ball is at position `i` and the second-to-last black ball is at position `j`.

With N=10000 and M=100, positions j would be within M of i, so this is O(N*M) states.

**Transition**: From state `(i, j)` (last two black balls at positions i and j), we move to state `(k, i)` where k > i is the next black ball.

The constraint is: every window must have 2 black balls. The critical constraint when adding ball k is that for all windows `[w, w+M-1]` that are now fully processed, we need 2 black balls. 

With last two black balls at j and i, and next at k: any window that includes j must have at least one more black ball. Windows are `[w, w+M-1]`. The window starting at `w = j-M+1` to `w = j` includes j. But since j and i are both black and i > j, any window containing j also contains i (if i - j < M). 

The key constraint: between consecutive black balls, the gap can't be too large. Specifically, if the last two black balls are at positions `j` and `i`, the next black ball must be at some position `k` such that: every window is covered. 

Windows that only see one black ball (at i) after position j: these are windows `[w, w+M-1]` where `w > j` and `w+M-1 < k` (so k is not in them) — wait, k is the next black ball so it's not painted yet in this window. So the window `[j+1, j+M]` needs at least 2 black balls, but j+1 > j so j is not in it, and if k > j+M, then k is also not in it. This window would have 0 black balls, which violates the constraint.

So: between consecutive black balls i and k, we need `k - i < M` (otherwise window `[i+1, i+M]` has no black balls from the left side... wait, i is black so window `[i+1, i+M]` doesn't include i).

Actually the constraint is simpler: for any two consecutive black balls at positions `a` and `b` (a < b), there should be no window of M balls entirely between them with 0 black balls. The window `[a+1, a+M]` must contain at least 2 black balls. Since a is not in `[a+1, a+M]`, we need at least 2 black balls in `[a+1, a+M]`. So b must be ≤ a+M, AND there must be another black ball in `[a+1, a+M]`.

Let me restate: `dp[i][j]` where `i` is current last black ball, `j` is second-to-last (j < i, i-j < M). Next black ball k must satisfy: `k - i < M` (so window `[i+1, i+M]` is covered... but we need 2 in it, meaning k ≤ i+M-1 and some other constraint on j and i for the window `[j+1, j+M]`).

Window `[j+1, j+M]` needs 2 black balls. It contains i (since i < j+M, i.e., i-j < M). The second one must be ≤ j+M, meaning k ≤ j+M, OR the second black ball in that window is k... so k ≤ j+M-1 (0-indexed adjustment needed).

So next ball k must satisfy: k ≤ j + M - 1 (to cover window starting at j+1) and k - i < M.

Since j < i, constraint k ≤ j+M-1 is tighter. So k can range from i+1 to j+M-1.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int N, M; cin >> N >> M;
    vector<int> c(N+1); for(int i=1;i<=N;i++) cin>>c[i];
    const int INF=1e9;
    // dp[i][j]: min cost, last black=i, second last=j, j<i, i-j<M
    vector<vector<int>> dp(N+2, vector<int>(N+2, INF));
    // Initialize: first two black balls at j and i
    for(int j=1;j<=N;j++) for(int i=j+1;i<j+M&&i<=N;i++) dp[i][j]=c[i]+c[j];
    int ans=INF;
    for(int i=2;i<=N;i++) for(int j=max(1,i-M+1);j<i;j++){
        if(dp[i][j]==INF) continue;
        if(i+(M-1)>N){ // check if done
            if(j+M-1>N) ans=min(ans,dp[i][j]);
        }
        for(int k=i+1;k<=min(N,(int)(j+M-1));k++) dp[k][i]=min(dp[k][i],dp[i][j]+c[k]);
    }
    cout<<ans<<endl;
}
```
