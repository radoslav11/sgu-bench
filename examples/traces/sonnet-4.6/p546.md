Looking at this problem, I need to:

1. Take a password of length n and make it have exactly `a` zeros, `b` ones, and `n-a-b` twos.
2. If `a+b > n`, output -1.
3. Otherwise, find minimum replacements.

Strategy:
- Count current zeros, ones, twos in the password.
- The target is `a` zeros, `b` ones, `n-a-b` twos.
- If `n-a-b < 0`, output -1.

To minimize replacements, I want to keep as many correct characters as possible. 

Let me think about it greedily:
- Let c0, c1, c2 be current counts of 0, 1, 2.
- Target: a zeros, b ones, c = n-a-b twos.
- Characters we keep: min(c0, a) zeros we can keep, min(c1, b) ones we can keep, min(c2, c) twos we can keep.

But we need to be careful - we might keep too many of one type if we're not careful. Actually, the greedy approach:

We want to maximize characters kept (which minimizes replacements).

The number of zeros we keep = min(c0, a), ones kept = min(c1, b), twos kept = min(c2, n-a-b).

But these might sum to less than n - that's fine, the rest need replacement.

Wait, actually the minimum replacements = n - (number we keep). But we need to ensure that what we keep doesn't conflict. 

Actually since each kept character is a different position, and we're just deciding which ones to keep:
- Keep up to `a` zeros from current zeros
- Keep up to `b` ones from current ones  
- Keep up to `n-a-b` twos from current twos

The total kept = min(c0,a) + min(c1,b) + min(c2, n-a-b), and this is valid since each position is counted once.

Replacements = n - total_kept.

Let me implement this:

1. Check if n-a-b < 0: output -1.
2. Go through the password, decide which chars to keep:
   - For each '0': keep it if we still need more zeros (remaining_a > 0), decrement remaining_a.
   - For each '1': keep it if remaining_b > 0, decrement remaining_b.
   - For each '2': keep it if remaining_c > 0, decrement remaining_c.
   - Otherwise mark position as "needs replacement".
3. Then fill positions needing replacement with remaining needed zeros, ones, twos.

Let me trace through example 1:
- n=6, a=1, b=3, password="012022", c=n-a-b=2
- Process: '0' -> keep (remaining_a: 1->0), '1' -> keep (remaining_b: 3->2), '2' -> keep (remaining_c: 2->1), '0' -> replace (remaining_a=0), '2' -> keep (remaining_c: 1->0), '2' -> replace (remaining_c=0)
- Positions to replace: index 3 (was '0'), index 5 (was '2')
- Need: remaining_a=0, remaining_b=2, remaining_c=0
- Fill index 3 with '1', index 5 with '1'
- Result: "011021" - 2 replacements. That's valid (1 zero, 3 ones, 2 twos). ✓

The sample shows "111022" but any valid answer works.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, a, b;
    cin >> n >> a >> b;
    string s;
    cin >> s;
    
    int c = n - a - b;
    if (c < 0) {
        cout << -1 << endl;
        return 0;
    }
    
    int rem_a = a, rem_b = b, rem_c = c;
    string result = s;
    vector<int> to_replace;
    
    for (int i = 0; i < n; i++) {
        if (s[i] == '0' && rem_a > 0) {
            rem_a--;
        } else if (s[i] == '1' && rem_b > 0) {
            rem_b--;
        } else if (s[i] == '2' && rem_c > 0) {
            rem_c--;
        } else {
            to_replace.push_back(i);
        }
    }
    
    int t = (int)to_replace.size();
    cout << t << "\n";
    
    int idx = 0;
    for (int i = 0; i < rem_a; i++) {
        result[to_replace[idx++]] = '0';
    }
    for (int i = 0; i < rem_b; i++) {
        result[to_replace[idx++]] = '1';
    }
    for (int i = 0; i < rem_c; i++) {
        result[to_replace[idx++]] = '2';
    }
    
    cout << result << "\n";
    return 0;
}
```
