Looking at this problem, I need to simulate a process where numbers 1 to N are arranged in a circle, and I repeatedly:
1. Count Q-th integer in current direction and erase it
2. Move to next integer clockwise
3. Check if that next integer is odd/even to set direction

Let me trace through the example N=5, Q=3:
- Start at position 1 (value 1), moving clockwise
- Count 3 clockwise from 1: 1,2,3 → erase 3
- Move to next clockwise from where 3 was: that's 4
- 4 is even → direction = counter-clockwise
- Count 3 counter-clockwise from 4: 4,2,1 → erase 1
- Move to next clockwise from where 1 was: that's 2
- 2 is even → direction = counter-clockwise
- Count 3 counter-clockwise from 2: 2,5,4 → erase 4
- Move to next clockwise from where 4 was: that's 5
- 5 is odd → direction = clockwise
- Count 3 clockwise from 5: 5,2,5... wait, remaining are {2,5}
- Count 3 clockwise from 5: 5,2,5 → erase 5
- Move to next clockwise: 2
- 2 is even → counter-clockwise
- Only 2 left, erase 2. Last = 2. ✓

For N up to 2,000,000, I need an efficient data structure. An order-statistics tree or a Fenwick tree for finding k-th element efficiently would work.

I'll use a Fenwick tree (BIT) to maintain counts and find the k-th element efficiently.

Key simulation details:
- Current position is tracked as an index in the current list
- Direction: clockwise = forward, counter-clockwise = backward
- After erasing, move to next clockwise (regardless of direction)
- Then set direction based on that next element's parity

Let me implement with a BIT for O(log N) per operation, total O(N log N).

When counting Q-th in direction from current position:
- Clockwise: move forward Q steps from current
- Counter-clockwise: move backward Q steps from current

Current position starts at index 1 (value 1) in the circle.

After erasing element at position p (1-indexed in current list of size m):
- Next clockwise is at position p in the new list (of size m-1), but if p > m-1, wrap to 1.

Let me track "current index" in the 1..N original numbering using the BIT.

#include <bits/stdc++.h>
using namespace std;

int bit[2000002];
int N;

void update(int i, int v) { for (; i <= N; i += i&(-i)) bit[i] += v; }
int query(int i) { int s=0; for (; i>0; i -= i&(-i)) s += bit[i]; return s; }
int total() { return query(N); }

// Find k-th element (1-indexed)
int kth(int k) {
    int pos = 0;
    for (int pw = 1<<21; pw; pw >>= 1)
        if (pos+pw <= N && bit[pos+pw] < k) { pos += pw; k -= bit[pos]; }
    return pos+1;
}
