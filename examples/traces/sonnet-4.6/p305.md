Looking at this problem, I need to assign unique numbers from 1 to M to N progressions, maximizing the number of progressions that receive a number belonging to their progression.

A number X belongs to progression i if X = Ai*K + Bi for some non-negative integer K ≥ 0.

So X is in progression i if:
- If Ai > 0: X ≥ Bi and (X - Bi) % Ai == 0
- If Ai == 0: X == Bi (only if Bi is valid, K=0 gives Bi, K=1 gives Bi again... wait Ai=0 means X=Bi always)
- If Ai < 0: X = Ai*K + Bi, for K≥0, so X ≤ Bi and (X - Bi) % Ai == 0

This is a matching problem: we want maximum bipartite matching between progressions and numbers 1..M, but M can be up to 10^9 so we can't enumerate all numbers.

Key insight: We only need to consider "useful" numbers - numbers that belong to at least one progression. For each progression i, the valid numbers in [1,M] form an arithmetic sequence. We generate all candidate numbers from all progressions, then find maximum matching.

The number of candidates: for each progression, there could be up to M/|Ai| numbers. If Ai=1, that's up to 10^9 candidates - too many.

But N ≤ 300, and we need at most N assignments. So we only need at most N numbers per progression to consider (actually just enough for matching). Since we're matching N progressions to numbers, we need at most N distinct numbers total.

Better approach: For each progression, enumerate only enough candidate numbers. Since we need at most N total assignments, for each progression we might need just a few candidates. But we can't know which ones will work.

Actually, let's think: the total number of candidates is at most N * (number per progression). For progressions with |Ai| ≥ 1, if |Ai| is large, few candidates. The problem is when |Ai| = 1, there are M candidates.

But we only need N numbers in total for the matching. So for progressions with many valid numbers, we don't need all of them - just pick N representative ones.

For a progression with |Ai|=1, valid numbers are Bi, Bi+|Ai|, ... We can limit to first N+1 valid numbers ≥ 1 in [1,M].

Let me generate for each progression at most N+1 valid numbers in [1,M], collect all unique candidates (at most N*(N+1) = ~90000 candidates), then do bipartite matching.

For each progression i:
- Find first X ≥ 1 in progression with X ≤ M
- Generate up to N+1 such values

Then run maximum bipartite matching on progressions vs candidate numbers.

For Ai=0: only X=Bi if 1≤Bi≤M.
For Ai>0: X = Bi + K*Ai ≥ 1, X ≤ M. First K = max(0, ceil((1-Bi)/Ai)).
For Ai<0: X = Bi + K*Ai ≤ M and ≥ 1. As K increases, X decreases.

Let me implement this with Hopcroft-Karp for efficiency.

```cpp
#include <bits/stdc++.h>
using namespace std;

// Hopcroft-Karp
struct HopcroftKarp {
    int n, m;
    vector<vector<int>> adj;
    vector<int> matchL, matchR, dist;
    HopcroftKarp(int n, int m) : n(n), m(m), adj(n), matchL(n,-1), matchR(m,-1), dist(n) {}
    void addEdge(int u, int v) { adj[u].push_back(v); }
    bool bfs() {
        queue<int> q;
        for(int u=0;u<n;u++) { if(matchL[u]==-1){dist[u]=0;q.push(u);}else dist[u]=1e9;}
        bool found=false;
        while(!q.empty()){int u=q.front();q.pop();for(int v:adj[u]){int w=matchR[v];if(w==-1)found=true;else if(dist[w]>1e8){dist[w]=dist[u]+1;q.push(w);}}}
        return found;
    }
    bool dfs(int u){for(int v:adj[u]){int w=matchR[v];if(w==-1||dist[w]==dist[u]+1&&dfs(w)){matchL[u]=v;matchR[v]=u;return true;}}dist[u]=1e9;return false;}
    int maxMatch(){int r=0;while(bfs())for(int u=0;u<n;u++)if(matchL[u]==-1)r+=dfs(u);return r;}
};

int main(){
    int N; long long M;
    cin>>N>>M;
    vector<long long> A(N),B(N);
    for(int i=0;i<N;i++) cin>>A[i]>>B[i];
    vector<long long> cands;
    for(int i=0;i<N;i++){
        long long a=A[i],b=B[i];
        if(a==0){if(b>=1&&b<=M)cands.push_back(b);}
        else if(a>0){long long k=max(0LL,(long long)ceil((1.0-b)/a));int cnt=0;while(cnt<=N){long long x=b+k*a;if(x>M)break;if(x>=1){cands.push_back(x);cnt++;}k++;}}
        else{long long k=max(0LL,(long long)ceil((b-M)/(-a)));int cnt=0;while(cnt<=N){long long x=b+k*a;if(x<1)break;if(x<=M){cands.push_back(x);cnt++;}k++;}}
    }
    sort(cands.begin(),cands.end());cands.erase(unique(cands.begin(),cands.end()),cands.end());
    int C=cands.size();
    HopcroftKarp hk(N,C);
    for(int i=0;i<N;i++){long long a=A[i],b=B[i];for(int j=0;j<C;j++){long long x=cands[j];if(a==0){if(x==b)hk.addEdge(i,j);}else{long long d=x-b;if(d%a==0&&d/a>=0)hk.addEdge(i,j);}}}
    hk.maxMatch();
    // assign unmatched a number not in cands
    set<long long> used(cands.begin(),cands.end());
    vector<long long> ans(N);
    for(int i=0;i<N;i++) ans[i]=(hk.matchL[i]==-1)?-1:cands[hk.matchL[i]];
    long long nxt=1;
    for(int i=0;i<N;i++){if(ans[i]==-1){while(used.count(nxt))nxt++;ans[i]=nxt;used.insert(nxt);nxt++;}}
    for(int i=0;i<N;i++) cout<<ans[i]<<" \n"[i==N-1];
}
```
