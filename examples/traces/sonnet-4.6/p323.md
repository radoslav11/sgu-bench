Looking at this problem, I need to:

1. Find a spanning tree of N towns using flights from multiple airlines
2. Choose one airline R to keep
3. Transfer all flights not belonging to R (that are in the spanning tree) to R, paying their taxes
4. Minimize total tax paid

The key insight: We need to find a spanning tree of the graph where we minimize the total cost of edges not belonging to the chosen airline R.

For each possible airline R, I want to find a spanning tree that maximizes the use of airline R's edges (with 0 cost) while using other edges only when necessary (paying their tax).

**Approach:**
- For each airline R, find the minimum cost spanning tree where edges of airline R have cost 0 and other edges have their tax cost p_i.
- Pick the airline R that gives the minimum spanning tree cost.

But M can be up to 2000, and K up to 200000, N up to 2000. Running MST for each airline would be O(M * K log K) which is 2000 * 200000 * 17 ≈ 6.8 billion — too slow.

**Better approach:** Use a modified MST. 

Think of it differently: We want a spanning tree. For a fixed R, edges of airline R cost 0. We want MST with these modified weights.

**Key observation:** For each airline R, the MST first connects components using R's edges (cost 0), then uses cheapest available edges from other airlines.

**Algorithm:**
- For each airline R (1 to M):
  - Build spanning forest using R's edges (all free)
  - Then add minimum cost edges from other airlines to connect remaining components
  - Total cost = sum of non-R edges selected

With M=2000, N=2000, K=200000: For each R, Kruskal takes O(K log K). Total O(M*K log K) = too slow.

**Optimization:** Pre-sort all edges. For each airline R:
- First union all edges of airline R
- Then process remaining edges in order of cost using Kruskal

Pre-sort non-airline-specific edges once: O(K log K). Then for each R, reset DSU, union R's edges, then greedily add from sorted non-R edges.

With N=2000, DSU operations are nearly O(1). For each R: O(N + K) total work. Total: O(M*(N+K)) = 2000*(2000+200000) = 400M operations. Should be feasible in 2.25s.

Let me implement this:

1. Sort all edges by cost.
2. For each airline R from 1 to M:
   - Initialize DSU
   - Add all edges of airline R (cost 0)
   - Go through sorted edges (excluding R's edges), add if connects components
   - Compute total cost
3. Pick best R, output result.
