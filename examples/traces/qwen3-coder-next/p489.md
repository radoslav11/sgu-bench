To solve this problem, we need to count the number of extremal permutations of integers from 1 to \( n \), where an extremal permutation is defined as a permutation where every internal element (except the first and last) is a local extremum (either a local maximum or minimum). The result should be output modulo \( m \).

### Approach
1. **Problem Analysis**: 
   - For \( n = 1 \), there's only one permutation, and since there are no internal elements, it is trivially extremal.
   - For \( n = 2 \), both permutations \([1, 2]\) and \([2, 1]\) are valid as there are no internal elements.
   - For \( n \geq 3 \), the permutations must alternate between increasing and decreasing (i.e., they are alternating permutations). The count of such permutations is \( 2 \times E(n) \), where \( E(n) \) is the Euler number for up-down permutations of length \( n \).

2. **Euler Numbers Calculation**:
   - The Euler numbers \( E(n) \) can be computed using a recurrence relation:
     - \( E(0) = 1 \)
     - \( E(1) = 1 \)
     - For \( n \geq 2 \), \( E(n) = \sum_{\substack{k=1 \\ k \text{ odd}}}^{n-1} \binom{n-1}{k} \times E(k) \times E(n-1-k) \)
   - To compute binomial coefficients modulo \( m \), we use Pascal's triangle, updating the current row based on the previous row to save memory.

3. **Algorithm Selection**:
   - Precompute Euler numbers up to \( n \) using dynamic programming.
   - For each \( i \) from 2 to \( n \), compute the binomial coefficients for the \((i-1)\)-th row of Pascal's triangle on the fly using the previous row.
   - Sum the products of relevant binomial coefficients and previously computed Euler numbers for odd indices \( k \).

4. **Complexity Analysis**:
   - **Time Complexity**: \( O(n^2) \) due to nested loops for computing Euler numbers and binomial coefficients.
   - **Space Complexity**: \( O(n) \) for storing Euler numbers and the current row of Pascal's triangle.

### Solution Code
```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    long long n_input, m;
    cin >> n_input >> m;

    if (n_input == 1) {
        cout << (1 % m) << '\n';
        return 0;
    }

    long long n = n_input;
    vector<long long> E(n + 1, 0);
    E[0] = 1 % m;
    E[1] = 1 % m;

    vector<long long> prev = {1 % m};

    for (int i = 2; i <= n; i++) {
        vector<long long> curr(i, 1 % m);
        for (int j = 1; j < i - 1; j++) {
            curr[j] = (prev[j - 1] + prev[j]) % m;
        }

        for (int k = 1; k < i; k += 2) {
            long long term = curr[k];
            term = (term * E[k]) % m;
            term = (term * E[i - 1 - k]) % m;
            E[i] = (E[i] + term) % m;
        }

        prev = curr;
    }

    long long ans = (2 * E[n]) % m;
    cout << ans << '\n';

    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads \( n \) and \( m \). If \( n = 1 \), it directly outputs \( 1 \mod m \).
2. **Euler Numbers Initialization**: For \( n \geq 2 \), it initializes \( E[0] = 1 \) and \( E[1] = 1 \).
3. **Dynamic Programming Setup**: The algorithm iterates from \( i = 2 \) to \( n \), computing the \((i-1)\)-th row of Pascal's triangle using the previous row.
4. **Euler Number Calculation**: For each \( i \), it sums the products of binomial coefficients (from the current Pascal row) and previously computed Euler numbers for odd indices \( k \).
5. **Result Calculation**: The result for \( n \) is \( 2 \times E[n] \mod m \), which is printed as the output.

This approach efficiently computes the number of extremal permutations using dynamic programming and properties of Euler numbers, ensuring optimal performance within given constraints.
