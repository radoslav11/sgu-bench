To solve this problem, we need to determine the minimum number of tables required to seat all groups of friends in a cafe such that no group member is seated alone at any table. Each table can accommodate up to `r` people, and for any table that includes members from a group, there must be at least two members from that group at the table.

### Approach
1. **Initial Table Calculation**: For each group, calculate the number of full tables needed (each full table seats exactly `r` people) using integer division. The remaining members (those not fitting into full tables) form the remainder for each group.
2. **Handling Remainders**: Collect all remainders (members left after filling full tables) into a list. Sort this list in descending order to facilitate efficient pairing.
3. **Pairing Remainders**: Use a two-pointer technique to pair the largest remainder with the smallest possible remainder such that their sum does not exceed `r`. This minimizes the number of additional tables needed:
   - If the sum of the largest and smallest remainders fits in one table, pair them.
   - If they don't fit, the largest remainder requires its own table.
   - If only one remainder remains, it requires its own table.
4. **Total Tables Calculation**: The total number of tables is the sum of full tables and the additional tables needed for the remainders.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <algorithm>

int main() {
    int n, r;
    std::cin >> n >> r;
    std::vector<int> a(n);
    for (int i = 0; i < n; i++) {
        std::cin >> a[i];
    }

    int full_tables = 0;
    std::vector<int> remainders;
    for (int i = 0; i < n; i++) {
        full_tables += a[i] / r;
        int rem = a[i] % r;
        if (rem > 0) {
            remainders.push_back(rem);
        }
    }

    std::sort(remainders.rbegin(), remainders.rend());

    int tables = full_tables;
    int left = 0;
    int right = remainders.size() - 1;
    while (left <= right) {
        if (left == right) {
            tables++;
            left++;
        } else if (remainders[left] + remainders[right] <= r) {
            tables++;
            left++;
            right--;
        } else {
            tables++;
            left++;
        }
    }

    std::cout << tables << std::endl;

    return 0;
}
```

### Explanation
1. **Reading Input**: The code reads the number of groups `n` and the table capacity `r`, followed by the sizes of each group.
2. **Full Tables Calculation**: For each group, it calculates how many full tables (each seating `r` people) are needed. The remaining members (after filling full tables) are stored in a list.
3. **Sorting Remainders**: The remainders are sorted in descending order to prioritize pairing the largest remainders first, which helps in minimizing the number of tables.
4. **Two-Pointer Pairing**: Using two pointers (one at the start and one at the end of the sorted remainders list), the code checks if the sum of the largest and smallest remainders fits in one table. If yes, they are paired; otherwise, the largest remainder gets its own table. If only one remainder remains, it gets its own table.
5. **Output Result**: The total number of tables, which is the sum of full tables and the additional tables for remainders, is printed.

This approach efficiently minimizes the number of tables by optimally pairing remainders while adhering to the constraint that each group at a table must have at least two members. The sorting and two-pointer technique ensure optimal performance with a time complexity of \(O(n \log n)\).
