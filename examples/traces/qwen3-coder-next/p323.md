To solve this problem, we need to determine the minimum cost for Don Berlione to consolidate all flights under a single airline such that the resulting airline can connect all towns in Berland. The solution involves evaluating each airline as a potential candidate for consolidation and selecting the one with the minimal total tax cost required to ensure connectivity.

### Approach
1. **Problem Analysis**: The problem requires us to find an airline such that, after transferring some flights (by paying taxes) to this airline, all towns remain connected. The key insight is that for any candidate airline \( R \), the minimal cost is derived from:
   - Using all existing flights of airline \( R \) for free.
   - Adding the cheapest flights from other airlines to connect the remaining disconnected components formed by the flights of airline \( R \).

2. **Algorithm Selection**:
   - **Preprocessing**: Read all flights and group them by their respective airlines. Sort all flights globally by their tax cost to facilitate efficient selection of the cheapest edges during connectivity checks.
   - **Evaluation per Airline**: For each airline \( R \) (from 1 to \( M \)):
     - **Step 1**: Use a Union-Find (Disjoint Set Union, DSU) data structure to connect all towns via flights belonging to airline \( R \) (these cost nothing).
     - **Step 2**: Iterate through the globally sorted list of flights (excluding those of airline \( R \)), and for each flight, if it connects two disconnected components, include it in the solution (adding its tax to the total cost).
   - **Optimal Selection**: Track the airline \( R \) that results in the minimal total tax cost. If multiple airlines yield the same minimal cost, any can be chosen.

3. **Complexity Analysis**:
   - **Time Complexity**: \( O(M \times K \times \alpha(N)) \), where \( \alpha \) is the inverse Ackermann function (effectively constant). Here, \( M \) is the number of airlines, \( K \) is the number of flights, and \( N \) is the number of towns. In the worst case, this is \( O(2000 \times 200000 \times 5) \approx 2 \times 10^9 \) operations, which is feasible given the constraints (2.25 seconds per test).
   - **Space Complexity**: \( O(N + K + M) \), primarily for storing flights and airline-specific edge lists.

#include <iostream>
#include <vector>
#include <algorithm>
#include <cstdio>

using namespace std;

struct Edge {
    int a, b, c, p, idx;
};

struct DSU {
    vector<int> parent;
    vector<int> rank;
    int comp;
    DSU(int n) {
        parent.resize(n);
        rank.resize(n, 0);
        comp = n;
        for (int i = 0; i < n; i++) {
            parent[i] = i;
        }
    }
    int find(int x) {
        if (parent[x] != x) {
            parent[x] = find(parent[x]);
        }
        return parent[x];
    }
    bool unite(int x, int y) {
        x = find(x);
        y = find(y);
        if (x == y) {
            return false;
        }
        if (rank[x] < rank[y]) {
            swap(x, y);
        }
        parent[y] = x;
        if (rank[x] == rank[y]) {
            rank[x]++;
        }
        comp--;
        return true;
    }
};

int main() {
    scanf("%d %d %d", &N, &M, &K);
    
    vector<Edge> edges;
    vector<vector<Edge>> airlineEdges(M + 1);
    
    for (int i = 0; i < K; i++) {
        int a, b, c, p;
        scanf("%d %d %d %d", &a, &b, &c, &p);
        Edge e = {a, b, c, p, i + 1};
        edges.push_back(e);
        airlineEdges[c].push_back(e);
    }
    
    vector<Edge> global_sorted = edges;
    sort(global_sorted.begin(), global_sorted.end(), [](const Edge& x, const Edge& y) {
        return x.p < y.p;
    });
    
    long long best_cost = 1e18;
    int best_R = -1;
    vector<int> best_ops;
    
    for (int R = 1; R <= M; R++) {
        DSU dsu(N);
        long long cost = 0;
        vector<int> ops;
        
        for (Edge& e : airlineEdges[R]) {
            dsu.unite(e.a - 1, e.b - 1);
        }
        
        if (dsu.comp == 1) {
            if (cost < best_cost) {
                best_cost = cost;
                best_R = R;
                best_ops = ops;
            }
            continue;
        }
        
        for (Edge& e : global_sorted) {
            if (e.c == R) continue;
            int u = e.a - 1;
            int v = e.b - 1;
            if (dsu.find(u) == dsu.find(v)) continue;
            dsu.unite(u, v);
            cost += e.p;
            ops.push_back(e.idx);
            if (dsu.comp == 1) break;
        }
        
        if (dsu.comp != 1) continue;
        
        if (cost < best_cost) {
            best_cost = cost;
            best_R = R;
            best_ops = ops;
        }
    }
    
    printf("%lld %d %d\n", best_cost, best_R, (int)best_ops.size());
    for (int idx : best_ops) {
        printf("%d\n", idx);
    }
    
    return 0;
}

1. **Input Reading**: The input is read and stored, grouping flights by their respective airlines. Flights are also sorted globally by tax cost for efficient processing.
2. **Airline Evaluation**: For each airline \( R \):
   - **Initial Connectivity**: Using DSU, all flights belonging to airline \( R \) are processed (免费) to form initial connected components.
   - **补充 Connectivity**: The remaining flights (from other airlines) are processed in order of increasing tax cost. Each flight that connects two disconnected components is added to the solution, with its tax added to the total cost.
3. **Optimal Selection**: The airline \( R \) with the minimal total tax cost is selected. The solution outputs this cost, the chosen airline, the number of operations, and the indices of flights to be transferred.

This approach efficiently checks each airline's viability by leveraging sorted edge processing and Union-Find for dynamic connectivity checks, ensuring optimal performance even for the upper constraint limits.
