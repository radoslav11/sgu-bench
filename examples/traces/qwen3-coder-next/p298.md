### Approach
1. **Problem Analysis**: The problem involves constraints derived from the promises made by the king. Each promise specifies that daughter Xi must receive at least Ci burles more than daughter Yi, which translates to the constraint \( A[Xi] \geq A[Yi] + Ci \). Additionally, each dowry \( A[i] \) must satisfy \( -10000 \leq A[i] \leq 10000 \).

2. **Graph Representation**: We model the problem as a system of difference constraints. We introduce a dummy node (node 0) to handle the bounds on the dowries. For each daughter \( i \), we add edges:
   - From node 0 to \( i \) with weight 10000 (ensuring \( A[i] \leq 10000 \)).
   - From \( i \) to node 0 with weight 10000 (ensuring \( A[i] \geq -10000 \)).
   
   For each promise \( (Xi, Yi, Ci) \), we add an edge from \( Xi \) to \( Yi \) with weight \( -Ci \) (ensuring \( A[Yi] \leq A[Xi] - Ci \), which is equivalent to \( A[Xi] \geq A[Yi] + Ci \)).

3. **Bellman-Ford Algorithm**: We use the Bellman-Ford algorithm to solve the system of difference constraints. The algorithm computes the shortest paths from the dummy node (node 0) to all other nodes. If a negative cycle is detected during the algorithm, it means no feasible solution exists, and we return -1. Otherwise, the shortest path distances from node 0 to nodes 1 through N give the dowry values that satisfy all constraints and minimize the difference \( A[N] - A[1] \).
