        After checking Timus 190, I see that the intended solution is to use bipartite matching.

        Steps for bipartite matching:

          Let left set: black squares (where (i+j) % 2 == 0) [if we consider (1,1): 1+1=2 -> even -> black]
          right set: white squares ( (i+j) % 2 == 1 )

          But note: the grid is not necessarily starting with (1,1) as black? but the bipartition is fixed.

          However, the problem: the board is oriented with row1 at bottom. But the coloring by (i+j) is independent of the physical orientation.

          We'll use:
            For a square (i, j) [i: row from 1 to N (1=bottom, N=top), j: column from 1 to N (1=left, N=right)].
            Color = (i+j) % 2.

          But note: 
            (1,1): 1+1=2 -> even -> black.
            (1,2): 1+2=3 -> odd -> white.
            (2,1): 2+1=3 -> odd -> white.

          So adjacent squares have different colors.

          We build a graph: for each remaining black square, connect to adjacent (up, down, left, right) remaining white squares.

          Then run matching.

          If the matching size * 2 != total remaining, then "No".

        How to output:

          If matching exists, then:
            We need to output:
              "Yes"
              Nh: number of horizontal dominoes, then for each: the left square of the domino.
              Nv: number of vertical dominoes, then for each: the bottom square of the domino.

          How to classify a domino?
            A domino covering (i,j) and (i,j+1) is horizontal -> the left square is (i,j).
            A domino covering (i,j) and (i+1,j) is vertical -> the bottom square is (i,j) (because row i is below row i+1).

          Note: In our coordinate system, row i is below row i+1.

          So for a vertical domino: 
             if the domino covers (i,j) and (i+1,j), then (i,j) is the bottom square.

          How do we store the matching?
            We can have an array: 
               pair[i][j] = the partner square (if any) for (i,j).

          Then:
            For each black square (i,j) that is matched to a white square:
               if the partner is (i, j+1) -> horizontal, and the left square is (i,j) [if j+1 is to the right] -> but note: 
                 (i,j) and (i,j+1): then (i,j) is left.
               if the partner is (i, j-1) -> then the left square would be (i, j-1) but we are iterating on black squares.

          Alternatively, we can store for each domino the two squares.

          However, the problem: 
            We must output the horizontal dominoes by the left square, and vertical by the bottom square.

          So:
            For a horizontal domino: 
                it covers (i, j) and (i, j+1). 
                We can output (i, j) as the left square.

            For a vertical domino:
                it covers (i, j) and (i+1, j).
                We output (i, j) as the bottom square.

          How to get the dominoes? 
            After matching, for each black square (i,j) that has a match (ni, nj):
                if ni == i and nj == j+1: then horizontal -> output (i,j) as left square.
                if ni == i and nj == j-1: then horizontal -> but then the left square is (i, j-1) -> so we should output (i, j-1) but we are at (i,j) as black.
                Similarly, if ni == i+1 and nj == j: then vertical -> output (i,j) as bottom.
                if ni == i-1 and nj == j: then vertical -> then the bottom square is (i-1, j) -> but we are at (i,j) which is the top part.

To solve this problem, we need to determine if a given chessboard with some removed squares can be completely covered by dominoes (2x1 tiles) such that each domino covers exactly two adjacent squares (horizontally or vertically), and no two dominoes overlap. The solution involves checking if the remaining squares can be perfectly matched in pairs where each pair consists of adjacent squares, which translates to a bipartite matching problem on the chessboard graph.

### Approach
1. **Problem Analysis**: The chessboard is an N x N grid where some squares are removed. The remaining squares must be even in number for a domino covering to be possible. Each domino covers two adjacent squares, so the problem reduces to finding a perfect matching in a bipartite graph where:
   - The two partitions are the black and white squares of the chessboard (colored in a checkerboard pattern).
   - Edges exist between adjacent black and white squares (up, down, left, right).

2. **Key Insight**: The chessboard can be divided into two sets based on the sum of coordinates (x + y) modulo 2. Squares where (x + y) is even form one set (black), and those where (x + y) is odd form the other set (white). Each domino must cover one black and one white square.

3. **Algorithm Selection**: We use a depth-first search (DFS) based approach for bipartite matching (Hungarian algorithm for bipartite matching). For each black square, we try to find an augmenting path to match it to an adjacent white square.

4. **Output Construction**: After successful matching, we classify each domino as horizontal or vertical:
   - **Horizontal Domino**: Covers squares in the same row (same y-coordinate). The left square (smaller x-coordinate) is recorded.
   - **Vertical Domino**: Covers squares in the same column (same x-coordinate). The bottom square (smaller y-coordinate) is recorded.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

bool dfs(int x, int y, vector<vector<bool>>& vis, 
        int d[4][2], 
        const vector<vector<bool>>& bad, int N,
        int blackMatchX[50][50], int blackMatchY[50][50],
        int whiteMatchX[50][50], int whiteMatchY[50][50]) {
    for (int i = 0; i < 4; i++) {
        int nx = x + d[i][0];
        int ny = y + d[i][1];
        if (nx < 1 || nx > N || ny < 1 || ny > N) 
            continue;
        if (bad[nx][ny]) 
            continue;
        if ((nx + ny) % 2 != 1) 
            continue;

        if (vis[nx][ny]) 
            continue;
        vis[nx][ny] = true;

        if (whiteMatchX[nx][ny] == 0 && whiteMatchY[nx][ny] == 0) {
            blackMatchX[x][y] = nx;
            blackMatchY[x][y] = ny;
            whiteMatchX[nx][ny] = x;
            whiteMatchY[nx][ny] = y;
            return true;
        } else {
            int bx = whiteMatchX[nx][ny];
            int by = whiteMatchY[nx][ny];
            if (dfs(bx, by, vis, d, bad, N, blackMatchX, blackMatchY, whiteMatchX, whiteMatchY)) {
                blackMatchX[x][y] = nx;
                blackMatchY[x][y] = ny;
                whiteMatchX[nx][ny] = x;
                whiteMatchY[nx][ny] = y;
                return true;
            }
        }
    }
    return false;
}

int main() {
    int N, P;
    cin >> N >> P;

    vector<vector<bool>> bad(N+1, vector<bool>(N+1, false));
    int total = N * N - P;
    if (total % 2 != 0) {
        cout << "No" << endl;
        return 0;
    }

    for (int i = 0; i < P; i++) {
        int x, y;
        cin >> x >> y;
        bad[x][y] = true;
    }

    int blackMatchX[50][50] = {{0}};
    int blackMatchY[50][50] = {{0}};
    int whiteMatchX[50][50] = {{0}};
    int whiteMatchY[50][50] = {{0}};

    int d[4][2] = {{1, 0}, {-1, 0}, {0, 1}, {0, -1}};

    int count = 0;
    for (int x = 1; x <= N; x++) {
        for (int y = 1; y <= N; y++) {
            if ((x + y) % 2 == 0 && !bad[x][y]) {
                vector<vector<bool>> vis(N+1, vector<bool>(N+1, false));
                if (dfs(x, y, vis, d, bad, N, blackMatchX, blackMatchY, whiteMatchX, whiteMatchY)) {
                    count++;
                }
            }
        }
    }

    if (count * 2 != total) {
        cout << "No" << endl;
        return 0;
    }

    vector<pair<int, int>> hor, ver;
    for (int x = 1; x <= N; x++) {
        for (int y = 1; y <= N; y++) {
            if ((x + y) % 2 == 0 && !bad[x][y] && blackMatchX[x][y] != 0) {
                int nx = blackMatchX[x][y];
                int ny = blackMatchY[x][y];
                if (ny == y) {
                    int left_x = min(x, nx);
                    hor.push_back(make_pair(left_x, y));
                } else if (nx == x) {
                    int bottom_y = min(y, ny);
                    ver.push_back(make_pair(x, bottom_y));
                }
            }
        }
    }

    cout << "Yes" << endl;
    cout << hor.size() << endl;
    for (auto p : hor) {
        cout << p.first << " " << p.second << endl;
    }
    cout << ver.size() << endl;
    for (auto p : ver) {
        cout << p.first << " " << p.second << endl;
    }

    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the board size (N) and the number of removed squares (P). It marks the removed squares in a 2D array `bad`.
2. **Initial Check**: If the total remaining squares is odd, it immediately outputs "No" since dominoes cover two squares each.
3. **Bipartite Matching**: The algorithm uses DFS to find a maximum matching between black and white squares (based on (x + y) modulo 2). The matching is stored in arrays tracking partners for each square.
4. **Result Validation**: If the matching size does not cover all remaining squares, it outputs "No".
5. **Output Preparation**: For successful matching, it classifies each domino as horizontal (same row) or vertical (same column). Horizontal dominoes report the left square (min x-coordinate), and vertical dominoes report the bottom square (min y-coordinate).
6. **Output**: Prints "Yes" followed by counts and coordinates of horizontal and vertical dominoes if a solution exists.

This approach efficiently checks for domino tiling using bipartite matching and constructs the solution by leveraging chessboard coloring properties and adjacency constraints.
