To solve this problem, we need to find the N-th number in the sequence of positive integers that do not contain the substring "13" in their decimal representation. Given the constraints where N can be as large as \(10^{18}\), a brute-force approach is infeasible. Instead, we use a combination of digit dynamic programming (digit DP) and binary search to efficiently determine the N-th valid number.

### Approach
1. **Precomputation**: We precompute the count of valid numbers for each digit length (1 to 60) using a recurrence relation. The recurrence relation is derived from the observation that the count of valid n-digit numbers without "13" follows \(T[n] = 10 \times T[n-1] - T[n-2]\), with base cases \(T[1] = 9\) and \(T[2] = 89\).

2. **Digit Dynamic Programming (Digit DP)**: For a given number \(x\), the function `count(x)` calculates how many valid numbers exist from 1 to \(x\). This involves:
   - Summing the counts of valid numbers with fewer digits than \(x\).
   - Using a DP approach to count valid numbers with the same number of digits as \(x\) that are less than or equal to \(x\), ensuring no "13" substring appears.

3. **Binary Search**: For each test case, we perform a binary search over possible values of \(x\) to find the smallest \(x\) such that `count(x) >= N`. The search space is bounded by adjusting the upper limit until `count(high) >= N`.

4. **Handling Large Numbers**: Since \(N\) can be up to \(10^{18}\), we use `__int128` to handle large numbers during computations, converting results to strings for output.

    if (x == 0) return "0";
    string s = "";
    while (x > 0) {
        s = char('0' + (x % 10)) + s;
        x /= 10;
    }
    return s;
}

__int128 count(__int128 x) {
    if (x == 0) return 0;
    string s = toString(x);
    int n = s.size();
    __int128 total = 0;
    for (int len = 1; len < n; len++) {
        total += T[len];
    }

    __int128 dp[61][2][2];
    memset(dp, 0, sizeof(dp));

    for (int d = 1; d <= (s[0] - '0'); d++) {
        int state = (d == 1) ? 1 : 0;
        int tight = (d == (s[0] - '0')) ? 1 : 0;
        dp[1][state][tight] = 1;
    }

    for (int i = 1; i < n; i++) {
        for (int state = 0; state < 2; state++) {
            for (int tight = 0; tight < 2; tight++) {
                if (dp[i][state][tight] == 0) continue;
                int limit = tight ? (s[i] - '0') : 9;
                for (int d = 0; d <= limit; d++) {
                    if (state == 1 && d == 3) continue;
                    int new_state = (d == 1) ? 1 : 0;
                    int new_tight = tight && (d == limit);
                    dp[i+1][new_state][new_tight] += dp[i][state][tight];
                }
            }
        }
    }

    for (int state = 0; state < 2; state++) {
        for (int tight = 0; tight < 2; tight++) {
            total += dp[n][state][tight];
        }
    }
    return total;
}

### Explanation
1. **Precomputation**: The `precompute_T` function calculates the number of valid numbers for each digit length up to 60 using the recurrence relation \(T[n] = 10 \times T[n-1] - T[n-2]\), which accounts for numbers that avoid the substring "13".

2. **Digit DP (`count` function)**:
   - **Initialization**: Converts the number \(x\) to a string to process digit by digit.
   - **Summing Shorter Numbers**: Adds counts of valid numbers with fewer digits than \(x\).
   - **DP for Same Length**: Uses a 3-dimensional DP array to count valid numbers with the same number of digits as \(x\), where states track the last digit (to avoid "13") and whether the current number is still tight (constrained by \(x\)'s digits).

3. **Binary Search**: For each test case, the binary search efficiently locates the smallest number \(x\) such that the count of valid numbers up to \(x\) is at least \(N\). The search starts with an exponentially increasing upper bound to cover large values of \(N\).

4. **Output Handling**: Results are converted from `__int128` to strings for correct decimal representation, ensuring compatibility with standard output.

This approach efficiently handles the large constraints by leveraging mathematical insights and dynamic programming, ensuring optimal performance even for the upper limits of the problem's requirements.
