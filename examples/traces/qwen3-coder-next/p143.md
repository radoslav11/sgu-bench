This problem asks for the maximum profit subset of towns such that the subset is connected (i.e., induces a connected subgraph). Since the original graph is a tree, any connected subset must also be a tree (a subtree).

This is the "Maximum Subtree Sum" problem, which can be solved using tree DP.

The idea:
- Root the tree at an arbitrary node (say node 0 or 1).
- For each node u, compute dp[u] = maximum profit we can get from the subtree rooted at u, **given that the selected connected subgraph includes u**.
- Then the answer is the maximum dp[u] over all nodes.

The recurrence:
- For a leaf node u: dp[u] = profit[u]
- For an internal node u:
  - Start with dp[u] = profit[u]
  - For each child v of u, if dp[v] > 0, we can include the subtree rooted at v (since adding it increases the profit), so:
    dp[u] += max(0, dp[v])

But note: since the subset must be connected and include u, we can choose to include any subset of children's subtrees that are profitable (and connected through u).

However, there is a subtle point: the problem asks for any non-empty connected subset, not necessarily rooted at a specific node. So we compute dp[u] as the maximum profit of a connected subgraph that includes u and is contained in the subtree rooted at u.

Then answer = max(dp[u] for all u).

Steps:
1. Read N and profits.
2. Build adjacency list for the tree.
3. Root the tree at node 1 (arbitrary).
4. Do DFS from root, computing dp values.
5. Track the maximum dp value during DFS.

Note: Since profits can be negative, we must consider that the best connected subgraph might be just a single node with positive profit, or even if all are negative, we must pick the least negative (since non-empty). But our DP naturally handles this: for a node, if all children have negative dp, we just take the node alone.

Let's implement accordingly.

Constraints: N up to 16000, so DFS recursion should be fine in C++ (but we might need to increase stack size or use iterative DFS; however, 16000 is usually okay for recursion in C++ in online judges).

We'll use 1-indexing.

Sample check:
- Towns: 5, profits = [-1, 1, 3, 1, -1] (for towns 1..5)
- Roads: 4-1, 1-3, 1-2, 4-5.

Let's draw:
- 1 connected to 4, 3, 2.
- 4 connected to 5.

If we root at 1:
- Children of 1: 2, 3, 4.
- Node 2: leaf -> dp[2] = 1
- Node 3: leaf -> dp[3] = 3
- Node 4: has child 5 -> dp[5] = -1 (so we don't include it) => dp[4] = 1 + max(0, dp[5]) = 1 + 0 = 1
- Then for node 1: dp[1] = profit[1] + max(0, dp[2]) + max(0, dp[3]) + max(0, dp[4])
          = -1 + 1 + 3 + 1 = 4

Then max over dp: 
dp[2]=1, dp[3]=3, dp[4]=1, dp[5]=-1, dp[1]=4 -> max=4.

So sample matches.

Implementation:

- Build graph (0-indexed internally, but input is 1-indexed).
- Use DFS (recursive) with parent tracking to avoid going back.

Let's code accordingly.


```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<int> profit(n + 1);
    for (int i = 1; i <= n; i++) {
        cin >> profit[i];
    }

    vector<vector<int>> graph(n + 1);
    for (int i = 0; i < n - 1; i++) {
        int a, b;
        cin >> a >> b;
        graph[a].push_back(b);
        graph[b].push_back(a);
    }

    vector<int> dp(n + 1, 0);
    int maxProfit = -1000 * n; // worst case: all negative

    function<void(int, int)> dfs = [&](int u, int parent) {
        dp[u] = profit[u];
        for (int v : graph[u]) {
            if (v == parent) continue;
            dfs(v, u);
            if (dp[v] > 0) {
                dp[u] += dp[v];
            }
        }
        maxProfit = max(maxProfit, dp[u]);
    };

    dfs(1, 0);

    cout << maxProfit << endl;

    return 0;
}
```
