To solve this problem, we need to help fans of the band "North-East" determine which cities the band might visit (list A) and which cities they must visit for sure (list B) during their tour. The band's tour involves visiting cities in a strictly north-east direction, meaning each subsequent city must have both coordinates strictly greater than the previous one. The tour is planned to maximize the number of cities visited.

### Approach
1. **Problem Analysis**: The problem reduces to finding the longest increasing subsequence (LIS) in 2D, where each city must have strictly greater x and y coordinates than the previous city. 
2. **Key Insights**:
   - **List A**: Cities that can be part of at least one longest chain (LIS of length M).
   - **List B**: Cities that are part of every longest chain (must-visit cities).
3. **Algorithm Selection**:
   - **Dynamic Programming (DP) with Coordinate Compression**: We use DP to compute:
     - `dp1[i]`: Length of the longest chain ending at city `i`.
     - `dp2[i]`: Length of the longest chain starting at city `i`.
   - **Fenwick Tree for Efficient Range Queries**: To efficiently compute `dp1` and `dp2`, we use Fenwick Trees (Binary Indexed Trees) for range maximum queries after compressing the y-coordinates.
4. **Complexity Analysis**:
   - **Time Complexity**: O(n log n) due to sorting and Fenwick Tree operations (each query/update is O(log n)).
   - **Space Complexity**: O(n) for storing cities, DP arrays, and Fenwick Trees.

### Explanation
1. **Input Handling**: The cities are read and stored with their coordinates and original indices.
2. **Coordinate Compression**: The y-coordinates are compressed to a smaller range to optimize Fenwick Tree operations.
3. **DP Array Calculation**:
   - **`dp1_arr`**: Computed by processing cities in increasing x and y order. For each city, the longest chain ending at it is found using a Fenwick Tree to query the maximum chain length for cities with smaller y-coordinates.
   - **`dp2_arr`**: Computed by processing cities in decreasing x and y order. For each city, the longest chain starting at it is found using a Fenwick Tree on reversed y-coordinates (z-coordinates).
4. **Finding Longest Chain Length (M)**: The maximum value in `dp1_arr` gives the length of the longest chain.
5. **Constructing Lists**:
   - **List A**: Cities where `dp1[i] + dp2[i] - 1 == M` are included, as they can be part of some longest chain.
   - **List B**: For each level (from 1 to M), if only one city exists in that level, it must be in every longest chain and is added to list B.
6. **Output**: The lists are sorted and printed.

This approach efficiently computes the required lists by leveraging dynamic programming and Fenwick Trees for optimal performance.
