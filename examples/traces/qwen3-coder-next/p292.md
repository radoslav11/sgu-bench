The graves must be placed parallel to the sides of the grid, meaning they can be oriented either horizontally or vertically.

### Approach
1. **Problem Analysis**: The key observation is that the maximum number of graves is constrained by the grid dimensions and the grave size \(n\). Specifically:
   - Horizontally placed graves (each occupying \(n\) columns in a single row) allow \(\lfloor c / n \rfloor\) graves per row, leading to \(q \times \lfloor c / n \rfloor\) graves.
   - Vertically placed graves (each occupying \(n\) rows in a single column) allow \(\lfloor q / n \rfloor\) graves per column, leading to \(c \times \lfloor q / n \rfloor\) graves.
   - However, placing graves in both orientations might lead to overlapping in the overlapping region. To avoid double-counting, we adjust by subtracting \(n \times \lfloor q / n \rfloor \times \lfloor c / n \rfloor\), which accounts for the overlapping region where both orientations would have been counted.

2. **Mathematical Formula**: The solution is given by:
   \[
   \text{ans} = q \times \lfloor c / n \rfloor + c \times \lfloor q / n \rfloor - n \times \lfloor q / n \rfloor \times \lfloor c / n \rfloor
   \]
   This formula ensures we count graves without overlap, leveraging integer division to handle large numbers.

3. **Handling Large Numbers**: Since \(q\), \(c\), and \(n\) can be as large as \(10^{1000}\), we use string manipulation to perform arithmetic operations (division, multiplication, addition, subtraction) on these large numbers. The operations include:
   - **Division**: Long division to compute \(\lfloor q / n \rfloor\) and \(\lfloor c / n \rfloor\).
