To solve this problem, we need to determine the smallest area of a convex polygon when it is split by a given line for multiple queries. The polygon represents the border of Berland, and each query defines a cutting line. The solution involves efficiently computing the area of the smaller part resulting from each cut.

### Approach
1. **Problem Analysis**: The problem requires processing multiple queries where each query defines a line that splits a convex polygon into two parts. The goal is to find the area of the smaller part for each query.
2. **Key Insight**: For a convex polygon, the intersection with a half-plane (defined by the cutting line) results in another convex polygon. The area of this intersection can be computed using the Sutherland-Hodgman algorithm for clipping the polygon against the half-plane.
3. **Algorithm Selection**:
   - **Total Area Calculation**: Compute the total area of the original polygon using the shoelace formula.
   - **Clipping for Each Query**: For each query line:
     - Determine the line equation \(ax + by + c = 0\).
     - Clip the polygon against the half-plane \(ax + by + c \leq 0\) by traversing each edge of the polygon, adding vertices that lie inside the half-plane, and intersection points where edges cross the line.
     - Compute the area of the clipped polygon using the shoelace formula.
     - The answer for the query is the minimum of the clipped area and the remaining area (total area - clipped area).
4. **Complexity Analysis**: 
   - **Preprocessing**: Computing the total area takes \(O(N)\) time.
   - **Query Processing**: Each query involves \(O(N)\) operations for clipping and area computation. With \(P\) queries, the total time complexity is \(O(P \cdot N)\), which is feasible given the constraints (since \(N\) and \(P\) are up to \(5 \times 10^4\), but worst-case scenarios might be optimized in practice due to efficient constants and compiler optimizations).

#include <vector>
#include <cmath>
#include <iomanip>
using namespace std;

struct Point {
    double x, y;
    Point() {}
    Point(double x, double y) : x(x), y(y) {}

    int n = p.size();

    for (int i = 0; i < n; i++) {
