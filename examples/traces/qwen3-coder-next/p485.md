              - For A and C: maximize the dot product -> sort A and C in the same order.
              - For B and C: minimize the dot product -> sort B increasingly and C decreasingly.

          But the catch: the same C is used in both.

        Final insight:

          S = sum_i c_i * (a_i - b_{N-1-i})

          and we know b_{N-1-i} for i=0..N-1: 
            b_{N-1} is the largest in B, b_{N-2} is the next largest, etc.

          In fact, let d_i = b_{N-1-i} for i=0..N-1. Then d is the B array sorted in decreasing order.

          So S = sum_i c_i * (a_i - d_i)

          = sum_i c_i * a_i - sum_i c_i * d_i.

          Now, to maximize this, we want to maximize the first sum and minimize the second sum.

          For the first sum: maximize sum c_i * a_i -> sort C and A in the same order.
          For the second sum: minimize sum c_i * d_i -> since d is fixed (B sorted decreasingly), we want C sorted in the opposite order to d to minimize the sum. But d is decreasing, so to minimize the sum, C should be increasing.

          So for the second sum, we want C increasing and d decreasing -> which is already what we have for the first sum? 

          Actually, if we sort C increasingly and A increasingly, then for the first sum we have the maximum possible for sum c_i * a_i.
          For the second sum, with C increasingly and d decreasingly, we have the minimum possible for sum c_i * d_i.

          Therefore, the assignment is:

            - B = the smallest N numbers, sorted increasingly: b0, b1, ..., b_{N-1}
            - Let d = B sorted in decreasingly: d_i = b_{N-1-i}
            - The remaining 2N numbers: sort them increasingly: y0, y1, ..., y_{2N-1}
            - Split into A and C: 
                  A = y_{N}, y_{N+1}, ..., y_{2N-1}  (the largest N of the 2N)
                  C = y_{0}, y_{1}, ..., y_{N-1}      (the smallest N of the 2N)

            Why? 
              Then A sorted increasingly = [y_N, ..., y_{2N-1}]
              C sorted increasingly = [y_0, ..., y_{N-1}]

              But then the pairing: 
                 a_i = y_{N+i} for i=0..N-1
                 c_i = y_i for i=0..N-1

              Then S = sum_i c_i * a_i - sum_i c_i * d_i
                     = sum_i y_i * y_{N+i} - sum_i y_i * d_i

            Let's test with the example:
              x = [0,1,2,4,5,8]
              B = [0,1] -> d = [1,0] (decreasingly: d0=1, d1=0)
              Remaining = [2,4,5,8] -> sorted: [2,4,5,8]
              C = [2,4] (smallest N=2)
              A = [5,8] (largest N=2)
              Then:
                sum c_i * a_i = 2*5 + 4*8 = 10+32 = 42
                sum c_i * d_i = 2*1 + 4*0 = 2+0 = 2
                S = 42 - 2 = 40.
