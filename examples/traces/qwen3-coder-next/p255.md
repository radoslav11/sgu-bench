To solve this problem, we need to determine for a given number \( M \) whether there exists exactly one positive integer \( K \) such that the number of ports in the range \([K+1, 2K]\) with exactly three non-zero bits (i.e., popcount = 3) is exactly \( M \).

### Approach
1. **Precompute Popcount-3 Numbers**: Generate all numbers up to \( 2^{60} \) that have exactly three 1s in their binary representation. This is done by iterating over all possible triples \((a, b, c)\) where \( a > b > c \geq 0 \) and forming the number \( 2^a + 2^b + 2^c \).

2. **Event-Based Counting**: For each generated number \( x \), determine the range of \( K \) values for which \( x \) lies in \([K+1, 2K]\). Specifically, \( K \) must satisfy \( \lceil x/2 \rceil \leq K \leq x-1 \). We use a difference map to mark the start and end of these intervals.

3. **Sweep Line Algorithm**: Process the critical points (start and end of intervals) to compute the number of valid \( K \) values for each possible \( M \). For segments between critical points where the count \( M \) is within a reasonable bound (we use 1000 as the threshold), accumulate the segment lengths into a frequency array.

4. **Query Processing**: For each test case, check the precomputed frequency array. If \( M \) is within bounds and occurs exactly once, output "YES"; otherwise, output "NO".

### Solution Code
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAX_M = 1000;

int main() {
    vector<unsigned long long> S;
    for (int a = 2; a < 60; a++) {
        for (int b = 1; b < a; b++) {
            for (int c = 0; c < b; c++) {
                unsigned long long x = (1ULL << a) | (1ULL << b) | (1ULL << c);
                S.push_back(x);
            }
        }
    }
    sort(S.begin(), S.end());

    map<unsigned long long, int> diff_map;
    for (auto x : S) {
        unsigned long long L = (x + 1) / 2;
        unsigned long long R = x;
        diff_map[L]++;
        diff_map[R]--;
    }

    vector<unsigned long long> keys;
    for (auto& p : diff_map) {
        keys.push_back(p.first);
    }
    sort(keys.begin(), keys.end());

    vector<long long> freq(MAX_M + 1, 0);
    unsigned long long current = 0;
    unsigned long long prev_k = 0;

    for (int i = 0; i < keys.size(); i++) {
        unsigned long long k = keys[i];
        if (prev_k < k) {
            if (current <= MAX_M) {
                freq[current] += (k - prev_k);
            }
        }
        current += diff_map[k];
        prev_k = k;
    }

    int N;
    cin >> N;
    while (N--) {
        long long M;
        cin >> M;
        if (M > MAX_M) {
            cout << "NO" << endl;
        } else {
            if (freq[M] == 1) {
                cout << "YES" << endl;
            } else {
                cout << "NO" << endl;
            }
        }
    }
    return 0;
}
```

### Explanation
1. **Precomputation**: The code first generates all numbers with exactly three 1s in their binary representation (popcount = 3) up to a sufficiently large value (\( < 2^{60} \)). This is done using three nested loops over possible bit positions \( a, b, c \).

2. **Event Marking**: For each generated number \( x \), the code calculates the interval \([L, R)\) where \( L = \lceil x/2 \rceil \) and \( R = x \). It marks the start of this interval with +1 and the end with -1 in a difference map.

3. **Sweep Processing**: The critical points (keys of the difference map) are sorted. The algorithm then processes these points to compute the number of \( K \) values for each possible \( M \) by sweeping through the segments between critical points. For segments where the count \( M \) is within the predefined bound (1000), it accumulates the segment length into a frequency array.

4. **Query Handling**: For each test query \( M \), the solution checks if \( M \) is within bounds. If \( M \) exceeds 1000, it defaults to "NO" (since large \( M \) values are unlikely to have exactly one \( K \)). Otherwise, it checks the frequency array: if \( M \) occurs exactly once, it outputs "YES"; otherwise, "NO".

This approach efficiently handles the constraints by leveraging precomputation and a sweep line algorithm to avoid brute-force checks, ensuring optimal performance even for the upper limits of input sizes.
