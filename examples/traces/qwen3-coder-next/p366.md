To solve this problem, we need to select exactly K opponents from N opponents such that the absolute difference between the total pleasure (A) and total score points (B) is minimized. If multiple solutions exist, we choose the one that maximizes the sum of A and B. 

### Approach
1. **Problem Analysis**: The problem requires selecting K opponents such that the absolute difference |A - B| is minimized, where A is the sum of pleasure values (a_i) and B is the sum of score points (b_i) of the selected opponents. If there are multiple solutions with the same minimal |A - B|, we select the one with the maximum A + B.

2. **Key Insight**: The difference (A - B) can be represented as the sum of individual differences (a_i - b_i) for the selected opponents. Let `da_i = a_i - b_i` and `ab_sum_i = a_i + b_i`. The goal is to find a subset of K opponents where the sum of `da_i` is as close to zero as possible, and among such subsets, the sum of `ab_sum_i` is maximized.

3. **Dynamic Programming (DP) Setup**: We use a DP approach where `dp[j][d]` represents the maximum sum of `ab_sum_i` for selecting `j` opponents with a total difference `d` (where `d` is shifted by a base value to handle negative indices). The base value is `50*K` to ensure all possible differences (ranging from `-50*K` to `50*K`) are non-negative.

4. **DP State Transition**: For each opponent, we update the DP table in reverse order (to avoid reusing the same opponent) for `j` from K down to 1. For each possible difference `d`, we consider adding the current opponent's `da_i` and update the DP state if a better sum is found.

5. **Parent Tracking**: To reconstruct the solution, we maintain parent arrays that track the previous difference and the opponent index used to reach the current state.

6. **Solution Reconstruction**: After processing all opponents, we find the state with `j = K` that has the smallest absolute difference. If multiple states exist with the same minimal absolute difference, we choose the one with the highest sum. We then backtrack using the parent arrays to collect the indices of selected opponents.

        int abs_diff = abs(d_val);
        long long sum_ab = dp[K][d_index];
        if (abs_diff < best_abs_diff || (abs_diff == best_abs_diff && sum_ab > best_sum)) {
            best_abs_diff = abs_diff;
            best_sum = sum_ab;
            best_d_index = d_index;
        }
    }

    vector<int> chosen_indices;
    int j = K;
    int d_idx = best_d_index;
    while (j > 0) {
        int opponent_index = parent_opponent[j][d_idx];
        chosen_indices.push_back(opponent_index);
        int prev_d_idx = parent_d[j][d_idx];
        d_idx = prev_d_idx;
        j = j - 1;
    }

    long long A = (best_sum + (best_d_index - base)) / 2;
    long long B = (best_sum - (best_d_index - base)) / 2;
    cout << A << " " << B << "\n";

    sort(chosen_indices.begin(), chosen_indices.end());
    for (int i = 0; i < chosen_indices.size(); i++) {
        if (i > 0) cout << " ";
        cout << chosen_indices[i] + 1;
    }
    cout << "\n";

    return 0;
}

### Explanation
1. **Initialization**: We read the input values for the number of opponents (N) and the number to select (K), followed by the pleasure and score values for each opponent.
2. **DP Setup**: We initialize a DP table where `dp[j][d]` stores the maximum sum of `a_i + b_i` for selecting `j` opponents with a difference `d` (shifted by `base` to handle negatives).
3. **Processing Opponents**: For each opponent, we update the DP table in reverse order (to avoid reuse) by considering the opponent's contribution to the difference (`a_i - b_i`) and the sum (`a_i + b_i`).
4. **Finding Optimal Solution**: After processing all opponents, we find the state with `j = K` that has the smallest absolute difference. If multiple states exist, we choose the one with the highest sum.
5. **Reconstruction**: Using parent arrays, we backtrack to collect the indices of selected opponents, sort them, and convert to 1-based indexing.
6. **Output**: We compute and output the total pleasure (A) and score (B), followed by the sorted indices of selected opponents.

This approach efficiently explores all possible subsets of size K using dynamic programming, ensuring optimal performance even for large N by leveraging the constraints on K. The solution minimizes the absolute difference and maximizes the sum when ties occur, providing the correct output as required.
