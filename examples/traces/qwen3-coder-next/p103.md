## Problem Understanding

This is a shortest path problem with time-dependent edge weights. The key insight is that the ability to traverse an edge depends on the synchronized colors of the two junctions at the departure time.

Key elements:
1. Each junction has a traffic light that alternates between blue and purple
2. We can only traverse an edge when both endpoints have the same color at departure time
3. We can wait at junctions
4. Travel times are positive integers

## Algorithm Approach

This is a variation of Dijkstra's algorithm where the "distance" to a node depends on when we arrive there. Since we can wait at nodes, we need to consider the arrival time as part of our state.

The state will be: `(arrival_time, current_node)`

For each node, we need to compute the earliest time we can reach it. For edges, we need to compute the earliest time we can traverse them given our arrival time at the source node of the edge.

## Dijkstra Implementation

We'll use:
- `dist[i]`: earliest arrival time at node i
- `parent[i]`: previous node in the shortest path to i
- Priority queue: (time, node)

For each edge (u, v) with travel time l:
- If we're at u with arrival time `dist[u]`, compute `start_time = same_color_time(u, v, dist[u])`
- If we can improve `dist[v]` by going through this edge: 
  - `new_time = start_time + l`
  - If `new_time < dist[v]`, update

Note: The graph is undirected, so edges work both ways.

using namespace std;

typedef long long ll;
typedef pair<ll, int> pli; // (distance, node)

const ll INF = 1LL << 60;

int gcd(int a, int b) {
    while (b != 0) {
        int temp = b;
        b = a % b;
        a = temp;
    }
    return a;
}

int lcm(int a, int b) {
    if (a == 0 || b == 0) return 0;
    return a * b / gcd(a, b);
}

struct Node {
    int init_color; // 0 for 'B', 1 for 'P'
    int rem_time;
    int dur[2]; // dur[0] = blue duration, dur[1] = purple duration
};

int get_color(const vector<Node>& nodes, int u, ll t) {
    const Node& node = nodes[u];
    
    // Initial phase: color remains init_color for rem_time
    if (t < node.rem_time) {
        return node.init_color;
    }
    
    // After initial phase, the cycle is periodic
    ll cycle = node.dur[0] + node.dur[1];
    ll elapsed = t - node.rem_time;
    
    // Handle the case where cycle might be 0 (shouldn't happen per constraints)
    if (cycle == 0) return node.init_color;
    
    elapsed %= cycle;
    
    // After rem_time, the color alternates
    if (node.init_color == 0) { // started with blue
        // First after rem_time: purple for dur[1]
        if (elapsed < node.dur[1]) {
            return 1; // purple
        } else {
            return 0; // blue
        }
    } else { // started with purple
        // First after rem_time: blue for dur[0]
        if (elapsed < node.dur[0]) {
            return 0; // blue
        } else {
            return 1; // purple
        }
    }
}

ll same_color_time(const vector<Node>& nodes, int u, int v, ll t) {
    // Check times up to t + lcm(cycle[u], cycle[v])
    int cycle_u = nodes[u].dur[0] + nodes[u].dur[1];

            return check_time;
        }
    }
    
    // Fallback (shouldn't happen in practice)
    return t + combined_cycle;
}
