I need to solve a competitive programming problem where I need to distribute M coins among N robbers to minimize total unfairness.

The unfairness for robber i is |Xi/Y - Ki/M|, and I need to minimize the sum of these unfairnesses, subject to the constraint that sum(Ki) = M and each Ki is a non-negative integer.

Let me think about this mathematically:
- The ideal share for robber i would be (Xi/Y) * M coins
- Let's call this value `ideal[i] = (Xi * M) / Y` (as a real number)

Since we need integer values that sum to M, a common approach is:
1. Compute the floor of each ideal value: `floor[i] = floor(ideal[i])`
2. Compute the remaining coins: `remaining = M - sum(floor[i])`
3. Distribute the remaining coins to those who would benefit most from an extra coin

The key insight is that adding one coin to robber i changes the unfairness by:
- Current unfairness: |Xi/Y - Ki/M|
- New unfairness: |Xi/Y - (Ki+1)/M|

The difference depends on whether Ki/M is less than or greater than Xi/Y.

Actually, a more systematic approach is to use the concept of "marginal cost" - which robber would benefit most (or have the smallest increase in unfairness) from getting an additional coin.

For each robber i, if we have currently assigned `ki` coins, the marginal cost of giving them one more coin is:
- If `ki/M <= Xi/Y`: then adding a coin reduces the unfairness (since we're getting closer to the target)
- If `ki/M > Xi/Y`: then adding a coin increases the unfairness

The amount of reduction or increase can be calculated as:
- Current unfairness: `|Xi/Y - ki/M|`
- New unfairness: `|Xi/Y - (ki+1)/M|`
- Change: `|Xi/Y - (ki+1)/M| - |Xi/Y - ki/M|`

Since we want to minimize total unfairness, we should give extra coins to those for whom this change is smallest (most negative or least positive).

A standard approach for this type of problem is:
1. Start with `ki = floor(Xi * M / Y)` for each i
2. Let `rem = M - sum(ki)` be the remaining coins to distribute
3. For each extra coin, give it to the robber i with the minimum value of `|Xi/Y - (ki+1)/M| - |Xi/Y - ki/M|`
4. Or equivalently, give coins to those with the largest "excess" `Xi*M - ki*Y` (since this represents how much they're short of their ideal share in terms of Y*M units)

Actually, let me think in terms of fractions to avoid floating point precision issues:
- We want to minimize `sum(|Xi/Y - Ki/M|) = sum(|(Xi*M - Ki*Y)/(Y*M)|) = (1/(Y*M)) * sum(|Xi*M - Ki*Y|)`
- So minimizing `sum(|Xi*M - Ki*Y|)` is equivalent

Now, for each robber, if we've assigned `Ki` coins, the "deficit" in terms of Y units is `Xi*M - Ki*Y`.
- If this is positive, the robber needs more coins
- If this is negative, the robber has too many coins

When we give an extra coin to robber i, `Ki` increases by 1, so `Xi*M - Ki*Y` decreases by Y.

The change in `|Xi*M - Ki*Y|` when we add a coin is:
- If `Xi*M - Ki*Y >= 0`: the absolute value decreases by Y (until it might become negative)
- If `Xi*M - Ki*Y < 0`: the absolute value increases by Y

So for each robber, the "priority" for getting an extra coin is based on `Xi*M - Ki*Y` - the higher this value, the more benefit (or less harm) from giving an extra coin.

Algorithm:
1. Compute initial `Ki = floor(Xi * M / Y)` using integer arithmetic: `Ki = (Xi * M) / Y` (integer division truncates toward 0, but since all values are positive, this is floor)
2. Compute remaining coins: `rem = M - sum(Ki)`
3. Create a priority queue or sort based on the value `(Xi * M) - (Ki * Y)` (the residual)
4. For each of the `rem` coins, give it to the robber with the highest residual value, and update their residual by subtracting Y

Implementation:
1. Read N, M, Y and the X array
2. Compute initial K[i] = (X[i] * M) / Y for each i
3. Compute remaining = M - sum(K)
4. Create an array of indices sorted by (X[i] * M - K[i] * Y) in descending order
5. For i from 0 to remaining-1, increment K[index[i]]
6. Output K

Note: Since we're dealing with integer arithmetic, we can compute the residual as `X[i] * M - K[i] * Y` without floating point.

Let me write the code accordingly.

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    int N, M, Y;
    cin >> N >> M >> Y;
    
    vector<int> X(N);
    for (int i = 0; i < N; i++) {
        cin >> X[i];
    }
    
    vector<int> K(N, 0);
    long long sumK = 0;
    
    // Compute initial K[i] = floor(X[i] * M / Y)
    for (int i = 0; i < N; i++) {
        K[i] = (X[i] * M) / Y;
        sumK += K[i];
    }
    
    // Calculate remaining coins to distribute
    int remaining = M - sumK;
    
    // Create indices sorted by residual (X[i] * M - K[i] * Y) in descending order
    vector<int> indices(N);
    for (int i = 0; i < N; i++) {
        indices[i] = i;
    }
    
    // Sort by residual in descending order
    sort(indices.begin(), indices.end(), [&](int i, int j) {
        long long residual_i = 1LL * X[i] * M - 1LL * K[i] * Y;
        long long residual_j = 1LL * X[j] * M - 1LL * K[j] * Y;
        if (residual_i != residual_j) {
            return residual_i > residual_j;
        }
        // For tie-breaking, we can use any consistent order
        return i < j;
    });
    
    // Distribute remaining coins
    for (int i = 0; i < remaining; i++) {
        K[indices[i]]++;
    }
    
    // Output the result
    for (int i = 0; i < N; i++) {
        if (i > 0) cout << " ";
        cout << K[i];
    }
    cout << endl;
    
    return 0;
}
```
