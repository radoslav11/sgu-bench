To solve this problem, we need to determine all possible thimbles that can have the ball after performing exactly M swap operations in any order. The ball starts under the first thimble (position 1), and each swap operation exchanges the positions of two thimbles. The key insight is that the ball can only move to thimbles within the connected component of the initial position (thimble 1) in the graph formed by the swap operations. This is because swaps only affect the positions of thimbles directly involved in the swaps, and the ball can only be moved to positions reachable through a sequence of swaps starting from the initial position.

### Approach
1. **Graph Construction**: Treat each thimble as a node in a graph and each swap operation as an undirected edge between two nodes.
2. **Connected Component Identification**: Perform a Depth-First Search (DFS) or Breadth-First Search (BFS) starting from node 1 (thimble 1) to identify all nodes reachable from the initial position. These nodes form the connected component containing the initial thimble.
3. **Result Collection**: The thimbles (nodes) within this connected component are the only possible positions where the ball can end up after any sequence of the M swap operations, as swaps outside this component do not affect the ball's position.

### Solution Code

#include <vector>
#include <algorithm>

    for (int i = 0; i < m; i++) {
        int a, b;
        cin >> a >> b;
        adj[a].push_back(b);
        adj[b].push_back(a);
    }

    vector<bool> visited(n + 1, false);
    vector<int> component;
    
    for (int i = 1; i <= n; i++) {
        if (!visited[i]) {
            vector<int> comp;
            // BFS for component starting at i
            if (i == 1) {
                vector<bool> vis(n + 1, false);
                vector<int> q;
                q.push_back(1);
                vis[1] = true;
                while (!q.empty()) {
                    int u = q.back();
                    q.pop_back();
                    comp.push_back(u);
                    for (int v : adj[u]) {
                        if (!vis[v]) {
                            vis[v] = true;
                            q.push_back(v);

                    }

2. **Connected Component Identification**: Using BFS starting from thimble 1, the code identifies all thimbles reachable from the initial position. This is stored in the `component` vector.
3. **Result Collection**: The thimbles in the connected component containing thimble 1 are collected into the result list. 4. **Output**: The result is printed as space-separated integers representing the thimble positions.
