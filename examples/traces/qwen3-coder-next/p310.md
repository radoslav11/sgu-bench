To solve this problem, we need to count the number of valid binary strings of length `n` (where each bit represents a board: 0 for wooden, 1 for iron) such that every contiguous segment of `m` boards contains at least `k` iron boards. The solution involves dynamic programming (DP) with a state that tracks the last `m-1` boards to efficiently enforce the constraint for each new board added.

### Approach
1. **Problem Analysis**: The problem requires counting binary strings of length `n` where every window of `m` consecutive boards has at least `k` ones. Given the constraints (n ≤ 60, m ≤ 15), a brute-force approach is infeasible. Instead, we use DP where the state represents the last `m-1` boards.
2. **Dynamic Programming Setup**: The DP state `dp[i][state]` counts the number of valid sequences of `i` boards ending with the bitmask `state` (representing the last `min(i, m-1)` boards). The state size is `2^(m-1)`, which is manageable since `m` is small (≤ 15).
3. **State Transition**: For each position `i` from 0 to `n-1`, and for each possible state:
   - **New Board Selection**: Try adding either a wooden (0) or iron (1) board.
   - **Constraint Check**: If `i+1 >= m`, verify that the new window of `m` boards (formed by the last `m-1` boards in the current state and the new board) contains at least `k` ones. If valid, update the next state.
4. **Initialization and Result Calculation**: Start with an empty sequence (state `0` with count `1`). After processing all `n` boards, sum the counts of all valid end states.

### Solution Code
```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    long long n, m, k;
    cin >> n >> m >> k;

    int num_states = 1 << (m - 1);
    vector<long long> dp(num_states, 0);
    dp[0] = 1;

    for (int i = 0; i < n; i++) {
        vector<long long> new_dp(num_states, 0);
        for (int state = 0; state < num_states; state++) {
            if (dp[state] == 0) 
                continue;
            for (int b = 0; b < 2; b++) {
                if (i + 1 < m) {
                    int new_state = ((state << 1) | b) & (num_states - 1);
                    new_dp[new_state] += dp[state];
                } else {
                    int window = (state << 1) | b;
                    if (__builtin_popcount((unsigned int)window) >= k) {
                        int new_state = ((state << 1) | b) & (num_states - 1);
                        new_dp[new_state] += dp[state];
                    }
                }
            }
        }
        dp = new_dp;
    }

    long long ans = 0;
    for (int state = 0; state < num_states; state++) {
        ans += dp[state];
    }
    cout << ans << endl;

    return 0;
}
```

### Explanation
1. **Reading Input**: The integers `n`, `m`, and `k` are read from standard input.
2. **DP Initialization**: The DP array `dp` is initialized to store counts of sequences ending with each possible state (bitmask of the last `m-1` boards). The initial state (no boards) has count `1`.
3. **Processing Boards**: For each board position:
   - **State Transition**: For each state, try adding a wooden (`0`) or iron (`1`) board.
   - **Constraint Handling**: If enough boards have been placed (`i+1 >= m`), check that the new window of `m` boards meets the requirement (at least `k` iron boards). Valid transitions update the next state's count.
4. **Result Calculation**: After processing all boards, the result is the sum of counts for all valid end states in `dp`.
5. **Output**: The total count of valid sequences is printed.

This approach efficiently leverages dynamic programming with bitmask states to handle the constraint, ensuring optimal performance even for the upper constraint limits. The complexity is O(n * 2^(m-1) * 2), which is feasible given the problem constraints.
