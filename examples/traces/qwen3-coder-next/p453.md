To solve this problem, we need to determine the number of times a cyclist and a pedestrian meet on a straight road between two cities, A and B, over a given time period. The cyclist starts from city A towards city B, and the pedestrian starts from city B towards city A. Both move at constant speeds and reverse direction upon reaching either city. The solution involves calculating the meetings by considering both opposite-direction and same-direction encounters, while avoiding double-counting meetings that occur at the cities.

### Approach
1. **Opposite-Direction Meetings**: These occur when the cyclist and pedestrian are moving towards each other. The number of such meetings is given by:
   \[
   \text{count1} = \left\lfloor \frac{(v1 + v2) \times t + l}{2 \times l} \right\rfloor
   \]
   This formula accounts for all meetings where the sum of distances traveled equals an odd multiple of the road length \( l \).

2. **Same-Direction Meetings**: These occur when one catches up to the other (e.g., the faster person catches the slower one after both have turned around). The number of such meetings is:
   \[
   \text{count2} = \left\lfloor \frac{|v1 - v2| \times t + l}{2 \times l} \right\rfloor
   \]
   This formula accounts for meetings where the difference in distances traveled equals an odd multiple of \( l \).

3. **Double-Counted Meetings**: Meetings that occur exactly at the cities (A or B) might be counted in both the above categories. To correct for this, we compute the number of such overlapping meetings:
   \[
   \text{double\_count} = \left\lfloor \frac{t \times g + l}{2 \times l} \right\rfloor
   \]
   where \( g \) is the greatest common divisor (gcd) of \( (v1 + v2) \) and \( |v1 - v2| \).

4. **Final Count**: The total number of meetings is:
   \[
   \text{ans} = \text{count1} + \text{count2} - \text{double\_count}
   \]

#include <iostream>
#include <algorithm>
using namespace std;

long long gcd(long long a, long long b) {
    if (b == 0)
        return a;
    return gcd(b, a % b);
}

int main() {
    long long l, v1, v2, t;
    cin >> l >> v1 >> v2 >> t;

    long long total_relative = v1 + v2;
    long long count1 = (total_relative * t + l) / (2 * l);

    long long diff = abs(v1 - v2);
    long long count2 = 0;
    long long g = 0;
    long long double_count = 0;

    if (diff > 0) {
        count2 = (diff * t + l) / (2 * l);
        g = gcd(total_relative, diff);
        double_count = (t * g + l) / (2 * l);
    }

    long long ans = count1;
    if (diff > 0) {
        ans = count1 + count2 - double_count;
    }

    cout << ans << endl;

    return 0;
}

3. **Same-Direction Meetings (count2)**: For cases where speeds differ, calculates meetings where the difference in distances traveled is an odd multiple of \( l \).
4. **Double-Count Correction (double_count)**: Uses gcd to find overlapping meetings at cities, ensuring they are counted only once.
5. **Result Calculation**: Combines counts from both types of meetings and subtracts overlaps to get the final count of meetings within time \( t \).
