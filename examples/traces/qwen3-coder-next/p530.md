We are given n programmers (with geniuses at 1,2,3) and m friendship edges.
The hiring process:
- Two companies alternate, company 1 starts.
- A company can only hire a programmer if:
    - It's the first hire of the company (then arbitrary), OR
    - The programmer has at least one friend already in that company.

Both companies play optimally to maximize the number of geniuses they hire.
We are to determine which company (1 or 2) will end up with two geniuses.

Important: It is guaranteed that one company will hire two geniuses.

Observations:
1. Since the geniuses are only 3, and both companies want as many as possible, the optimal play will focus on securing the geniuses.

2. Note that once a company hires a genius, they can then hire others who are adjacent to that genius (or adjacent to any already hired in the company, but the genius is the most valuable).

3. The key idea: the companies will try to secure the geniuses. Since the first company starts, it can choose a genius first. But then the second company can choose another genius. Then the first company wants a third, but the second might block.

However, note the constraint: a new hire must have a friend already in the company.

So to hire a genius, the company must either:
   - Start with a genius (so first move: choose any genius), or
   - Have a friend of that genius already in the company.

But note: the geniuses (1,2,3) might be connected to each other or not.

Let’s denote the geniuses as G = {1,2,3}.

We need to analyze the graph structure around the geniuses.

Important known fact: In both examples, the geniuses are connected to a common node (programmer 4 in the first, and 4,5,6 form a cycle but each genius connected to two non-genius nodes).

What matters is the connectivity of the geniuses and the possibility to "branch out" from one genius to others.

However, note the problem says: "It is guaranteed that one company will hire two geniuses."

So we don't have to handle cases where it's a tie.

To solve this problem, we need to determine which company (1 or 2) will successfully hire two out of the three geniuses (numbered 1, 2, and 3) when both companies employ optimal strategies. The companies alternate hiring programmers, starting with company 1, and each new hire must have at least one friend already in their company (except for the first hire, which can be any programmer).

### Approach
1. **Problem Analysis**: The key observation is that the companies aim to maximize the number of geniuses they hire. The optimal strategy involves securing geniuses early, but the initial choice might involve a non-genius if it blocks the opponent more effectively. However, the solution leverages graph properties around the geniuses to determine the winner without simulating all possible moves.

2. **Insight**: For company 1 to secure two geniuses, there must exist a node (programmer) such that:
   - It is closer to one genius than to the other two geniuses.
   - The distance from this node to one of the other geniuses is at most the sum of the distances from the node to all geniuses. This ensures company 1 can hire the initial genius and use this node to subsequently hire another genius before company 2 can block.

3. **Algorithm**:
   - Perform BFS from each of the three geniuses to compute the shortest distances from every programmer to each genius.
   - For each programmer, check if they satisfy the condition where they are closer to one genius than the others, and the distance to another genius is within the sum of distances to all geniuses. If such a programmer exists, company 1 can secure two geniuses; otherwise, company 2 can.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <cstring>
using namespace std;

const int MAXN = 100010;

vector<int> g[MAXN];
int dist[3][MAXN];

void bfs(int s, int *d, int n) {
    queue<int> q;
    memset(d, -1, sizeof(int) * (n + 1));
    d[s] = 0;
    q.push(s);
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        for (int v : g[u]) {
            if (d[v] == -1) {
                d[v] = d[u] + 1;
                q.push(v);
            }
        }
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
        g[v].push_back(u);
    }

    bfs(1, dist[0], n);
    bfs(2, dist[1], n);
    bfs(3, dist[2], n);

    int ans = 2;
    for (int i = 1; i <= n; i++) {
        if (dist[0][i] <= dist[1][i] && dist[0][i] <= dist[2][i]) {
            if (dist[1][i] <= dist[0][i] + dist[2][i]) {
                ans = 1;
            }
        }
        if (dist[1][i] <= dist[0][i] && dist[1][i] <= dist[2][i]) {
            if (dist[0][i] <= dist[1][i] + dist[2][i]) {
                ans = 1;
            }
        }
        if (dist[2][i] <= dist[0][i] && dist[2][i] <= dist[1][i]) {
            if (dist[0][i] <= dist[1][i] + dist[2][i]) {
                ans = 1;
            }
        }
    }

    cout << ans << endl;
    return 0;
}
```

1. **Input Handling**: The code reads the number of programmers (n) and friendships (m), then constructs the friendship graph.
2. **BFS for Shortest Paths**: For each genius (1, 2, 3), it computes the shortest path distances from every programmer to that genius using BFS.
3. **Checking Optimal Conditions**: For each programmer, it checks if they can serve as a strategic node where company 1 can first secure one genius and then use the node to subsequently secure another genius before company 2 can interfere. If such a node exists for any genius, company 1 wins; otherwise, company 2 wins.
4. **Output**: The result (1 or 2) is printed based on whether the optimal condition is met.

This approach efficiently leverages graph properties to determine the winner without exhaustive simulation, ensuring optimal performance even for large inputs.
