We are given n and an array a[1..n]. We need to construct a bipermutation of order n: 
 a linear order of the 2n elements: 1, 1', 2, 2', ... , n, n' (we'll represent i' as -i)

 Conditions:
 1. For each i: i <. i'  (i comes before i')
 2. For each i, j: i' <. j'  <=>  i <. j  (the relative order of the primed elements is the same as the natural order of their indices)

 Definition: 
   bij = 
        j'   if j < i
        j    if j > i

 Then a(i) is defined as the number of j such that i <. bij.

 Note: bii is undefined, so we consider j ≠ i.

 Let's break down the definition of a(i):

 For a fixed i, we consider all j ≠ i.

   Case 1: j < i
        bij = j' 
        Condition: i <. j' 
        But note: since j < i, by condition 2 we have j' <. i'? 
        Actually condition 2 says: j' <. i'  <=> j < i -> which is true. So j' <. i'. 
        Therefore, for j < i: we have j' <. i', so i cannot be <. j'? 
        But wait: condition 2 only tells about the relative order of primed elements. 
        However, we also have condition 1: i <. i'. So the positions of i and i' are fixed relative to each other.

        Actually, the condition 2 says the primed elements are in the same order as the natural indices. 
        So the primed elements appear in increasing index order? Actually, it says: 
          i' <. j'  <=> i < j   [if we interpret the natural order of indices as the standard one]

        So the primed elements are in increasing order: 1', 2', ..., n' appear in that order? 
        But note: the condition is: i' <. j' if and only if i < j. So yes, the primed elements must appear in increasing index order.

        Now, what about non-primed? They can be arbitrary as long as for each i: i <. i'.

        Now, for j < i: 
          bij = j'
          Condition: i <. j'   -> meaning the element i must come before j' in the bipermutation.

        But note: since j < i, by condition 2 we have j' <. i'. 
        And by condition 1: i <. i'. 
        So we have j' <. i' and i <. i'. But we don't know the relation between i and j'.

        So for j < i, we require i <. j'. 

   Case 2: j > i
        bij = j
        Condition: i <. j

        So for j > i, we require i <. j.

 Therefore, a(i) = 
     (# of j such that j < i and i <. j')   +   (# of j such that j > i and i <. j)

 Let:
   Let L(i) = the number of j < i such that i <. j'
   Let R(i) = the number of j > i such that i <. j

 Then a(i) = L(i) + R(i).

 Now, note that the entire set of elements is: {1, 2, ..., n} and {1', 2', ..., n'}.

 Also, the relative order of the primed elements is fixed: 1', 2', ..., n' must appear in that order.

 Alternative insight from the hint: 
   "If i <. bij and j <. bjk, then i <. bik"

 Let's expand:
   bij = { j' if j < i; j if j > i }
   bjk = { k' if k < j; k if k > j }

 Then the condition says: 
   If (i <. bij) and (j <. bjk) then (i <. bik)

 This is a kind of transitivity condition.

 But note: the problem is asking for a bipermutation satisfying the given a(i).

1. For each \( i \), \( i \) appears before \( i' \) (i.e., \( i < i' \) in the sequence).
2. For any \( i \) and \( j \), \( i' < j' \) if and only if \( i < j \) (i.e., the primed elements appear in increasing order of their indices).

The array \( a \) provides constraints where \( a(i) \) is the number of indices \( j \) such that \( i < \text{bij} \), with \( \text{bij} \) defined as \( j' \) if \( j < i \) and \( j \) if \( j > i \).

### Approach
1. **Initial Validation**: Check if any value in \( a \) is out of the valid range \([0, n-1]\). If so, output "NO".
2. **Compute Auxiliary Array**: For each \( i \), compute \( d[i] = n - 1 - a[i] \). This array helps in determining the number of elements that must precede \( i \) in the sequence.
3. **Construct Sequence**: 
   - Process indices from \( n \) down to 1.
   - For each \( i \), push \( d[i] \) negative elements (starting from the smallest available index) followed by \( i \).
   - This ensures the constraints on the order of elements and the counts specified by \( a \) are satisfied.
4. **Output Result**: If the construction is successful, output "YES" followed by the sequence; otherwise, output "NO".

### Solution Code

#include <iostream>
#include <vector>
#include <set>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<int> a(n + 1);
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
        if (a[i] < 0 || a[i] >= n) {
            cout << "NO" << endl;
            return 0;
        }
    }

    vector<int> d(n + 1);
    for (int i = 1; i <= n; i++) {
        d[i] = n - 1 - a[i];
    }

    set<int> s;
    for (int i = 1; i <= n; i++) {
        s.insert(i);
    }

    vector<int> res;
    for (int i = n; i >= 1; i--) {
        if (d[i] > (int)s.size()) {
            cout << "NO" << endl;
            return 0;
        }
        for (int j = 0; j < d[i]; j++) {
            res.push_back(-(*s.begin()));
            s.erase(s.begin());
        }
        res.push_back(i);
        s.erase(i);
    }

    if (!s.empty()) {
        cout << "NO" << endl;
        return 0;
    }

    cout << "YES" << endl;
    for (int i = 0; i < res.size(); i++) {
        if (i > 0) cout << " ";
        cout << res[i];
    }
    cout << endl;

    return 0;
}
