## Problem Understanding

The game works as follows:
- We start with K black cells on an infinite grid
- A move consists of finding 3 black cells that form 3 vertices of a rectangle (with sides parallel to axes), where the 4th vertex is white
- We then paint that 4th vertex black
- We repeat until no more moves are possible

## Key Insight

For a rectangle with sides parallel to axes, if we have three vertices: (x1,y1), (x1,y2), (x2,y1), then the 4th vertex must be (x2,y2).

The operation is: if we have points (x1,y1), (x1,y2), (x2,y1), we can add (x2,y2).

This is equivalent to: if we have points at (x1,y1), (x1,y2), (x2,y1), then we can "complete" the rectangle by adding (x2,y2).

## Mathematical Interpretation

Let's think about what points we can generate. Notice that:

1. If we have two points with the same x-coordinate: (x, y1) and (x, y2)
2. And two points with the same y-coordinate: (x1, y) and (x2, y)
3. Then we can generate more points

Actually, this process is related to closure under the operation of completing rectangles. 

Let me consider the set of x-coordinates X = {x1, x2, ...} and y-coordinates Y = {y1, y2, ...}.

If we have points at (x_i, y_j) for some pairs (i,j), when can we generate more points?

The key insight is that this process generates the Cartesian product of connected components. More precisely:

- Build a bipartite graph with X-coordinates on one side and Y-coordinates on the other
- Connect x to y if (x,y) is initially black
- The connected components in this bipartite graph determine the final set

If we have a connected component with n_x x-coordinates and n_y y-coordinates, then in the end we will have all n_x * n_y points in that component (because we can keep completing rectangles within the component).

## Algorithm

1. Map coordinates to smaller indices (since they can be up to 10^9)
2. Build bipartite graph: left nodes = x-coordinates, right nodes = y-coordinates
3. For each black cell (x,y), add an edge between node x and node y
4. Find connected components in this bipartite graph
5. For each component with cnt_x x-nodes and cnt_y y-nodes, the final number of black cells from this component is cnt_x * cnt_y
6. Sum over all components

Wait, but we need to be careful: each connected component in the bipartite graph corresponds to a set of x-coordinates and y-coordinates that are connected through the initial black cells. In the closure, we get all possible pairs within each connected component.

## Example Verification

First example: (1,1), (1,2), (2,2)
- X = {1,2}, Y = {1,2}
- Edges: 1-1, 1-2, 2-2
- Connected component: all nodes connected
- cnt_x = 2, cnt_y = 2 → 2*2 = 4 ✓

Second example: (0,0), (1,0), (0,1), (1,2), (2,1)
- X = {0,1,2}, Y = {0,1,2}
- Edges: 0-0, 1-0, 0-1, 1-2, 2-1
- Let's see connectivity:
  - 0 connected to 0,1
  - 1 connected to 0,2
  - 2 connected to 1
  - So all x-nodes {0,1,2} are connected, all y-nodes {0,1,2} are connected
- cnt_x = 3, cnt_y = 3 → 3*3 = 9 ✓

## Implementation

I'll use union-find or DFS/BFS on the bipartite graph. Since the graph can be large (up to 2*10^5 edges and up to 4*10^5 nodes), BFS/DFS is fine.

Steps:
1. Read K
2. Read points, collect all x and y coordinates
3. Map coordinates to indices (0-indexed)
4. Build adjacency list for bipartite graph (size = total distinct x + distinct y)
5. For x-coordinate i, its node index is i; for y-coordinate j, its node index is num_x + j
6. BFS/DFS to find connected components
7. For each component, count how many x-nodes and y-nodes
8. Sum cnt_x * cnt_y for all components

```cpp
#include <iostream>
#include <vector>
#include <map>
#include <set>
#include <queue>
#include <algorithm>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int K;
    cin >> K;
    
    vector<pair<long long, long long>> points;
    set<long long> xs, ys;
    
    for (int i = 0; i < K; i++) {
        long long x, y;
        cin >> x >> y;
        points.push_back({x, y});
        xs.insert(x);
        ys.insert(y);
    }
    
    // Map coordinates to indices
    map<long long, int> x_to_idx;
    map<long long, int> y_to_idx;
    
    int idx = 0;
    for (long long x : xs) {
        x_to_idx[x] = idx++;
    }
    
    int num_x = idx;
    for (long long y : ys) {
        y_to_idx[y] = idx++;
    }
    
    int total_nodes = num_x + (int)ys.size();
    vector<vector<int>> adj(total_nodes);
    
    // Build bipartite graph
    for (auto& p : points) {
        int x_idx = x_to_idx[p.first];
        int y_idx = y_to_idx[p.second];
        // x node connects to y node
        adj[x_idx].push_back(y_idx);
        adj[y_idx].push_back(x_idx);
    }
    
    vector<bool> visited(total_nodes, false);
    long long result = 0;
    
    for (int i = 0; i < total_nodes; i++) {
        if (!visited[i]) {
            queue<int> q;
            q.push(i);
            visited[i] = true;
            
            int cnt_x = 0, cnt_y = 0;
            
            if (i < num_x) {
                cnt_x++;
            } else {
                cnt_y++;
            }
            
            while (!q.empty()) {
                int u = q.front();
                q.pop();
                
                for (int v : adj[u]) {
                    if (!visited[v]) {
                        visited[v] = true;
                        q.push(v);
                        
                        if (v < num_x) {
                            cnt_x++;
                        } else {
                            cnt_y++;
                        }
                    }
                }
            }
            
            result += (long long)cnt_x * cnt_y;
        }
    }
    
    cout << result << endl;
    
    return 0;
}
```
