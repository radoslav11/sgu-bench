How to check if the intersection of a set of circles is empty? For a set S, the intersection is non-empty if and only if for every pair of circles in S, they intersect (not necessarily, because three circles might have empty intersection even if pairwise they intersect), but for disks, the intersection is non-empty if and only if the set has a common point. There is an algorithm: the intersection of disks is non-empty if and only if there is a point that is in all disks, which can be found by checking the intersection points of the circles in S and also the centers of the circles in S (but centers might not be in the intersection).

Actually, the intersection of disks is a convex set, and it is non-empty if and only if for every subset of three disks, their intersection is non-empty (by Helly's theorem, since in R^2, Helly number is 3), but that doesn't help for computation.

But for a set S of k disks, the intersection is non-empty if and only if the following holds: the intersection of the disks is non-empty, and this can be checked by solving a quadratic programming problem, but that is heavy.

There is a simpler way: the intersection of k disks is non-empty if and only if there is no empty circle (in the sense of the smallest enclosing circle) that contains the centers but not the intersection, but it's complicated.

Given the constraints (n<=100), and that the worst-case might be borderline, but note that in practice the number of non-empty intersections for k>=3 might be O(n^2) because the intersection of three circles is determined by three circles and might be empty for most triplets.

In fact, the number of non-empty intersections of k circles is at most O(n^2) for k=2, and for k>=3, it is O(n^2) as well, because the intersection region for k circles is determined by at most 3 circles (the boundary is formed by arcs from at most 3 circles). So the number of non-empty intersections for k>=3 is O(n^2) (one for each triple that has a non-empty intersection, and then the intersection of more circles would be a subset).

But wait, the intersection of a set S of circles is non-empty only if the intersection of the minimal set that defines the region is non-empty, and the minimal set has size at most 3. Therefore, the number of non-empty intersections for |S|>=3 is O(n^2) (because each non-empty intersection region is determined by at most 3 circles, and there are O(n^3) triples, but many triples yield empty intersection).

So total number of non-empty intersections is O(n^3), which for n=100 is 1e6, which is acceptable.

    if (d <= abs(c1.r - c2.r)) {
        double r_min = min(c1.r, c2.r);
        return PI * r_min * r_min;
    }
    
    // General case: two intersection points
    double r1 = c1.r;
    double r2 = c2.r;
    
    // Using the formula with acos and sqrt
    double part1 = r1 * r1 * acos((d * d + r1 * r1 - r2 * r2) / (2 * d * r1));
    double part2 = r2 * r2 * acos((d * d + r2 * r2 - r1 * r1) / (2 * d * r2));
    double part3 = 0.5 * sqrt((-d + r1 + r2) * (d + r1 - r2) * (d - r1 + r2) * (d + r1 + r2));
    
    return part1 + part2 - part3;
}

int main() {
    int n;
    cin >> n;
    vector<Circle> circles(n);
    for (int i = 0; i < n; i++) {
        cin >> circles[i].x >> circles[i].y >> circles[i].r;
    }

    double S1 = 0.0;
    for (int i = 0; i < n; i++) {
        S1 += PI * circles[i].r * circles[i].r;
    }

    double S2 = 0.0;
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            S2 += intersectionArea(circles[i], circles[j]);
        }
    }

    double burned = S1 - 2 * S2;
    double inverse = S2;

    cout << fixed << setprecision(5) << burned << " " << inverse << endl;
    return 0;
}
