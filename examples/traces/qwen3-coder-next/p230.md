We are given N types of coins (values 1 to N) and N boxes (numbered 1 to N). 
Each box contains exactly one coin (a permutation of [1, N]).
The weight of a coin of value i is i grams.

We are also given M weighings: each weighing is a pair (P, Q) meaning "box P is lighter than box Q".
This implies: weight(box P) < weight(box Q)  =>  value(coin in box P) < value(coin in box Q).

We need to assign a distinct coin value (from 1 to N) to each box such that for every weighing (P, Q): 
   value[P] < value[Q].

This is a classic topological sorting problem on a directed graph:
   Nodes: boxes 1..N
   Edges: for each weighing (P, Q) we have an edge P -> Q (because P must be lighter, so value[P] < value[Q])

But note: the condition is only about the relative order of the values. We need to assign the values 1..N (which are distinct) to the boxes such that if there is an edge u->v then value[u] < value[v].

However, note that the condition does not require that we use the topological order directly as the assignment? 
Actually, we can assign the values in increasing order according to a topological order.

But caution: the topological order gives a linear order of the boxes such that if there's an edge u->v then u comes before v. 
Then we can assign the smallest available value (1) to the first box in the topological order, then 2 to the next, etc.

But is that the only possibility? Actually, any assignment that satisfies the constraints is acceptable.

However, note: the problem does not require the lexicographically smallest assignment, but any valid one.

But there's a catch: the constraints are only the given inequalities. We don't have any other constraints.

Steps:
1. Build a directed graph with N nodes (boxes 1..N) and M edges: for each (P, Q) add edge P -> Q.
2. Check if the graph has a cycle. If yes, output "No solution".
3. Otherwise, do a topological sort to get an ordering of the boxes.
4. Then assign the value 1 to the first box in the topological order, 2 to the second, etc.

But note: the problem says "the K-th number means the type of coin in K-th box". So we need an array `ans` of size N+1 (1-indexed) such that `ans[i]` is the coin value in box i.

How?
   Let `topo_order` be a list of boxes in topological order (from smallest to largest in terms of constraints: if u->v then u comes before v).
   Then for index i (0-indexed) in `topo_order`, the box `topo_order[i]` should get the coin value i+1.

So:
   ans[ topo_order[i] ] = i+1

But note: the constraints are only the given edges. However, is this assignment valid?
   For an edge (P, Q): we require value[P] < value[Q]. 
   In our assignment, if P comes before Q in the topological order, then value[P] = position_of_P + 1 < position_of_Q + 1 = value[Q]. So yes.

However, what if the topological order does not have P before Q? But by definition of topological sort, if there is an edge P->Q, then P must come before Q.

But note: the graph might not be connected? Topological sort works for DAGs (directed acyclic graphs) even if not connected.

Implementation:
   We'll use Kahn's algorithm for topological sort (BFS) or DFS. Since N is at most 100, either is fine.

Steps for Kahn:
   - Compute in-degree for each node.
   - Queue all nodes with in-degree 0.
   - While queue not empty:
        pop a node u, append to topo_order.
        for each neighbor v of u:
            in_degree[v]--
            if in_degree[v] == 0, push v.

   If the size of topo_order != N, then there's a cycle -> "No solution".

Let's code accordingly.

Important: the input uses 1-indexed for boxes and coin values.

Example sample: 
   Input: "3 2\n2 1\n1 3"
   Graph: 
        Box2 -> Box1 (so edge 2->1)
        Box1 -> Box3 (so edge 1->3)

   Adjacency list:
        2: [1]
        1: [3]

   In-degrees:
        box1: 1 (from 2)
        box2: 0
        box3: 1 (from 1)

   Queue starts with [2]. 
        Pop 2 -> topo_order = [2]
        Then for neighbor 1: in_degree[1] becomes 0 -> push 1.
        Pop 1 -> topo_order = [2,1]
        Then for neighbor 3: in_degree[3] becomes 0 -> push 3.
        Pop 3 -> topo_order = [2,1,3]

   Then assign:
        box2 -> 1
        box1 -> 2
        box3 -> 3

   So the output for box1, box2, box3: 
        box1: 2, box2: 1, box3: 3 -> "2 1 3"

   Which matches the sample.

So we'll do:

   Read N, M.
   Create graph (adjacency list) of size N+1.
   in_degree array of size N+1, initialized to 0.

   For each of the M lines:
        read P, Q -> add edge P->Q: adj[P].push_back(Q); in_degree[Q]++;

   Then Kahn's algorithm.

   If topo_order size != N -> "No solution"

   Else:
        Create an array `res` of size N+1 (1-indexed).
        For i from 0 to N-1:
            box = topo_order[i]
            res[box] = i+1

        Then output: res[1], res[2], ... res[N]

Note: We assume the graph is built correctly.

Let's code accordingly.


```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>

using namespace std;

int main() {
    int N, M;
    cin >> N >> M;

    vector<vector<int>> adj(N + 1);
    vector<int> in_degree(N + 1, 0);

    for (int i = 0; i < M; i++) {
        int P, Q;
        cin >> P >> Q;
        adj[P].push_back(Q);
        in_degree[Q]++;
    }

    queue<int> q;
    for (int i = 1; i <= N; i++) {
        if (in_degree[i] == 0) {
            q.push(i);
        }
    }

    vector<int> topo_order;
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        topo_order.push_back(u);

        for (int v : adj[u]) {
            in_degree[v]--;
            if (in_degree[v] == 0) {
                q.push(v);
            }
        }
    }

    if (topo_order.size() != N) {
        cout << "No solution" << endl;
    } else {
        vector<int> ans(N + 1);
        for (int i = 0; i < N; i++) {
            ans[topo_order[i]] = i + 1;
        }

        for (int i = 1; i <= N; i++) {
            if (i > 1) cout << " ";
            cout << ans[i];
        }
        cout << endl;
    }

    return 0;
}
```
