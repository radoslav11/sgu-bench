We are counting k-digit numbers n such that:
  1. n is "good": 
        (a) P(n) != 0  (i.e., no digit is 0)
        (b) n mod P(n) == 0
  2. n+1 is also "good".

Note: Since n is a k-digit number, we have 10^(k-1) <= n <= 10^k - 1.
Then n+1 is either a k-digit number (if n < 10^k - 1) or (k+1)-digit (if n = 10^k - 1).
But note: if n = 10^k - 1 (which is a number with k nines), then n+1 = 10^k, which has k+1 digits and contains a '1' followed by k zeros -> so P(n+1)=0 -> not good.

Therefore, for n to be perfect and k-digit, we require:
  - n must be a k-digit number (so n in [10^(k-1), 10^k-1])
  - n+1 must also be good -> so n+1 must be a k-digit number? Actually, if n = 10^k - 1, then n+1 = 10^k (which is k+1 digits and has zeros) -> not good. So n must be at most 10^k - 2.

Hence, n must be in [10^(k-1), 10^k - 2].

But note: the problem says "n contains exactly K digits", and "n+1" must be good. The definition of good requires that the product of digits of n+1 is non-zero. So n+1 must not contain the digit 0. Therefore, n+1 must be a number without zero digits. However, if n is the largest k-digit number (i.e., 999...9), then n+1 is 1000...0 which has zeros. So n cannot be the largest k-digit number.

Thus, n must be in [10^(k-1), 10^k - 2] and both n and n+1 must have no zero digit and be divisible by their digit products.

However, note the constraints: k can be up to 1000000 (1e6). We cannot iterate over all k-digit numbers.

To solve this problem, we need to count the number of k-digit numbers \( n \) such that both \( n \) and \( n+1 \) are "good" numbers. A good number is defined as a number where the product of its digits is non-zero and the number is divisible by this product.

### Approach
1. **Problem Analysis**: 
   - For \( n \) to be a good number, it must not contain any zero digits (so the product of digits \( P(n) \neq 0 \)) and \( n \mod P(n) = 0 \).
   - Similarly, \( n+1 \) must also be a good number.
   - Given \( n \) is a k-digit number, \( n \) ranges from \( 10^{k-1} \) to \( 10^k - 2 \) (since \( n = 10^k - 1 \) would make \( n+1 = 10^k \), which contains zeros).

2. **Key Insight**:
   - For consecutive numbers \( n \) and \( n+1 \) to both be good, the only possible candidates are numbers where the first \( k-1 \) digits are 1, and the last digit \( d \) (from 1 to 8) satisfies specific conditions derived from modular arithmetic.
   - The conditions reduce to checking if \( 10^K \equiv 10 \mod (9 \times d \times (d+1)) \) for each \( d \) in \( 1..8 \).

3. **Algorithm Selection**:
   - For each digit \( d \) from 1 to 8, compute the modulus \( 9 \times d \times (d+1) \).
   - Use modular exponentiation to check if \( 10^K \mod \text{modulus} = 10 \).
   - Count how many such \( d \) values satisfy the condition.

4. **Complexity Analysis**:
   - The loop runs 8 times (for \( d = 1 \) to \( 8 \)).
   - Each modular exponentiation operation is \( O(\log K) \), which is efficient even for \( K = 1,000,000 \) (since \( \log_2(1,000,000) \approx 20 \)).
   - Overall complexity is \( O(1) \) per test case, making it suitable for large \( K \).

### Solution Code
```cpp
#include <iostream>
using namespace std;

int main() {
    long long K;
    cin >> K;

    int count = 0;
    for (int d = 1; d <= 8; d++) {
        long long modulus = 9LL * d * (d + 1);
        long long res = 1;
        long long base = 10 % modulus;
        long long exp = K;
        while (exp > 0) {
            if (exp & 1) {
                res = (res * base) % modulus;
            }
            base = (base * base) % modulus;
            exp >>= 1;
        }
        long long target = 10 % modulus;
        if (res == target) {
            count++;
        }
    }
    cout << count << endl;
    return 0;
}
```

### Explanation
1. **Reading Input**: The program reads the integer \( K \), representing the number of digits.
2. **Loop Through Digits**: For each digit \( d \) from 1 to 8 (inclusive), the program calculates the modulus \( 9 \times d \times (d+1) \).
3. **Modular Exponentiation**: The program computes \( 10^K \mod \text{modulus} \) using fast exponentiation to handle large \( K \) efficiently.
4. **Check Condition**: It checks if the result of the modular exponentiation equals \( 10 \mod \text{modulus} \). If true, it increments the count.
5. **Output Result**: Finally, the program prints the total count of valid \( k \)-digit numbers \( n \) where both \( n \) and \( n+1 \) are good numbers.

This approach efficiently checks the necessary conditions using modular arithmetic, ensuring optimal performance even for very large values of \( K \).
