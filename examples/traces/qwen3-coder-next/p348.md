To solve this problem, we need to find the minimum generator \( p \) such that the union of sets \( W(1), W(2), \ldots, W(p) \) contains all integers from 1 to \( n \). The set \( W(p) \) consists of all cyclic shifts of the binary representation of \( p \) (with the same number of bits as \( p \)), interpreted as numbers.

### Approach
1. **Problem Analysis**: 
   - For any integer \( p \) with \( b \) bits (where the highest bit is 1), \( W(p) \) includes all numbers formed by cyclically shifting the binary representation of \( p \). Some shifts might result in numbers with leading zeros, but they are still valid positive integers when interpreted.
   - The key observation is that numbers from 1 to \( 2^b - 1 \) can be covered by generators with fewer than \( b \) bits. Thus, to cover numbers up to \( n \), we first determine the bit length \( b \) of \( n \), so \( 2^b \leq n < 2^{b+1} \). The numbers from 1 to \( 2^b - 1 \) are covered by generators \( \leq 2^b - 1 \).
   - The remaining task is to cover numbers from \( 2^b \) to \( n \) using generators of exactly \( b+1 \) bits. The minimal generator \( p \) for this range is the largest number \( m \) in \( [2^b, n] \) such that \( m \) is the smallest number in its orbit (i.e., the minimal cyclic shift of \( m \) that is \( \geq 2^b \)).

2. **Algorithm Selection**:
   - Compute \( b \) as \( \lfloor \log_2(n) \rfloor \).
   - The base answer for numbers \( \leq 2^b - 1 \) is \( 2^b - 1 \).
   - For numbers from \( 2^b \) to \( n \), iterate from \( n \) down to \( 2^b \) to find the largest \( m \) where \( m \) is the minimal generator of its orbit. This is checked by verifying that \( m \) is the smallest cyclic shift of itself that is \( \geq 2^b \).

3. **Complexity Analysis**:
   - The number of bits \( b \) is \( O(\log n) \), specifically up to 60 for \( n \leq 10^{18} \).
   - Checking each candidate \( m \) involves \( O(b^2) \) operations (generating all cyclic shifts and comparing them).
   - Expected number of candidates checked is \( O(b) \) due to the probabilistic nature of minimal generators (probability \( 1/b \) per candidate).
   - Overall complexity is \( O(b^3) \), which is feasible for \( b \leq 60 \).

### Solution Code
```cpp
#include <iostream>
#include <string>
#include <climits>
#include <cctype>
#include <cmath>
#include <algorithm>
using namespace std;

typedef long long ll;

bool is_minimal_generator(ll m, int b) {
    ll low = 1LL << b;
    string s = "";
    for (int i = b; i >= 0; i--) {
        if (m & (1LL << i)) {
            s += '1';
        } else {
            s += '0';
        }
    }
    string t = s + s;
    int len = b + 1;
    ll min_val = (1LL << (b + 2));
    for (int i = 0; i < len; i++) {
        if (t[i] == '1') {
            ll num = 0;
            for (int j = 0; j < len; j++) {
                num = num * 2 + (t[i + j] - '0');
            }
            if (num >= low) {
                if (num < min_val) {
                    min_val = num;
                }
            }
        }
    }
    return (min_val == m);
}

int main() {
    ll n;
    cin >> n;
    if (n == 1) {
        cout << 1 << endl;
        return 0;
    }
    int leading_zeros = __builtin_clzll((unsigned long long)n);
    int b = 63 - leading_zeros;
    ll low = 1LL << b;
    ll ans = low - 1;
    for (ll m = n; m >= low; m--) {
        if (is_minimal_generator(m, b)) {
            ans = m;
            break;
        }
    }
    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the integer \( n \). If \( n \) is 1, it directly outputs 1 since \( W(1) = \{1\} \).
2. **Bit Length Calculation**: Using `__builtin_clzll`, it calculates the number of leading zeros in \( n \)'s binary representation to determine \( b \) (the exponent such that \( 2^b \leq n < 2^{b+1} \)).
3. **Base Answer**: The value \( 2^b - 1 \) covers all numbers from 1 to \( 2^b - 1 \), forming the initial answer.
4. **Finding Minimal Generator**: The loop checks numbers from \( n \) down to \( 2^b \) to find the largest \( m \) where \( m \) is the smallest cyclic shift (of its \( b+1 \)-bit representation) that is \( \geq 2^b \). This is verified by generating all cyclic shifts and selecting the smallest valid shift.
5. **Output**: The largest valid \( m \) found (or the base answer if no larger \( m \) is found) is printed as the minimal generator.

This approach efficiently narrows down the search space using properties of cyclic shifts and bit manipulation, ensuring optimal performance even for large \( n \).
