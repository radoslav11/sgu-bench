### Approach
1. **Problem Analysis**: 
   - In a round-robin tournament, each player must play every other player exactly once.
   - The minimal number of tours (rounds) required is \( T = N-1 \) if \( N \) is even, and \( T = N \) if \( N \) is odd. This is derived from the fact that each tour can accommodate at most \( \lfloor N/2 \rfloor \) matches, leading to the minimal tours being \( \lceil \frac{N(N-1)/2}{\lfloor N/2 \rfloor} \rceil \), which simplifies to \( N-1 \) for even \( N \) and \( N \) for odd \( N \).

2. **Algorithm Selection**:
   - **Circle Method**: This is a standard algorithm for generating round-robin schedules. For odd \( N \), we introduce a dummy player (by) to make the total number of players even. The algorithm works by fixing one player and rotating the rest to generate matches for each tour.
   - **Steps**:
     - If \( N \) is odd, set \( M = N + 1 \) (including the dummy player); otherwise, \( M = N \).
     - The number of tours \( T \) is \( M - 1 \).
     - Initialize an array `arr` with players \( 1 \) to \( M-1 \).
     - For each tour \( r \) (from \( 0 \) to \( T-1 \)):
       - Pair player \( 0 \) with player \( \text{arr}[r] \) (if not the dummy).
       - For each \( i \) from \( 1 \) to \( (size-1)/2 \), pair players at positions \( (r + i) \mod size \) and \( (r + size - i) \mod size \) in `arr` (if both are real players).
     - Populate the schedule matrix where each entry \( (i, j) \) stores the tour number for the match between players \( i \) and \( j \).

### Solution Code
```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    int N;
    cin >> N;

    if (N == 1) {
        cout << 0 << endl;
        cout << 0 << endl;
        return 0;
    }

    int T = (N % 2 == 0) ? (N - 1) : N;
    cout << T << endl;

    vector<vector<int>> ans(N, vector<int>(N, 0));

    int M = N;
    int size = N - 1;
    if (N % 2 == 1) {
        M = N + 1;
        size = N;
    }

    vector<int> arr;
    for (int i = 1; i < M; i++) {
        arr.push_back(i);
    }

    for (int r = 0; r < T; r++) {
        if (arr[r] < N) {
            ans[0][arr[r]] = r + 1;
            ans[arr[r]][0] = r + 1;
        }

        int num_pairs = (size - 1) / 2;
        for (int i = 1; i <= num_pairs; i++) {
            int idx1 = (r + i) % size;
            int idx2 = (r + size - i) % size;
            int p1 = arr[idx1];
            int p2 = arr[idx2];
            if (p1 < N && p2 < N) {
                ans[p1][p2] = r + 1;
                ans[p2][p1] = r + 1;
            }
        }
    }

    for (int i = 0; i < N; i++) {
        for (int j = 0; j < N; j++) {
            if (j > 0) 
                cout << " ";
            cout << ans[i][j];
        }
        cout << endl;
    }

    return 0;
}
```

### Explanation
1. **Input Handling**: The code reads the number of players \( N \). For \( N = 1 \), it outputs 0 tours and a single zero.
2. **Tour Calculation**: The minimal number of tours \( T \) is determined based on whether \( N \) is even (\( T = N-1 \)) or odd (\( T = N \)).
3. **Schedule Generation**:
   - **Dummy Player Handling**: If \( N \) is odd, a dummy player is added to make \( M = N + 1 \).
   - **Array Initialization**: An array `arr` is initialized with players \( 1 \) to \( M-1 \).
   - **Matrix Population**: For each tour \( r \):
     - Player \( 0 \) is paired with \( \text{arr}[r] \) (if not the dummy).
     - Other matches are generated by pairing players symmetrically positioned around \( \text{arr}[r] \) in the array.
4. **Output**: The schedule matrix is printed, where each entry \( (i, j) \) indicates the tour number for the match between players \( i \) and \( j \), with zeros on the diagonal (no self-matches).

This approach efficiently generates a valid tournament schedule by leveraging the circle method, ensuring optimal tour count and correctness for all valid inputs.
