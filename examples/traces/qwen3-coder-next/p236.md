To solve this problem, we need to find a cyclic path in a directed graph that maximizes the ratio of the total cost to the total time of the path. The solution involves using binary search on the possible ratio values and verifying if a cycle exists with a ratio at least as large as the current candidate ratio using dynamic programming.

### Approach
1. **Binary Search on Ratio**: We perform binary search on the possible values of the ratio (from 0 to 5000, as the maximum possible ratio given constraints is 5000). For each candidate ratio `R`, we check if there exists a cycle where the sum of costs minus `R` times the sum of times is non-negative.
2. **Dynamic Programming Check**: For each candidate ratio `R`, we compute edge weights as `cost - R * time`. Using dynamic programming, we then check for a cycle where the total weight is non-negative:
   - **DP State**: `d[k][v]` represents the maximum weight achievable to reach vertex `v` in exactly `k` steps.
   - **Transition**: For each step `k` from 1 to `N`, update `d[k][v]` by considering all incoming edges to `v`.
   - **Cycle Detection**: After filling the DP table, for each vertex `v`, check if `d[N][v] - d[k][v] >= 0` for some `k < N`. If found, reconstruct the cycle from the path information stored during the DP.
3. **Cycle Reconstruction**: When a valid cycle is found during the check, reconstruct the path by backtracking from the end vertex and identify the cycle by finding the first occurrence of the start vertex in the path.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <cmath>
#include <climits>

using namespace std;

struct Edge {
    int to;
    int cost;
    int time;
};

const double INF = 1e18;
const double NEG_INF = -1e18;
const double EPS = 1e-9;

bool check(double R, int n, const vector<vector<Edge>>& edges, vector<int>& cycle) {
    vector<vector<double>> d(51, vector<double>(n, NEG_INF));
    vector<vector<int>> parent(51, vector<int>(n, -1));

    for (int v = 0; v < n; v++) {
        d[0][v] = 0;
    }

    for (int k = 1; k <= n; k++) {
        for (int u = 0; u < n; u++) {
            if (d[k-1][u] == NEG_INF) continue;
            for (const Edge& e : edges[u]) {
                double weight = e.cost - R * e.time;
                if (d[k-1][u] + weight > d[k][e.to]) {
                    d[k][e.to] = d[k-1][u] + weight;
                    parent[k][e.to] = u;
                }
            }
        }
    }

    for (int v = 0; v < n; v++) {
        for (int k = 0; k < n; k++) {
            if (d[n][v] - d[k][v] >= -EPS) {
                vector<int> path;
                int cur = v;
                for (int i = n; i >= 1; i--) {
                    path.push_back(cur + 1);
                    cur = parent[i][cur];
                }
                path.push_back(cur + 1);
                reverse(path.begin(), path.end());

                int first_occurrence = -1;
                for (int i = 0; i < n; i++) {
                    if (path[i] == v + 1) {
                        first_occurrence = i;
                        break;
                    }
                }
                if (first_occurrence == -1) {
                    continue;
                }
                vector<int> cycle_vertices;
                for (int i = first_occurrence; i < n; i++) {
                    cycle_vertices.push_back(path[i]);
                }
                cycle = cycle_vertices;
                return true;
            }
        }
    }
    return false;
}

int main() {
    int n, m;
    cin >> n >> m;

    vector<vector<Edge>> edges(n);
    for (int i = 0; i < m; i++) {
        int a, b, c, t;
        cin >> a >> b >> c >> t;
        a--; b--;
        edges[a].push_back({b, c, t});
    }

    double low = 0.0;
    double high = 5000.0;
    vector<int> best_cycle;

    for (int iter = 0; iter < 50; iter++) {
        double mid = (low + high) / 2.0;
        vector<int> cycle;
        if (check(mid, n, edges, cycle)) {
            best_cycle = cycle;
            low = mid;
        } else {
            high = mid;
        }
    }

    if (best_cycle.empty()) {
        cout << "0" << endl;
    } else {
        cout << best_cycle.size() << endl;
        for (int i = 0; i < best_cycle.size(); i++) {
            if (i > 0) cout << " ";
            cout << best_cycle[i];
        }
        cout << endl;
    }

    return 0;
}
```

### Explanation
1. **Binary Search Setup**: The binary search is performed over the possible ratio values (0 to 5000). For each midpoint ratio `mid`, we check if a valid cycle exists.
2. **Dynamic Programming Check**: The `check` function computes the maximum weight path for steps 1 to `N` (where `N` is the number of vertices). The weight of each edge is `cost - mid * time`.
3. **Cycle Detection**: After filling the DP table, for each vertex `v`, we check if the difference `d[N][v] - d[k][v]` is non-negative for any `k < N`. This indicates a cycle with non-negative total weight.
4. **Path Reconstruction**: When a valid cycle is found, the path is reconstructed by backtracking from the end vertex. The cycle is identified by the first occurrence of the start vertex in the reconstructed path, ensuring the path is cyclic.
5. **Output**: The longest valid cycle found during the binary search is output. If no cycle is found, "0" is printed.

This approach efficiently narrows down the maximum ratio using binary search and verifies candidate ratios with dynamic programming, ensuring optimal performance even for the upper constraint limits.
