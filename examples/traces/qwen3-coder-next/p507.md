To solve this problem, we need to compute for each internal node in a rooted tree the minimum absolute difference between any two leaf values in its subtree. If the subtree contains only one leaf, we output \(2^{31} - 1\) (2147483647) for that node.

### Approach
1. **Tree Construction**: Read the input to construct the tree using an adjacency list where each node points to its children.
2. **Leaf Values Assignment**: Assign the given leaf values to the respective leaf nodes (nodes numbered from \(n - m + 1\) to \(n\)).
3. **Postorder Traversal**: Process the tree in postorder (from leaves to root) to ensure that we process children before their parents. This is achieved by performing a BFS to get the order and then reversing it.
4. **Merge Sets with Small-to-Large Optimization**: For each internal node, merge the sets of leaf values from its children. To optimize, always merge the smallest set into the largest one to minimize the number of operations. This ensures that each leaf value is processed \(O(\log m)\) times, leading to an overall \(O(m \log m)\) complexity.
5. **Compute Minimum Absolute Difference**: For each internal node, after merging the sets of its children, compute the minimum absolute difference between consecutive elements in the sorted merged set (since the minimum difference in a sorted array lies between adjacent elements).
6. **Output Results**: After processing all nodes, output the results for internal nodes (nodes 1 to \(n - m\)) in order.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>
#include <climits>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int n, m;
    cin >> n >> m;

    vector<vector<int>> children(n + 1);
    for (int i = 2; i <= n; i++) {
        int p;
        cin >> p;
        children[p].push_back(i);
    }

    vector<long long> leaf_values(n + 1, 0);
    for (int i = n - m + 1; i <= n; i++) {
        cin >> leaf_values[i];
    }

    vector<int> order;
    queue<int> q;
    q.push(1);
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        order.push_back(u);
        for (int v : children[u]) {
            q.push(v);
        }
    }
    reverse(order.begin(), order.end());

    vector<vector<long long>> sets(n + 1);
    vector<long long> ans(n + 1, 0);

    for (int u : order) {
        if (children[u].empty()) {
            sets[u].push_back(leaf_values[u]);
        } else {
            int idx = -1;
            for (int v : children[u]) {
                if (idx == -1 || sets[v].size() > sets[idx].size()) {
                    idx = v;
                }
            }
            sets[u] = move(sets[idx]);
            for (int v : children[u]) {
                if (v == idx) continue;
                vector<long long> temp;
                temp.reserve(sets[u].size() + sets[v].size());
                merge(sets[u].begin(), sets[u].end(), sets[v].begin(), sets[v].end(), back_inserter(temp));
                sets[u] = move(temp);
            }
            if (sets[u].size() == 1) {
                ans[u] = 2147483647;
            } else {
                long long min_diff = (1LL << 60);
                for (int i = 0; i < (int)sets[u].size() - 1; i++) {
                    long long diff = sets[u][i + 1] - sets[u][i];
                    if (diff < min_diff) {
                        min_diff = diff;
                    }
                }
                ans[u] = min_diff;
            }
        }
    }

    for (int i = 1; i <= n - m; i++) {
        cout << ans[i];
        if (i < n - m) {
            cout << ' ';
        } else {
            cout << '\n';
        }
    }

    return 0;
}
```

### Explanation
1. **Tree Construction**: The tree is built using an adjacency list where each node stores its children. The parent information for nodes 2 to \(n\) is read and used to populate the adjacency list.
2. **Leaf Values Assignment**: The values for the leaf nodes (from \(n - m + 1\) to \(n\)) are read and stored in an array.
3. **Postorder Traversal**: A BFS is performed starting from the root to create an order of nodes. This order is reversed to process leaves before their parents.
4. **Set Merging**: For each internal node, the sets of leaf values from its children are merged. The smallest set is merged into the largest one to optimize performance. This ensures that each leaf value is moved \(O(\log m)\) times, maintaining efficiency.
5. **Minimum Difference Calculation**: After merging, the merged set is sorted. The minimum absolute difference is found by checking adjacent elements in this sorted set. If the set contains only one element, \(2^{31} - 1\) is stored.
6. **Result Output**: The results for internal nodes (1 to \(n - m\)) are printed in order, with spaces separating them and a newline at the end.

This approach efficiently computes the required minimum absolute differences by leveraging postorder traversal and optimized set merging, ensuring optimal performance even for large trees.
