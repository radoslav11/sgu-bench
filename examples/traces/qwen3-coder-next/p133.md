We are given N outposts, each with an interval [Ak, Bk].
An outpost i is redundant if there exists some outpost j such that Aj < Ai and Bi < Bj.

Note: The condition is strict in both parts: Aj < Ai and Bi < Bj.

Let's rephrase: 
  Outpost i is redundant if there exists j with:
      Aj < Ai and Bi < Bj.

But note: since intervals are [Ak, Bk] and we know Ak < Bk for each.

We cannot have Aj >= Ai or Bi >= Bj for the same j to make i redundant.

 Alternate interpretation: 
   We are looking for outposts i for which there is another outpost j that "dominates" it in the sense:
      Aj < Ai and Bj > Bi.

Why? Because if j has a start that is strictly less than i's start and an end that is strictly greater than i's end, then j covers more to the left and right. But note the condition says: Aj < Ai and Bi < Bj.

Wait, the condition is: 
   Aj < Ai and Bi < Bj.

But note: if j covers [Aj, Bj] and i is [Ai, Bi], then for j to "dominate" i in the required way, we require:
   Aj < Ai (so j starts before i) and Bj > Bi (so j ends after i). 
However, the condition in the problem says: "Bi < Bj", which is equivalent to Bj > Bi.

So indeed, we require: Aj < Ai and Bj > Bi.

Therefore, an outpost i is redundant if there exists j such that:
   Aj < Ai and Bj > Bi.

We can think of it as: 
   For each outpost i, check if there is another outpost j that has a smaller A (Aj < Ai) and a larger B (Bj > Bi).

Note: The problem states that all Ak are distinct and all Bk are distinct.

Approach:

1. Sort the outposts by A (the start) in increasing order. 
   However, note: if two outposts have the same A? But the problem says all Ak are different. So we can sort by A.

2. After sorting by A, for an outpost i (in the sorted order), all outposts j that come before i in the sorted order have Aj < Ai (because A is strictly increasing). 
   Therefore, for outpost i (at position i in the sorted array), we only need to check: is there any j < i (in the sorted array) such that Bj > Bi?

   Why? Because if there is a j with Aj < Ai (which is guaranteed for all j < i in the sorted-by-A array) and Bj > Bi, then i is redundant.

3. So the problem reduces to: 
   Given an array of B values in the order of increasing A, count the number of indices i for which there exists j < i such that Bj > Bi.

   This is equivalent to: 
        Count the number of elements that are not "record high" from left to right? 
   Actually, if we traverse from left to right (in increasing A order), then an element Bi is non-redundant if it is greater than all the B values we have seen so far? 

   But note: 
        For i to be non-redundant, there should be no j < i with Bj > Bi.
        That means: Bi must be >= all B values for j < i? 
        However, note the condition is strict: we require Bj > Bi to make i redundant. So if there is a j with Bj == Bi, that does not count because the problem states all Bk are distinct. So we don't have to worry about equality.

   Therefore, in the sorted-by-A array, an outpost i is non-redundant if and only if Bi is greater than all the B values for indices 0 to i-1.

   Then, the number of redundant outposts = total outposts - number of non-redundant outposts.

   How to compute non-redundant?
        We traverse the sorted array (by A) and keep track of the maximum B encountered so far.
        For the first outpost: 
            maxB = -infinity (but since B>=0, we can set to -1)
        Then for i=0: 
            if maxB < B[0] -> then non-redundant, and update maxB = B[0].
        For i>=1:
            if B[i] > maxB -> then non-redundant, and update maxB = B[i].
            else -> redundant.

   However, note: the condition for non-redundant is that there is no j with Aj < Ai and Bj > Bi. 
        In the sorted array by A, for i, all j < i have Aj < Ai. So if B[i] <= maxB (which is the maximum of all Bj for j < i), then there exists a j (the one that set maxB) such that Bj > Bi (if B[i] < maxB) or equal? But distinct B, so if B[i] < maxB then redundant. If B[i] == maxB? but distinct so not possible.

   Therefore, we can do:

        sort the outposts by A (increasing)
        non_redundant = 0
        current_max_B = -1 (since B>=0, but to be safe we can use -1 or 0)

        for each outpost in the sorted list:
            if B[i] > current_max_B:
                non_redundant += 1
                current_max_B = B[i]
            // else: redundant, skip

        answer = N - non_redundant

Let's test with the sample:

Input: 
  5
  0 10
  2 9
  3 8
  1 15
  6 11

Sort by A:
  (0,10)
  (1,15)   -> because 1 < 2
  (2,9)
  (3,8)
  (6,11)

Now traverse:
  i=0: A=0, B=10 -> current_max_B = -1 -> 10 > -1 -> non_redundant=1, maxB=10.
  i=1: A=1, B=15 -> 15>10 -> non_redundant=2, maxB=15.
  i=2: A=2, B=9 -> 9<15 -> redundant.
  i=3: A=3, B=8 -> 8<15 -> redundant.
  i=4: A=6, B=11-> 11<15 -> redundant.

So non_redundant = 2, redundant = 5-2 = 3 -> matches sample.

Therefore, the plan is:

  Step 1: Read N.
  Step 2: Read N lines, each with Ak and Bk.
  Step 3: Sort the list of pairs by Ak (the first element).
  Step 4: Traverse the sorted list, keeping track of the maximum B seen so far.
          Count how many times we update the maximum (which gives non_redundant count).
  Step 5: Answer = N - non_redundant.

Note: Since the constraints say N up to 16000, sorting and a linear scan is efficient.

Let's code accordingly.

We'll use vector of pairs, sort by first element (Ak), then traverse.

Important: The problem says Ak and Bk can be as large as 2e9, so we use long? But for comparison, int might be enough (since 2e9 is within int for many systems? but note: 2e9 is about 2*10^9, and int is typically -2^31 to 2^31-1 which is about -2e9 to 2e9, so 2e9 is the limit for unsigned? Actually, 2,000,000,000 is less than 2^31-1 (which is about 2.1e9). So we can use int.

But to be safe, the problem says "0<= Ak < Bk <= 2 000 000 000", so we can use int (since 2e9 is 2000000000, and 2000000000 < 2^31-1 = 2147483647).

So we'll use int.

Code:

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    int N;
    cin >> N;
    vector<pair<int, int>> outposts;
    for (int i = 0; i < N; i++) {
        int a, b;
        cin >> a >> b;
        outposts.push_back({a, b});
    }

    // Sort by the start (A)
    sort(outposts.begin(), outposts.end());

    int non_redundant = 0;
    int maxB = -1; // since Bk>=0, we can use -1

    for (int i = 0; i < N; i++) {
        int b = outposts[i].second;
        if (b > maxB) {
            non_redundant++;
            maxB = b;
        }
    }

    cout << N - non_redundant << endl;

    return 0;
}
