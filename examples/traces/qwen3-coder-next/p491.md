So the condition is not simply "gcd divides M and M>=0".

Given the complexity, and the time limit of 0.75 seconds, and that N is likely not too large (the sample is 10, and the problem might have N up to 200,000), we need an efficient method.

Alternate efficient approach:

We can iterate over A from 1 to N-1, and for each A, iterate over multiples.

Specifically, for a fixed A, the condition is: there exists y>=1 such that:
   (N - B*y) is divisible by A, and 1<=y<= (N-A)/B, and B>A.

But B = (N - A*x) / y, and also B>A, so (N - A*x)/y > A  => N - A*x > A*y  => N > A*(x+y).

Also, x>=1, so let x = k, then for fixed A and y, we have:
   B = (N - A*k) / y, and we require:
        B > A, and B is integer, and y>=1, k>=1.

So: N - A*k > 0, and y must divide (N - A*k), and (N - A*k) / y > A.

Let M = N - A*k, then M > 0, and y must be a divisor of M, and y < M/A (because B = M/y > A => y < M/A).

Also, since y>=1, we have M/y >=1.

So for fixed A and k, M = N - A*k must be positive, and then the number of y that work is the number of divisors y of M such that 1<= y < M/A.

But note: for each such y, we get a pair (A, B) with B = M/y.

However, we are counting pairs (A,B), and for each (A,B) we might count it once (since B is determined).

But the issue: for fixed A, different k might yield the same B? Let's see: 
   B = (N - A*k) / y.
   If for the same A and B, there are multiple representations, we would count it multiple times. But the problem asks for distinct pairs (A,B), so we want to count each (A,B) at most once.

However, for fixed (A,B), the representation might not be unique, but we only care if there is at least one solution. So we want to know, for fixed (A,B), if there exists at least one solution.

So for fixed A, we want to know if there exists k>=1 and y>=1 such that:
   A*k + B*y = N, with B>A, and B = (N - A*k)/y > A.

But we can reindex by B: for fixed A, B must be in (A, N-A] (because B*y >= B > A, and A*k >= A, so A+B <= N), and also there exists k>=1 such that (N - B*y) = A*k for some y>=1, which means that A divides (N - B*y) for some y>=1.

But note: for fixed A and B, we require that there exists y in [1, floor((N-A)/B)] such that B*y ≡ N (mod A).

This is equivalent to: the residue of N modulo A must be in the set { B*y mod A : y=1,2,..., floor((N-A)/B) }.

The set { B*y mod A : y=1,2,...,A } is the set of multiples of g = gcd(A,B) modulo A, and it repeats every A/g steps.

So the condition is: 
   g = gcd(A,B) must divide N, and the smallest positive solution y0 to B*y ≡ N (mod A) must satisfy y0 <= floor((N-A)/B).

So for fixed A, we can iterate over g that divide A, and then over B that are multiples of g, with B>A and B<=N-A, and gcd(A,B)=g, and then check if g|N and if the smallest solution y0 to (B/g)*y ≡ (N/g) (mod (A/g)) is <= (N-A)/B.

But this is still heavy.

However, note that the condition "g|N" is independent, and then we require that the smallest solution y0 to (B/g)*y ≡ (N/g) (mod (A/g)) is <= (N-A)/B.

The smallest solution y0 is in the range [1, A/g].

So for fixed A and g|gcd(A, N), let A' = A/g, N' = N/g.
Then B = g * b, where b > A' (since B>A) and b <= (N - A)/g = N' - A'.
Also, gcd(A', b) = 1 (because gcd(A,B)=g, so gcd(A',b)=1).

The congruence: b * y ≡ N' (mod A').
Since gcd(b, A')=1, there is a unique solution modulo A': y0 = N' * b^{-1} mod A'.

We require y0 <= (N - A)/B = (g*(N' - A')) / (g*b) = (N' - A')/b.

So for fixed A and g (with g|A and g|N), let A' = A/g, N' = N/g.
Then we iterate over b such that:
   b > A', 
   b <= N' - A',
   gcd(A', b) = 1,
   and the solution y0 = (N' * inv(b, A')) mod A' is in [1, min(A', floor((N'-A')/b))].

But note: y0 is in [1, A'-1] (since if y0=0, we take y0 = A', but usually the modular inverse solution is in [0, A'-1], and if it's 0, then we use A' because we want positive y).

Actually, the smallest positive solution is in [1, A'].

So the condition is: y0 <= floor((N'-A')/b).

Now, how to count this fast? 
   We iterate A from 1 to N-1.
   For each A, iterate g over divisors of gcd(A, N) [which is the same as g|A and g|N].
   Let A' = A/g, N' = N/g.
   Then iterate b over integers in (A', N'-A'] that are coprime with A', and for which the modular inverse condition holds.

But the range for b is up to N' - A' which can be up to N, and for each A, the number of b is O(N), so total O(N^2) which is too slow for N=10^5.

To solve this problem, we need to count the number of distinct pairs (A, B) such that 1 ≤ A < B ≤ N and there exist integers x, y ≥ 1 satisfying the equation A·x + B·y = N. 

### Approach
1. **Problem Analysis**: The problem requires finding all pairs (A, B) where A is less than B, and the equation A·x + B·y = N has a solution with both x and y at least 1. The key insight is that for each valid pair (A, B), the value N - A·x must be divisible by B for some x ≥ 1, and the resulting B must be greater than A and not exceed N - A (since A + B ≤ N).

2. **Intuition and Insight**: For each A from 1 to floor((N-1)/2), we consider values of M = N - k·A for k ≥ 1 (ensuring M > 0). For each M, we check all divisors d of M. If a divisor d is greater than A and ≤ M, then the pair (A, d) is valid because d can serve as B (i.e., B = d, and y = M/d ≥ 1, x = k ≥ 1).

3. **Algorithm Selection**: 
   - **Precomputation**: Precompute divisors for all numbers from 1 to N using a sieve-like approach. This allows O(1) access to divisors of any number during the main computation.
   - **Counting Valid Pairs**: For each A, iterate over possible k values (from 1 to floor((N-1)/A)), compute M = N - k·A, and for each divisor d of M, check if d > A. Use a boolean array to track which B values have been counted for the current A to avoid duplicates.

4. **Complexity Analysis**: 
   - **Precomputation**: O(N log N) time and space, as each number i contributes to N/i divisors.
   - **Main Loop**: For each A, there are O(N/A) values of M, and for each M, O(number of divisors of M) operations. The total operations sum to O(N (log N)^2) due to the harmonic series and average divisor count.

### Solution Code
```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    long long N;
    cin >> N;

    if (N < 2) {
        cout << 0 << endl;
        return 0;
    }

    vector<vector<int>> divisors(N + 1);
    for (int i = 1; i <= N; i++) {
        for (int j = i; j <= N; j += i) {
            divisors[j].push_back(i);
        }
    }

    vector<bool> in_set(N + 1, false);
    vector<int> to_clear;
    long long ans = 0;

    for (long long A = 1; A * 2 < N; A++) {
        to_clear.clear();
        for (long long k = 1; k * A < N; k++) {
            long long M = N - k * A;
            for (int d : divisors[M]) {
                if (d > A && d <= M) {
                    if (!in_set[d]) {
                        in_set[d] = true;
                        to_clear.push_back(d);
                        ans++;
                    }
                }
            }
        }
        for (int d : to_clear) {
            in_set[d] = false;
        }
    }

    cout << ans << endl;
    return 0;
}
```

### Explanation
1. **Precomputation**: The code first precomputes all divisors for every number up to N using a nested loop. This allows efficient access to divisors during the main processing phase.
2. **Main Processing**: For each A (from 1 to just below N/2), the code iterates over possible multiples k such that M = N - k·A remains positive. For each M, it checks all divisors of M. If a divisor d is greater than A and ≤ M, the pair (A, d) is counted as valid.
3. **Avoiding Duplicates**: A boolean array `in_set` tracks which B values have been counted for the current A, ensuring each B is counted only once per A. The `to_clear` vector stores these B values for efficient reset after processing each A.
4. **Output**: The total count of valid pairs (A, B) is printed as the result.

This approach efficiently counts all valid pairs by leveraging precomputation and careful tracking to avoid duplicates, ensuring optimal performance even for large N.
