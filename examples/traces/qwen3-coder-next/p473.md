To solve this problem, we need to find the smallest positive integer \( N \) such that the number of divisors of \( N \) is exactly \( K \). The number of divisors of \( N \) corresponds to the number of ways to arrange \( N \) droids in rows with each row having the same number of droids, as described in the problem.

### Approach
1. **Problem Analysis**: The problem reduces to finding the smallest \( N \) such that the number of divisors of \( N \) is \( K \). The number of divisors of \( N \) is determined by its prime factorization. If \( N = p_1^{e_1} p_2^{e_2} \ldots p_m^{e_m} \), then the number of divisors is \( (e_1 + 1)(e_2 + 1) \ldots (e_m + 1) \).

2. **Key Insight**: To minimize \( N \), we should use the smallest primes available and assign the largest exponents to the smallest primes. This means the exponents in the prime factorization should be non-increasing (i.e., \( e_1 \geq e_2 \geq \ldots \geq e_m \)).

3. **Algorithm Selection**: We generate all possible multiplicative partitions of \( K \) (i.e., all ways to express \( K \) as a product of integers greater than or equal to 2, where the factors are in non-increasing order). For each partition, we compute \( N \) by assigning the largest factor (minus one) to the smallest prime, the next largest factor (minus one) to the next smallest prime, and so on. The smallest \( N \) across all partitions is our answer.

4. **Optimization**: 
   - Precompute a list of the first 25 primes (sufficient since \( \log_2(10^5) \approx 17 \)).
   - Use depth-first search (DFS) to generate multiplicative partitions of \( K \) in non-increasing order.
   - During computation, prune paths where the current candidate \( N \) exceeds the current best solution to avoid unnecessary calculations.

#include <vector>

void dfs_fac(int rem, int max_factor) {
    if (rem == 1) {
        long long cur = 1;
        for (int i = 0; i < (int)factors.size(); i++) {
            long long p = 1;
            int exp = factors[i] - 1;

### Explanation
1. **Prime List**: We precompute the first 25 primes to ensure we have enough primes for constructing \( N \) (since \( \log_2(10^5) \approx 17 \)).
2. **DFS for Multiplicative Partitions**: The function `dfs_fac` generates all non-increasing factorizations of \( K \). For each factorization, it computes \( N \) by raising each prime to the power of (factor - 1) and multiplying the results.
3. **Minimization**: During the DFS, if the computed \( N \) for a partition is smaller than the current best solution, it updates the answer. The search prunes paths where \( N \) exceeds the current best solution to optimize performance.
4. **Output**: After processing all partitions, the smallest \( N \) is printed. If no solution is found (though theoretically impossible for \( K \geq 1 \)), it outputs 0 as a safeguard.

This approach efficiently explores all possible configurations of \( N \) by leveraging multiplicative partitions and prime assignment, ensuring the solution is both correct and optimal. The DFS with pruning handles the constraints within feasible time limits.
