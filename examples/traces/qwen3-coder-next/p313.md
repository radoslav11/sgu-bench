To solve this problem, we need to establish a one-to-one correspondence between employees' houses and offices located along a circular railway such that the total travel time is minimized. The travel time between any two adjacent stations is 1 minute, and the railway is circular, meaning the distance between stations can be measured in both clockwise and counter-clockwise directions.

### Approach
1. **Problem Analysis**: The problem involves matching houses to offices on a circular railway to minimize the sum of the shortest paths between matched pairs. The shortest path between two stations on a circle is the minimum of the clockwise or counter-clockwise distance.
2. **Key Insight**: The optimal matching can be found by considering sorted lists of house and office locations. Due to the circular nature, we need to consider cyclic shifts of the office list relative to the house list to find the shift that minimizes the total travel time.
3. **Algorithm Selection**: 
   - **Sorting and Matching**: Sort both the house and office locations. For each candidate shift (a rotation of the office list), compute the total travel time by matching the i-th house in the sorted list to the (i + shift) mod n-th office in the sorted list.
   - **Efficiency Optimization**: To avoid checking all possible shifts (which would be O(n^2)), we limit the candidate shifts to the top 10 offices closest (in circular distance) to the first house in the sorted list. This reduces the number of shifts to check to O(1) per test case, making the solution efficient with O(n log n) complexity due to sorting.
4. **Complexity Analysis**: Sorting the house and office lists takes O(n log n) time. Checking up to 10 candidate shifts, each requiring O(n) time, results in O(n) time for the matching step. Thus, the overall complexity is O(n log n), which is efficient for n up to 50,000.

        cin >> a[i];
    }
    for (int i = 0; i < n; i++) {
        cin >> b[i];
    }

    vector<pair<ll, int>> houses_sorted, offices_sorted;
    for (int i = 0; i < n; i++) {
        houses_sorted.push_back({a[i], i});
    }
    for (int i = 0; i < n; i++) {
        offices_sorted.push_back({b[i], i});
    }
    sort(houses_sorted.begin(), houses_sorted.end());
    sort(offices_sorted.begin(), offices_sorted.end());

    vector<ll> a_sorted(n), b_sorted(n);
    for (int i = 0; i < n; i++) {
        a_sorted[i] = houses_sorted[i].first;
        b_sorted[i] = offices_sorted[i].first;
    }

    vector<pair<ll, int>> candidates;
    for (int j = 0; j < n; j++) {
        ll diff = abs(a_sorted[0] - b_sorted[j]);
        ll circular_diff = min(diff, L - diff);
        candidates.push_back({circular_diff, j});
    }
    sort(candidates.begin(), candidates.end());

    ll best_cost = 1e18;
    int best_shift = 0;
    int try_count = min(10, n);
    for (int t = 0; t < try_count; t++) {
        int j = candidates[t].second;
        int shift = (j - 0 + n) % n;

        ll cost = 0;
        for (int i = 0; i < n; i++) {
            int office_index = (i + shift) % n;
            ll diff = abs(a_sorted[i] - b_sorted[office_index]);
            cost += min(diff, L - diff);
        }
        if (cost < best_cost) {
            best_cost = cost;
            best_shift = shift;
        }
    }

    vector<int> ans(n);
    for (int i = 0; i < n; i++) {
        int office_sorted_index = (i + best_shift) % n;
        int house_original_index = houses_sorted[i].second;
        int office_original_index = offices_sorted[office_sorted_index].second;
        ans[house_original_index] = office_original_index + 1;
    }

    cout << best_cost << "\n";
    for (int i = 0; i < n; i++) {
        if (i > 0) cout << " ";
        cout << ans[i];
    }
    cout << "\n";
    return 0;
}

### Explanation
1. **Input Reading**: The code reads the number of employees (n) and the number of stations (L), followed by the locations of houses and offices.
2. **Sorting and Indexing**: The house and office locations are sorted, and their original indices are preserved to facilitate the final mapping.
3. **Candidate Selection**: For the first house in the sorted list, the code identifies the top 10 closest offices (in terms of circular distance) to serve as candidate shifts for the office list.
4. **Cost Calculation**: For each candidate shift, the code calculates the total travel time by matching each house to the corresponding office in the shifted list and sums the minimum of the clockwise or counter-clockwise distances.
5. **Optimal Matching**: The shift yielding the minimum total travel time is selected. The matching is then reconstructed using the original indices of houses and offices.
6. **Output**: The minimal total travel time and the assignment of offices to houses (1-based indices) are printed.

This approach efficiently narrows down the search space to a manageable number of candidate shifts, leveraging sorting and circular distance properties to ensure optimal performance.
