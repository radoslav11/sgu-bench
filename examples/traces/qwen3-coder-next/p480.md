To solve this problem, we need to determine if it's possible to construct a big "corner" of size \(2^k\) using given smaller corners of various sizes and colors. The solution involves checking feasibility based on the total units of smallest corners (size \(2^0\)) available and then distributing the required corners across available colors.

### Approach
1. **Problem Analysis**: The big corner of size \(2^k\) requires \(8^k\) units of the smallest corners (size \(2^0\)). Each corner of size \(2^j\) contributes \(8^j\) units. The problem reduces to checking if the sum of units from available corners (considering their sizes) meets or exceeds \(8^k\).

2. **Feasibility Check**: 
   - Compute the total available corners per size \(j\) (denoted \(T[j]\)).
   - Starting from the largest size (index \(k-1\)) down to the smallest (index \(0\)), check if the required units can be covered:
     - For each size \(j\), the required units are \(8 \times \text{need}\) (initially \(\text{need} = 1\) for the big corner).
     - If the required units exceed available corners \(T[j]\), subtract \(T[j]\) from the requirement and carry over the remainder multiplied by 8 to the next smaller size.
     - If at any point the remaining requirement is zero, it's feasible; otherwise, it's impossible.

3. **Distribution**: 
   - Once feasibility is confirmed, distribute the required corners per size across available colors. For each size, allocate corners starting from the first color, taking as many as needed without exceeding available counts.

### Solution Code
```cpp
#include <iostream>
#include <vector>
#include <tuple>
using namespace std;

int main() {
    int c, k;
    cin >> c >> k;

    vector<vector<long long>> a(c, vector<long long>(k));
    for (int i = 0; i < c; i++) {
        for (int j = 0; j < k; j++) {
            cin >> a[i][j];
        }
    }

    vector<long long> T(k, 0);
    for (int j = 0; j < k; j++) {
        for (int i = 0; i < c; i++) {
            T[j] += a[i][j];
        }
    }

    const long long BOUND = 1000000000000000LL;
    vector<long long> use(k, 0);
    long long need = 1;
    bool feasible = true;

    for (int j = k - 1; j >= 0; j--) {
        if (need > BOUND / 8) {
            need = BOUND + 1;
            break;
        }
        need *= 8;
        if (need <= T[j]) {
            use[j] = need;
            need = 0;
            break;
        } else {
            use[j] = T[j];
            need -= T[j];
        }
    }

    if (need > 0) {
        cout << "NO SOLUTION" << endl;
        return 0;
    }

    vector<tuple<int, int, long long>> result;
    for (int j = 0; j < k; j++) {
        long long rem = use[j];
        for (int i = 0; i < c; i++) {
            if (rem <= 0) break;
            long long take = min(rem, a[i][j]);
            if (take > 0) {
                result.push_back(make_tuple(i + 1, j, take));
                rem -= take;
            }
        }
    }

    cout << result.size() << endl;
    for (auto &t : result) {
        cout << get<0>(t) << " " << get<1>(t) << " " << get<2>(t) << endl;
    }

    return 0;
}
```

### Explanation
1. **Reading Input**: The code reads the number of colors \(c\) and the size index \(k\), followed by a matrix \(a\) where \(a[i][j]\) represents the count of corners of color \(i\) and size \(2^j\).

2. **Total per Size**: For each size \(j\), it computes the total available corners \(T[j]\) by summing across all colors.

3. **Feasibility Check**: 
   - Starting from size \(k-1\) down to \(0\), it checks if the required units (initially \(8^k\)) can be met. The requirement is computed as \(\text{need} = 8 \times \text{previous need}\).
   - If \(\text{need}\) exceeds available corners \(T[j]\), it subtracts \(T[j]\) and carries over the remainder (multiplied by 8 for the next smaller size).
   - A bound (\(10^{15}\)) prevents overflow during large computations.

4. **Distribution**: 
   - If feasible, it distributes the required corners per size across colors, prioritizing colors with higher availability.
   - The solution outputs the number of corner types used, followed by each type's color, size index, and count.
