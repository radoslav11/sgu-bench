/**
 * Problem Analysis:
 * A number n is "good" if P(n) != 0 and n mod P(n) == 0, where P(n) is the product of its digits.
 * A number n is "perfect" if both n and n+1 are good numbers.
 * 
 * Let's analyze the condition for n being perfect.
 * If n ends in 9, then n+1 will end in 0, so P(n+1) = 0, which makes n+1 not a good number.
 * Thus, for n to be perfect, n cannot end in 9.
 * Let n = 10m + d, where d is the last digit and d belongs to {1, 2, 3, 4, 5, 6, 7, 8}.
 * Since n and n+1 do not contain 0 as a digit (otherwise P(n) or P(n+1) would be 0),
 * the digits of n are the digits of m followed by d, and the digits of n+1 are the digits of m followed by d+1.
 * Thus, P(n) = P(m) * d and P(n+1) = P(m) * (d+1).
 * 
 * The conditions for n and n+1 to be good are:
 * 1) (10m + d) % (P(m) * d) == 0
 * 2) (10m + d + 1) % (P(m) * (d+1)) == 0
 * 
 * Let 10m + d = A * P(m) * d and 10m + d + 1 = B * P(m) * (d+1).
 * Subtracting the two equations gives:
 * 1 = B * P(m) * (d+1) - A * P(m) * d
 * 1 = P(m) * [B * (d+1) - A * d]
 * 
 * Since P(m) is a product of digits and must be a positive integer, the only possibility is P(m) = 1.
 * If P(m) = 1, then all digits of m must be 1.
 * Thus, m must be of the form 11...1 (a sequence of K-1 ones).
 * 
 * For m = 11...1, the equations simplify to:
 * 1) (10m + d) % d == 0  => 10m % d == 0
 * 2) (10m + d + 1) % (d+1) == 0 => 10m % (d+1) == 0
 * 
 * Since d and d+1 are coprime, 10m must be a multiple of d*(d+1).
 * For K=1, n is a 1-digit number. The perfect numbers are {1, 2, 3, 4, 5, 6, 7, 8} (total 8).
 * For K>=2:
 * m = 11...1 (K-1 ones), so 10m = 11...10.
 * We check d from {1, 2, 3, 4, 5, 6, 7, 8}:
 * - d=1: d(d+1)=2. 11...10 is always a multiple of 2. (Always a solution)
 * - d=2: d(d+1)=6. 11...10 is a multiple of 6 if it's a multiple of 3 (since it's always even).

 *        Sum of digits is K-1. So (K-1) % 3 == 0.
 * - d=6: d(d+1)=42. 11...10 is a multiple of 42 if 11...10 is multiple of 2, 3, and 7.
 *        11...10 is multiple of 2 and 3 if (K-1) % 3 == 0.
 *        11...10 is multiple of 7 if 11...1 is multiple of 7.
 *        A number consisting of L ones is a multiple of 7 if L is a multiple of 6.
 *        So (K-1) % 6 == 0.
 * - d=7: d(d+1)=56. 11...10 is not a multiple of 56 (not a multiple of 8).
 * - d=8: d(d+1)=72. 11...10 is not a multiple of 72 (not a multiple of 8).
 * 
 * Summary for K>=2:
 * - If (K-1) % 6 == 0: Solutions are d=1, 2, 5, 6. (Total 4)
 * - Else if (K-1) % 3 == 0: Solutions are d=1, 2, 5. (Total 3)
