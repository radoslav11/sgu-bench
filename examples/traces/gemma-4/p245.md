The correct logic to implement:
For each piece type $T$:
1. Find connected components of "safe" cells. A cell is safe if it is not '#' and not `unsafe`. Two safe cells are connected if there is a $T$-move between them that doesn't pass through any '#' or *other* pieces. (Actually, you can move through *empty* safe cells, and you *stop* at a piece).
2. A component's score is the sum of bonuses of all pieces in it.
3. A component is "reachable" if '@' can reach it.
4. For any piece (safe or unsafe), if it's reachable from '@' via safe cells, its bonus minus penalty (if unsafe) or just bonus (if safe) is a candidate.
5. If an unsafe piece $p$ is reachable from a reachable component $C$ in one $T$-move, $B(C) + bonus(p) - penalty$ is a candidate.

Wait, let's simplify even more:
A piece is reachable if there's a path from '@' through safe cells.
The score is the sum of bonuses of all *safe* pieces in a component, plus at most one unsafe piece.

Actually, the simplest way is to find all components of safe cells and see which one is reachable from '@'.
If a component is reachable, $B(C)$ is a candidate.
Also, if any cell in a reachable component can move to an unsafe piece, $B(C) + bonus(p) - penalty$ is a candidate.
Wait, if '@' is in a component, then any piece in that component is also reachable.
