Looking at this problem:

Each player height in micrometers, range 1950000..2050000. Average exactly 2000mm. Let d_i = height_i - 2000000 (in micrometers), so each d_i in [-50000, 50000], and sum of d_i = 0.

We need an ordering where for any contiguous subsegment, the sum of d_i over that segment has absolute value <= 100000 (10cm = 100000 micrometers).

Actually condition: |H - 2000*K| <= 100mm = 100000 micrometers. H - 2000*K = sum of d_i over segment.

So we need: every contiguous subarray has |partial sum| <= 100000.

Let prefix sums P_0=0, P_1, ..., P_N. Subarray sum from i+1 to j = P_j - P_i. We need |P_j - P_i| <= 100000 for all i<j... but actually only for segments selected by two players (segment of length >=... at least 2 players? "two players and players between them" - so length >=2). But length 1 trivially satisfies (|d_i|<=50000<=100000). So we need all pairs of prefix sums within 100000.

So we need to order the d_i so that all prefix sums stay within a window of width 100000. Since each d in [-50000,50000], and we want max prefix - min prefix <= 100000.

Classic greedy: keep running sum, if current sum > 0, add a negative element, else add a positive element. Since |each| <= 50000, the running sum stays in [-50000, 50000], giving range of prefix sums (including 0) within [-50000,50000], width <=100000. 

This always works! Sum is 0 so balance possible. Let me verify: start sum=0. If sum<=0 pick a positive (d>=0) element; sum increases by at most 50000, stays <=50000. If sum>0 pick a negative element, decreases by at most 50000, stays > -50000 hence >=-50000+... Actually after adding negative to positive sum: sum was in (0,50000], adding d in [-50000,0] gives sum in [-50000,50000]. Good. So always within [-50000,50000].

But what if sum<=0 and no positive elements left? Then remaining all negative but total sum must be 0, current sum<=0, remaining sum = -current sum >=0, but all negative means sum<0 unless empty — contradiction unless remaining empty. So whenever we need a positive one, one exists (or done). Similarly for negative. So solution always exists ("yes" always).

#include <bits/stdc++.h>
using namespace std;
int main(){
    int n;
    if(scanf("%d",&n)!=1) return 0;
    vector<long long> d(n);
    vector<int> idx(n);
    for(int i=0;i<n;i++){
        double x;
        scanf("%lf",&x);
        // convert to micrometers, round
        long long um = llround(x*1000000.0);
        d[i]=um-2000000;
        idx[i]=i+1;
    }
    // separate positives (>=0) and negatives (<0)
    vector<int> pos, neg;
    for(int i=0;i<n;i++){
        if(d[i]>=0) pos.push_back(i);
        else neg.push_back(i);
    }
    int pi=0, ni=0;
    long long sum=0;
    vector<int> order;
    for(int k=0;k<n;k++){
        if(sum<=0){
            if(pi<(int)pos.size()){
                int j=pos[pi++];
                sum+=d[j];
                order.push_back(idx[j]);
            } else {
                int j=neg[ni++];
                sum+=d[j];
                order.push_back(idx[j]);
            }
        } else {
            if(ni<(int)neg.size()){
                int j=neg[ni++];
                sum+=d[j];
                order.push_back(idx[j]);
            } else {
                int j=pos[pi++];
                sum+=d[j];
                order.push_back(idx[j]);
            }
        }
    }
    printf("yes\n");
    for(int i=0;i<n;i++){
        printf("%d%c", order[i], i+1<n?' ':'\n');

    return 0;
}
