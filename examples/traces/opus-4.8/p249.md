Looking at this problem:

We need a 2^N × 2^M matrix where the number is a combination of a row part (N bits) and column part (M bits). 

The idea: use Gray code for rows and columns. The value at cell (i,j) = gray(i) * 2^M + gray(j), where gray() is the standard Gray code. This way:
- Horizontally adjacent cells (same row, adjacent columns) differ only in gray(j) by one bit.
- Vertically adjacent cells differ only in gray(i) by one bit.
- Cyclic adjacency works because Gray code is cyclic (gray(0) and gray(2^k-1) differ by one bit).

Let me verify with sample N=1,M=1:
- gray(0)=0, gray(1)=1
- cell(0,0)=0*2+0=0, cell(0,1)=0*2+1=1... but expected output is:
```
0 2
1 3
```

So cell(0,0)=0, cell(0,1)=2, cell(1,0)=1, cell(1,1)=3. That means row is the low bits and column is the high bits. value = gray(j)*2^N + gray(i).

Check: cell(0,1): j=1→gray=1, i=0→gray=0 → 1*2+0=2 ✓. cell(1,0): j=0,i=1→0*2+1=1 ✓. cell(1,1): 1*2+1=3 ✓.

So value(i,j) = gray(j) * 2^N + gray(i).

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    int N, M;
    scanf("%d %d", &N, &M);
    int rows = 1<<N;
    int cols = 1<<M;
    
    // precompute gray codes
    vector<long long> grayRow(rows), grayCol(cols);
    for(int i=0;i<rows;i++) grayRow[i] = i ^ (i>>1);
    for(int j=0;j<cols;j++) grayCol[j] = j ^ (j>>1);
    
    long long base = 1LL<<N;
    
    // build output efficiently
    string out;
    out.reserve((size_t)rows*cols*4);
    char buf[24];
    for(int i=0;i<rows;i++){
        for(int j=0;j<cols;j++){
            long long val = grayCol[j]*base + grayRow[i];
            int len = sprintf(buf, "%lld", val);
            out.append(buf, len);
            if(j+1<cols) out.push_back(' ');
        }
        out.push_back('\n');
    }
    fputs(out.c_str(), stdout);
    return 0;
}
```
