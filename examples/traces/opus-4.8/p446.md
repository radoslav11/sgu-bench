Looking at this problem:

We have two point sets that are the same shape (one is a rotated/translated version of the other). We need to find the rotation angle, normalized to [0, π].

Approach: 
- Compute centroid of each set, subtract to center them.
- For each point, the rotation maps point i in set1 to some point in set2 (with same distance from centroid). The rotation angle is consistent.
- Pick a point far from centroid (to avoid numerical issues), find its corresponding point in set2 (same distance), compute angle difference.

But matching is ambiguous. Better: use the fact that sum of rotated vectors. Actually a robust way: rotation R that maps centered set1 to centered set2. We can compute R via least squares (Kabsch-like), since both are rigid same shape.

Given centered points p_i (set1) and q_i (set2), but ordering unknown. However we don't even need ordering if we use a rotation-invariant approach: 

The rotation angle θ. Consider the sum over a matching. Instead, use Kabsch but need correspondence.

Simplest robust: pick the point with max distance from centroid in set1. Find the point in set2 with closest matching distance. The angle between these vectors gives θ. Then normalize to [0,π].

Let me find correspondence by distance matching for the farthest point.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int n; 
    if(!(cin>>n)) return 0;
    vector<double> x1(n),y1(n),x2(n),y2(n);
    for(int i=0;i<n;i++) cin>>x1[i]>>y1[i];
    for(int i=0;i<n;i++) cin>>x2[i]>>y2[i];
    double cx1=0,cy1=0,cx2=0,cy2=0;
    for(int i=0;i<n;i++){cx1+=x1[i];cy1+=y1[i];cx2+=x2[i];cy2+=y2[i];}
    cx1/=n;cy1/=n;cx2/=n;cy2/=n;
    for(int i=0;i<n;i++){x1[i]-=cx1;y1[i]-=cy1;x2[i]-=cx2;y2[i]-=cy2;}
    // pick farthest point in set1
    int idx=0; double best=-1;
    for(int i=0;i<n;i++){double d=x1[i]*x1[i]+y1[i]*y1[i]; if(d>best){best=d;idx=i;}}
    double dist1=sqrt(best);
    // find matching point in set2 with closest distance
    int jdx=0; double bdiff=1e18;
    for(int j=0;j<n;j++){double d=sqrt(x2[j]*x2[j]+y2[j]*y2[j]); double df=fabs(d-dist1); if(df<bdiff){bdiff=df;jdx=j;}}
    double a1=atan2(y1[idx],x1[idx]);
    double a2=atan2(y2[jdx],x2[jdx]);
    double diff=a2-a1;
    // normalize to [0,pi]
    const double PI=acos(-1.0);
    while(diff<0) diff+=2*PI;
    while(diff>=2*PI) diff-=2*PI;
    if(diff>PI) diff=2*PI-diff;
    printf("%.10f\n",diff);
    return 0;
}
```

Wait—the farthest point might have ties (multiple points at same distance), causing wrong correspondence. To be safe, I should find the matching point in set2 that minimizes distance error among candidates, but ambiguity in which corresponds. To handle ties better, I'll consider all set2 points near dist1 and pick the one giving a consistent rotation by verifying. But for simplicity and given constraints, let me verify the rotation across all points and choose the candidate minimizing total error.

Let me make it robust by trying each candidate j with matching distance and computing how well the resulting rotation aligns both sets (using set2 as a set of distances/angles).

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int n; if(!(cin>>n)) return 0;
    vector<double> x1(n),y1(n),x2(n),y2(n);
    for(int i=0;i<n;i++) cin>>x1[i]>>y1[i];
    for(int i=0;i<n;i++) cin>>x2[i]>>y2[i];
    double cx1=0,cy1=0,cx2=0,cy2=0;
    for(int i=0;i<n;i++){cx1+=x1[i];cy1+=y1[i];cx2+=x2[i];cy2+=y2[i];}
    cx1/=n;cy1/=n;cx2/=n;cy2/=n;
    for(int i=0;i<n;i++){x1[i]-=cx1;y1[i]-=cy1;x2[i]-=cx2;y2[i]-=cy2;}
    int idx=0; double best=-1;
    for(int i=0;i<n;i++){double d=x1[i]*x1[i]+y1[i]*y1[i]; if(d>best){best=d;idx=i;}}
    double dist1=sqrt(best);
    double a1=atan2(y1[idx],x1[idx]);
    const double PI=acos(-1.0);
    double resDiff=0, resErr=1e18;
    for(int j=0;j<n;j++){
        double d=sqrt(x2[j]*x2[j]+y2[j]*y2[j]);
        if(fabs(d-dist1)>1e-3) continue;
        double a2=atan2(y2[j],x2[j]);
        double theta=a2-a1;
        double c=cos(theta), s=sin(theta), err=0;
        for(int i=0;i<n;i++){
            double rx=x1[i]*c - y1[i]*s, ry=x1[i]*s + y1[i]*c;
            double bm=1e18;
            for(int k=0;k<n;k++){double dx=rx-x2[k],dy=ry-y2[k]; bm=min(bm,dx*dx+dy*dy);}
            err+=bm;
        }
        if(err<resErr){resErr=err; double diff=theta;
            while(diff<0)diff+=2*PI; while(diff>=2*PI)diff-=2*PI; if(diff>PI)diff=2*PI-diff; resDiff=diff;}
    }
    printf("%.10f\n",resDiff);
    return 0;
}
```
