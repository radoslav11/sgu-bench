Looking at this problem, these are "Euclid strings" / Christoffel words.

The construction: a funny string with N elements summing to K (with gcd(N,K)=1) can be built using a balanced word / Christoffel word approach. Each element is `S[i] = floor((i+1)*K/N) - floor(i*K/N)`... but we need it to be funny.

The known construction for Christoffel words gives a balanced sequence. Let me use: `S[i] = floor(((i+1)*K)/N) - floor((i*K)/N)` for i from 0 to N-1.

Let me verify with N=9, K=16:
- i=0: floor(16/9)-0 = 1
- i=1: floor(32/9)-floor(16/9) = 3-1 = 2
- i=2: floor(48/9)-floor(32/9) = 5-3 = 2
- i=3: floor(64/9)-5 = 7-5 = 2
- i=4: floor(80/9)-7 = 8-7 = 1
- i=5: floor(96/9)-8 = 10-8 = 2
- i=6: floor(112/9)-10 = 12-10 = 2
- i=7: floor(128/9)-12 = 14-12 = 2
- i=8: floor(144/9)-14 = 16-14 = 2

Result: 1 2 2 2 1 2 2 2 2 — matches the sample exactly!

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    long long N, K;
    if(!(cin >> N >> K)) return 0;
    for(long long i = 0; i < N; i++){
        long long val = ((i+1)*K)/N - (i*K)/N;
        cout << val;
        if(i+1 < N) cout << ' ';
    }
    cout << '\n';
    return 0;
}
```
