Looking at this problem:

- Add segment [L,R], print count of existing segments contained in [L,R] (i.e., L ≤ l and r ≤ R), then add it.
- Remove one such segment if exists.
- At most 1000 segments present at once, up to 250000 queries.

With at most 1000 segments present, a query that needs counting can scan all present segments in O(1000) time. With 250000 queries that's 2.5×10^8 which should be fine within 2.25s. But we can also be smart. Let me just do the simple O(1000) per query scan—but we need a multiset to handle duplicates for removal.

For counting on '+', we count segments [l,r] with L≤l and r≤R. Let me just store a multiset/map of segments and scan.

Actually a map from segment to count, iterating over all distinct segments. Number of distinct segments ≤ 1000, and total counts ≤ 1000. So iterating over distinct entries is at most 1000 per query → 2.5×10^8 simple operations. Fine.

For removal: just decrement/remove from the map.

Let me implement with `std::map<pair<int,int>, int>`.

```cpp
#include <bits/stdc++.h>
using namespace std;
```

For each '+ L R': iterate over map entries, sum counts where L<=l && r<=R. Print. Then add.
For each '- L R': if present, decrement.

```cpp
for(auto&kv:mp){ int l=kv.first.first,r=kv.first.second; if(L<=l&&r<=R) cnt+=kv.second; }
```

This is correct. Let me write it with fast IO.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    map<pair<int,int>,int> mp;
    char c; int L,R;
    string out;
    while(scanf(" %c %d %d",&c,&L,&R)==3){
        if(c=='+'){
            long long cnt=0;
            for(auto&kv:mp){ if(L<=kv.first.first && kv.first.second<=R) cnt+=kv.second; }
            out+=to_string(cnt); out+='\n';
            mp[{L,R}]++;
        } else {
            auto it=mp.find({L,R});
            if(it!=mp.end()){ if(--it->second==0) mp.erase(it); }
        }
    }
    fputs(out.c_str(),stdout);
    return 0;
}
```

Let me verify with example:
- `+ 1 2`: no segments → 0. add [1,2].
- `+ 1 2`: count where 1<=l, r<=2: [1,2] yes → 1. add another [1,2] (count 2).
- `+ 0 3`: count where 0<=l, r<=3: [1,2] count 2 → 2. add [0,3].
- `- 1 2`: remove one [1,2] (count→1).
- `+ 1 2`: count where 1<=l,r<=2: [1,2] count 1 → 1. add.

Output: 0,1,2,1. Correct!
