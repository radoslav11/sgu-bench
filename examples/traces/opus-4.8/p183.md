Looking at this problem:

Among any M successive balls, at least 2 must be black. 

Let me think of DP where we track the positions of the last two black balls. If we paint balls and consider state by the two most recent black balls.

Let `dp[i][j]` = minimum cost where `i` is the position of the last painted ball, `j` is the position of the second-to-last painted ball, with the constraint satisfied up to position `i`.

The constraint: among any M consecutive, at least 2 black. Equivalently, considering the gaps. If last two black balls are at positions `i` and `j` (j<i), then for the window covering positions... we need that any window of M balls has 2 black.

Let me define: paint ball positions. Two consecutive black positions `p < q`: between them, the window of size M must contain 2 blacks. Actually any window of M consecutive containing only positions strictly between... The condition that gap is okay: if two consecutive black balls are at distance, we need that within any M-window there are 2. The critical: between three consecutive black balls a<b<c, the window must... Actually the requirement on two consecutive black balls at i and j (the two latest): the next black ball must be within M of... 

Let me use: `dp[i][k]` where k = i - j (distance between last two blacks), k from 1 to M-1. We need any M consecutive to have 2 blacks. With last two at i and j=i-k, the window ending at i+M-1 ... Let me just enforce: when adding a new black at position `p` after last two at `i,j`: the window [i+1, i+M] must contain at least 2 of {p and future}, but simplest known formulation: the new ball p must satisfy p <= i + (M-1) and also ensure window between j and p... Actually if consecutive blacks are i and p (p>i), any window of size M between them must have 2 blacks; the window starting at i+1 of size M is [i+1,i+M] needs 2 blacks, so p<=i+1 and one more... 

Standard: distance between two consecutive black balls must be ≤ M-1, AND consider windows: with blacks at j,i,p consecutive, window of size M must always contain 2. The binding constraint is i - j <= M-1 isn't enough. Actually window [i, i+M-1]: contains i and p, need p <= i+M-1. Window [j+1, j+M]: contains i and p? need p <= j+M. The tightest is p <= j+M where j is second-to-last. So **next black p satisfies p ≤ j + M** where j is the position before the last black i.

So state = (last black i, prev black j), transition to p where p ≤ j+M (and p>i). At the end positions [N-M+1...] must have 2 blacks: last two must satisfy constraint for final window. Let me implement with dp[i][k], k=i-prev, and handle boundaries.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int N,M; scanf("%d %d",&N,&M);
    vector<long long> C(N+1);
    for(int i=1;i<=N;i++) scanf("%lld",&C[i]);
    const long long INF=1e18;
    // dp[i][k]: i = last black, k=i-j (1..M-1), j=prev black
    // we use j+M >= next constraint. We'll iterate.
    // dp[i][k] min cost with last two blacks at i and i-k, all windows up to ... satisfied
    vector<vector<long long>> dp(N+1, vector<long long>(M+1, INF));
    // initialize first two blacks at j<i, both within first window: need window [1,M] has 2 blacks
    // first black j>=1, second i, i-j<=M-1, and j<=M? window[1,M] needs 2 blacks => i<=M (since must have 2 in first M)
    for(int i=2;i<=N && i<=M;i++){
        for(int j=1;j<i;j++){
            int k=i-j;
            if(k<=M-1){
                dp[i][k]=min(dp[i][k], C[i]+C[j]);
            }
        }
    }
    long long ans=INF;
    for(int i=2;i<=N;i++){
        for(int k=1;k<=M-1;k++){
            if(dp[i][k]==INF) continue;
            int j=i-k;
            // last window [N-M+1,N] must contain 2 blacks. If i is last black, prev j.
            // valid ending: positions i and j both >= N-M+1 OR there are blacks covering. 
            // ending valid if j >= N-M+1 (both i,j in last window) -- need last window 2 blacks
            if(j>=N-M+1){
                ans=min(ans,dp[i][k]);
            }
            // transition: next black p, constraint p<=j+M, p>i
            int pmax=min(N, j+M);
            for(int p=i+1;p<=pmax;p++){
                int nk=p-i;
                if(nk<=M-1){
                    if(dp[p][nk]>dp[i][k]+C[p]){
                        dp[p][nk]=dp[i][k]+C[p];
                    }
                }
            }
        }
    }
    printf("%lld\n",ans);
    return 0;
}
```

The transition is O(N*M*M) = 10000*100*100 = 10^8, acceptable.
