For each starting position, find the minimal ending position needed, then count.

For each left endpoint `i` (0-indexed start of substring), find the smallest `j` such that the message is a subsequence of `text[i..j]`. Then the number of valid right endpoints is `n - j` (we can end anywhere from `j` to `n-1`). Sum over all valid `i`.

To compute efficiently: for each `i`, greedily match the message. But that's O(n*m). With n up to maybe 1e5 and m up to 100, that's 1e7 — fine.

Actually let me precompute the minimal end for each start. As `i` increases, the minimal end is non-decreasing, so we can use a two-pointer approach, but greedy per-i is simplest and O(n*m) which should be ok.

Let me use the approach: for each i, match greedily. To avoid worst case, use precomputed next-occurrence table: `nxt[pos][c]` = next position >= pos with character c. Then matching message takes O(m) per start. 

```
for i in 0..n-1:
    pos = i
    ok = true
    for each char in message:
        pos = nxt[pos][c]
        if pos == n: ok=false; break
        pos++
    if ok: answer += n - (pos-1)  // last matched index is pos-1, j = pos-1, ways = n - j... 
```

Wait, end index j = pos-1 (the position of last matched char). Right endpoint can be any from pos-1 to n-1, giving n - (pos-1) choices = n - pos + 1.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    string t, m;
    getline(cin, t);
    getline(cin, m);
    int n = t.size();
    // nxt[pos][c]
    vector<array<int,26>> nxt(n+1);
    for(int c=0;c<26;c++) nxt[n][c]=n;
    for(int i=n-1;i>=0;i--){
        nxt[i]=nxt[i+1];
        nxt[i][t[i]-'a']=i;
    }
    long long ans=0;
    for(int i=0;i<n;i++){
        int pos=i;
        bool ok=true;
        for(char ch:m){
            int p=nxt[pos][ch-'a'];
            if(p==n){ok=false;break;}
            pos=p+1;
        }
        if(ok){
            int j=pos-1;
            ans += (long long)(n - j);
        }
    }
    cout<<ans<<endl;
    return 0;
}
```
