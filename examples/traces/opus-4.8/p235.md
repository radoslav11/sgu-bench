Looking at this problem:

- Queen moves like a chess queen. White pieces ('W') block movement and cannot be captured/landed on. Black pieces ('B') can be captured (Queen lands on them, but then they're removed... actually it captures and stops there).
- After exactly M moves, count distinct cells reachable.

Key insight: A Queen can move along 8 directions. When it captures a black piece, it stays on that cell. The black piece is removed. But since we want cells reachable after exactly M moves, and we track which cells are reachable at move count k.

Important: parity/extra moves. If a cell is reachable in k moves with k<=M, can we "waste" moves? A queen can move back and forth, so once reachable in k moves, reachable in k+2 moves (move to a free adjacent reachable cell and back). Actually we need exactly M moves.

Let me think with BFS by move count. Let `dist[cell]` = min moves to reach. Cells reachable in exactly M moves = cells where dist <= M and (M - dist) achievable via back-and-forth. Since a queen on any cell with at least one neighbor empty cell can waste 2 moves, any cell reachable in d<=M with same parity (M-d even) OR if there's flexibility... Actually moving to adjacent and back gives +2. Also some positions allow +1 differences via triangle moves. 

Given M up to 50, board up to 300, let me just do a BFS layered: dp[k] = set of cells reachable in exactly k moves. dp[0] = start. From each cell in dp[k], generate all cells reachable in one queen move (considering blocking). Union over reachable. But cells can be revisited — set grows. With M=50 and up to 90000 cells, transitions could be expensive.

But once a cell is in dp[k] and dp[k+1] (both parities present), it stays forever. The set stabilizes quickly. Let me just simulate: maintain boolean arrays cur (reachable in exactly k). For each step compute next. Number of distinct sets converges; but worst case 50 steps * 90000 cells * each cell expanding O(N) per direction = 90000*300=2.7e7 per step *50 = 1.3e9, too much maybe. But reachable set saturates; we can break when both parities of full reachable set achieved. Practically fine. Let me precompute for each cell its queen-move neighbors lazily.

Let me just do it directly with arrays, computing moves on the fly.

```cpp
#include <bits/stdc++.h>
using namespace std;
int N,M;
vector<string> g;
int dx[8]={-1,-1,-1,0,0,1,1,1};
int dy[8]={-1,0,1,-1,1,-1,0,1};
int main(){
    cin>>N>>M;
    g.resize(N);
    for(auto&s:g)cin>>s;
    int qx,qy;
    for(int i=0;i<N;i++)for(int j=0;j<N;j++)if(g[i][j]=='Q'){qx=i;qy=j;}
    vector<char> cur(N*N,0), nxt(N*N,0);
    cur[qx*N+qy]=1;
    for(int step=0;step<M;step++){
        fill(nxt.begin(),nxt.end(),0);
        for(int i=0;i<N;i++)for(int j=0;j<N;j++){
            if(!cur[i*N+j])continue;
            for(int d=0;d<8;d++){
                int x=i+dx[d],y=j+dy[d];
                while(x>=0&&x<N&&y>=0&&y<N){
                    if(g[x][y]=='W')break;
                    if(g[x][y]=='B'){nxt[x*N+y]=1;break;}
                    nxt[x*N+y]=1;
                    x+=dx[d];y+=dy[d];
                }
            }
        }
        swap(cur,nxt);
        // early termination if stabilized
        // check if cur == previous cur with same content twice... skip
    }
    int cnt=0;
    for(int i=0;i<N*N;i++)if(cur[i])cnt++;
    cout<<cnt<<endl;
}
```

If M=0, answer is 1 (the queen's cell). The loop runs 0 times, cur has only start. Good.

To avoid TLE, add early stop: if the set didn't change from two steps ago (period 2 stable), we can compute final based on parity. Let me add detection comparing cur to state two steps prior.

        }

    for(int i=0;i<N*N;i++)if(cur[i])cnt++;
    cout<<cnt<<endl;
}
