1. Build all constraint groups (21 lines + 7 marked groups).
2. Find the N-th lexicographic solution via backtracking, counting solutions but capping to avoid overflow.

Let me build adjacency: for each cell, which cells must differ from it (union of all groups containing that cell).

I'll do DFS filling cells 1..31 in order, trying values 1..K. For each cell, the constraint is it differs from already-filled cells in its conflict set. I count complete solutions; when count reaches N, output.

But counting all solutions could be huge. N up to 1e18. I'll cap counting at N (subtract as I go). Standard approach: DFS, when reaching a complete assignment count++; but to find N-th, I traverse in lex order and stop when the N-th is reached.

The challenge: counting solutions in subtrees could be enormous (up to many). I traverse cells in order trying values ascending; for each value branch I recursively count solutions; if count < N skip that branch (subtract), else descend.
