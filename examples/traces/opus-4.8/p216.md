After merging children's pending lists and adding self, whenever pending size >= B, flush them into a new province with capital = current node. The remaining pending (< B) bubble up. At root, any leftover pending get attached to the last created province (which is valid since adding < B keeps it <= 3B... need care).

Since each flush has size between B and 2B-1 (we flush as soon as >= B, and per-merge we add at most... actually we add child's leftover < B plus self... could exceed). Let me bound: we collect children leftovers each < B, but we flush whenever total >= B. Process children one by one, after each child's leftover added, if size>=B flush. So before adding a child leftover (<B), size < B, after adding size < 2B, flush leaving 0. Then add self. Max province size < 2B <= 3B. Good. Leftover at root < B gets merged into last province → size < 2B + B = 3B, OK. If no province exists and leftover at root, then total N < B → impossible.
