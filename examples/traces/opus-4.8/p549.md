Looking at this problem:

- Group dumbbells by mass. For each mass, sort costs descending.
- We need to pick k distinct masses. Number of sets = minimum count among chosen masses (each set needs one of each chosen mass).
- For t sets, we need k masses each having at least t dumbbells available, and we use the top t costs from each.

So: find max t such that at least k masses have count ≥ t. Then among masses with count ≥ t, we want to choose k of them to maximize total cost (sum of top t costs).

For each candidate t, masses with count ≥ t contribute their sum of top t costs. Pick the k largest such sums.

To maximize t: t can range. The max possible t is the largest t where number of masses with count≥t is ≥ k.

For each t from 1 to max, check feasibility and compute cost. But we want maximum t (primary), then max cost.

Let me just iterate t from high to low, find first feasible t, compute best cost.

Max t: at most n/k. Let's iterate over all possible t values, but to be efficient: the maximum t is determined. For that t, compute cost.

Number of distinct masses ≤ 4000, counts... Let me compute for the optimal t only.

For each mass with sorted-descending prefix sums, prefix[t] = sum of top t. For a given t, collect prefix[t] for all masses with count≥t, take top k sum.

Complexity: total work summing over t of (number of masses with count≥t) = total dumbbells = n. So iterating all t and for each computing is fine if we organize well. But selecting top k each time naively could be costly. Total elements across all t is n=4000, so it's fine.

We only need the maximum feasible t. Find it, then compute cost for that t.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int n,k;
    scanf("%d %d",&n,&k);
    vector<vector<int>> byMass(4001);
    for(int i=0;i<n;i++){
        int m,c; scanf("%d %d",&m,&c);
        byMass[m].push_back(c);
    }
    // build prefix sums for each mass (sorted desc)
    vector<vector<long long>> pref;
    for(int m=1;m<=4000;m++){
        if(byMass[m].empty()) continue;
        sort(byMass[m].rbegin(), byMass[m].rend());
        vector<long long> p(byMass[m].size()+1,0);
        for(size_t j=0;j<byMass[m].size();j++) p[j+1]=p[j]+byMass[m][j];
        pref.push_back(move(p));
    }
    // find max t: max value such that #masses with count>=t >= k
    int maxT=0;
    // for each t, count masses with size-1>=t i.e. (p.size()-1)>=t
    // sizes
    int M=pref.size();
    // determine max t
    int bestT=0;
    // t up to n/k
    int upper = n/k;
    for(int t=upper;t>=1;t--){
        int cnt=0;
        for(int i=0;i<M;i++) if((int)pref[i].size()-1>=t) cnt++;
        if(cnt>=k){ bestT=t; break;}
    }
    if(bestT==0){ printf("0 0\n"); return 0;}
    int t=bestT;
    vector<long long> sums;
    for(int i=0;i<M;i++){
        if((int)pref[i].size()-1>=t) sums.push_back(pref[i][t]);
    }
    sort(sums.rbegin(), sums.rend());
    long long total=0;
    for(int i=0;i<k;i++) total+=sums[i];
    printf("%d %lld\n", t, total);
    return 0;
}
```

The loop over t down from upper checking count is O(upper * M). upper ≤ n/k, M ≤ number of masses. Worst case k=1, upper=n=4000, M=4000 → 16M, fine.
