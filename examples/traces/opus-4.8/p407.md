Looking at this problem:

We have a wheel graph: capital (vertex 0) connected to all n country towns arranged in a cycle. We need number of closed walks of length m starting and ending at vertex 0.

This is the (0,0) entry of A^m where A is the adjacency matrix of the wheel graph.

n up to 1000, m up to 5000. Matrix exponentiation is too big. But we can just do iterated vector multiplication: start with vector e_0, multiply by A m times, read entry 0. Each multiplication is O(edges) ~ O(n). Total O(n*m) = 5,000,000 operations, but with big integers.

The numbers can be huge though. We need big integer arithmetic. The answer grows roughly like (max degree)^m... degree of capital is n=1000, so values ~ 1000^5000, around 15000 digits. With n*m vector multiplications each involving big int additions... that's expensive.

Let me think about symmetry. By rotational symmetry, after operations the values on all cycle vertices reachable symmetrically... Starting from vertex 0, the state is symmetric under rotation. So all cycle vertices have the same value at each step! 

So state: (c, r) where c = value at capital, r = value at each cycle vertex (all equal by symmetry).

Adjacency: capital connects to all n cycle vertices. Each cycle vertex connects to capital + 2 neighbors.

New capital value = sum over cycle vertices = n * r.
New cycle vertex value (for vertex i) = capital + left neighbor + right neighbor = c + 2*r.

So recurrence:
- c' = n * r
- r' = c + 2*r

Start: c=1, r=0 (length 0, at capital). After m steps answer = c.

We need big integers. Only 2 values per step, m=5000 steps. Easy with big int.

Let me verify: n=4, m=3.
Step0: c=1,r=0
Step1: c=4*0=0, r=1+0=1
Step2: c=4*1=4, r=0+2*1=2
Step3: c=4*2=8, r=4+2*2=8 → answer c=8. ✓

n=3,m=4:
Step0: c=1,r=0
Step1: c=0, r=1
Step2: c=3, r=2
Step3: c=6, r=3+4=7
Step4: c=21, r=6+14=20 → answer 21. ✓

Now implement big integers with base 1e9 or use simple decimal. I'll use base 10^9 vectors.

```cpp
#include <bits/stdc++.h>
using namespace std;
typedef vector<long long> Big; // base 1e9, little endian
const long long BASE=1000000000LL;

Big mulSmall(const Big&a, long long k){
    Big res(a.size()+1,0);
    long long carry=0;
    for(size_t i=0;i<a.size();i++){
        __int128 cur=(__int128)a[i]*k+carry;
        res[i]=(long long)(cur%BASE);
        carry=(long long)(cur/BASE);
    }
    size_t i=a.size();
    while(carry){ res[i]+=carry%BASE; carry/=BASE; i++; }
    while(res.size()>1&&res.back()==0) res.pop_back();
    return res;
}
Big add(const Big&a,const Big&b){
    Big res; long long carry=0;
    size_t n=max(a.size(),b.size());
    for(size_t i=0;i<n||carry;i++){
        long long cur=carry;
        if(i<a.size())cur+=a[i];
        if(i<b.size())cur+=b[i];
        res.push_back(cur%BASE);
        carry=cur/BASE;
    }
    while(res.size()>1&&res.back()==0) res.pop_back();
    return res;
}

int main(){
    long long n,m;
    cin>>n>>m;
    Big c(1,1), r(1,0);
    for(long long step=0;step<m;step++){
        Big nc=mulSmall(r,n);
        Big nr=add(c, mulSmall(r,2));
        c=nc; r=nr;
    }
    // print c
    string s=to_string(c.back());
    for(int i=(int)c.size()-2;i>=0;i--){
        string t=to_string(c[i]);
        s+=string(9-t.size(),'0')+t;
    }
    cout<<s<<"\n";
}
```
