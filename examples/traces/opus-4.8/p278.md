Looking at this problem:

We buy m_i kilograms of fuel i. Constraints: sum(m_i * a_i) <= A, sum(m_i * b_i) <= B, m_i >= 0. Maximize sum(m_i * c_i).

This is an LP with 2 constraints. The optimal solution uses at most 2 fuels (since only 2 constraints). 

For a single fuel i: max amount limited by A/a_i (if a_i>0) and B/b_i (if b_i>0). Intensity = c_i * min.

For combinations: with 2 constraints, optimum lies at a vertex using at most 2 variables. We can think of it as: for each fuel, it has a "rate" vector (a_i, b_i) and value c_i. We want to combine to use up budget (A, B) maximally.

Standard approach: The answer corresponds to maximizing c·m subject to (a·m <= A, b·m <= B). By LP duality, consider dual variables. Actually with 2 resources, the convex hull approach: the best uses at most 2 fuels mixing the two constraints.

Let me think via the dual: minimize A*y1 + B*y2 subject to a_i*y1 + b_i*y2 >= c_i for all i, y1,y2>=0. The minimum equals the answer.

The dual feasible region: y1,y2>=0, and for each i, a_i*y1+b_i*y2 >= c_i. Minimize A*y1+B*y2. The optimum is at a vertex defined by 2 constraints (or axis). 

Since N up to 75000, vertices from pairs is too many (N^2). But the constraints define an upper envelope of lines in (y1,y2)... Actually we need feasible region = intersection of half-planes a_i*y1+b_i*y2>=c_i (each excludes region below a line). The feasible region is the intersection — minimizing a linear function. The boundary relevant is the lower-left envelope.

We minimize over the upper-right region bounded by these lines. The optimal vertex is where two binding constraints meet on the lower-left boundary of feasible set, which is the upper envelope of lines (since each constraint requires being above its line). Computing upper envelope of N lines = convex hull, O(N log N).

Lines: y2 = (c_i - a_i*y1)/b_i. Feasibility requires y2 >= each line. The boundary is upper envelope. We minimize A*y1+B*y2 over points above this envelope with y1,y2>=0. Optimum on envelope or axis.

Let me just take upper envelope of lines (for b_i>0), plus handle b_i=0 (constraint a_i*y1>=c_i => y1>=c_i/a_i) and edge cases. Then walk the envelope, evaluate objective at vertices, also at intersections with axes considering constraints.

Given complexity, simpler robust approach given small coordinate values: build upper envelope of lines, evaluate objective at all breakpoints and clip to y1>=y1min (from b=0 constraints) and y2>=0 region.

Let me implement upper envelope (max of lines) using convex hull trick, then evaluate.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int N; double A,B;
    scanf("%d %lf %lf",&N,&A,&B);
    // dual: min A*y1+B*y2 s.t. a*y1+b*y2>=c, y1,y2>=0
    // lines y2 >= (c-a*y1)/b for b>0. y2 = -a/b * y1 + c/b
    // also b=0: a*y1>=c => y1>=c/a (if a>0,c>0). if a=0,b=0,c>0 infeasible? c<=100,a,b can be 0
    vector<double> slope, inter; // y2 = slope*y1+inter, take max (upper envelope)
    double y1min=0;
    bool infeasible=false;
    for(int i=0;i<N;i++){
        double a,b,c; scanf("%lf %lf %lf",&a,&b,&c);
        if(c<=0) continue;
        if(b>0){ slope.push_back(-a/b); inter.push_back(c/b);}
        else {
            // b==0, a*y1>=c
            if(a>0) y1min=max(y1min,c/a);
            else infeasible=true; // can't satisfy
        }
    }
    if(infeasible){ printf("inf\n"); return 0;}
    // upper envelope: sort by slope, build hull keeping max
    // include line y2=0 (since y2>=0)
    slope.push_back(0); inter.push_back(0);
    int n=slope.size();
    vector<int> idx(n); for(int i=0;i<n;i++) idx[i]=i;
    sort(idx.begin(),idx.end(),[&](int x,int y){ if(slope[x]!=slope[y])return slope[x]<slope[y]; return inter[x]>inter[y];});
    vector<double> sl,in;
    for(int k=0;k<n;k++){
        double s=slope[idx[k]], c=inter[idx[k]];
        if(!sl.empty() && sl.back()==s) continue; // same slope, lower inter (already max kept first)
        while(sl.size()>=1){
            int m=sl.size();
            if(m>=2){
                // intersection of new with sl[m-2] vs sl[m-1] removal
                double x1=(in[m-1]-c)/(s-sl[m-1]);
                double x2=(in[m-2]-in[m-1])/(sl[m-1]-sl[m-2]);
                if(x1<=x2){ sl.pop_back(); in.pop_back(); } else break;
            } else {
                double x1=(in[m-1]-c)/(s-sl[m-1]);
                // need check vs first? just break, allow
                break;
            }
        }
        sl.push_back(s); in.push_back(c);
    }
    // now evaluate min A*y1+B*y2 over y1>=y1min, on upper envelope (which gives min y2)
    // breakpoints between consecutive lines
    int m=sl.size();
    vector<double> bp; bp.push_back(y1min);
    for(int i=0;i+1<m;i++){
        double x=(in[i+1]-in[i])/(sl[i]-sl[i+1]);
        if(x>y1min) bp.push_back(x);
    }
    // also a large value to evaluate slope ends, but objective min won't be at infinity if slopes ok
    double best=1e18;
    auto envY2=[&](double y1)->double{
        double v=-1e18;
        for(int i=0;i<m;i++) v=max(v, sl[i]*y1+in[i]);
        return max(v,0.0);
    };
    for(double x:bp){
        double y2=envY2(x);
        if(y2<0) y2=0;
        best=min(best, A*x+B*y2);
    }
    printf("%.6f\n", best);
    return 0;
}
```
