Looking at this problem, I need to find the shortest path along a hyperboloid surface between two points.

The constraint -1 ≤ z ≤ 1 means we're dealing with the portion of the hyperboloid near the "waist". With z=0, x²+y²=1 (unit circle), and the hyperboloid widens as |z| grows.

The example: A=(1,0,0), B=(0,1,0), both on the unit circle at z=0. Distance is π/2, which is exactly the arc length along the circle (quarter circle, radius 1). So for points on the same z-level circle, the geodesic might just be along the circle... but actually geodesics can dip in z.

Let me think about geodesics. The surface can be parametrized by angle θ and z: x = √(1+z²)cosθ, y = √(1+z²)sinθ. 

Let me use numerical approach. I'll discretize and find shortest path via a relaxation/optimization. Given accuracy ±0.1, I can use a numerical geodesic computation.

I'll parametrize the path by z going... but path may not be monotonic in z. Better: parametrize by angle. Let me set up the path as a function z(θ) and minimize the arc length integral, then optimize.

Actually, simplest robust approach: build a fine grid mesh on the surface (over θ and z) and run Dijkstra/shortest path between nearest grid points to A and B.

Let me create grid: θ in [0, 2π), z in [-1, 1]. Compute 3D positions, connect neighbors, run Dijkstra. With grid fine enough, accuracy within 0.1 is achievable.

Metric distances between adjacent grid points use Euclidean 3D distance (good approximation for fine grid). Let me use ~200 θ steps and ~100 z steps = 20000 nodes, Dijkstra feasible.

Let me write it carefully with Dijkstra.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    double xa,ya,za,xb,yb,zb;
    cin>>xa>>ya>>za>>xb>>yb>>zb;
    int NT=360, NZ=160;
    auto P=[&](int it,int iz){
        double z=-1.0+2.0*iz/(NZ-1);
        double r=sqrt(1+z*z);
        double th=2*M_PI*it/NT;
        return array<double,3>{r*cos(th),r*sin(th),z};
    };
    // find nearest node to A and B
    auto findN=[&](double x,double y,double z){
        int bi=0,bj=0; double bd=1e18;
        for(int it=0;it<NT;it++)for(int iz=0;iz<NZ;iz++){
            auto p=P(it,iz);
            double d=(p[0]-x)*(p[0]-x)+(p[1]-y)*(p[1]-y)+(p[2]-z)*(p[2]-z);
            if(d<bd){bd=d;bi=it;bj=iz;}
        }
        return bi*NZ+bj;
    };
    int src=findN(xa,ya,za), dst=findN(xb,yb,zb);
    int N=NT*NZ;
    vector<double> dist(N,1e18);
    priority_queue<pair<double,int>,vector<pair<double,int>>,greater<>> pq;
    dist[src]=0; pq.push({0,src});
    int dt[]={-1,-1,-1,0,0,1,1,1};
    int dz[]={-1,0,1,-1,1,-1,0,1};
    while(!pq.empty()){
        auto[d,u]=pq.top();pq.pop();
        if(d>dist[u])continue;
        int it=u/NZ, iz=u%NZ;
        auto pu=P(it,iz);
        for(int k=0;k<8;k++){
            int nt=(it+dt[k]+NT)%NT, nz=iz+dz[k];
            if(nz<0||nz>=NZ)continue;
            int v=nt*NZ+nz;
            auto pv=P(nt,nz);
            double w=sqrt((pu[0]-pv[0])*(pu[0]-pv[0])+(pu[1]-pv[1])*(pu[1]-pv[1])+(pu[2]-pv[2])*(pu[2]-pv[2]));
            if(dist[u]+w<dist[v]){dist[v]=dist[u]+w;pq.push({dist[v],v});}
        }
    }
    printf("%.10f\n",dist[dst]);
}
```
