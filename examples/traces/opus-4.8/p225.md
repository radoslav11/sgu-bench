Looking at this: place k non-attacking knights on n×n board.

Standard approach: DP row by row, keeping track of last two rows' configurations (since knight attacks span 2 rows). Use bitmask DP.

```cpp
#include <bits/stdc++.h>
using namespace std;

int n, k;
// dp[prevprev][prev][count]
long long dp[1<<10][2]; // we'll handle differently

int main(){
    cin>>n>>k;
    int full=1<<n;
    // states: pair (prevprev row mask, prev row mask)
    // map index
    // We'll use dp indexed by (ppmask, pmask) and count
    // dimension: full*full*(k+1) could be big for n=10: 1024*1024*101 ~ 1e8 too big in longlong
    // but masks limited... let's just do it with vectors
    
    // Use map from (pp,p) -> array over counts. Actually total knights <= n^2 <=100.
    // Memory: full*full=1M entries each with k+1 longs -> too much for n=10.
    // Instead iterate: dp over rows, state = (prev2 row, prev1 row), but we sum counts.
    
    // Let's reduce: only need counts up to k. Use 3D but compress.
    // For n=10, full=1024. full*full=1048576. times (k+1) up to 101 *8 bytes = huge.
    
    // Better: process counts separately? Number of knights placed differs per state path.
    // Alternative: dp[pp][p] = vector of counts. Use unordered_map for sparse.
    
    // popcount of valid placements... let's just use 3D with manageable since many states invalid.
    
    // Use map<pair<int,int>, vector<long long>>
    map<pair<int,int>, vector<long long>> cur;
    // initial: before any row, prev rows are 0,0 with 0 knights
    {
        vector<long long> v(k+1,0);
        v[0]=1;
        cur[{0,0}]=v;
    }
    
    for(int row=0; row<n; row++){
        map<pair<int,int>, vector<long long>> nxt;
        for(auto &e: cur){
            int pp=e.first.first;
            int p=e.first.second;
            vector<long long>& cnt=e.second;
            // try all masks for current row
            for(int m=0;m<full;m++){
                // check attack with prev row p: knight moves (1,2) and (2,1)
                // within same two adjacent rows: distance 1 row, columns differ by 2
                bool ok=true;
                // p is row-1, columns shift by 2
                if( ((m<<2)&p) || ((m>>2)&p) ) ok=false;
                // pp is row-2, columns shift by 1
                if(ok && ( ((m<<1)&pp) || ((m>>1)&pp) )) ok=false;
                if(!ok) continue;
                int pc=__builtin_popcount(m);
                auto &target=nxt[{p,m}];
                if(target.empty()) target.assign(k+1,0);
                for(int c=0;c+pc<=k;c++){
                    if(cnt[c]) target[c+pc]+=cnt[c];
                }
            }
        }
        cur=move(nxt);
    }
    
    long long ans=0;
    for(auto &e:cur){
        ans+=e.second[k];
    }
    cout<<ans<<endl;
    return 0;
}
```

Let me verify masks within mask m itself: two knights in same row can't attack (distance in one direction min 1, need 2+1). Same row distance is 0 rows apart, knight needs row diff of 1 or 2, so same row never attacks. Good.

For columns: shift by 2 means `m<<2` aligns column j of m with column j+2; comparing with p detects pairs (row diff 1, col diff 2). And `m<<1` with pp detects (row diff 2, col diff 1). The shifts should be masked to full width but extra high bits beyond board are 0 in p/pp anyway. Good.
