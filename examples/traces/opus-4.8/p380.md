Looking at this problem:

Two alpinists start at positions 1 (left/west) and n (right/east). One moves right, one moves left. They must always be at the same height. They want to meet at the same point.

Let me think of this as a BFS/DFS on states. The mountain border is a polyline. An alpinist at a point on the polyline can move to adjacent points along the polyline, but actually they move continuously along the surface.

State: left alpinist's position on the polyline, right alpinist's position, with constraint same height. The positions are along segments. Let me model state as which segment each is on, plus we need them at same height.

Let me think: alpinist A is on segment between vertex i and i+1, alpinist B is on segment between j and j+1. They must be at the same height h. For a given pair of segments, there's a range of heights where both can be on those segments simultaneously. We can move continuously.

The standard approach: state = (i, j) meaning A is on edge i (vertices i,i+1) and B is on edge j (vertices j,j+1), and they're at some common height. We do BFS over which (segment, segment) pairs are reachable, with the set of achievable common heights being an interval (intersection of height ranges of the two edges). 

A starts at vertex 1 (edge 0), B starts at vertex n (edge n-2), both at height 0. Goal: they meet at same x position. They meet when their x-coordinates are equal at the same height — meaning A and B occupy the same point.

Transitions between (i,j) and adjacent edges share continuity if height intervals overlap. Meeting happens when on the same edge i==j... actually they meet when positions coincide. If both on edge i with same height, they're at the same point. So goal: reachable state where i==j (same edge), since same edge + same height = same x = same point.

Let me do BFS where node (i,j) is valid if height range of edge i intersects height range of edge j. Edge i covers heights [min(y_i,y_{i+1}), max(...)]. Move from (i,j) to (i±1,j) or (i,j±1) if intervals overlap. Start (0, n-2). If we reach any (i,i), answer YES.

```cpp
#include <bits/stdc++.h>
using namespace std;
int n;
int y[100005];
bool vis[2005][2005];
int main(){
    scanf("%d",&n);
    for(int i=0;i<n;i++) scanf("%d",&y[i]);
    int m=n-1; // edges 0..m-1
    auto lo=[&](int e){return min(y[e],y[e+1]);};
    auto hi=[&](int e){return max(y[e],y[e+1]);};
    auto overlap=[&](int a,int b){
        return max(lo(a),lo(b))<=min(hi(a),hi(b));
    };
    queue<pair<int,int>>q;
    if(overlap(0,m-1)){vis[0][m-1]=1;q.push({0,m-1});}
    bool ans=false;
    while(!q.empty()){
        auto[i,j]=q.front();q.pop();
        if(i==j){ans=true;break;}
        int di[]={-1,1,0,0};
        int dj[]={0,0,-1,1};
        for(int d=0;d<4;d++){
            int ni=i+di[d],nj=j+dj[d];
            if(ni<0||ni>=m||nj<0||nj>=m)continue;
            if(vis[ni][nj])continue;
            if(overlap(ni,nj)){vis[ni][nj]=1;q.push({ni,nj});}
        }
    }
    printf(ans?"YES\n":"NO\n");
    return 0;
}
```

The constraint on n must be small (≤2000 likely) given the memory for vis array. Let me verify with the example: n=7, edges with heights ranges. Start (0,5). The example outputs YES.

This approach treats reachability correctly since within a (segment,segment) pair you can move continuously over the overlapping height interval, and adjacency requires overlap to pass the shared vertex height.
