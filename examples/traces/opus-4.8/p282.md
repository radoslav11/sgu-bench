Looking at this problem, I need to count edge-colored complete graphs on N vertices up to isomorphism, using Burnside's lemma.

By Burnside: answer = (1/N!) * Σ_{permutations} M^(number of edge-orbits under permutation).

For a permutation with given cycle structure, I need to count the orbits of edges (pairs of vertices) induced by the permutation.

The number of permutations with a given cycle type, and the edge-orbit count, can be computed by iterating over partitions of N.

**Edge orbits formula** for cycle type with cycles of lengths $c_1, c_2, ...$:
- Within a single cycle of length L: number of edge orbits = floor(L/2).
- Between two cycles of lengths a and b: number of edge orbits = gcd(a,b).

**Number of permutations** with cycle type (using $a_i$ = number of cycles of length $i$):
$$\frac{N!}{\prod_i i^{a_i} \cdot a_i!}$$

So Burnside term = N!/∏(i^{a_i} a_i!) * M^(edges) divided by N!, meaning each term contributes $\frac{1}{\prod i^{a_i} a_i!} M^{E}$.

I need everything mod P (prime). The division by N! is tricky since P may be small and N! may share factors. But the final answer is an integer. I'll compute the sum of (count * M^E) where count = N!/(∏ i^{a_i} a_i!), then divide by N!.

To handle division mod P safely (P could divide N!), I'll compute the total sum as a big integer divided... Actually, better: compute each coefficient $\frac{N!}{\prod i^{a_i} a_i!}$ which is an integer (multinomial-like), multiply by $M^E \bmod P$, sum mod P, then divide by N!. But N!/N! = need to divide total by N!.

Let me restructure: answer = Σ (1/(∏ i^{a_i} a_i!)) * M^E. The coefficient $\frac{1}{∏ i^{a_i} a_i!}$ times N! is integer (it's the count). So answer = (1/N!) Σ count_i * M^E_i. Since answer is integer, I compute S = Σ count_i * M^{E_i} as a big number... but counts can be huge.

Instead use Python-style big integers via __int128? Counts up to 53! ~ huge. I'll use a fraction-free approach: keep the sum as exact rational with denominator N!, i.e. accumulate Σ count_i * M^E_i exactly using big integers. Since N≤53, partitions are few but numbers huge — I'll implement big integer with mod... 

Better: answer mod P. Use that answer = S/N! where S = Σ count_i * M^E_i is divisible by N!. Compute S mod (P * N!)? N! too big.

I'll just use Python big int simulation in C++ with __int128 won't fit. Let me use a simple bignum (vector of digits) for S, divide by N!, then mod P. Partition count of 53 is 329931 — manageable.
