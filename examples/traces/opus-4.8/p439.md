Looking at this problem:

1. Find the lexicographically smallest cyclic shift of the second string (Booth's algorithm or simple comparison since M is small).

2. Then shift the first string (cyclically, left/right) so that its beginning matches the second string with at most one differing letter. We want minimal number of shifts, and among ties, prefer left shift.

Let me think about "minimal number of shifts" and "prefer left". A left shift by k or right shift by k — the number of shifts is k in both cases. We try k=0,1,2,... and for each k check left shift (which gives starting position k) then right shift (starting position N-k). We pick first match.

For each starting position p of the first string (cyclic), check the prefix of length M matches second string with at most 1 difference.

Left shift by k → starts at position k. Right shift by k → starts at position (N-k)%N.

Order of checking for increasing shift count: k=0: pos 0. k=1: left pos1, right pos N-1. k=2: left pos2, right pos N-2... For each k, left first.

```cpp
#include <bits/stdc++.h>
using namespace std;

int main(){
    int N,M;
    cin>>N>>M;
    string A,B;
    cin>>A>>B;
    // smallest cyclic shift of B
    string best=B;
    for(int i=0;i<M;i++){
        string s=B.substr(i)+B.substr(0,i);
        if(s<best) best=s;
    }
    B=best;
    
    auto matches=[&](int p)->bool{
        int diff=0;
        for(int j=0;j<M;j++){
            if(A[(p+j)%N]!=B[j]){
                diff++;
                if(diff>1) return false;
            }
        }
        return true;
    };
    
    int found=-1;
    for(int k=0;k<N && found<0;k++){
        // left shift by k -> start pos k
        int pl=k%N;
        if(matches(pl)){ found=pl; break;}
        if(k>0){
            int pr=((N-k)%N+N)%N;
            if(pr!=pl && matches(pr)){ found=pr; break;}
        }
    }
    
    string res;
    for(int j=0;j<N;j++) res+=A[(found+j)%N];
    cout<<B<<"\n"<<res<<"\n";
    return 0;
}
```
