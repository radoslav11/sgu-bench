Looking at this problem:

- Lines start with markers (`#` or `*`).
- A group of 2+ consecutive lines with the same starting marker form a list.
- If a single line, it's not a list — it's output as-is.

Let me think recursively. We process a set of lines, each with a "prefix" of markers already consumed. At each level, we look at the current marker character of each line.

The key rule: a group of two or more consecutive list elements sharing the same marker becomes a list. So we partition lines by their current marker char; but only groups of size ≥2 (with that marker) form a list. A single line with a marker — is it treated as list element? 

From sample 1: `FirstLine` (no marker) printed plain. The `*ItemZ` at end is single, printed plain `*ItemZ`. So a single line doesn't form a list — printed verbatim (with markers intact).

So algorithm: given lines (each is a string, considering remaining content), group consecutive lines. For a group of lines, look at first char. If first char is a marker AND there are ≥2 consecutive lines starting with the same marker, they form a list. Otherwise the line is output verbatim.

Process: iterate, find maximal run of consecutive lines whose first char is the same marker (`#` or `*`). If run length ≥2, emit list with that tag, and recurse on the lines with first char stripped. If run length 1 (or first char not a marker), output line verbatim.
