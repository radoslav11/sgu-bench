Looking at this problem:

We have A and B balls. A move: from one box move as many balls as the other contains (doubling the other box).

We want all balls in one box, i.e., one box becomes 0.

Total S = A+B. For it to be possible, we need... let's think. Each move doubles one box. To get one box to 0, we move from the larger to smaller until... Actually let's simulate: at each step, if A==B impossible to continue unless one is 0. We want one to become 0.

Standard approach: keep moving from larger to smaller (smaller doubles, larger loses that amount). Let total = A+B. The sum must be divisible appropriately. Let's just simulate: while both nonzero, take the smaller one, double it by moving from larger. If smaller > larger, impossible (can't move that many). Count moves.

Let me simulate: if A==0 or B==0 done. Otherwise let small=min, large=max. We move from large to small: small becomes 2*small, large becomes large-small. This requires large>=small. If large<small impossible. If A==B and both nonzero, we'd need to move but result large-small=0... actually if A==B, move makes one 0. So that's fine (one becomes 0).

Need to bound iterations. Total S divisible by some power. The number of moves is bounded — actually it relates to S = G * 2^k where G is odd... The answer terminates when small divides such that large-small=0, meaning large==small. Number of moves bounded by ~32 since values double. Let's cap at like 60 iterations.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    long long A,B;
    cin>>A>>B;
    long long cnt=0;
    while(A!=0 && B!=0){
        long long small=min(A,B), large=max(A,B);
        if(large<small){cnt=-1;break;}
        // move from large to small
        // small doubles
        long long nl=large-small;
        long long ns=small*2;
        if(A<B){A=ns;B=nl;} else {B=ns;A=nl;}
        cnt++;
        if(cnt>100){cnt=-1;break;}
    }
    cout<<cnt<<endl;
}
```
