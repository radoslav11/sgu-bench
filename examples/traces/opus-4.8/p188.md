Looking at this problem:

Two soldiers meet when they're at the same point with opposite velocities (moving toward each other). They count meetings during [0, T] minutes inclusive.

For each pair (i,j), we need to count the number of times they meet within time T.

Position of soldier i at time t: `(Li + Vi*t) mod 1000`.

They meet when positions equal AND velocities are opposite directions (one clockwise, one counter-clockwise, i.e., they're moving toward each other at that point).

Actually "meet" = same point with opposite velocities. Since velocities are constant, the relative direction is fixed. Two soldiers moving in same direction never "meet" (they could catch up but that's same velocity direction). Wait - opposite velocities means signs of Vi and Vj differ... but actually they need to be approaching each other.

Let me think: positions on a circle. They're at the same point with opposite velocities means their velocity vectors point in opposite directions along the circle. Since each moves either CW or CCW, opposite velocities means one CW, one CCW (different signs). When two move in opposite directions and meet, they pass through each other.

So only pairs with opposite sign velocities can meet. For such pairs, count times t in [0,T] where positions coincide.

Position coincide: `Li + Vi*t ≡ Lj + Vj*t (mod 1000)`, so `(Li-Lj) + (Vi-Vj)*t ≡ 0 (mod 1000)`.

Let me use fractions/scale. Multiply t by appropriate. Let's solve: `(Vi-Vj)*t ≡ (Lj-Li) (mod 1000)`. Let D = Vi-Vj (nonzero since opposite signs). For t in [0,T], find number of solutions.

To avoid floating point, multiply everything. Times when meeting: t = (Lj - Li + 1000*k) / (Vi - Vj) for integer k, need 0 <= t <= T.

But must ensure it's a valid meeting (opposite velocities - already guaranteed by sign difference). Also two soldiers at same point moving same direction don't count.

Let me count integer k such that t is in [0,T]. Since we want each meeting counted, and t can be non-integer, just enumerate k over feasible range.

For pair with D = Vi-Vj: t = (Lj-Li+1000k)/D, need 0<=t<=T. Count valid k. Each such t is a distinct meeting.

Let me iterate k over a sufficient range. D up to 200 in magnitude, T up to 50, so 1000k/D up to 50 → k up to 50*200/1000 = 10, plus offsets. Use wide range like -100000..100000 to be safe, using rational comparison.

For ordered pairs: Bi counts how many times soldier i asks — but each meeting i asks j and j asks i. So both increment. So Bi = total meetings involving i.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int n; long long T;
    cin>>n>>T;
    vector<long long> L(n), V(n);
    for(auto&x:L)cin>>x;
    for(auto&x:V)cin>>x;
    vector<long long> B(n,0);
    for(int i=0;i<n;i++)for(int j=i+1;j<n;j++){
        // opposite velocities
        if((V[i]>0) == (V[j]>0)) continue; // same direction sign
        long long D=V[i]-V[j];
        // t=(L[j]-L[i]+1000k)/D in [0,T]
        long long cnt=0;
        for(long long k=-200000;k<=200000;k++){
            long long num=(L[j]-L[i]+1000*k);
            // t=num/D, need 0<=num/D<=T
            // multiply by D: careful sign
            // condition: 0 <= num/D <= T
            long long Dd=D;
            // compute num/D >=0 and <=T using rationals
            // num/Dd >= 0
            __int128 nn=num, dd=Dd;
            if(dd<0){nn=-nn; dd=-dd;}
            if(nn<0) continue; // t<0
            if(nn > (__int128)T*dd) continue; // t>T
            cnt++;
        }
        B[i]+=cnt; B[j]+=cnt;
    }
    for(int i=0;i<n;i++){cout<<B[i];if(i+1<n)cout<<" ";}
    cout<<"\n";
    return 0;
}
```
