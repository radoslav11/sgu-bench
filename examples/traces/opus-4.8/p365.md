Simpler safe approach: DP counting. State: position, current value, phase (0=ascending, 1=descending). When phase=0 we can stay or increase or switch to phase=1 (decrease). To avoid double counting equal-value transitions at peak, when switching to descending we must strictly decrease (since equal continues ascending phase). 

DP: dp[pos][val][phase]. Transitions:
- phase 0: next val' >= val stays phase 0; next val' < val goes phase 1.
- phase 1: next val' <= val stays phase 1.

This uniquely classifies each sequence (the peak is the last position of the maximal run going up). Let me verify S=1: phase0, all 10 values =10. Good.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int S; cin>>S;
    // dp[val][phase]
    long long dp[10][2];
    for(int v=0;v<10;v++){dp[v][0]=1;dp[v][1]=0;}
    for(int pos=1;pos<S;pos++){
        long long ndp[10][2];
        memset(ndp,0,sizeof(ndp));
        for(int v=0;v<10;v++){
            // phase0
            long long a=dp[v][0];
            if(a){
                for(int nv=v;nv<10;nv++) ndp[nv][0]+=a; // stay ascending
                for(int nv=0;nv<v;nv++) ndp[nv][1]+=a; // switch descending strictly less
            }
            long long b=dp[v][1];
            if(b){
                for(int nv=0;nv<=v;nv++) ndp[nv][1]+=b;
            }
        }
        memcpy(dp,ndp,sizeof(dp));
    }
    long long ans=0;
    for(int v=0;v<10;v++){ans+=dp[v][0]+dp[v][1];}
    cout<<ans<<endl;
}
```
