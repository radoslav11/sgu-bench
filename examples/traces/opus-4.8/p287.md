Looking at this problem:

- The machine confirms "Correct!" only `c` steps after the correct number was input. So total questions = (position where correct guess is made in worst case) + c.
- Let `f(n)` = maximum number of values for which we can guarantee the correct guess is made within `n` queries. With delay `c`, the recurrence is:

`f(n) = f(n-1) + f(n-1-c) + 1`, with `f(k)=0` for `k<=0`.

Reasoning: the first query directly identifies 1 value (+1). The "less"/"greater" outcomes split the rest, but feedback arrives `c+1` queries later, giving the `f(n-1-c)` term, while continuing immediately gives `f(n-1)`.

Verification:
- c=0: f(n)=2^n−1 (classic binary search).
- Test1 q=21,c=2: f(8)=27≥21, n=8 → 8+2=10 ✓
- Test6 q=21,c=20: n=21 → 21+20=41 ✓
- Test5 q=111,c=1: n=10 → 11 ✓
- Test2 q=1,c=1: n=1 → 2 ✓
- Test3/4 q=65535/65536,c=0 → 16/17 ✓

Answer = (min n with f(n)≥q) + c.

Implementation uses a circular buffer of size `c+1`, since `(n-1-c) ≡ n (mod c+1)`. Values stay below 3·10^15 (we stop once ≥q), fitting in 64-bit. Loop length is at most a few million.
