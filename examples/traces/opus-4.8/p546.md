Looking at this problem:

We need exactly `a` zeros, `b` ones, and `n-a-b` twos. If `a+b > n`, impossible.

Strategy: Keep as many existing correct characters as possible. We want to minimize replacements.

Let me think greedily. We need to assign target characters to positions to meet counts, maximizing positions where target equals original.

This is an assignment problem but simple: we have target counts for '0','1','2'. For each position, keeping original char of type X "uses up" one of the needed count of X if available.

Greedy: First, keep characters that match needed counts. For each digit type, count how many we have. We can keep up to `need[d]` characters of digit `d`. The rest need to be changed.

So: count occurrences. For each digit, keep min(count[d], need[d]) of them as-is. Mark others as "needs change". Then fill the deficit positions with the surplus needed characters.

Let me implement:

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int n,a,b;
    cin>>n>>a>>b;
    string s; cin>>s;
    int c=n-a-b;
    if(c<0){ cout<<-1<<"\n"; return 0; }
    int need[3]={a,b,c};
    int keep[3]={0,0,0};
    // mark which positions to keep
    vector<bool> kept(n,false);
    for(int i=0;i<n;i++){
        int d=s[i]-'0';
        if(keep[d]<need[d]){ keep[d]++; kept[i]=true; }
    }
    // remaining needs
    int rem[3];
    for(int d=0;d<3;d++) rem[d]=need[d]-keep[d];
    int t=0;
    int cur=0;
    for(int i=0;i<n;i++){
        if(!kept[i]){
            t++;
            while(cur<3 && rem[cur]==0) cur++;
            s[i]='0'+cur;
            rem[cur]--;
        }
    }
    cout<<t<<"\n"<<s<<"\n";
    return 0;
}
```
