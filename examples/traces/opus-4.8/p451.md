We need to model people in a family tree. Each "relation" step expands into possible primitive moves. We track the set of possible positions (people identities) reachable, then compute min/max distance.

Let me represent each person by coordinates in the family tree. Since there are no cycles, I can model people relative to A as a path. Actually, let me think of representing each person by a position relation to A using a tree of generated nodes.

I'll represent the family tree as nodes. Each person is a node. Relations: father/mother/son/daughter/husband/wife are primitive. Brother etc. expand.

Let me build dynamically: start with set containing A's node. Apply each relation, which from a node produces a set of resulting nodes (creating new nodes as needed). I'll model the tree with parent pointers and spouse links.

Each node: parent (the couple's representation). Let me use a structure: nodes have father, mother, spouse, and children. I'll generate nodes lazily but must avoid infinite. Since at most 10 relations and each expands to bounded primitives, the tree stays finite.

I'll implement: each node has id. Functions to get father, mother, son(new), daughter(new), spouse. For brother: father's son (excluding self) or mother's son. To handle "not identical to X", we create a sibling. Father and mother of a node are the same married couple — a person's father's other children = mother's other children.

To get min/max degree, after collecting all final possible nodes, compute distance from A's node to each, where husband/wife edges cost 0, parent/child cost 1. BFS with 0/1 weights.

Let me build the tree on the fly with explicit node creation, sharing father&mother as a couple.

Each node: fatherId, motherId, spouseId, list of childIds. When creating a sibling, attach to same father/mother. New son/daughter creates child with this node as one parent and a new spouse as other.

For brother(X): collect existing sons of X's father/mother != X, plus create a new one (since sibling may exist). For min/max we want all possibilities, so include creating new sibling too. Actually to get all possible distances we should consider both existing relatives and new ones. Let me always also create fresh ones for siblings/uncles etc.
