Looking at this problem:

- Each percentage `p_i = 100*A_i/total`. We need integers that are either floor or ceil, summing to 100.
- If total is 0... all A_i are 0. Then percentages undefined. Hmm, edge case.

Let me think. Each real value is `100*A_i/total`. We assign floor to each, sum the floors = S. We need to add 1 to (100 - S) of them, but only those that have a fractional part (otherwise ceil=floor, can't add 1). Actually if it's an integer, floor=ceil, we can't round up. We need to round up exactly (100-S) candidates that have non-zero fractional part.

Number with non-zero fractional part must be >= (100-S). Since sum of fractions = 100 - S, and each fraction < 1, the count of nonzero fractions >= 100-S. So always solvable when total>0.

If total == 0: all zero votes. Sum of parts must be 100 but each real part is 0/0 undefined. Probably "No solution" unless... Let's output "No solution" for total==0.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    int n; 
    if(!(cin>>n)) return 0;
    vector<long long> a(n);
    long long total=0;
    for(int i=0;i<n;i++){cin>>a[i]; total+=a[i];}
    if(total==0){ cout<<"No solution\n"; return 0; }
    vector<long long> fl(n);
    long long S=0;
    for(int i=0;i<n;i++){ fl[i]=(100*a[i])/total; S+=fl[i]; }
    long long need=100-S;
    // round up `need` candidates with nonzero remainder
    for(int i=0;i<n && need>0;i++){
        long long rem=(100*a[i])%total;
        if(rem!=0){ fl[i]++; need--; }
    }
    for(int i=0;i<n;i++){ cout<<fl[i]; if(i+1<n) cout<<' '; }
    cout<<"\n";
    return 0;
}
```
