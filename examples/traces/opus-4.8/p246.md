Looking at this problem.

We have a necklace (circular) of M = 2N-1 beads. Two beads chosen, between them strictly one part contains exactly N beads. Since total is 2N-1, if one part between them has N beads, the other has (2N-1) - 2 - N = N-3... let me think about positions instead.

Beads positions 0..M-1 on a circle. Choosing two black beads at positions i,j. The arc strictly between them (one side) contains exactly N beads. So the distance: positions i and j with N beads between meaning j = i + N + 1 (mod M), i.e., separation of N+1 steps. Or the other direction separation M-(N+1).

Actually "two black beads (not necessarily different)" - same bead allowed: then going around, part between contains... if same bead, the part strictly between (whole circle minus that bead) = M-1 = 2N-2 beads, not N (unless N=2N-2, N=2). So generally need beads at distance N+1 apart on circle.

M = 2N-1, so N+1 and M-(N+1) = 2N-1-N-1 = N-2. So a chord connects beads at circular distance N+1 or equivalently N-2.

So beautiful iff there exist two black beads at circular distance N+1 (= distance N-2 the other way).

Consider gcd(N-2, M) cycles. Stepping by N-2 around circle of M beads. We need to find min K such that ANY placement of K black beads has two at this distance — i.e., max independent set (no two at distance N-2) size = K-1, answer = (max set avoiding the distance) + 1.

The structure: graph where vertices 0..M-1, edges between i and i+(N-2) mod M. This forms cycles. gcd(N-2, 2N-1). Note 2N-1 = 2(N-2)+3, so gcd(N-2,2N-1)=gcd(N-2,3). 

So d = gcd(N-2,3) which is 1 or 3. If N-2 ≡0 mod 3, d=3 else d=1.

If d=1: single cycle of length M=2N-1 (odd). Max independent set in odd cycle of length L is (L-1)/2 = (2N-2)/2 = N-1. Answer = N-1+1 = N.

Wait check N=3 (M=5): answer N=3. ✓. N=4(M=7): N-2=2, gcd(2,3)... 2 mod3≠0 so d=1, cycle length7, indep set 3, answer 4 ✓.

If d=3: three cycles each of length M/3 = (2N-1)/3. Cycle length parity? (2N-1)/3 odd. Max indep set per cycle (odd cycle length L): (L-1)/2. Total = 3*(L-1)/2, answer +1.

Let me verify with N where N-2 divisible by 3, e.g. N=5, M=9. cycles length 3 each, indep set (3-1)/2=1 per cycle, total 3, answer 4. Let me trust.

Special case: N=2 -> M=3 but M≥5 so ignore. Also distance could be such that N-2=0 when N=2.

Let me code: read M, N=(M+1)/2. d=gcd(N-2,3) (handle N-2 could be 0? N≥3 so N-2≥1). 
- if d==1: answer N
- else: L=M/3, ans=3*((L-1)/2)+1.

L odd always since M=2N-1 odd and divisible by 3 -> L odd. (L-1)/2 integer.

```cpp
#include <bits/stdc++.h>
using namespace std;
int main(){
    long long M; cin>>M;
    long long N=(M+1)/2;
    long long d=__gcd(N-2,3LL);
    long long ans;
    if(d==1) ans=N;
    else { long long L=M/3; ans=3*((L-1)/2)+1; }
    cout<<ans<<endl;
}
```
